Examples Overview

The portfolio sample mimics a web-based portfolio management application. The goal is to enable users to view all their portfolios, navigate through them, display them graphically, and have them updated on a regular basis with current stock prices as delivered to the page by a Web service. Users will also be able to change quantities on their stocks for analysis purposes.

The hello world sample demonstrates the power of the Faces Client framework by showing an MVC (Model View Controller) setup on a page with a very simple model and using two input fields as the controls.

Running the Faces Client Example

To run and test the Faces Client examples just follow these steps:

  1. In the Project Navigator, select the FacesClientTutorial project and choose Run on Server from its context menu.
  2. On the Server Selection dialog create a new WebSphere v5.1 Test Environment and Finish.
  3. The Server will be created and the FacesClientTutorial project will be published to it. After publishing, a web browser window will be opened displaying an initial index page. You can run the JSF-based Portfolio or hello world example from this page.

Portfolio Application Documentation

The portfolio sample mimics a web-based portfolio management application. The model for this application has a Root object, which contains a collection of Users and a collection of Stocks. It also contains a reference to a single Stock object used as a place holder to send and receive data from web-services in the sample. Then there are Users. Each user can have zero or more portfolios, and each portfolio can have zero or more positions. Each position points to a Stock.

Here is an explanation on the setup of the different controls on the page.

The page has two tabbed panels. The first tabbed panel has 3 panels. The Tree Panel shows a TreeView bound to a list of users connected to the Root and for each user his list of portfolios is shown. The DataGrid panel shows a DataGrid bound to the positions of a selected portfolio. The Full Panel shows the Tree and the DataGrid with the same setup as in the Tree Panel and the DataGrid Panel side by side.
The second tabbed panel has two panels. The Graph Panel shows a graph control bound to the positions of a selected portfolio. The x-axis of the graph shows a list of stock symbols and the y-axis shows the corresponding price and quantity for each stock. The WebServices Panel shows some input and output text field bound to properties of the temporary place holder stock. The command button in this panel will be used to invoke a Stock Web Service.

Here is a brief summary of the interactions between the controls

Note that each control is independently mapped to the model, so that any changes in the model data will be reflected in the control. In addtion to these implicit bindings there are some explicit bindings between the controls. When you highlight a portfolio in the TreeView, the DataGrid and the GraphDraw are updated with that Portfolio object and refreshed to show the positions in that portfolio object. We create this kind of an event system between the TreeView in the Tree Panel and the DataGrid in the DataGrid Panel and also between the TreeView in the Full Panel and the DataGrid in the same panel and the GraphDraw in the GraphDraw Panel. A second type of event that we create is between the DataGrid in the Full Panel and the place holder stock object. When a particular row in the DataGrid is highlighted the all the fields in the stock object for that position are copied to the corresponding fields of the place holder stock object. As a result, the values in the input and output fields in the WebService Panel are refreshed.

In order to invoke the Stock Web Service to get the stock quote for a particular symbol, click on the "Fetch Info" button in the WebServices Panel. The values displayed by the form controls in the Web Service Panel will be updated with the values recieved from the Stock Web Service .

Hello World Application Documentation

For this application on the server side we have a simple class called Hello which contains an attribute called text of type String. On the client side, we have exported this same model in JavaScript and in the browser we have two input text fields, both of which we want to bind to the text attribute of the Hello object. When we update the value of one control, this will change the text field of the Hello object, which will then result in the value being propagated to the other control without making a round trip back to the server

Try changing the value of the string inside the first input text box control to “Hello World!!!” the control will fire an event which updates the underlying data model (on the client side) and if you move out of the context of the first text box (click outside the box or navigate to the second text box) the value in the second text box will change to “Hello World!!!” since it is also bound to the same attribute of the same object as the first control.
