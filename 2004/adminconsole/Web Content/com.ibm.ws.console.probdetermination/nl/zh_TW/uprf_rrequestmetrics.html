效能監視基礎架構要求度量

請利用這個頁面來啟用效能監視基礎架構 (PMI) 要求度量、啟用要求度量應用程式回應度量 (ARM)，以及設定追蹤層次。

如果要檢視這個管理主控台頁面，請按一下疑難排解 > PMI 要求度量。

「配置」標籤

要求度量
啟用 PMI 要求度量。

如果它停用時，要求度量也會停用。

應用程式回應測量 (ARM)
啟用 PMI 要求度量來呼叫基本的 ARM 代理程式。

在啟用 ARM 之前，請先在所有的 WebSphere Application Server 節點上，安裝適當的 ARM 實作。 請與您的 ARM 代理程式提供者一起驗證，ARM 代理程式實作方式是否支援「要求度量」。 ARM 支援取決於「要求度量」支援。

追蹤層次
指定給定要求要累計多少追蹤資料。

包括下列項目之一： NONE - 不追蹤；HOPS - 只累計主要程序中繼點；PERF_DEBUG - 啟用中繼點以外的其他資訊，但不到 DEBUG 的效能密集程度；DEBUG - 完整詳細的追蹤。
