Performance Monitoring Infrastructure Request Metrics

Hier können Sie Performance Monitoring Infrastructure Request Metrics und Request Metrics Application Response Measurement (ARM) aktivieren und Trace-Stufen festlegen.

Klicken Sie zum Anzeigen dieser Seite der Administrationskonsole auf Fehlerbehebung > PMI Request Metrics.

Register "Konfiguration"

Request Metrics
Aktiviert PMI Request Metrics.

Wenn diese Einstellung inaktiviert ist, ist die Funktion Request Metrics inaktiviert.

Application Response Measurement (ARM)
Ermöglicht PMI Request Metrics den Aufruf eines zugrundeliegenden ARM-Agenten.

Installieren Sie vor der Aktivierung von ARM eine entsprechende ARM-Implementierung auf allen Knoten des WebSphere Application Server. Vergewissern Sie sich beim Provider des ARM-Agenten, dass Request Metrics von der Implementierung des ARM-Agenten unterstützt wird. Voraussetzung für die ARM-Unterstützung ist die Unterstützung von Request Metrics.

Trace-Stufe
Gibt an, wie viele Trace-Daten für eine bestimmte Anforderung erfasst werden sollen.

Folgende Werte sind gültig: NONE - Kein Tracing; HOPS - Tracing nur bei wichtigen Prozess-Hops; PERF_DEBUG - Zusätzliche Informationen zu Hops werden erfasst, jedoch weniger als bei der Einstellung DEBUG; DEBUG - Vollständiges Tracing mit allen Einzeldaten.
