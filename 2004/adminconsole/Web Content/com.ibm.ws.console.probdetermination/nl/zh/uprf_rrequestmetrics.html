性能监视基础结构请求度量

使用此页面启用指定性能监控基础结构（PMI）请求度量，启用请求度量应用程序响应测量（ARM）并设置跟踪级别。

要查看此管理控制台页面，单击故障诊断 > PMI 请求度量。

“配置”选项卡

请求度量
启用 PMI 请求度量。

禁用时，禁用请求度量功能。

应用程序响应测量（ARM）
允许 PMI 请求度量调用基础 ARM 代理程序。

启用 AMR 之前，在所有 WebSphere Application Server 节点上安装适当的 ARM 实现。 用 ARM 代理程序提供程序验证请求度量受 ARM 代理程序实现支持。 ARM 支持从属于请求度量支持。

跟踪级别
指定为给定的请求累加多少跟踪数据。

包含下列一种：NONE － 无跟踪；HOPS － 仅累加主要进程跳跃；PERF_DEBUG － 启用跳跃中的其它信息，但不象 DEBUG 那样性能强大；DEBUG － 全细节跟踪。
