诊断跟踪服务设置

使用此页面复查和修改诊断跟踪服务的属性。

要查看此页面，单击故障诊断 > 日志和跟踪 > server > 诊断跟踪。

“配置”选项卡

启用跟踪
启用跟踪服务。

如果未选择此选项，则在服务器启动时下列配置属性不会被传递到应用程序服务器跟踪服务。

跟踪规范
指定跟踪详细信息。

输入指定要跟踪的组件、软件包或组的跟踪字符串。跟踪字符串必须符合下面描述的特定语法。您可以直接输入跟踪字符串或使用图形跟踪界面生成跟踪字符串。单击修改启动图形跟踪界面。

如果您从配置选项卡启动图形跟踪界面，将显示一张熟悉的组件、软件包和组的列表。此列表可能会有遗漏。

如果您从运行时选项卡启动图形跟踪界面，组件、软件包和组的列表将显示位于正在运行的服务器上的所有当前已注册的组件。

跟踪规范的格式是：

<component> = <trace_type>= enabled | disabled
其中：<component> 是要启用或禁用跟踪的组件，而 <trace_type> 是启用或禁用的跟踪类型。用冒号（:）分隔多个跟踪规范。

相应于 Java 软件包和类或 Java 软件包集合的组件。使用 * 作为通配符以表明一些组件，这些组件包括指定组件包含的所有软件包中的所有类。例如：

*
指定所有运行在应用程序服务器中的可跟踪代码，包括 WebSphere Application Server 系统代码和客户代码。
com.ibm.ws.*
指定其软件包名以 com.ibm.ws 开始的所有类。
com.ibm.ws.classloader.JarClassLoader
仅指定 JarClassLoader 类。

要获取更多有关跟踪字符串语法的信息，请参阅 WebSphere Application Server 信息中心的启用跟踪一文。

注: 如果选择是从“组”和“组件”列表同时完成的，则从管理控制台设置跟踪规范时可能发生错误。 在有些情况下，从一个列表完成的选择不会在从其它列表添加选择时丢失。要解决此问题，将期望的跟踪规范直接输入到“跟踪规范”输入字段中。

跟踪输出  
指定跟踪输出应该写到哪里。

跟踪输出可以直接写入输出文件，或根据需要使用可在运行时页面上找到的“转储”按钮存储到内存和写入文件。

内存缓冲区
指定跟踪输出应该写到内存中的循环缓冲区。如果您选择此选项，您必须指定下列参数：
最大缓冲区大小
指定在缓冲区中可以高速缓存的条目数（以千计）。超过此数时，新的条目将覆盖旧的条目。
转储文件名
内存缓冲区转储时将写入的文件名。此选项仅在“运行时”选项卡中可用。

文件
指定以写跟踪输出到自管理日志文件。

自管理日志文件写消息到文件直到达到文件大小条件。当文件达到指定大小时，将临时地暂挂记录并关闭此日志文件和重命名。新的名称基于原始文件名，加上时间戳记限定符以表明发生重命名的时间。一旦重命名完成，将重新打开以原始文件名命名的新的空日志文件并继续记录。尽管单个消息可能被分割到两个文件中，但是该切换不会丢失消息。

如果您选择此选项，您必须指定下列参数：

最大文件大小
指定允许输出文件增长到的最大的大小（以兆字节计）。

此属性仅在选择了文件大小属性时才有效。当文件达到该大小时，将像上面所描述的一样滚动。

最大历史文件数
指定要保留的最大滚动文件数。
文件名
指定跟踪输出写入的文件名。

跟踪输出格式  
指定跟踪输出的格式。

您可指定跟踪输出的三种级别之一：

基本（兼容的）
仅保存基本跟踪信息。选择此选项最小化跟踪输出占用的空间数。
高级
保存更多特定跟踪信息。选择此选项查看详细的跟踪信息以用于故障诊断和问题确定。
日志分析器
以日志分析器工具兼容的格式保存跟踪信息。如果您要将跟踪输出用作到日志分析器工具的输入，则选择此选项。

“运行时”选项卡  
指定跟踪输出的格式。
保存跟踪
同时保存在“运行时”选项卡上对于跟踪配置的更改。

选择此框也将运行时跟踪更改复制到跟踪配置设置。保存这些更改到跟踪配置将导致即使重新启动应用程序，更改还将保留。
