Performance Monitoring Infrastructure 要求メトリック

このページを使用して、Performance Monitoring Infrastructure (PMI) 要求メトリック、 および要求メトリック Application Response Measurement (ARM) を使用可能にし、 トレース・レベルを設定します。

この管理コンソール・ページを表示するには、「トラブルシューティング」>「PMI 要求メトリック」をクリックします。

「構成」タブ

要求メトリック
PMI 要求メトリックを使用可能にします。

これを使用不可にすると、要求メトリック機能は使用できなくなります。

アプリケーション応答測定 (ARM)
PMI 要求メトリックが、基礎をなす ARM エージェントを呼び出すことができるようにします。

ARM を使用可能にするにはまず、 すべての WebSphere Application Server のノードに、適切な ARM インプリメンテーションをインストールします。 ARM エージェント・プロバイダーを使用して、 ARM エージェントのインプリメンテーションが要求メトリックをサポートすることを検証します。 ARM サポートは、要求メトリック・サポートに依存しています。

Trace Level
所定の要求に対して集計するトレース・データの量を指定します。

次のいずれかを指定します。NONE - トレースなし。HOPS - 主要なプロセス・ホップの場合にだけ集計する。 PERF_DEBUG - ホップに関する詳細情報を利用できるようにするが、DEBUG ほどパフォーマンスに特化していない。 DEBUG - トレースの完全な詳細。
