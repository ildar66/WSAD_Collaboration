Boutons de la console d'administration

Les choix de boutons suivants sont disponibles sur différentes pages de la console d'administration selon les options du produit que vous avez activées.

  • Abandonner. Abandonne une transaction qui n'est pas encore à l'état prévu. Toutes les opérations achevées par la transaction seront annulées.
  • Ajouter. Ajoute l'élément sélectionné ou saisi à une liste ou ouvre une boîte de dialogue permettant d'ajouter un élément à une liste.
  • Valider. Enregistre les modifications que vous apportez à une page sans la quitter.
  • Précédent. Affiche la page ou l'élément précédent d'une séquence. La console d'administration ne prend pas en charge les boutons Précédent et Suivant du navigateur. Ces boutons peuvent générer des problèmes par intermittence. Utilisez à la place les boutons Précédent ou Annuler dans les panneaux de la console d'administration.
  • Parcourir. Ouvre une boîte de dialogue permettant de rechercher un fichier de votre système.
  • Annuler. Quitte la page ou la boîte de dialogue en cours, en supprimant les modifications non sauvegardées. La console d'administration ne prend pas en charge les boutons Précédent et Suivant du navigateur. Ces boutons peuvent générer des problèmes par intermittence. Utilisez à la place le bouton Annuler dans les panneaux de la console d'administration.
  • Modifier. Dans le contexte de sécurité, permet de rechercher dans le registre d'utilisateurs un ID utilisateur sous lequel une exécution va s'exécuter. Dans le contexte des propriétés de conteneur, permet de modifier la source de données utilisée par le conteneur.
  • Effacer. Efface vos modifications et restaure les dernières valeurs sauvegardées.
  • Désélectionner. Annule la sélection de cellules dans les tableaux de l'onglet.
  • Fermer. Ferme la boîte de dialogue.
  • Valider. Libère tous les verrous détenus par une transaction préparée et force la validation de la transaction.
  • Copier. Crée des copies des serveurs d'applications sélectionnés.
  • Créer. Enregistre les modifications que vous avez apportées aux onglets d'une boîte de dialogue et quitte cette dernière.
  • Supprimer. Supprime l'instance sélectionnée.
  • Détails. Affiche les détails relatifs à une transaction.
  • Terminé. Enregistre les modifications que vous avez apportées aux onglets d'une boîte de dialogue et quitte cette dernière.
  • Bas. Se déplace dans une liste en allant vers le bas de cette dernière.
  • Vidage. Active le vidage d'un serveur d'applications tracé.
  • Editer. Permet d'éditer l'élément sélectionné dans une liste ou ouvre une boîte de dialogue dans laquelle l'élément peut être édité.
  • Exporter. Accède à une page permettant l'exportation de fichiers EAR pour une application d'entreprise.
  • Exporter les fichiers DDL. Accède à une page permettant l'exportation de fichiers DDL pour une application d'entreprise.
  • Exporter les clés. Exporte les clés LTPA dans d'autres domaines.
  • Filtrage. Ouvre une boîte de dialogue permettant de définir les ressources à afficher dans les tableaux de l'onglet.
  • Terminer. Provoque la fin d'une transaction, que sa sortie ait été rapportée à toutes les applications participantes ou non.
  • Premier. Affiche le premier enregistrement d'une série d'enregistrement.
  • Génération de clés. Génère de nouvelles clés LTPA. Lorsque la sécurité est activée pour la première fois avec LTPA comme mécanisme d'authentification, les clés LTPA sont automatiquement générées avec le mot de passe saisi dans le panneau. Si de nouvelles clés doivent être générées, utilisez ce bouton après la sauvegarde du serveur avec la sécurité activée. Lorsque vous cliquez sur ce bouton, les clés sont générées et elles sont propagées à tous les serveurs actifs (cellule, noeud et serveurs d'applications). Les nouvelles clés peuvent être utilisées pour chiffrer et déchiffrer les jetons LTPA. Cliquez sur Sauvegarder dans la barre des tâches de la console pour enregistrer les nouvelles clés et le mot de passe dans le référentiel.
  • Arrêt immédiat. Arrête le serveur mais ignore le processus de mise au repos normal de celui-ci qui permet aux demandes en cours d'être exécutées avant la fermeture de l'ensemble du processus serveur. Ce mode de fermeture est plus rapide que l'arrêt normal du serveur, mais certains clients d'application peuvent recevoir des exceptions.
  • Importer les clés. Importe de nouvelles clés LTPA d'autres domaines. Pour prendre en charge l'ouverture de session unique (SSO) dans WebSphere sur plusieurs domaines (cellules) WebSphere, des clés LTPA et un mot de passe doivent être partagés entre les domaines. Après l'exportation des clés de l'une des cellules dans un fichier, cliquez sur ce bouton pour importer les clés dans tous les serveurs actifs (cellule, noeud et serveurs d'application). Les nouvelles clés peuvent être utilisées pour chiffrer et déchiffrer le jeton LTPA. Cliquez sur Sauvegarder dans la barre des tâches de la console pour enregistrer les nouvelles clés et le mot de passe dans le référentiel.
  • Installer. Affiche la page Préparation à l'installation de l'application, qui permet de déployer une application ou un composant EJB ou Web sur un serveur d'applications.
  • Install RAR.Le bouton Installation d'un fichier RAR permet d'ouvrir une boîte de dialogue utilisée pour installer un connecteur JCA et créer l'adaptateur de ressources associé.
  • Gestion des transactions. Affiche une liste de transactions actives s'exécutant sur un serveur. Vous pouvez mettre fin de manière forcée à toute transaction dont le traitement s'est arrêté en raison de l'indisponibilité d'une ressource transactionnelle.
  • Modifier. Ouvre une boîte de dialogue permettant de modifier une spécification.
  • Déplacer. Déplace les serveurs d'applications sélectionnés à un autre emplacement de la cellule d'administration. A l'invite du système, indiquez l'emplacement cible.
  • Nouveau. Affiche une page permettant de définir une nouvelle instance. Par exemple, en cliquant sur Nouveau dans la page des serveurs d'applications, une page s'affiche, sur laquelle vous pouvez configurer un nouveau serveur d'applications.
  • Suivant. Affiche la page, le cadre ou l'élément suivant d'une séquence.
  • OK. Enregistre vos modifications et quitte la page.
  • Ping. Tente d'entrer en contact avec les serveurs d'applications sélectionnés.
  • Précédent. Affiche la page, le cadre ou l'élément précédent d'une séquence.
  • Quitter. Quitte une boîte de dialogue en supprimant toutes les modifications non sauvegardées.
  • Régénérer. Régénère la vue des données pour les instances figurant actuellement dans cet onglet.
  • Retirer. Supprime l'élément sélectionné.
  • Restaurer. Efface vos modifications de l'onglet et restaure les valeurs le plus récemment enregistrées.
  • Redémarrer. Arrête les objets sélectionnés et les démarre à nouveau.
  • Extraire un nouvel élément. Extrait un nouvel enregistrement.
  • Sauvegarder. Sauvegarde les modifications apportées à la configuration locale dans la configuration maîtresse.
  • Sélectionner. Pour l'analyse des ressources, permet de sélectionner une portée dans laquelle contrôler les ressources.
  • Définir. Sauvegarde les modifications apportées aux paramètres dans une boîte de dialogue.
  • Paramètres. Ouvre une boîte de dialogue pour l'édition des paramètres de ressources relatifs au servlet.
  • Paramètres actuellement configurés. Ouvre une boîte de dialogue affichant les paramètres en cours d'utilisation.
  • Démarrer. Dans le contexte des serveurs d'applications, démarre les serveurs d'application sélectionnés. Dans le contexte de collecte de données, démarre la collecte des données pour les tables de cet onglet.
  • Arrêter. Dans le contexte des composants serveur, tels que des serveurs d'applications, arrête les composants serveur sélectionnés. Dans le contexte d'une collecte de données, arrête la collecte des données pour les tables de cet onglet.
  • Désinstaller. Supprime une application déployée du référentiel de configuration de WebSphere Application Server. Supprime également les fichiers binaires d'application du système de fichiers.
  • Mettre à jour. Remplace une application déployée sur un serveur par une application mise à jour. Lors du processus de mise à jour, il se peut que vous deviez effectuer des opérations sur les pages de préparation à l'installation de l'application et de mise à jour de l'application.
  • Mettre à jour la liste de ressources. Met à jour les données d'une table. Découvre et ajoute les nouvelles instances à la table.
  • Afficher. Affiche une boîte de dialogue relative à un fichier.
