Performance Monitoring Infrastructure Request Metrics

Cette page permet d'activer Performance Monitoring Infrastructure (PMI) Request Metrics ainsi que Request Metrics Application Response Measurement (ARM) et de définir des niveaux de trace.

Pour afficher cette page de la console d'administration, cliquez sur Résolution des incidents > PMI Request Metrics.

Onglet Configuration

Request Metrics
Active PMI Request Metrics.

Si cette option est désactivée, la fonction Request Metrics l'est également.

Application Response Measurement (ARM)
Active PMI Request Metrics pour appeler un agent ARM sous-jacent.

Avant d'activer ARM, installez une implémentation ARM appropriée sur tous les noeuds WebSphere Application Server. Vérifiez auprès de votre fournisseur d'agent ARM que Request Metrics est pris en charge par l'implémentation d'agent ARM. Le support ARM dépend du support Request Metrics.

Niveau de trace
Spécifie la quantité de données de trace à accumuler pour une demande spécifique.

Les options possibles sont les suivantes : NONE - Pas de trace ; HOPS - Accumule uniquement au niveau des sauts du traitement principal ; PERF_DEBUG - Active des informations supplémentaires sur les sauts, mais les performances ne sont pas aussi étendues qu'avec DEBUG ; DEBUG - Trace entièrement détaillée.
