ログイン設定

このページを使用して、WebSphere Application Server 管理コンソールのユーザーを指定します。 グローバル・セキュリティーを使用している場合は、パスワードも指定します。

ユーザーを指定すると、その製品を使用して以前に実行した作業を再開することができます。 ユーザー ID を入力したら、「OK」をクリックして次のページに進み、 管理コンソールにアクセスしてください。

このページを表示するには、管理コンソールを始動します。

ユーザー ID
ユーザーを識別するストリングを指定します。 ユーザー ID は、管理サーバーに対して固有のものでなければなりません。

この製品を使用して実行し、製品を終了する前に保管した作業は、 ここで入力するユーザー ID で識別される構成に保管されます。 そのユーザー ID で行った作業にあとでアクセスする場合は、「ログイン」ページでユーザー ID を指定します。

管理コンソールで現在、ブラウザーが開いているクライアント・マシンからログインし、 同じサーバーに接続する場合は、 別のユーザー ID でログインするかどうかにかかわらず、2 つの接続で同じ HTTP セッション・オブジェクトを共用することになる可能性があります。 同じユーザー ID を使用するかどうかにかかわらず、 2 つの異なる HTTP セッション・オブジェクトを使用して作業するためには、 同じクライアント・マシンから管理コンソールにログインするのに、 2 つの異なる種類のブラウザー (例えば、Microsoft Internet Explorer と Netscape Communicator) を使用する必要があります。 つまり、同じクライアント・マシンに 2 つの並行セッションが必要な場合は、 2 つの異なるブラウザー・タイプから管理コンソールにアクセスします。

同じブラウザー・タイプを使用して、コンソールに再度ログインすることはできますが、別の物理ホストからログインし、別のユーザー ID を使用する必要があるので注意してください。 例えば、Joe が、Internet Explorer ブラウザーを使用して物理ホスト myhost1 からユーザー ID joe で管理コンソールにログインするとします。 Joe は、Internet Explorer ブラウザーを使用して同じコンソールに再度ログインできますが、 別のユーザー ID (例えば、peggy) と別の物理ホスト (例えば、myhost2) を使用する必要があります。

データ型 ストリング
現在、別のユーザーが同じユーザー名でログインしています
そのユーザーをログアウトさせて指定したユーザー ID で作業を継続するか、「 ログイン」ページに戻って別のユーザー ID を指定するか、 あるいはそのユーザーがログアウトするのを待機するかを指定します。

ユーザーが管理コンソールをブラウズしているときにログアウトしないで Web ブラウザーを閉じ、 その後新たにブラウザーを開いて管理コンソールにアクセスしようとした場合に、このフィールドが表示されます。

マスター構成を編集する
これが使用可能になっている場合は、デフォルトの管理構成を使用して、 そのユーザーのセッションで最後に使用された構成は使用しないように指定されています。 この場合、管理構成を最後に保管した後にユーザーのセッションで行った変更は保管されません。

このフィールドが表示されるのは、ユーザーが管理構成を変更した後に、 変更を保管せずにログアウトした場合に限られます。

データ型 ブール
デフォルト false
前のセッションで行った変更をリカバリーする
これが使用可能になっている場合は、 そのユーザーのセッションで最後に使用されたものと同じ管理構成を使用するように指定されています。 ユーザーのセッションの管理構成を最後に保管した後にユーザーが行った変更をリカバリーします。

このフィールドが表示されるのは、ユーザーが管理構成を変更した後に、 変更を保管せずにログアウトした場合に限られます。

データ型 ブール
デフォルト true
