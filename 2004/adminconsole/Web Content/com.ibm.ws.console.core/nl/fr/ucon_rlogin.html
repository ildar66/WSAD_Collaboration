Paramètres de connexion

Cette page permet d'indiquer l'utilisateur de la console d'administration de WebSphere Application Server. Si vous utilisez la sécurité globale, vous devez également indiquer un mot de passe.

La spécification d'un utilisateur permet de reprendre un travail commencé avec le produit. Une fois l'ID utilisateur entré, cliquez sur OK pour passer à la page suivante et accéder à la console d'administration.

Pour afficher cette page, démarrez la console d'administration.

ID utilisateur
Indique une chaîne identifiant l'utilisateur. L'ID utilisateur doit être unique pour le serveur d'administration.

Le travail que vous effectuez dans ce produit et que vous sauvegardez avant de quitter l'application est enregistré dans une configuration pouvant être identifiée par l'ID utilisateur que vous avez saisi. Pour accéder ultérieurement au travail effectué sous cet ID utilisateur, spécifiez celui-ci sur la page de connexion.

Si vous vous connecter à partir d'une machine client qui dispose d'un navigateur ouvert dans la console d'administration et que vous vous connectez au même serveur, vos deux connexions peuvent partager le même objet de session HTTP, que vous vous connectiez en utilisant ou non des ID utilisateur différents. Vous devez utiliser deux types de navigateur (par exemple, Microsoft Internet Explorer et Netscape Communicator) pour vous connecter à la console d'administration à partir de la même machine client afin d'utiliser deux objets de session HTTP, que vous utilisiez ou non le même ID utilisateur. Cela signifie que si vous avez besoin de deux sessions simultanées sur la même machine client, accédez à la console d'administration à partir de deux types de navigateur.

Vous pouvez utiliser le même type de navigateur pour vous connecter deux fois à la console mais vous devez vous connecter à différents hôtes physiques et utiliser différents ID utilisateur. Par exemple, supposons que Paul se connecte à la console d'administration sous l'ID utilisateur paul sur l'hôte physique monhôte1 à l'aide d'un navigateur Internet Explorer. Paul peut se connecter à la même console une deuxième fois à l'aide d'un navigateur Internet Explorer mais il peut utiliser un ID utilisateur différent (tel que peggy) et un hôte physique différent (tel que monhôte2).

Utiliser la configuration maîtresse
Si l'option est activée, cela signifie que vous souhaitez utiliser la configuration administrative par défaut et non celle utilisée en dernier pour la session de l'utilisateur. Les modifications apportées à la session de l'utilisateur depuis la dernière sauvegarde de la configuration d'administration seront perdues.

Cette zone apparaît uniquement si l'utilisateur a modifié la configuration d'administration et s'est ensuite déconnecté sans sauvegarder les modifications.

Type de données Elément booléen
Valeur par défaut false
Récupérer les modifications apportées lors de la session précédente
Si l'option est activée, cela signifie que vous souhaitez utiliser la même configuration administrative que celle utilisée en dernier pour la session de l'utilisateur. Les modifications apportées par l'utilisateur depuis la dernière sauvegarde de la configuration d'administration pour la session de l'utilisateur sont récupérées.

Cette zone apparaît uniquement si l'utilisateur a modifié la configuration d'administration et s'est ensuite déconnecté sans sauvegarder les modifications.

Type de données Elément booléen
Valeur par défaut true
