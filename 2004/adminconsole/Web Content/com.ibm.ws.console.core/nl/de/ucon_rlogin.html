Einstellungen für Anmeldung

Auf dieser Seite können Sie den Benutzer für die Administrationskonsole des WebSphere Application Server angeben. Wenn Sie die globale Sicherheitseinrichtung verwenden, müssen Sie auch ein Kennwort angeben.

Geben Sie einen Benutzer an, haben Sie zu einem späteren Zeitpunkt die Möglichkeit, die mit dem Produkt bereits durchgeführte Arbeit wieder aufzunehmen. Klicken Sie nach der Eingabe einer Benutzer-ID auf OK, um zur nächsten Seite zu wechseln und die Administrationskonsole aufzurufen.

Starten Sie zum Anzeigen dieser Seite die Administrationskonsole.

Benutzer-ID
Gibt eine Zeichenfolge zur Bezeichnung des Benutzers an. Die Benutzer-ID muss auf dem Administrationsserver eindeutig sein.

Arbeit, die Sie mit dem Produkt durchgeführt und dann vor dem Schließen des Produkts gespeichert haben, wird in einer Konfiguration gespeichert, die über den Benutzernamen, den Sie eingeben, identifiziert wird. Wenn Sie später auf die Arbeit, die unter dieser Benutzer-ID durchgeführt wurde, zugreifen möchten, geben Sie die Benutzer-ID auf der Seite "Anmelden" an.

Wenn Sie sich über eine Clientmaschine anmelden, auf der gegenwärtig ein Browser in der Administrationskonsole geöffnet ist, und Sie eine Verbindung zu demselben Server herstellen, können die zwei Verbindungen dasselbe HTTP-Sitzungsobjekt verwenden, unabhängig davon, ob Sie sich unter verschiedenen Benutzer-IDs anmelden. Sie müssen zwei verschiedene Browsertypen verwenden (z. B. Microsoft Internet Explorer und Netscape Communicator), um sich von einer Clientmaschine an der Administrationskonsole anmelden und mit zwei verschiedenen HTTP-Sitzungsobjekten arbeiten zu können, unabhängig davon, ob Sie dieselbe Benutzer-ID verwenden. Das heißt, wenn Sie mit zwei gleichzeitigen Sitzungen auf einer Clientmaschine arbeiten möchten, müssen Sie mit zwei verschiedenen Browsertypen auf die Administrationskonsole zugreifen.

Beachten Sie, dass Sie denselben Browsertyp verwenden können, um sich zweimal an der Konsole anzumelden, die Anmeldungen müssen jedoch mit verschiedenen physischen Hosts und Benutzer-IDs durchgeführt werden. Nehmen Sie z. B. an, dass Joe sich mit dem Browser Internet Explorer unter der Benutzer-ID joe auf dem physischen Host meinhost1 anmeldet. Joe kann sich kann sich mit dem Internet Explorer ein zweites Mal an derselben Konsole anmelden, muss jedoch eine andere Benutzer-ID (z. B. peggy) und einen anderen physischen Host (z. B. meinhost2) verwenden.

Datentyp String
Es ist bereits ein anderer Benutzer unter dieser Benutzer-ID angemeldet
Gibt an, ob der Benutzer abgemeldet und die Arbeit mit der angegebenen Benutzer-ID fortgesetzt werden soll oder ob die Seite "Anmelden" erneut aufgerufen und eine andere Benutzer-ID eingegeben bzw. auf das Abmelden des Benutzers gewartet werden soll.

Dieses Feld erscheint, wenn der Benutzer einen Webbrowser beim Durchsuchen der Administrationskonsole geschlossen hat, ohne sich vorher abzumelden, dann einen neuen Browser geöffnet und anschließend versucht hat, auf die Administrationskonsole zuzugreifen.

Mit der Master-Konfiguration arbeiten
Gibt bei Aktivierung an, dass Sie die Standardkonfiguration der Administration und nicht die Konfiguration der letzten Benutzersitzung verwenden möchten. Änderungen, die seit dem letzten Speichern der Administrationskonfiguration vorgenommen wurden, gehen verloren.

Das Feld erscheint nur, wenn der Benutzer die Administrationskonfiguration geändert und sich dann ohne Speichern der Änderungen abgemeldet hat.

Datentyp Boolescher Wert
Standardwert false
In der vorherigen Sitzung vorgenommene Änderungen wiederherstellen
Gibt bei Aktivierung an, dass Sie die Konfiguration der letzten Benutzersitzung verwenden möchten. Stellt Änderungen, die der Benutzer seit dem letzten Speichern der Administrationskonfiguration für die Benutzersitzung vorgenommen hat, wieder her.

Das Feld erscheint nur, wenn der Benutzer die Administrationskonfiguration geändert und sich dann ohne Speichern der Änderungen abgemeldet hat.

Datentyp Boolescher Wert
Standardwert true
