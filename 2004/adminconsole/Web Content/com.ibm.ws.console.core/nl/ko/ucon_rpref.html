환경 설정

환경 설정 페이지를 사용하여 관리 콘솔 작업영역의 변경 이후 자동으로 새로 고치기 수행할지 여부, 관리 콘솔 노드로의 기본 범위, 확인 대화 상자 표시 여부 및 관리 콘솔 노드가 되는 기본 범위를 지정하십시오.

이 관리 콘솔 페이지를 보려면 콘솔 작업 표시줄에서 환경 설정을 클릭하십시오.

WorkSpace 자동 새로 고치기 사용 가능
관리 구성이 변경된 후 관리 콘솔 workspace가 자동으로 다시 그리기 원하는지 여부를 지정합니다.

기본값은 자동으로 갱신하는 것입니다. Application Server의 새 인스턴스를 작성하도록 콘솔에 지시한 경우 Application Server 페이지가 자동으로 새로 고쳐지고 서버 콜렉션에 새 서버의 이름을 표시합니다.

workspace가 자동으로 다시 그리지 않도록 지정하는 것은 사용자가 관리 구성에 대해 수행된 변경사항을 보기 위해 콘솔 탐색 트리나 콜렉션 페이지의 링크를 클릭하여 페이지에 다시 액세스해야 함을 의미합니다.

데이터 유형 부울
기본값 true
WorkSpace 삭제 확인 안함
오브젝트를 삭제하는 요청 이후에 확인 대화 상자가 표시되는지 여부를 지정합니다. 기본값은 확인 대화 상자가 표시되지 않는것 입니다.
데이터 유형 부울
기본값 false
기본 범위 사용
기본 범위가 관리 콘솔 노드일지 여부를 지정합니다. 기본값은 범위가 콘솔 노드가 아닙니다.
데이터 유형 부울
기본값 false
배너 숨기기/표시
관리 콘솔의 맨 위에 있는 WebSphere Application Server 배너의 표시 여부를 지정합니다. 기본값은 표시하려는 배너에 대한 것입니다.
데이터 유형 부울
기본값 true
설명 숨기기/표시
콘솔 페이지 정보 및 페이지 내의 필드의 표시 여부를 선택할 수 있습니다. 작업 표시줄 오른쪽의 숨기기/표시 필드 및 페이지 설명 아이콘은 동일한 기능을 제공합니다. 기본값은 페이지 및 필드 설명을 표시하는 것입니다.
데이터 유형 부울
기본값 true
