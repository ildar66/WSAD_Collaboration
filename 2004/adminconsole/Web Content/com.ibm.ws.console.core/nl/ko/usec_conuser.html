콘솔 사용자 설정 및 CORBA 네임 서비스 사용자 설정

관리 콘솔 또는 wsadmin 스크립팅과 같은 도구를 사용하여 WebSphere Application Server를 관리할 수 있는 특정 권한을 사용자에게 제공하려면 콘솔 사용자 설정 페이지를 사용하십시오. 권한 요구사항은 글로벌 보안이 사용 가능할 때만 적용됩니다. CORBA 네임 서비스 사용자 설정을 관리하려면 CORBA 네임 서비스 사용자 설정을 사용하십시오.

콘솔 사용자 관리 콘솔 페이지를 보려면 시스템 관리 > 콘솔 사용자를 클릭하십시오.

CORBA 네임 서비스 사용자 관리 콘솔 페이지를 보려면 환경 > 이름 지정 > CORBA 네임 서비스 사용자를 클릭하십시오.

사용자(콘솔 사용자)
사용자를 지정합니다.

입력된 사용자가 구성된 활성 사용자 레지스트리에 있어야 합니다.

데이터 유형: 문자열
사용자(CORBA 네임 서비스 사용자)
CORBA 네임 서비스 사용자를 지정합니다.

입력된 사용자가 구성된 활성 사용자 레지스트리에 있어야 합니다.

데이터 유형: 문자열
역할(콘솔 사용자)
사용자 역할을 지정합니다.

다음의 관리 역할은 특정 WebSphere Application Server 관리 기능 수행에 필요한 권한에 대한 다양한 등급을 제공합니다.

관리자
관리자 역할에는 조작자 사용 권한, 구성자 사용 권한 및 서버 암호, LTPA 암호 및 키 등의 민감한 데이터를 액세스하는데 필요한 사용 권한이 포함됩니다.
구성자
구성자 역할에는 모니터 사용 권한 및 WebSphere Application Server 구성을 변경할 수 있는 능력이 포함됩니다.
조작자
조작자 역할에는 모니터 권한 및 런타임 상태를 변경할 수 있는 능력이 포함됩니다. 예를 들어, 조작자는 서비스를 시작하거나 중지할 수 있습니다.
모니터
모니터 역할은 최소한의 사용 권한을 가집니다. 이 역할은 주로 사용자가 WebSphere Application Server 구성 및 현재 상태를 보는 것으로 제한합니다.

데이터 유형: 문자열
범위: 관리자, 구성자, 조작자 및 모니터
역할(CORBA 네임 서비스 사용자)
네임 서비스 사용자 역할을 지정합니다.

특정 WebSphere 네임 서비스 기능을 수행하기 위해 필요한 권한의 정도를 제공하기 위해 많은 이름 역할이 정의됩니다. 권한 정책은 글로벌 보안이 사용 가능할 때만 강제 실행됩니다. 다음의 역할이 유효합니다. CosNamingRead, CosNamingWrite, CosNamingCreate, and CosNamingDelete.

네 가지 역할의 이름은 WebSphere Application Server Advanced Edition 버전 4.0.2와 동일합니다. 그러나 현재 역할에는 다음과 같이 낮은 단계에서 높은 단계까지의 권한 레벨이 있습니다.

CosNamingRead
사용자가 예를 들면 JNDI(Java Naming and Directory Interface) 찾아보기 메소드를 사용하여 WebSphere 네임 스페이스를 조회할 수 있습니다. 특정 주제인 EVERYONE이 이 역할에 대한 기본 정책입니다.
CosNamingWrite
사용자가 JNDI 바인드, 리바인드 또는 언바인드등의 쓰기 조작과 CosNamingRead 조작을 수행할 수 있습니다. 특정 주제 ALL AUTHENTICATED는 이 역할에 대한 기본 정책입니다.
CosNamingCreate
사용자가 JNDI createSubcontext와 CosNamingWrite 등의 조작을 통해 네임 스페이스에서 새 오브젝트를 작성할 수 있습니다. 특정 주제 ALL AUTHENTICATED는 이 역할에 대한 기본 정책입니다.
CosNamingDelete
사용자가 예를 들면 JNDI destroySubcontext 메소드와 CosNamingCreate 조작을 사용하여 네임 스페이스에 있는 오브젝트를 삭제할 수 있습니다. 특정 주제 ALL AUTHENTICATED는 이 역할에 대한 기본 정책입니다.

데이터 유형: 문자열
범위: CosNamingRead, CosNamingWrite, CosNamingCreate 및 CosNamingDelete
