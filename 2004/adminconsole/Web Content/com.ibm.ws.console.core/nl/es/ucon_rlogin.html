Valores de inicio de sesión

Utilice esta página para especificar el usuario de la consola administrativa de WebSphere Application Server. Si está utilizando la seguridad global, también debe especificar una contraseña.

Si especifica un usuario podrá reanudar el trabajo realizado anteriormente con el producto. Después de escribir un ID de usuario, pulse Aceptar para continuar con la siguiente página y acceder a la consola administrativa.

Para ver esta página, inicie la consola administrativa.

ID de usuario
Especifica una serie que identifica el usuario. El ID de usuario debe ser exclusivo para el servidor de administración.

El trabajo que realice con el producto se puede guardar antes de salir en una configuración identificada por el ID de usuario que especifique. Para acceder posteriormente al trabajo realizado bajo ese ID de usuario, especifique el ID de usuario en la página Inicio de sesión.

Si va a iniciar la sesión desde una máquina cliente que actualmente tiene abierto un navegador en la consola administrativa y se conecta al mismo servidor, las dos conexiones pueden compartir el mismo objeto de sesión HTTP, aunque los ID de usuario con los que inicie la sesión sean distintos. Debe utilizar dos tipos de navegadores distintos (por ejemplo, Microsoft Internet Explorer y Netscape Communicator) para iniciar la sesión en la consola administrativa desde la misma máquina cliente para poder trabajar desde dos objetos de sesión HTTP distintos, tanto si utiliza o no el mismo ID de usuario. Es decir, si necesita dos sesiones simultáneas en la misma máquina cliente, acceda a la consola administrativa desde dos tipos de navegadores diferentes.

Tenga en cuenta que puede utilizar el mismo tipo de navegador para iniciar la sesión dos veces en la consola, aunque debe hacerlo desde sistemas principales físicos distintos y utilizar ID de usuario distintos. Por ejemplo, suponga que Juan inicia la sesión en la consola administrativa con el ID de usuario juan en el sistema principal físico misistppal1 utilizando un navegador Internet Explorer. Juan puede iniciar la sesión en la misma consola una segunda vez utilizando un navegador Internet Explorer, pero debe utilizar un ID de usuario distinto (por ejemplo alicia) y un sistema principal físico diferente (por ejemplo misistppal2).

Tipo de datos String
Otro usuario ha iniciado una sesión con el mismo nombre de usuario
Especifica si debe terminar la sesión del usuario y continuar trabajando con el ID de usuario especificado, si debe volver a la página de Inicio de sesión y especificar otro ID de usuario, o si debe esperar a que el usuario termine la sesión.

Este campo aparece si el usuario ha cerrado un navegador Web mientras examinaba la consola administrativa y no ha cerrado antes la sesión y, a continuación, ha abierto un navegador nuevo y ha intentado acceder a la consola administrativa.

Acceder a la configuración maestra
Cuando esta opción está habilitada, especifica que desea utilizar la configuración administrativa por omisión y no la configuración utilizada por última vez para la sesión de usuario. Los cambios efectuados en la sesión de usuario desde la última vez que se ha guardado la configuración administrativa se perderán.

Este campo aparece sólo si el usuario ha cambiado la configuración administrativa y, a continuación, ha cerrado la sesión sin guardar los cambios.

Tipo de datos Boolean
Valor por omisión false
Recuperar los cambios efectuados en la sesión anterior
Cuando esta operación está habilitada, especifica que desea utilizar la misma configuración administrativa utilizada por última vez para la sesión de usuario. Recupera los cambios efectuados por el usuario desde la última vez que se ha guardado la configuración administrativa para la sesión de usuario.

Este campo aparece sólo si el usuario ha cambiado la configuración administrativa y, a continuación, ha cerrado la sesión sin guardar los cambios.

Tipo de datos Boolean
Valor por omisión true
