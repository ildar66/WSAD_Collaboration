CORBA 오브젝트 바인딩 설정

이 페이지를 참조하여 CORBA 오브젝트 바인딩의 새 이름 바인딩을 구성하거나 기존의 CORBA 오브젝트 바인딩을 보기 또는 편집하십시오.

이 관리 콘솔 페이지를 보려면 환경 > 네이밍 > 네임 스페이스 바인딩 > CORBA_namespace_binding을 클릭하십시오.

구성 탭

범위
구성된 바인딩의 범위를 표시합니다. 이 값은 namebindings.xml 파일의 구성 위치를 표시합니다. 이 필드는 정보를 위해서만 제공되며 갱신할 수 없습니다.

구성된 바인딩이 셀 범위인 경우, 시작 컨텍스트는 셀 지속 루트 컨텍스트입니다. 구성된 바인딩이 노드 범위인 경우, 시작 컨텍스트는 노드 지속 루트 컨텍스트입니다. 구성된 바인딩이 서버 범위인 경우, 시작 컨텍스트는 서버의 서버 루트 컨텍스트입니다.

바인딩 유형
바인딩 구성의 유형을 표시합니다. 가능한 선택사항은 String, EJB, CORBA 및 Indirect입니다. 이 필드는 정보를 위해서만 제공되며 갱신할 수 없습니다.
바인딩 ID
이 구성 바인딩을 고유하게 식별하는 이름을 지정합니다.
네임 스페이스의 이름
네임 스페이스에서 이 바인딩에 사용되는 이름을 지정합니다. 이 이름은 이 바인딩이 구성되는 네임 스페이스의 위치에 의존하는 단순 또는 복합 이름입니다.
Corbaname URL
오브젝트가 CosNaming 서버에서 바인드되는 장소를 식별하는 CORBA 이름 URL 문자열을 지정합니다.
연합 컨텍스트
대상이 CosNaming 컨텍스트인지(true) 리프 노드 오브젝트인지(false) 지정합니다.
값 결과
true 대상 오브젝트는 CORBA 바인딩 유형에 바인드됩니다. corbaname URL이 NamingContext로 해석되지 않을 경우, 맨 처음 바인딩이 사용될 때 (URL이 맨 처음 해석될 때) 오류가 발생합니다.
false 대상 오브젝트는 오브젝트 CORBA 바인딩 유형과 바인드됩니다.
