CORBA 对象绑定设置

使用此页面配置 CORBA 对象绑定的新名称绑定，或查看或编辑现有 CORBA 对象绑定。

要查看此管理控制台页面，单击环境 > 命名 > 名称空间绑定 > CORBA_namespace_binding。

“配置”选项卡

作用域
显示已配置绑定的作用域。此值表明 namebindings.xml 文件的配置位置。此字段仅用于信息目的，且不能更新。

如果配置的绑定是以单元为作用域，则开始上下文是单元持久根上下文。如果配置的绑定是以节点为作用域，则开始上下文是节点持久根上下文。如果配置的绑定是以服务器为作用域，则开始上下文是服务器的服务器根上下文。

绑定类型
显示已配置绑定的类型。可能的选项是 String、EJB、CORBA 和 Indirect。此字段仅用于信息目的，且不能更新。
绑定标识
指定唯一标识此已配置绑定的名称。
名称空间中的名称
指定用于名称空间中的此绑定的名称。此名称可以是简单名称也可以是复合名称，这取决于配置此绑定的名称空间部分。
Corbaname URL
指定标识对象绑定到 CosNaming 服务器中的位置的 CORBA 名称 URL 字符串。
联合上下文
指定目标是 CosNaming 上下文（真）还是叶节点对象（假）。
值 结果
true 目标对象是用上下文 CORBA 绑定类型绑定的。如果 corbaname URL 未解析到 NamingContext，那么当第一次使用绑定时（这是第一次解析 URL 时）会发生错误。
false 目标对象是用对象 CORBA 绑定类型绑定的。
