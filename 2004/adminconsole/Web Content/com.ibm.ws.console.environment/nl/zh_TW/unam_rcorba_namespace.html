CORBA 物件連結設定

請利用這個頁面來配置 CORBA 物件連結的新名稱連結，或檢視或編輯現有的 CORBA 物件連結。

如果要檢視這個管理主控台頁面，請按一下環境 > 命名 > 名稱空間連結 > CORBA_namespace_binding。

「配置」標籤

範圍
顯示已配置的連結之範圍。 這個值指出 namebindings.xml 檔的配置位置。這個欄位只供參考，無法更新。

如果配置的連結以 Cell 為範圍，起始環境定義就是 Cell 持續性根環境定義。 如果配置的連結不以節點為範圍，起始環境定義就是節點持續性根環境定義。 如果配置的連結以伺服器為範圍，起始環境定義就是伺服器的伺服器根環境定義。

連結類型
已配置的連結之類型。 可能選擇是「字串」、EJB、CORBA 及「間接」。這個欄位只供參考，無法更新。
連結 ID
指定用來唯一識別這個已配置的連結之名稱。
名稱空間中的名稱
指定名稱空間中的這個連結所用的名稱。 隨著這個連結配置所在的名稱空間部分而不同，這個名稱可以是簡式或複合名稱。
Corbaname URL
指定識別 CosNaming 伺服器中用來連結物件的位置之 CORBA 名稱 URL 字串。
聯合環境定義
指定目標是 CosNaming 環境定義 (true) 或葉節點物件 (false)。
值 結果
true 目標物件與環境定義 CORBA 連結類型一起連結。 如果 corbaname URL 無法解析出 NamingContext，第一次使用連結時（也就是第一次解析 URL 時），會發生錯誤。
false 目標物件與物件 CORBA 連結類型一起連結。
