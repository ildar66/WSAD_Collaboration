Visualizzazione del descrittore di distribuzione del client dei servizi Web

Utilizzare questa pagina per visualizzare il descrittore di distribuzione del client.

Per visualizzare questa pagina della console di gestione, completare le seguenti operazioni:

  1. Fare clic su Applicazioni > Applicazioni enterprise > nome_applicazione.
  2. In Elementi correlati, fare clic su Moduli Web > nome_file_URI > Visualizzazione del descrittore di distribuzione del client dei servizi Web.

WebSphere Application Server offre due livelli di collegamenti: a livello dell'applicazione e a livello del server. WebSphere Application Server Network Deployment offre tre livelli di collegamenti: a livello dell'applicazione, a livello del server e a livello della cella. Le informazioni contenute nelle seguenti descrizioni, relative all'implementazione, indicano come configurare i collegamenti a livello dell'applicazione. Per configurare i collegamenti a livello del server, ossia quelli predefiniti, completare le seguenti operazioni:

  1. Fare clic su Server > Server delle applicazioni > nome_server.
  2. In Elementi correlati, fare clic su Servizi Web: collegamenti predefiniti per la sicurezza dei servizi Web (WS-Security, Web Services Security).
  3. Per configurare i collegamenti a livello della cella, fare clic su Sicurezza > Servizi Web.

Nota: Se il server Web funziona come un client, vengono utilizzati i collegamenti predefiniti.

Se si sta utilizzando una qualsiasi delle seguenti configurazioni, verificare che il descrittore di distribuzione sia configurato in maniera appropriata:
  • Firma della richiesta
  • Codifica della richiesta
  • Autenticazione BasicAuth
  • Autenticazione asserzione di identità (ID)
  • Autenticazione asserzione di identità (ID) con la firma TrustMode
  • Verifica della firma digitale di risposta
  • Decodifica risposta

Firma della richiesta

Se sono stati specificati dei vincoli di integrità (firma digitale), verificare di aver configurato le informazioni relative alla firma nei file di collegamento.

Per configurare i parametri di firma, completare le seguenti operazioni:

  1. Fare clic su Applicazioni > Applicazioni enterprise > nome_applicazione.
  2. In Elementi correlati, fare clic su Moduli Web > nome_file_URI Servizi Web: collegamenti sicurezza client.
  3. Nella colonna Collegamento ricevente risposta, fare clic su Modifica > Informazioni sulla firma > Nuovo.

Per configurare gli indicatori di posizione della chiave, completare le seguenti operazioni:

  1. Fare clic su Server > Server delle applicazioni > nome_server.
  2. In Ulteriori proprietà, fare clic su Servizi Web: collegamenti predefiniti per la sicurezza dei servizi Web (WS-Security, Web Services Security) > Indicatore di posizione della chiave.

Autenticazione BasicAuth

Se BasicAuth viene configurato come il token di sicurezza necessario, specificare CallbackHandler nel file di collegamento in modo da raccogliere i dati di autenticazione di base. Il seguente elenco contiene le implementazioni di supporto CallBack:

com.ibm.wsspi.wssecurity.auth.callback.GuiPromptCallbackHandler
L'implementazione richiede informazioni BasicAuth (nome utente e password) in un'interfaccia.
com.ibm.wsspi.wssecurity.auth.callback.NonPromptCallbackHandler
Questa implementazione legge le informazioni su BasicAuth dal file di collegamento.
com.ibm.wsspi.wssecurity.auth.callback.StdPromptCallbackHandler
Questa implementazione richiede un nome utente e una password utilizzando standard in (stdin).

Per configurare le informazioni sul collegamento di login, completare le seguenti operazioni:

  1. Fare clic su Applicazioni > Applicazioni enterprise > nome_applicazione.
  2. In Elementi correlati, fare clic su Modulo Web > nome_file_URI > Servizi Web: collegamenti sicurezza client.
  3. In Collegamenti mittente richiesta, fare clic su Modifica > Collegamento di login.

Autenticazione asserzione d'identità (ID) con BasicAuth TrustMode

Configurare un collegamento di login nel file di collegamenti con un'implementazione com.ibm.wsspi.wssecurity.auth.callback.NonPromptCallbackHandler. Specificare un ID utente e una password BasicjAuth considerati attendibili da un TrustedIDEvaluator su un server downstream.

Per configurare le informazioni sul collegamento di login, completare le seguenti operazioni:

  1. Fare clic su Applicazioni > Applicazioni enterprise > nome_applicazione.
  2. In Elementi correlati, fare clic su Modulo Web > nome_file_URI > Servizi Web: collegamenti sicurezza client.
  3. In Collegamenti mittente richiesta, fare clic su Modifica > Collegamento di login.

Autenticazione asserzione di identità (ID) con la firma TrustMode

Configurare le informazioni relative alla firma nel file di collegamenti con una chiave di firma che indica un indicatore di posizione della chiave. L'indicatore di posizione della chiave contiene il certificato X.509 considerato affidabile dal server downstream.

Per configurare l'asserzione ID, completare le seguenti operazioni:

  1. Fare clic su Server > Server delle applicazioni > nome_server.
  2. In Ulteriori proprietà, fare clic su Servizi Web: collegamenti predefiniti per la sicurezza dei servizi Web (WS-Security, Web Services Security) > Mappature di login > Asserzione ID.

Per configurare le informazioni sul collegamento di login, completare le seguenti operazioni:

  1. Fare clic su Applicazioni > Applicazioni enterprise > nome_applicazione.
  2. In Elementi correlati, fare clic su Modulo Web > nome_file_URI > Servizi Web: collegamenti sicurezza client.
  3. In Collegamenti mittente richiesta, fare clic su Modifica > Collegamento di login.
