Raccolta di configurazioni SSL HTTP

Utilizzare questa pagina per configurare la sicurezza SSL (Secure Sockets Layer) del livello di trasporto. È possibile utilizzare questa configurazione quando un servizio Web funge da client per un altro servizio Web.

È possibile utilizzare la sicurezza del livello di trasporto per abilitare SSL HTTP (o HTTPS). La sicurezza del livello di trasporto può essere abilitata o disabilitata indipendentemente dalla sicurezza del livello di messaggio. Poiché la sicurezza del livello di trasporto fornisce una sicurezza minima, quando la sicurezza è essenziale per l'applicazione dei servizi Web, utilizzare la sicurezza del livello di messaggio.

Per visualizzare questa pagina della console di gestione, completare le seguenti operazioni:

  1. Fare clic su Applicazioni > Applicazioni Enterprise > nome_applicazione.
  2. In Elementi correlati, fare clic su Modulo Web > nome_file_URI > Servizi Web: collegamenti sicurezza client.
  3. In Configurazione SSL HTTP, fare clic su Modifica.

Inoltre, è possibile specificare l'autenticazione di base SSL HTTP completando le seguenti procedure:

  1. Fare clic su Applicazioni > Applicazioni Enterprise > nome_applicazione.
  2. In Elementi correlati, fare clic su Modulo Web > nome_file_URI > Servizi Web: collegamenti sicurezza client.
  3. In Configurazione di base HTTP, fare clic su Modifica.

SSL HTTP abilitato
Selezionare questa opzione per abilitare le comunicazioni socket protette per il trasporto HTTP per questa porta. Se abilitata, WebSphere Application Server utilizza l'impostazione Configurazione SSL HTTP.
Configurazione SSL HTTP
Specifica l'alias della configurazione SSL da utilizzare con il trasporto HTTP per questa porta.

Questa opzione viene utilizzata se si seleziona SSL HTTP abilitato. Gli alias SSL vengono definiti nel repertorio di configurazione SSL (Secure Sockets Layer), che può essere configurato facendo clic su Sicurezza > SSL.
