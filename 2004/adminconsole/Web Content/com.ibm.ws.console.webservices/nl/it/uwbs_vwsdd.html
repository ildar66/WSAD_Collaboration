Visualizzazione del descrittore di distribuzione del server dei servizi Web

Utilizzare questa pagina per visualizzare le impostazioni del descrittore di distribuzione del server.

Per visualizzare questa pagina della console di gestione, completare le seguenti operazioni:

  1. Fare clic su Applicazioni > Applicazioni enterprise > nome_applicazione.
  2. In Elementi correlati, fare clic su Moduli Web > nome_file_URI > Visualizzazione del descrittore di distribuzione del server dei servizi Web.

WebSphere Application Server presenta due livelli di collegamenti: a livello dell'applicazione e a livello del server. WebSphere Application Server Network Deployment presenta tre livelli di collegamenti: a livello dell'applicazione, a livello del server e a livello della cella. Le informazioni contenute nelle seguenti descrizioni relative all'implementazione indicano come configurare i collegamenti a livello dell'applicazione. Per configurare i collegamenti a livello del server, ossia quelli predefiniti, completare le seguenti operazioni:

  1. Fare clic su Server > Server delle applicazioni > nome_server.
  2. In Elementi correlati, fare clic su Servizi Web: collegamenti predefiniti per la sicurezza dei servizi Web (WS-Security, Web Services Security).
Per configurare i collegamenti a livello della cella, fare clic su Sicurezza > Servizi Web.

Verifica della firma digitale della richiesta

Se sono stati definiti i vincoli di integrità (firma necessaria), verificare di aver configurato le informazioni relative alla firma nei file di collegamento.

Per configurare i parametri di firma, completare le seguenti operazioni:

  1. Fare clic su Applicazioni > Applicazioni enterprise > nome_applicazione.
  2. In Elementi correlati, fare clic su Moduli Web > nome_file_URI> Servizi Web: collegamenti sicurezza server.
  3. Nella colonna Collegamento ricevente richiesta, fare clic su Modifica > Informazioni sulla firma.

Per configurare l'ancoraggio sicuro, completare le seguenti operazioni:

  1. Fare clic su Server > Server delle applicazioni > nome_server.
  2. In Ulteriori proprietà, fare clic su Servizi Web: collegamenti predefiniti per la sicurezza dei servizi Web (WS-Security, Web Services Security) > Ancoraggi sicuri.

Per configurare Raccolta archivio certificati, completare le seguenti operazioni:

  1. Fare clic su Server > Server delle applicazioni > nome_server.
  2. In Elementi correlati, fare clic su Servizi Web: collegamenti predefiniti per la sicurezza dei servizi Web (WS-Security, Web Services Security) > Raccolta archivio certificati.

Per configurare gli indicatori di posizione della chiave, completare le seguenti operazioni:

  1. Fare clic su Server > Server delle applicazioni > nome_server.
  2. In Ulteriori proprietà, fare clic su Servizi Web: collegamenti predefiniti per la sicurezza dei servizi Web (WS-Security, Web Services Security) > Indicatore di posizione della chiave.

Decodifica della richiesta

Se sono stati specificati i vincoli di riservatezza (codifica), verificare di aver definito le informazioni sulla codifica.

Per configurare i parametri di informazioni sulla codifica, completare le seguenti operazioni:

  1. Fare clic su Applicazioni enterprise > nome_applicazione.
  2. In Elementi correlati, fare clic su Modulo Web.
  3. In Ulteriori proprietà, fare clic su Servizi Web: collegamenti sicurezza server. In Collegamento ricevente richiesta, fare clic su Modifica > Informazioni di codifica.

Per configurare gli indicatori di posizione della chiave, completare le seguenti operazioni:

  1. Fare clic su Server > Server delle applicazioni > nome_server.
  2. In Ulteriori proprietà, fare clic su Servizi Web: collegamenti predefiniti per la sicurezza dei servizi Web (WS-Security, Web Services Security) > Indicatore di posizione della chiave.

Autenticazione BasicAuth

Se BasicAuth viene configurato come il token di sicurezza necessario, specificare CallbackHandler nel file di collegamento in modo da raccogliere i dati di autenticazione di base. Segue un elenco di implementazioni di supporto CallBack:

com.ibm.wsspi.wssecurity.auth.callback.GuiPromptCallbackHandler
L'implementazione richiede informazioni BasicAuth (nome utente e password) nel pannello di un'interfaccia.
com.ibm.wsspi.wssecurity.auth.callback.NonPromptCallbackHandler
Questa implementazione legge le informazioni su BasicAuth dal file di collegamento.
com.ibm.wsspi.wssecurity.auth.callback.StdPromptCallbackHandler
Questa implementazione richiede un nome utente e una password utilizzando un prompt standard in (stdin).

Per configurare le informazioni sulla mappatura del login, completare le seguenti operazioni:

  1. Fare clic su Server > Server delle applicazioni > nome_server.
  2. In Ulteriori proprietà, fare clic su Servizi Web: collegamenti predefiniti per la sicurezza dei servizi Web (WS-Security, Web Services Security) > Mappature del login.

Autenticazione asserzione identità (ID) con BasicAuth TrustMode

Configurare un collegamento di login nel file di collegamenti con un'implementazione com.ibm.wsspi.wssecurity.auth.callback.NonPromptCallbackHandler. Specificare un ID utente e una password BasicAuth considerati attendibili da un TrustedIDEvaluator su un server downstream.

Per configurare le informazioni sulla mappatura del login, completare le seguenti operazioni:

  1. Fare clic su Server > Server delle applicazioni > nome_server.
  2. In Ulteriori proprietà, fare clic su Servizi Web: collegamenti predefiniti per la sicurezza dei servizi Web (WS-Security, Web Services Security) > Mappature del login.

Autenticazione asserzione di identità (ID) con la firma TrustMode

Configurare le informazioni relative alla firma nel file di collegamenti con una chiave di firma che indica un indicatore di posizione della chiave. L'indicatore di posizione della chiave contiene il certificato X.509 considerato affidabile dal server downstream.

Per configurare le informazioni sulla mappatura del login, completare le seguenti operazioni:

  1. Fare clic su Server > Server delle applicazioni > nome_server.
  2. In Ulteriori proprietà, fare clic su Servizi Web: collegamenti predefiniti per la sicurezza dei servizi Web (WS-Security, Web Services Security) > Mappature del login.

WSLogin è il nome della configurazione di login utilizzato da JAAS (Java Authentication and Authorization Service). Per configurare JAAS, fare clic su Sicurezza > Configurazione JAAS > Login applicazione.

Il valore della tag <TrustedIDEvaluatorRef> nel collegamento deve corrispondere al valore del nome <TrustedIDEvaluator>.

Per configurare i programmi di valutazione ID sicuri, completare le seguenti operazioni:

  1. Fare clic su Server > Server delle applicazioni > nome_server.
  2. In Servizi aggiuntivi, fare clic su Servizi Web: collegamenti predefiniti per la sicurezza dei servizi Web (WS-Security, Web Services Security) > Programmi di valutazione ID sicuri.

Firma di risposta

Se sono stati definiti i vincoli di integrità (firma digitale), verificare di disporre delle informazioni relative alla firma configurate nei file di collegamento.

Per specificare le informazioni sulla firma, completare le seguenti operazioni:

  1. Fare clic su Applicazioni > Applicazioni Enterprise > nome_applicazione.
  2. In Elementi correlati, fare clic su Moduli Web > nome_file_URI > Servizi Web: collegamenti sicurezza server.
  3. Nella colonna Collegamento ricevente richiesta, fare clic su Modifica > Informazioni sulla firma.

Per configurare gli indicatori di posizione della chiave, completare le seguenti operazioni:

  1. Fare clic su Server > Server delle applicazioni > nome_server.
  2. In Ulteriori proprietà, fare clic su Servizi Web: collegamenti predefiniti per la sicurezza dei servizi Web (WS-Security, Web Services Security) > Indicatore di posizione della chiave.
