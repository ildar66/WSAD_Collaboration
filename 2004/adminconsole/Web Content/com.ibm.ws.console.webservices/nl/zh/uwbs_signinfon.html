标记信息配置设置

使用此页面配置新签名参数。

此页面上所列出签名方法、摘要方法和规范方法的规范位于标题为“XMLSignature Syntax and Specification: W3C Recommendation 12 Feb 2002”的 W3C 文档中。

要查看此管理控制台页面，单击企业应用程序 > application_name。在“相关项”下，单击 Web 模块 > URI_file_name > Web 服务：服务器安全性绑定。在“请求接收程序绑定”列中，单击编辑 > 签名信息 > 新建。

单击新建创建一个签名参数。

单击删除删除一个签名参数。

签名方法
指定签名方法的算法 URI。此方法包含使用 base64 编码的数字签名的实际值。

支持下列算法：

  • http://www.w3.org/2000/09/xmldsig#rsa-sha1
  • http://www.w3.org/2000/09/xmldsig#dsa-sha1

摘要方法
指定摘要方法的算法 URI。

支持 http://www.w3.org/2000/09/xmldsig#sha1 算法。

规范方法
指定规范方法的算法 URI。

支持下列算法：

  • http://www.w3.org/2001/10/xml-exc-c14n#
  • http://www.w3.org/2001/10/xml-exc-c14n#WithComments
  • http://www.w3.org/TR/2001/REC-xml-c14n-20010315
  • http://www.w3.org/TR/2001/REC-xml-c14n-20010315#WithComments

签名密钥
指定用于签名的密钥信息。这些字段在接收程序方配置忽略。

如果您指定密钥名和密钥定位器引用，为“证书路径”选择无。

证书路径
指定证书路径验证的设置。当您选择信任任何时，跳过此验证，并信任所有进入证书。这些字段在发送程序方配置忽略。

如果您单击信任任何，或选择认证锚和证书存储，则为前一字段中的“签名密钥”选择无。

信任锚

通过单击服务器 > 应用程序服务器 > server_name 指定可用于信任锚的选择。在“其它属性”下，单击 Web 服务：Web 服务安全性的缺省绑定 > 信任锚。

证书存储

通过单击服务器 > 应用程序服务器 > server_name 指定可用于集合存储的选择。在“相关项”下，单击 Web 服务：Web 服务安全性的缺省绑定 > 集合证书存储。
