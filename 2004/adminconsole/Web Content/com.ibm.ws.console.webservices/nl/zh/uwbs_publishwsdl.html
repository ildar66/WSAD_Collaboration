发布 WSDL 文件设置

使用此页面发布 Web 服务描述性语言（WSDL）文件。

要查看此页面，单击应用程序 > 企业应用程序 > application_instance > 发布 WSDL。

当您单击确定时，产生一个应用程序中所有已启用 Web 服务的模型的 zip 文件。已发布 zip 文件名为 application_name_WSDLFiles.zip。在已发布 zip 文件中，目录结构为 application_name/module_name/[META-INF|WEB-INF]/wsdl/wsdl_file_name。

在已发布 WSDL 文件中，服务 soap:address 节的位置属性包含通过其访问 Web 服务的 URL。您可指定在每个模块中为 Web 服务使用的 URl 部分。您可通过 HTTP 传输和／或 JMS 传输在模块中访问 Web 服务。您可为两种传输类型指定 URL 信息。

“配置”选项卡

指定 Web 服务的 URL 前缀
指定用于 URL 的协议（http 或 https）、主机名和端口号。

URL 前缀格式为：protocol://host_name:port_number，例如，http://myHost:9045。已发布 WSDL 文件中出现的实际 URL 由附加到模块的上下文根和 Web 服务 url 模式的前缀组成，例如 http://myHost:9045/services/myService。

选择 HTTP URL 前缀
指定与 URL 前缀的缺省列表相关联的下拉列表。此列表是模块的虚拟主机的端口集和模块的应用程序服务器的端口集的逻辑乘。如果直接访问 Web 服务应用程序服务器，则使用此列表中的项。

要设置 HTTP 前缀，选择 HTTP URL 前缀或定制 HTTP URL 前缀，输入值，选择要使用前缀的模块的复选框并单击应用。当您单击应用时，在选择 HTTP URL 前缀或定制 HTTP URL 前缀字段中的条目（这取决于您的选择）被复制到任何选择了其复选框（在最左列）的模块的 HTTP URL 前缀字段。不应用 HTTP 前缀到 JMS URL 前缀列中的字段。

定制 HTTP URL 前缀
如果通过中间节点（例如 Web Service 网关或 IHS 服务器）访问模块中的 Web 服务，指定中间服务的协议、主机和端口号。

要设置 HTTP 前缀，选择 HTTP URL 前缀或定制 HTTP URL 前缀，输入值，选择要使用前缀的模块的复选框并单击应用。当您单击应用时，在选择 HTTP URL 前缀或定制 HTTP URL 前缀字段中的条目（这取决于您的选择）被复制到任何选择了其复选框（在最左列）的模块的 HTTP URL 前缀字段。不应用 HTTP 前缀到 JMS URL 前缀列中的字段。

JMS URL 前缀
指定用于每个模块的 JMS URL 前缀字符串。

指定的 URL 前缀必须包含目标和 connectionFactory 属性。它可包含其它属性－值对，但是它不能包含 targetService 属性，其由系统在创建已发布 WSDL 文件时添加。JMS URL 前缀的格式为 jms:/[queue&topic]?destination=target_queue_or_topic_jndi_name&connectionFactory=factory_jndi_name, for example, jms:/queue?destination=jms/Q1&connectionFactory=jms/QCF。 已发布 WSDL 文件中出现的实际 URL 由附加到 Web 服务 targetService 的前缀组成，例如 jms:/queue?destination=jms/Q1&connectionFactory=jms/QCF&targetService=StockQuote。
