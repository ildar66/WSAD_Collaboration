登入連結配置設定

請利用這個頁面來配置加密和解密參數。

可外掛的記號利用 Java 鑑別和授權服務 (JAAS) CallBackHandler (javax.security.auth.callback.CallBackHandler) 介面來產生插入訊息的記號。 下列清單說明回呼支援實作：

com.ibm.wsspi.wssecurity.auth.callback.BinaryTokenCallback
這個實作用來產生插入為訊息中的 <wsse:BinarySecurityToken/@ValueType> 的二進位記號。
javax.security.auth.callback.NameCallback and javax.security.auth.callback.NameCallback
這個實作用來產生插入為訊息中的 <wsse:UsernameToken> 的使用者名稱記號。
com.ibm.wsspi.wssecurity.auth.callback.XMLTokenSenderCallback
這個實作用來產生可延伸標記語言 (XML) 記號，且插入為訊息中的 <SAML: Assertion> 元素。
com.ibm.wsspi.wssecurity.auth.callback.PropertyCallback
這個實作用來取得連結檔中所指定的內容。

如果要檢視這個管理主控台頁面，請完成下列步驟：

  1. 按一下應用程式 > 企業應用程式 > application_name。
  2. 在「相關項目」之下，按一下 Web 模組 > URI_file_name > Web 服務：用戶端安全連結。
  3. 在「要求傳送端連結」之下，按一下 編輯 > 登入連結。

如果沒有可用的加密資訊，請選取無。

如果有可用的加密資訊，請選取專用登入連結，在下列欄位中指定配置：

鑑別方法
指定鑑別方法的唯一名稱。
回呼處理常式
指定回呼處理常式的名稱。回呼處理常式必須實作 javax.security.auth.callback.CallbackHandler 介面。
基本鑑別使用者 ID
指定基本鑑別的使用者名稱。「基礎鑑別」這個鑑別方法提供在連結檔中定義使用者 ID 和密碼的功能。
基礎鑑別密碼
指定基本鑑別的密碼。
記號類型 URI
指定記號類型的統一資源識別碼 (URI)。 這項資訊插入為 XML 記號 <SAML: Assertion> 的 <wsse:BinarySecurityToken>/ValueType。
記號類型本端名稱
指定記號類型的本端名稱。 這個資訊插入為 WXL 記號 <SAML: Assertion> 的 <wsse:BinarySecurityToken>/ValueType。
