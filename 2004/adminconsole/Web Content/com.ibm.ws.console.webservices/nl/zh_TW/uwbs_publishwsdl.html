發佈 WSDL 檔設定

請利用這個頁面來發佈 Web 服務說明語言 (WSDL) 檔。

如果要檢視這個頁面，請按一下應用程式 > 企業應用程式 > application_instance > 發佈 WSDL。

當您按一下確定時，會產生應用程式中所有啟用 Web 服務的模組之 ZIP 檔。 發佈的 ZIP 檔的名稱是 application_name_WSDLFiles.zip。 在發佈的 ZIP 檔中，目錄結構是 application_name/module_name/[META-INF|WEB-INF]/wsdl/wsdl_file_name。

在發佈的 WSDL 檔中，服務 soap:address 段落的位置屬性含有用來存取 Web 服務的 URL。 您可以指定要用於每個模組中的各 Web 服務之 URL 部分。 您可以利用 HTTP 傳輸和/或 JMS 傳輸來存取模組中的 Web 服務。 您可以指定這兩類傳輸的 URL 資訊。

「配置」標籤

指定 Web 服務的 URL 字首
指定 URL 中要用的 protocol（http 或 https）、host_name 和 port_number。

URL 字首格式是 protocol://host_name:port_number，如 http://myHost:9045。 出現在發佈的 WSDL 檔中之實際 URL 由附加至模組環境定義起始目錄的字首和 Web 服務 URL 型樣組成，如 http://myHost:9045/services/myService。

選取 HTTP URL 字首
指定預設 URL 字首清單的相關下拉清單。 這份清單是模組虛擬主機埠集和模組應用程式伺服器埠集的交集。 如果直接存取 Web 服務應用程式，請使用這份清單中的項目。

如果要設定 HTTP 字首，您可以選取 HTTP URL 字首或自訂 HTTP URL 字首，輸入值，選取使用這個字首之模組的勾選框，再按一下套用。 當您按一下套用時，會依所選項目而定，將選取 HTTP URL 字首或自訂 HTTP URL 字首欄位中的項目複製到選取了勾選框（在最左側直欄）的任何模組的 HTTP URL 字首欄位中。HTTP 字首不會套用在 JMS URL 字首直欄中的各欄位上。

自訂 HTTP URL 字首
如果是利用中間節點（如 Web 服務閘道或 IHS 伺服器）存取模組中的 Web 服務，便指定中間服務的 protocol、host 和 port_number。

如果要設定 HTTP 字首，您可以選取 HTTP URL 字首或自訂 HTTP URL 字首，輸入值，選取使用這個字首之模組的勾選框，再按一下套用。 當您按一下套用時，會依所選項目而定，將選取 HTTP URL 字首或自訂 HTTP URL 字首欄位中的項目複製到選取了勾選框（在最左側直欄）的任何模組的 HTTP URL 字首欄位中。HTTP 字首不會套用在 JMS URL 字首直欄中的各欄位上。

JMS URL 字首
指定每個模組所用的 JMS URL 字首字串。

指定的 URL 字首必須含有 destination 和 connectionFactory 內容。 它可以含有其他內容值組，但不能含有發佈的 WSDL 檔建立時所加入的 targetService 內容。JMS URL 字首的格式是 jms:/[queue&topic]?destination=target_queue_or_topic_jndi_name&connectionFactory=factory_jndi_name, for example, jms:/queue?destination=jms/Q1&connectionFactory=jms/QCF。出現在發佈的 WSDL 檔中之實際 URL 由附加至 Web 服務 targetService 的字首組成，如 jms:/queue?destination=jms/Q1&connectionFactory=jms/QCF&targetService=StockQuote。
