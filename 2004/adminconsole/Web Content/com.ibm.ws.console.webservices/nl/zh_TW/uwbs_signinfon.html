簽章資訊配置設定

請利用這個頁面來配置新的簽章參數。

這個頁面列出的簽章方法、摘要方法和標準化方法規格都在 W3C 文件中，文件標題為 "XMLSignature Syntax and Specification: W3C Recommendation 12 Feb 2002"。

如果要檢視這個管理主控台頁面，請按一下企業應用程式 > application_name。 在「相關項目」之下，按一下 Web 模組 > URI_file_name > Web 服務：伺服器安全連結。 在「要求接收端連結」直欄中，按一下編輯 > 簽章資訊 > 新建。

按一下新建來建立簽章參數。

按一下刪除來刪除簽章參數。

簽章方法
指定簽章方法的演算法 URI。 這個方法包含利用 base64 來編碼的數位簽章值。

以下是支援的演算法：

  • http://www.w3.org/2000/09/xmldsig#rsa-sha1
  • http://www.w3.org/2000/09/xmldsig#dsa-sha1

摘要方法
指定摘要方法的演算法 URI。

支援 http://www.w3.org/2000/09/xmldsig#sha1 演算法。

標準化方法
指定標準化方法的演算法 URI。

以下是支援的演算法：

  • http://www.w3.org/2001/10/xml-exc-c14n#
  • http://www.w3.org/2001/10/xml-exc-c14n#WithComments
  • http://www.w3.org/TR/2001/REC-xml-c14n-20010315
  • http://www.w3.org/TR/2001/REC-xml-c14n-20010315#WithComments

簽章金鑰
指定簽章所用的金鑰資訊。 在接收端配置中，會忽略這些欄位。

如果您指定金鑰名稱和金鑰定位器參照，請在「憑證路徑」中，選取無。

憑證路徑
指定憑證路徑驗證的設定。 當您選取信任任何人時，會跳過這個驗證，並信任所有送入憑證。在傳送端配置中，會忽略這些欄位。

如果您按一下信任任何人或選取信任錨點和憑證儲存庫，請在先前欄位中，選取「簽章金鑰」為無。

信任錨點

信任錨點所能使用的選項是藉由按一下伺服器 > 應用程式伺服器 > server_name 來指定的。 在「其他內容」之下，按一下 Web 服務：Web 服務安全的預設連結 > 信任錨點。

憑證儲存庫

集合儲存庫所能使用的選項是藉由按一下伺服器 > 應用程式伺服器 > server_name 來指定的。 請在「相關項目」之下，按一下 Web 服務：Web 服務安全的預設連結 > 集合憑證儲存庫。
