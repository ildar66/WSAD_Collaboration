Web 服務：用戶端安全連結集合

請利用這個頁面來檢視 Web 服務安全的用戶端連結配置清單。 當 Web 服務是另一個 Web 服務的用戶端時，會使用這些連結。

如果要檢視這個管理主控台頁面，請完成下列步驟：

  1. 按一下企業應用程式 >application_name。
  2. 在「相關項目」之下，按一下 Web 模組 > URI_file_name > Web 服務：用戶端安全連結。

埠
指定用來傳送和接收伺服器發出的訊息的埠。
Web 服務
指定 Web 服務的名稱。
要求傳送端連結
指定用來將要求訊息傳送給要求接收端的連結配置。

請按一下編輯來配置要求傳送端的簽章資訊、加密資訊和登入連結，以及檢視金鑰儲存庫中的金鑰定位器清單。

指定給用戶端的要求傳送端連結資訊必須符合指定給伺服器的要求接收端連結資訊。

回應接收端連結
指定用來接收回應傳送端發出的回應訊息之連結配置。

請按一下編輯來配置簽章和加密資訊，以及檢視信任錨點、在集合憑證儲存庫中找到的中間憑證，以及回應接收端的金鑰定位器物件的清單。

指定給用戶端的回應要求端連結資訊必須符合指定給伺服器的回應傳送端連結資訊。

HTTP 基本鑑別
指定要搭配 HTTP 傳輸層基本鑑別供這個埠使用的使用者 ID 和密碼。 您可以在訊息層安全之外，獨立啟用傳輸層鑑別安全。

按一下編輯來配置傳輸層鑑別的基本鑑別 ID 和密碼。

HTTP SSL 配置
啟用和配置這個埠的傳輸層 SSL 安全。 您可以在訊息層次安全之外，獨立啟用傳輸層 SSL 安全。

請按一下編輯來指定這個埠的傳輸層 HTTP SSL 配置的設定。
