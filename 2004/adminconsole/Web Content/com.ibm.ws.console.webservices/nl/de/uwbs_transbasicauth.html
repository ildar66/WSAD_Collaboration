HTTP-Basisauthentifizierung

Verwenden Sie diese Seite, um eine Benutzer-ID und ein Kennwort für die Sicherheit der Basisauthentifizierung auf Transportebene an diesem Port festzulegen. Sie können diese Konfiguration verwenden, wenn ein Webservice ein Client eines anderen Webservice ist.

Sie können die Sicherheit auf Transportebene verwenden, um die Basisauthentifizierung zu aktivieren. Die Sicherheit auf Transportebene kann unabhängig von der Sicherheit auf Nachrichtenebene aktiviert und inaktiviert werden. Weil die Sicherheit auf Transportebene den niedrigsten Sicherheitsgrad bietet, sollten Sie die Sicherheit auf Nachrichtenebene verwenden, falls eine Webservice-Anwendung unbedingt geschützt werden muss.

Gehen Sie zum Anzeigen dieser Seite der Administrationskonsole wie folgt vor:

  1. Klicken Sie auf Anwendungen > Enterprise-Anwendungen > Anwendungsname.
  2. Klicken Sie unter "Zugehörige Elemente" auf Webmodule > Name_der_URI-Datei > Web Services: Sicherheitsbindungen des Clients.
  3. Klicken Sie unter "HTTP-Basisauthentifizierung" auf Bearbeiten.

Sie können die HTTP-SSL-Basisauthentifizierung auch wie folgt festlegen:

  1. Klicken Sie auf Anwendungen > Enterprise-Anwendungen > Anwendungsname.
  2. Klicken Sie unter "Zugehörige Elemente" auf Webmodule > Name_der_URI-Datei > Web Services: Sicherheitsbindungen des Clients.
  3. Klicken Sie unter "HTTP-SSL-Konfiguration" auf Bearbeiten.

ID für Basisauthentifizierung
Die Benutzer-ID für die HTTP-Basisauthentifizierung an diesem Port.
Kennwort für Basisauthentifizierung
Das Kennwort für die HTTP-Basisauthentifizierung an diesem Port.
