Trusted ID Evaluator

Verwenden Sie diese Seite, um eine Liste der Trusted ID Evaluator anzuzeigen. Der Trusted ID Evaluator bestimmt, ob eine identitätszusichernde Instanz als vertrauenswürdig eingestuft wird. Wenn die ID als vertrauenswürdig eingestuft wird, generiert WebSphere Application Server die erforderlichen Berechtigungsnachweise, die in Downstream-Aufrufen von Ressourcen verwendet werden. Der Trusted ID Evaluator implementiert das Interface com.ibm.wsspi.wssecurity.id.TrustedIDEvaluator.

Gehen Sie zum Anzeigen dieser Seite der Administrationskonsole wie folgt vor:

  1. Klicken Sie auf Server > Anwendungsserver > Servername.
  2. Klicken Sie unter "Weitere Merkmale" auf Web Services: Standardbindungen für die Sicherheit von Web Services > Trusted ID Evaluator.

Klicken Sie auf Neu, um einen neuen Trusted ID Evaluator zu erstellen.

Klicken Sie auf Löschen, um einen Trusted ID Evaluator zu löschen.

Führen Sie in dieser Anzeige die folgenden Schritte aus:

  1. Geben Sie einen Namen für den Trusted ID Evaluator und den Namen einer Trusted-ID-Evaluator-Klasse ein.
  2. Speichern Sie Ihre Änderungen. Klicken Sie dazu oben in der Administrationskonsole auf Speichern. Nachdem Sie auf Speichern geklickt haben, erscheint die Hauptanzeige der Administrationskonsole.
  3. Kehren Sie in die Anzeige "Trusted ID Evaluator" zurück und aktualisieren Sie die Laufzeit der Web-Services-Sicherheit mit den Daten der Standardbindungen, die Sie in der Datei ws_security.xml finden.
  4. Klicken Sie auf Laufzeit aktualisieren. Die an anderen Webservices vorgenommenen Konfigurationsänderungen werden ebenfalls in der Laufzeit der Web-Services-Sicherheit aktualisiert.
