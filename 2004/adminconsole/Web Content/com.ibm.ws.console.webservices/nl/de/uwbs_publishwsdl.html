Einstellungen der WSDL-Dateien veröffentlichen

Verwenden Sie diese Seite, um WSDL-Dateien (Web Services Description Language) zu veröffentlichen.

Klicken Sie zum Anzeigen dieser Seite auf Anwendungen >Enterprise-Anwendungen > Anwendungsinstanz > WSDL veröffentlichen.

Wenn Sie auf OK klicken, wird eine Zip-DAtei mit allen für Web Services aktivierten Modulen in der Anwendung erzeugt. Der Name der veröffentlichten Zip-Datei ist Anwendungsname_WSDLFiles.zip. In der veröffentlichten Zip-Datei ist die Verzeichnisstruktur wie folgt: Anwendungsname/Modulname/[META-INF|WEB-INF]/wsdl/Name_der_WSDL-Datei.

In einer veröffentlichten WSDL-Datei enthält das Verzeichnisattribut einer Servicezeilengruppe soap:address den URL, über den auf den Webservice zugegriffen wird. Sie können den Teil des URL angeben, der für die Webservices in den Modulen verwendet werden soll. Mögliche Arten für den Zugriff auf die Webservices in einem Modul sind HTTP-Transport und/oder JMS-Transport. Sie können URL-Informationen für beide Transporttypen angeben.

Register "Konfiguration"

URL-Präfixe für Web Services
Gibt das Protokoll (http oder https), den Hostnamen und die Port-Nummer an, die im URL verwendet werden sollen.

Das Format des URL-Präfix ist Protokoll://Hostname:Port-Nummer, z. B. http://meinHost:9045. Der echte URL, der in einer veröffentlichten WSDL-Datei angezeigt wird, enthält das Präfix, das dem Stammkontext des Moduls und dem URL-Muster des Webservice vorangestellt ist, z. B. http://meinHost:9045/services/meinService.

URL-Präfix für HTTP auswählen
Die Dropdown-Liste, die einer Standardliste von URL-Präfixen zugeordnet ist. Diese Liste ist der Schnittpunkt zwischen den Ports für den virtuellen Host des Moduls und den Ports für den Anwendungsserver des Moduls. Verwenden Sie die Einträge in der Liste, wenn Sie direkt auf die Web-Services-Anwendung zugreifen.

Wählen Sie zum Festlegen eines HTTP-Präfix HTTP-URL-Präfix oder Benutzerdefiniertes HTTP-URL-Präfix aus, geben Sie den Wert ein, wählen Sie die Markierungsfelder der Module aus, von denen dieses Präfix zu verwenden ist, und klicken Sie anschließend auf Anwenden. Wenn Sie auf Anwenden klicken, wird der Eintrag aus dem Feld HTTP-URL-Präfix bzw. Benutzerdefiniertes HTTP-URL-Präfix in das Feld HTTP-URL-Präfix der Module kopiert, deren Markierungsfeld (in der Spalte ganz links) ausgewählt wurde. Das HTTP-Präfix wird nicht auf die Felder in der Spalte "JMS-URL-Präfix" angewendet.

Benutzerdefiniertes HTTP-URL-Präfix
Gibt das Protokoll, den Host und die Port-Nummer des temporären Service an, wenn die Webservices in einem Modul über einen temporären Knoten aufgerufen werden, z. B. über den Web Services Gateway oder einen IHS-Server.

Wählen Sie zum Festlegen eines HTTP-Präfix HTTP-URL-Präfix oder Benutzerdefiniertes HTTP-URL-Präfix aus, geben Sie den Wert ein, wählen Sie die Markierungsfelder der Module aus, von denen dieses Präfix zu verwenden ist, und klicken Sie anschließend auf Anwenden. Wenn Sie auf Anwenden klicken, wird der Eintrag aus dem Feld HTTP-URL-Präfix bzw. Benutzerdefiniertes HTTP-URL-Präfix in das Feld HTTP-URL-Präfix der Module kopiert, deren Markierungsfeld (in der Spalte ganz links) ausgewählt wurde. Das HTTP-Präfix wird nicht auf die Felder in der Spalte "JMS-URL-Präfix" angewendet.

JMS-URL-Präfix
Der für jedes Modul zu verwendende JMS-URL-Präfix.

Das angegebene URL-Präfix muss die Merkmale Zieladresse und Verbindungs-Factory enthalten. Es kann auch weitere Merkmal/Wert-Paare enthalten. Das Merkmal targetService darf jedoch nicht im Präfix enthalten sein. Dieses Merkmal wird vom System hinzufügt, wenn die veröffentlichten WSDL-Dateien erstellt sind. Das Format des Präfix für JMS-URLs ist jms:/[queue&topic]?destination=JNDI-Name_der_Zielwarteschlange_bzw_Topics&connectionFactory=JNDI-Name_der_Factory, z. B. jms:/queue?destination=jms/Q1&connectionFactory=jms/QCF. Der echte URL, der in einer veröffentlichten WSDL-Datei erscheint, setzt sich aus dem Zielservice des Webservice (targetService) und dem vorangestellten Präfix zusammen, z. B. "jms:/queue?destination=jms/Q1&connectionFactory=jms/QCF&targetService=StockQuote".
