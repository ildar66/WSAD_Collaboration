Web Services: Sicherheitsbindungen des Clients

Verwenden Sie diese Seite, um eine Liste der clientseitigen Bindungskonfigurationen für die Web-Services-Sicherheit anzuzeigen. Diese Bindungen werden verwendet, wenn ein Webservice ein Client eines anderen Webservice ist.

Gehen Sie zum Anzeigen dieser Seite der Administrationskonsole wie folgt vor:

  1. Klicken Sie auf Enterprise-Anwendungen >Name_der_Anwendung.
  2. Klicken Sie unter "Zugehörige Elemente" auf Webmodule > Name_der_URI-Datei > Web Services: Sicherheitsbindungen des Clients.

Port
Der Port, der zum Senden und Empfangen von Nachrichten eines Servers verwendet wird.
Webservice
Der Name des Webservice.
Bindung für Anforderungssender
Die Bindungskonfiguration, die zum Senden von Anforderungsnachrichten an den Anforderungsempfänger verwendet wird.

Klicken Sie auf Bearbeiten, um die Signaturdaten, die Verschlüsselungsdaten und die Anmeldebindungen für den Anforderungssender zu konfigurieren und eine Liste der Key Locator im Keystore anzuzeigen.

Die Bindungsdaten für den Anforderungssender, die für den Client angegeben werden, müssen mit den Bindungsdaten für den Anforderungsempfänger, die für den Server angegeben werden, übereinstimmen.

Bindung für Antwortempfänger
Die Bindungskonfiguration, die für den Empfang von Antwortnachrichten vom Antwortsender verwendet wird.

Klicken Sie auf Bearbeiten, um die Signatur- und Verschlüsselungsdaten zu konfigurieren und eine Liste der Trust Anchor, temporären Zertifikate im Collection Certificate Store und der Key-Locator-Objekte für den Antwortempfänger anzuzeigen.

Die Bindungsdaten für den Antwortempfänger, die für den Client angegeben werden, müssen mit den Bindungsdaten für den Antwortsender, die für den Server angegeben werden, übereinstimmen.

HTTP-Basisauthentifizierung
Die Benutzer-ID/Kennwort-Kombination, die bei der HTTP-Basisauthentifizierung auf Transportebene für diesen Port zu verwenden ist. Sie können die Authentifizierungssicherheit auf Transportebene unabhängig von der Sicherheit auf Nachrichtenebene aktivieren.

Klicken Sie auf Bearbeiten, um die ID und das Kennwort für die Basisauthentifizierung auf Transportebene zu konfigurieren.

HTTP-SSL-Konfiguration
Aktiviert und konfiguriert die SSL-Sicherheit auf Transportebene für diesen Port. Sie können die SSL-Sicherheit auf Transportebene unabhängig von der Sicherheit auf Nachrichtenebene aktivieren.

Klicken Sie auf Bearbeiten, um die Einstellungen für die HTTP-SSL-Konfiguration auf Transportebene für diesen Port festzulegen.
