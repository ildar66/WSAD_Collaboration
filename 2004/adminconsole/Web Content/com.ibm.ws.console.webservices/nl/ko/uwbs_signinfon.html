서명 정보 구성 설정

이 페이지를 참조하여 새 서명 정보를 구성하십시오.

서명 메소드, 요약 메소드 및 정규 메소드에 대해 이 페이지에 나열된 스펙은 "XMLSignature Syntax and Specification: W3C Recommendation 12 Feb 2002" W3C 문서에 있습니다.

이 관리 콘솔 페이지를 보려면 엔터프라이즈 응용프로그램 > application_name을 클릭하십시오. 관련 항목 아래에서 웹 모듈 > URI_file_name > 웹 서비스: 서버 보안 바인딩을 클릭하십시오. 요청 수신자 바인딩 열에서 편집 > 서명 정보 > 새로 작성을 클릭하십시오.

새로 작성을 클릭하여 서명 매개변수를 작성하십시오.

삭제를 클릭하여 서명 매개변수를 삭제하십시오.

서명 메소드
서명 메소드의 알고리즘 URI를 지정합니다. 이 메소드는 base64를 사용하여 인코딩된 디지털 서명의 실제 값을 포함합니다.

다음은 지원되는 알고리즘입니다.

  • http://www.w3.org/2000/09/xmldsig#rsa-sha1
  • http://www.w3.org/2000/09/xmldsig#dsa-sha1

요약 메소드
요약 메소드의 알고리즘 URI를 지정합니다.

http://www.w3.org/2000/09/xmldsig#sha1 알고리즘은 지원됩니다.

정규 메소드
정규 메소드의 알고리즘 URI를 지정합니다.

다음은 지원되는 알고리즘입니다.

  • http://www.w3.org/2001/10/xml-exc-c14n#
  • http://www.w3.org/2001/10/xml-exc-c14n#WithComments
  • http://www.w3.org/TR/2001/REC-xml-c14n-20010315
  • http://www.w3.org/TR/2001/REC-xml-c14n-20010315#WithComments

서명 키
서명에 사용되는 키 정보를 지정합니다. 수신자측 구성에서는 이들 필드가 무시됩니다.

키 이름 및 키 위치 지정자 참조를 지정하는 경우, 인증 경로에 없음을 선택하십시오.

인증 경로
인증 경로 유효성 검증을 위한 설정을 지정합니다. 모두 신뢰를 선택하면, 이 유효성 검증이 생략되고 모든 수신 인증을 신뢰합니다. 송신자측 구성에서는 이들 필드가 무시됩니다.

모두 신뢰를 클릭하거나 신뢰 앵커 및 인증 저장을 선택하는 경우, 이전 필드에서 서명 키에 없음을 선택하십시오.

신뢰 앵커

신뢰 앵커에 사용 가능한 선택사항은 서버 > Application Server > server_name을 클릭하여 지정됩니다. 추가 등록 정보 아래에서 웹 서비스: 웹 서비스 보안을 위한 기본 바인딩 > 신뢰 앵커를 클릭하십시오.

인증 저장

콜렉션 저장에 사용 가능한 선택사항은 서버 > Application Server > server_name을 클릭하여 지정됩니다. 관련 항목 아래에서 웹 서비스: 웹 서비스 보안을 위한 기본 바인딩 > 콜렉션 인증 저장을 클릭하십시오.
