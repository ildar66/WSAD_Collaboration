HTTP SSL 구성 콜렉션

이 페이지를 참조하여 전송-레벨 SSL(Secure Socket Layer) 보안을 구성하십시오. 웹 서비스가 다른 웹 서비스에 대한 클라이언트일 때 이 구성을 사용할 수 있습니다.

전송 레벨 보안을 사용하여 HTTP SSL(또는 HTTPS)이 사용 가능하도록 설정할 수 있습니다. 전송-레벨 보안은 메시지 레벨 보안에서 사용 가능 또는 사용 불가능 상태로 만들 수 있습니다. 전송 레벨 보안은 최소 보안을 제공하기 때문에, 웹 서비스 응용프로그램에 보안이 필수적인 경우 메시지 레벨 보안을 사용하십시오.

이 관리 콘솔 페이지를 보려면, 다음 단계를 완료하십시오.

  1. 응용프로그램 > 엔터프라이즈 응용프로그램 > application_name을 클릭하십시오.
  2. 관련 항목 아래에서 웹 모듈 > URI_file_name> 웹 서비스: 클라이언트 보안 바인딩을 클릭하십시오.
  3. HTTP SSL 구성 아래에서 편집을 누르십시오.

다음 단계를 완료하여 HTTP SSL 기본 인증을 지정할 수도 있습니다.

  1. 응용프로그램 > 엔터프라이즈 응용프로그램 > application_name을 클릭하십시오.
  2. 관련 항목 아래에서 웹 모듈 > URI_file_name> 웹 서비스: 클라이언트 보안 바인딩을 클릭하십시오.
  3. HTTP 기본 구성 아래에서, 편집을 클릭하십시오.

HTTP SSL 사용 가능
이 포트의 HTTP 전송에 보안 소켓 통신이 사용 가능하도록 설정하려면 이 옵션을 선택하십시오. WebSphere Application Server가 사용 가능한 경우 HTTP SSL Configuration 설정을 사용합니다.
HTTP SSL 구성
이 포트의 HTTP 전송에 사용할 SSL 구성의 별명을 지정합니다.

HTTP SSL 사용 가능을 선택할 경우 이 옵션이 사용됩니다. SSL 별명은 보안 > SSL을 클릭하여 구성할 수 있는 Secure Sockets Layer 구성 레퍼터리에 정의됩니다.

관련 정보

웹 서비스: 웹 서비스 보안 콜렉션을 위한 기본 바인딩
웹 서비스: 클라이언트 보안 바인딩 콜렉션
웹 서비스: 서버 보안 바인딩 콜렉션
