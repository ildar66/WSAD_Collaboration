Raccolta di server

Utilizzare questa pagina per gestire l'applicazione e i server JMS e visualizzarne le informazioni correlate.

Server delle applicazioni

La pagina Server delle applicazioni elenca i server delle applicazioni nella cella e i nodi che li contengono.

Il prodotto Network Deployment visualizza inoltre lo stato dei server delle applicazioni. Lo stato indica se un server è in esecuzione, è arrestato o ha riscontrato dei problemi.

Per visualizzare questa pagina della console di gestione, fare clic su Server > Server delle applicazioni.

Server JMS

Ogni server JMS fornisce le funzioni del provider JMS per un nodo nel dominio di gestione. Nel dominio di gestione può esistere al massimo un server JMS su ogni nodo e il server delle applicazioni, all'interno del dominio, può accedere alle risorse JMS, supportate dai server JMS su qualsiasi nodo nel dominio.

Per visualizzare questa pagina della console di gestione, fare clic su Server > Server JMS.

Nome
Specifica un nome logico del server. I nomi server all'interno di un nodo devono essere univoci.
Nodo
Specifica il nome del nodo del server delle applicazioni.
Stato  
Indica se lo stato del server delle applicazioni è avviato o arrestato. (Solo Network Deployment)

Notare che, se lo stato è Non disponibile, l'agente nodo non è in esecuzione nel nodo specificato e sarà necessario riavviare l'agente nodo stesso prima di avviare il server.
