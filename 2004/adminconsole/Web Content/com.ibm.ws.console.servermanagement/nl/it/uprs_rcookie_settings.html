Impostazioni cookie

Utilizzare questa pagina per configurare le impostazioni cookie per la gestione delle sessioni.

Per visualizzare questa pagina della console di gestione, fare clic su Server > Server delle applicazioni > nome_server > Contenitore Web > Gestione sessioni > Abilita i cookie.

Scheda Configurazione

Nome cookie
Specifica un nome univoco per il cookie della gestione sessioni. La specifica del servlet richiede il nome JSESSIONID. Tuttavia, per una maggiore flessibilità, è possibile configurare tale valore.
Cookie protetti
Specifica che i cookie di sessione includono il campo protetto. L'abilitazione della funzione limita lo scambio di cookie alle sole sessioni HTTPS.
Dominio cookie
Specifica il campo del dominio di un cookie di traccia sessioni. Questo valore controlla se un browser invia un cookie a particolari server. Ad esempio, se si specifica un dominio particolare, i cookie di sessione verranno inviati agli host in quel dominio. Il dominio predefinito è il server.
Percorso cookie
Specifica che un cookie viene inviato all'URL specificato nel percorso. Specificare ogni stringa che rappresenta un percorso sul server. "/" indica la directory root. Specificare un valore per limitare i percorsi su cui il cookie verrà inviato. Limitando i percorsi, è possibile impedire al cookie di essere inviato a determinati URL sul server. Se si specifica la directory root, il cookie viene inviato a prescindere dal percorso a cui si accede sul server specificato.
Durata massima del cookie
Specifica la durata dell'attività del cookie nel browser del client. Specificare che la durata del cookie deve corrispondere a quella della sessione browser corrente o ad una durata massima. Se viene selezionata l'opzione di durata massima, specificare la durata in secondi. Questo valore corrisponde alla TTL (Time to Live/Durata dell'attività) descritta nella specifica Cookie.

L'impostazione predefinita è la sessione browser corrente, equivalente all'impostazione del valore a -1.
