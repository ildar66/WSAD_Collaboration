应用程序服务器设置

使用此页面查看或更改服务器组件实例的设置。

要查看此管理控制台页面，单击服务器 > 应用程序服务器 > server_name。

配置选项卡提供了可编辑字段，运行时选项卡提供了只读信息。运行时选项卡仅在服务器运行时可用。

“配置”选项卡

名称
指定服务器的逻辑名。在节点中，服务器名必须是唯一的。
数据类型 String
缺省 server1
初始状态
指定服务器第一次启动时请求的组件执行状态。
数据类型 String
缺省 已启动
应用程序类装入程序策略
指定是否使用单个类装入程序装入所有应用程序，还是为每个应用程序使用不同的类装入程序。

选择为 SINGLE 和 MULTIPLE。缺省是为每个应用程序使用单个类装入程序（MULTIPLE）。

数据类型 String
缺省 MULTIPLE
应用程序类装入方式
指定类装入程序是否首先在父类装入程序或应用程序类装入程序中搜索来装入类。JDK 类装入程序和 WebSphere 类装入程序的标准为 PARENT_FIRST。通过指定 PARENT_LAST，您的应用程序可以重设包含在父类装入程序中的类，但如果您混合使用了覆盖的类和未覆盖的类，则此操作可能潜在地导致 ClassCastException 或 LinkageErrors。

选项是 PARENT_FIRST 和 PARENT_LAST。缺省是在应用程序类装入程序装入一个类中搜索之前，在父类装入程序中搜索。

数据类型 String
缺省 PARENT_FIRST

“运行时”选项卡

进程标识  
指定标识进程的字符串。
数据类型 String
单元名称  
指定应用程序服务器的单元名。
数据类型 String
缺省 host_nameNetwork
节点名  
指定应用程序服务器的节点名。
数据类型 String
状态  
表明应用程序服务器已启动或已停止。
数据类型 String
缺省 已启动
