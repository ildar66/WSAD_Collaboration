WebSphere 内部 JMS 服务器设置

WebSphere Application Server 管理域中节点上的 JMS 服务器为该节点上的 JMS 函数提供服务。使用此面板查看或更改所选 JMS 服务器的配置属性。

您可以使用此面板来配置一组常规的 JMS 服务器属性，其添加到为嵌入式 WebSphere JMS 提供程序自动配置的属性缺省值。

注: 通常，WebSphere MQ 属性的缺省值对于 WebSphere 内部 JMS 服务器是足够的。但是，如果您正在运行高消息传递负载，则需要更改一些 WebSphere MQ 属性；例如，日志文件位置、文件页面和缓冲区页面的属性。要获得有关配置 WebSphere MQ 属性的更多信息，请参阅 WebSphere MQ System Administration 一书（SC33-1873），可从    IBM 出版物中心 或从 WebSphere MQ collection kit（SK2T-0730）获得。

名称
在 IBM WebSphere Application Server 中出于管理目的，用于标识 JMS 服务器的名称。

此名称不能更改。

数据类型 String
单位 不适用
缺省 WebSphere 内部 JMS 服务器
范围 不适用
描述
在 IBM WebSphere Application Server 中出于管理目的，对 JMS 服务器的描述。

此字符串不能更改。

数据类型 String
缺省 WebSphere 内部 JMS 服务器
线程数  
发布／预订匹配引擎使用的并发线程数

并发线程数应该仅设置为较小的数。

数据类型 整型
单位 线程
缺省 1
范围 大于或等于 1。
队列名称
此 JMS 服务器主管的队列名。每个队列名必须添加在单个行上。

此字段中列出的每个队列都必须有相同管理名称的独立队列管理对象。要使队列对应用程序是可用的， 定义一个 WebSphere 队列，并为主管队列的主机将这个 WebSphere 队列的名称添加到 JMS 服务器面板上的这个字段。

数据类型 ASCII
单位 队列名称
缺省 不适用
范围 此字段中的每个条目都是一个长达 45 个字符的队列名称，这个名称 必须完全匹配为这个队列而定义的 WebSphere 队列管理对象（包括大写和小写字符的使用）。
初始状态
当 JMS 服务器下一次重新启动时，您要它所处的状态。
数据类型 枚举
单位 不适用
缺省 已启动
范围
已启动
自动启动 JMS 服务器。
已停止
不自动启动 JMS 服务器。如果任何部署的企业应用程序使用 JMS 服务器提供的 JMS 服务器功能，则系统管理员必须手工启动 JMS 服务器，或选择此属性的“已启动”值，然后重新启动 JMS 服务器。

要重新启动应用程序服务器的 JMS 服务器，则停止，然后重新启动此应用程序服务器。
