Valores de servidor JMS interno de WebSphere

El servidor JMS sirve las funciones JMS en un nodo dentro del dominio de administración de WebSphere Application Server. Utilice este panel para ver o modificar las propiedades de configuración del servidor JMS seleccionado.

Puede utilizar este panel para configurar un conjunto general de propiedades del servidor JMS, que se añadirían a los valores por omisión de las propiedades configuradas automáticamente para el proveedor de JMS incorporado de WebSphere.

Nota: En general, los valores por omisión de las propiedades de WebSphere MQ resultan adecuados para servidores de JMS internos de WebSphere. No obstante, si ejecuta altas cargas de mensajes, es posible que tenga que cambiar algunas propiedades de WebSphere MQ, por ejemplo, las propiedades para ubicaciones de archivo de anotaciones cronológicas, páginas de archivo y páginas de almacenamiento intermedio. Para obtener más información sobre cómo configurar propiedades de WebSphere MQ, consulte el manual WebSphere MQ Administración del sistema, SC33-1873, que está disponible en    IBM Publications Center o en WebSphere MQ collection kit, SK2T-0730.

Nombre
Nombre por el que se conoce el servidor JMS para fines administrativos dentro de IBM WebSphere Application Server.

Este nombre no debe cambiarse.

Tipo de datos String
Unidades No se aplica
Valor por omisión Servidor JMS interno de WebSphere
Rango No se aplica
Descripción
Descripción del servidor JMS, para fines administrativos dentro de IBM WebSphere Application Server.

Esta serie no debe cambiarse.

Tipo de datos String
Valor por omisión Servidor JMS interno de WebSphere
Número de hebras  
Número de hebras simultáneas que el motor de publicación/suscripción coincidente va a utilizar.

El número de hebras simultáneas sólo debe establecerse en un número pequeño.

Tipo de datos Integer
Unidades Hebras
Valor por omisión 1
Rango Mayor o igual a 1
Nombres de cola
Nombres de las colas albergadas en este servidor JMS. Cada nombre de cola debe añadirse en una línea distinta.

Cada cola que aparece enumerada en este campo debe tener un objeto administrativo de colas independiente con el mismo nombre administrativo. Para que las aplicaciones puedan acceder a una cola, defina una cola WebSphere y añada su nombre a este campo en el panel Servidor JMS para el sistema principal donde desea que se albergue la cola.

Tipo de datos ASCII
Unidades Nombre de cola
Valor por omisión No se aplica
Rango Cada entrada de este campo es un nombre de cola de hasta 45 caracteres, que debe coincidir exactamente (incluido el uso de las mayúsculas y las minúsculas) con el objeto administrativo de colas definido por la cola.
Estado inicial
Estado que desea que tenga el servidor JMS la próxima vez que se reinicie.
Tipo de datos Enum
Unidades No se aplica
Valor por omisión Iniciado
Rango
Iniciado
El servidor JMS se inicia automáticamente.
Detenido
El servidor JMS no se inicia automáticamente. Si las aplicaciones de empresa desplegadas tienen que utilizar las funciones del servidor JMS proporcionadas por el servidor JMS, el administrador del sistema debe iniciar manualmente el servidor JMS o seleccionar el valor Iniciado de esta propiedad y después reiniciar el servidor JMS.

Para reiniciar el servidor JMS para un servidor de aplicaciones, detenga el servidor de aplicaciones y, a continuación, reinícielo.
