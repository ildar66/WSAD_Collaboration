Valores de contenedor de EJB

Utilice esta página para configurar y gestionar un contenedor de EJB específico.

Para ver esta página de la consola administrativa, pulse Servidores > Servidores de aplicaciones > nombreServidor > Contenedor de EJB.

Pestaña Configuración

Directorio de desactivación
Especifica el directorio donde el contenedor guarda el estado persistente de los beans de sesión con estados desactivados.

Los beans se desactivan cuando el número de instancias del bean activas supera el tamaño de antememoria especificado en la configuración del contenedor. Cuando se desactiva un bean con estado, el contenedor serializa la instancia del bean en un archivo del directorio de desactivación y descarta la instancia de la antememoria del bean. Si en el último momento llega una petición para la instancia del bean desactivado el contenedor lo recupera del directorio de desactivación, los deserializa y lo devuelve a la antememoria pasándole la petición. Si alguno de estos pasos falla (por ejemplo, si la instancia del bean ya no está en el directorio de desactivación) la invocación al método también fallará.

Intervalo de limpieza de la agrupación inactiva
Especifica el intervalo durante el cual el contenedor examina las agrupaciones de las instancias del bean disponibles para determinar si se pueden suprimir algunas y reducir así el uso de memoria.
Tipo de datos Integer
Unidades Milisegundos
Rango Mayor que 0
Nombre JNDI del origen de datos por omisión
Especifica el nombre JNDI de un origen de datos que se debe utilizar cuando no se ha especificado un origen de datos durante el despliegue de la aplicación. Este valor no se aplica a beans CMP compatibles con EJB 2.x.

Los servlets y los enterprise beans utilizan orígenes de datos para obtener estas conexiones. Al configurar un contenedor, puede especificar un origen de datos por omisión para el contenedor. Este origen de datos se convierte en el origen de datos por omisión que utilizan los beans de entidad instalados en el contenedor que usan CMP (persistencia gestionada por contenedor).

El origen de datos por omisión de un contenedor está protegido. Al especificarlo, debe proporcionar un ID de usuario y una contraseña para acceder a él.

La especificación de un origen de datos por omisión es opcional si cada bean de entidad CMP del contenedor tiene un origen de datos especificado en su configuración. Si no se ha especificado un origen de datos por omisión y se instala un bean de entidad CMP en el contenedor sin especificar un origen de datos para ese bean, las aplicaciones no pueden utilizar dicho bean de entidad CMP.

Estado inicial
Especifica el estado de ejecución solicitado cuando se inicia el servidor por primera vez.
Tipo de datos String
Valor por omisión Iniciado
Rango Los valores válidos son Iniciado y Detenido
