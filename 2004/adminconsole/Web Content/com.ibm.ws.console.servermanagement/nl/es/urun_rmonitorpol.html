Valores de política de supervisión

Utilice esta página para ver o modificar los valores que controlan cómo el agente de nodo supervisa y reinicia un proceso.

Para ver esta página de la consola administrativa, pulse Servidores > Servidores de aplicaciones > nombre_servidor > Definición de proceso > Políticas de supervisión.

Pestaña Configuración

Número máximo de intentos de arranque
Especifica el número máximo de intentos de arrancar el servidor de aplicaciones antes de abandonar.
Tipo de datos Integer
Intervalo de ping
Especifica la frecuencia de los intentos de comunicación entre el proceso padre, como por ejemplo un agente de nodo, y el proceso que ha generado, como por ejemplo un servidor de aplicaciones. Ajuste este valor según sus requisitos para reiniciar los servidores con anomalías. Si se reduce el valor, se detectan anomalías antes; si se incrementa el valor, se reduce la frecuencia del mandato PING y, así, se reduce la actividad general del sistema.
Tipo de datos Integer
Tiempo de espera de ping
Cuando un proceso padre genera un proceso hijo, como por ejemplo, cuando un gestor de procesos genera un servidor, el proceso padre efectúa PING en el proceso hijo para comprobar si éste se ha generado satisfactoriamente. Este valor especifica el tiempo en segundo que el proceso padre debe esperar (después de efectuar PING en el proceso hijo) antes de asumir que el proceso hijo no ha sido satisfactorio.
Tipo de datos Integer
Unidades Segundos
Rearranque automático
Especifica si se debe reiniciar el proceso automáticamente si se produce un error. El valor por omisión es reiniciar el proceso automáticamente.
Tipo de datos Boolean
Valor por omisión true
Estado de rearranque de nodo
Especifica el estado deseado para el proceso después de que el nodo se cierre completamente y se reinicie. Las opciones son: DETENIDO, EN EJECUCIÓN, ANTERIOR. El valor por omisión es DETENIDO.
Tipo de datos String
Valor por omisión DETENIDO
Rango Los valores válidos son DETENIDO, EN EJECUCIÓN o ANTERIOR.
