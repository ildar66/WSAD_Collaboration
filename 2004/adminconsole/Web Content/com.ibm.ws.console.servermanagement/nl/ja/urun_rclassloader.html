クラス・ローダーの集合

このページを使用して、アプリケーション・サーバー上のクラス・ローダー・インスタンスを管理します。 クラス・ローダーは、 アプリケーション・クラス・ローダーまたは親クラス・ローダーのどちらがアプリケーションの Java クラス・ファイルを検索してロードするかを判別します。

この管理コンソール・ページを表示するには、「サーバー」>「アプリケーション・サーバー」>「server_name」>「Classloader」をクリックします。

クラス・ローダー ID
クラス・ローダー・インスタンスを識別するサーバーに固有のストリングです。 この ID は製品によって割り当てられます。
クラス・ローダー・モード
アプリケーション・クラス・ローダー・ポリシーが SINGLE であるときに、 クラス・ローダー・モードを指定します。PARENT_FIRST を指定すると、 クラス・ローダーはローカル・クラスパスからクラスのロードを試行する前に、 親クラス・ローダーにクラスのロードを委任します。PARENT_LAST を指定した場合、クラス・ローダーは親にクラスのロードを委任する前に、 ローカルのクラスパスからクラスをロードしようとします。 これによりアプリケーション・クラス・ローダーは親クラス・ローダーに存在するクラスの独自のバージョンをオーバーライドして、 提供することができます。
