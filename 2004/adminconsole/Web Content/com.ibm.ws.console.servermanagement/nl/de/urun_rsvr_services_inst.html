Einstellungen für benutzerdefinierte Services

Auf dieser Seite können Sie einen Service konfigurieren, der auf einem Anwendungsserver ausgeführt wird.

Klicken Sie, um diese Seite der Administrationskonsole anzuzeigen, auf Server > Anwendungsserver > Servername > Benutzerdefinierte Services > Name_des_benutzerdefinierten_Service.

Register "Konfiguration"

Start
Gibt an, ob der Server versucht, den Service zu starten und zu initialisieren, wenn der übergeordnete Prozess (Server) gestartet wird. Standardmäßig wird der Service nicht aktiviert, wenn der übergeordnete Prozess gestartet wird. Zum Aktivieren des Service müssen Sie das Markierungsfeld auswählen.
Datentyp Boolescher Wert
Standardwert false
URL für externe Konfiguration
Gibt den URL für eine benutzerdefinierte Servicekonfigurationsdatei an.

Wenn für die Klasse des benutzerdefinierten Service eine Konfigurationsdatei erforderlich ist, geben Sie für den Wert den vollständigen Pfad zu dieser Datei ein. Dieser Dateiname wird an die Klasse des benutzerdefinierten Service übergeben.

Datentyp String
Einheiten URL
Klassenname
Gibt den Klassennamen der Serviceimplementierung an. Diese Klasse muss das Interface des benutzerdefinierten Service implementieren.
Datentyp String
Einheiten Java-Klassenname
Anzeigename
Gibt den Namen des Service an.
Datentyp String
Beschreibung
Beschreibt den benutzerdefinierten Service.
Datentyp String
Klassenpfad
Gibt den Klassenpfad an, der zum Lokalisieren der Klassen und JAR-Dateien für diesen Service verwendet wird.
Datentyp String
Einheiten Klassenpfad
