Server

Auf dieser Seite können Sie Informationen zu Anwendungs- und JMS-Servern anzeigen und diese Server verwalten.

Anwendungsserver

Auf der Seite "Anwendungsserver" sind die Anwendungsserver in der Zelle sowie die Knoten, auf denen die Anwendungsserver installiert sind, aufgelistet.

Network Deployment zeigt auch den Status der Anwendungsserver an. Der Status gibt an, ob ein Server ausgeführt wird, gestoppt wurde oder ob Probleme im Server auftreten.

Klicken Sie zum Anzeigen dieser Seite der Administrationskonsole auf Server > Anwendungsserver.

JMS-Server

Jeder JMS-Server stellt die Funktionen des JMS-Providers für einen Knoten in Ihrer Administrationsdomäne bereit. Pro Knoten in der Administrationsdomäne kann höchstens ein JMS-Server installiert sein, und jeder Anwendungsserver in der Domäne kann auf JMS-Ressourcen zugreifen, die von JMS-Servern, die auf einem in der Domäne befindlichen Knoten installiert sind, bereitgestellt werden.

Klicken Sie zum Anzeigen dieser Seite der Administrationskonsole auf Server > JMS-Server.

Name
Gibt einen logischen Namen für den Server an. Servernamen müssen auf einem Knoten eindeutig sein.
Knoten
Gibt den Namen des Knotens für den Anwendungsserver an.
Status  
Gibt an, ob der Anwendungsserver gestartet oder gestoppt wurde. (nur Network Deployment)

Beachten Sie Folgendes: Wenn der Status Nicht verfügbar ist, wird der Knotenagent nicht auf diesem Knoten ausgeführt, und Sie müssen den Knotenagenten vor dem Start des Servers erneut starten.
