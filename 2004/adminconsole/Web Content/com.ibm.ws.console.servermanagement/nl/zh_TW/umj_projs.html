WebSphere 內部 JMS 伺服器設定

WebSphere Application Server 管理領域內的節點，其 JMS 功能是由節點中的 JMS 伺服器來提供的。 請使用這個畫面來檢視或變更選取的 JMS 伺服器的配置內容。

您可以利用這個畫面來配置一組通用的 JMS 伺服器內容，以將它們新增到自動配置給內嵌 WebSphere JMS Provider 之內容的預設值。

附註: 一般而言，WebSphere MQ 內容的預設值已足夠供 WebSphere 內部 JMS 伺服器使用。 不過，如果您正在執行高傳訊負荷，您可能需要變更部分 WebSphere MQ 內容；比方說，日誌檔位置、檔案頁面和緩衝區頁面。 如果需要配置 WebSphere MQ 內容的詳細資訊，請參閱 WebSphere System Administration 一書 (SC33-1873)，您可以從    IBM Publications Center 或 WebSphere MQ collection kit (SK2T-0730) 中找到它。

名稱
用來識別 JMS 伺服器，供 IBM WebSphere Application Server 內管理使用的名稱。

這個名稱不應變更。

資料類型 字串
單位 不適用

範圍 不適用
說明
JMS 伺服器的說明，供 IBM WebSphere Application Server 內的管理使用。

這個字串不應變更。

資料類型 字串
預設值 WebSphere 內部 JMS 伺服器
執行緒數目  
發佈/訂閱相符引擎要用的並行執行緒的數目

並行執行緒的數目只應設為小數目。

資料類型 整數
單位 執行緒
預設值 1
範圍 大於或等於 1。
佇列名稱
在這個 JMS 伺服器中的佇列名稱。 每個佇列名稱都必須新增在個別的行中。

每一個列示在這個欄位的佇列都必須具有一個相同管理名稱的個別佇列管理物件。 如果要使佇列可供應用程式使用，請定義一個 WebSphere 佇列，再將它的名稱新增到「JMS 伺服器」畫面上的這個欄位，供要您在其中控管佇列的主機使用。

資料類型 ASCII
單位 佇列名稱
預設值 不適用
範圍 這個欄位中的每一個項目都是最多 45 個字元的佇列名稱，它們必須完全符合（包括所使用的大小寫字元）定義給佇列的 WebSphere 佇列管理物件。

資料類型 列舉
單位 不適用
預設值 已啟動
範圍
已啟動
自動啟動 JMS 伺服器。
已停止
不自動啟動 JMS 伺服器。 如果有任何部署的企業應用程式要使用 JMS 伺服器所提供的 JMS 伺服器功能，系統管理者必須以手動方式來啟動 JMS 伺服器，或選取這個內容的啟動值，再重新啟動 JMS 伺服器。

如果要重新啟動應用程式伺服器的 JMS 伺服器，請停止再重新啟動這個應用程式伺服器。
