診斷追蹤服務設定

請利用這個頁面來檢視和修改診斷追蹤服務的內容。

如果要檢視這個頁面，請按一下疑難排解 > 記載和追蹤 > server > 診斷追蹤。

「配置」標籤

啟用追蹤
啟用追蹤服務。

如果沒有選取這個選項，就不會在伺服器啟動時，將下列配置內容傳遞給應用程式伺服器追蹤服務。

追蹤規格
指定追蹤詳細資料。

請輸入追蹤字串來指定要追蹤的元件、套件或群組。 追蹤字串必須符合以下所說明特定文法。 您可以直接輸入追蹤字串，也可以利用圖形追蹤介面來產生它。 請按一下修改來啟動圖形追蹤介面。

如果您從配置標籤啟動圖形追蹤介面，就會顯示一份通用元件、套件和群組的清單。 這個清單不一定完整。

如果您從執行時期標籤啟動圖形追蹤介面，元件、套件和群組的清單會顯示執行中的伺服器目前所登錄的所有這類元件。

追蹤規格的格式為：

<component> = <trace_type>= enabled | disabled
其中：<component> 是要啟用或停用追蹤的元件，而 <trace_type> 是要啟用或停用的追蹤類型。請以冒號 (:) 區隔多個追蹤規格。

元件會對應於 Java 套件和類別，或對應於 Java 套件的集合。 使用 * 作為萬用字元，來指出包括指定的元件所包含的所有套件中所有類別的元件。比方說：

*
指定在應用程式伺服器中執行的所有可追蹤的程式碼，其中包括 WebSphere Application Server 系統程式碼和自訂程式碼。
com.ibm.ws.*
指定套件名稱以 com.ibm.ws 開頭的所有類別。
com.ibm.ws.classloader.JarClassLoader
僅指定 JarClassLoader 類別。

如果需要追蹤字串文法的詳細資訊，請參閱 WebSphere Application Server InfoCenter 中的啟用追蹤 一文。

附註: 如果您是從群組和元件清單決定選項，那麼當您在從管理主控台設定追蹤規格時，就會發生錯誤。 在某些情況下，在一份清單中所選的選項，不會因為從另一份清單加入另一個選項而遺失。 如果要暫時解決這個問題，請直接在「追蹤規格」輸入欄位中，輸入您所要的追蹤規格。

追蹤輸出  
指定追蹤輸出應該寫在哪裡。

追蹤輸出可以直接寫到輸出檔，或在執行時期頁面中，利用「傾出」按鈕來根據需求儲存在記憶體並寫入某個檔案中。

記憶體緩衝區
指定追蹤輸出應該寫入記憶體內循環緩衝區中。 如果您選取這個選項，您必須指定下列參數：
緩衝區大小上限
指定緩衝區所能快取的項目數，以千為單位。 當超出這個數目時，新項目會改寫較舊的項目。
傾出檔名稱
在傾出記憶體緩衝區時，要將它寫入的檔案名稱。 您只能從「執行時期」標籤使用這個選項。

檔案
指定將追蹤輸出寫入自我管理的日誌檔。

自我管理的日誌檔會將訊息寫入檔案中，直到到達某個大小準則為止。 當檔案到達指定的大小時，會暫停記載作業和關閉日誌檔，並將它重新命名。 新名稱是以檔案的原始名稱為基礎，再加上時間戳記限定元來指出重新命名的時間。 重新命名完成之後，就會重新開啟有原始名稱的新空白日誌檔，且會回復記載作業。 在輪替時，不會失去任何訊息，不過，可能會有單一訊息跨越兩個檔案。

如果您選取這個選項，您必須指定下列參數：

檔案大小上限
指定輸出檔所能增長的大小上限 (MB)。

這個屬性只在選取了「檔案大小」屬性時才有效。 當檔案到達這個大小時，就會依照上述說明來進行輪替。

歷程檔數目上限
指定要保留的輪替檔案數目上限。
檔案名稱
指定追蹤輸出要寫入其中的檔案名稱。

追蹤輸出格式  
指定追蹤輸出的格式。

您可以指定追蹤輸出的三個層次之一：

基礎（相容）
只保留基本追蹤資訊。請選取這個選項來將追蹤輸出所使用的空間量最小化。
進階
保留更明確的追蹤資訊。請選取這個選項來查看疑難排解和問題判斷時所用的詳細追蹤資訊。
日誌分析器
以相容於日誌分析器工具的格式來保留追蹤資訊。如果您要以追蹤輸出為日誌分析器工具的輸入,請選取這個選項。

「執行時期」標籤  
指定追蹤輸出的格式。
儲存追蹤
同時將執行時期標籤中的變更儲存在配置中。

請選取這個勾選框來同時將執行時期追蹤變更複製到追蹤配置設定中。 將這些變更儲存到追蹤配置中，會使變更持續存在，即使應用程式重新啟動也是如此。
