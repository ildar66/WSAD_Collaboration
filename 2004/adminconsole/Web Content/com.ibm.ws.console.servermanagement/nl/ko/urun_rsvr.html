서버 콜렉션

정보를 보고 응용프로그램 및 JMS 서버를 관리하려면 이 페이지를 참조하십시오.

Application Servers

Application Servers 페이지는 셀의 Application Server 및 해당 Application Server를 가지고 있는 노드를 나열합니다.

Network Deployment 제품도 Application Server의 상태를 표시합니다. 이 상태는 서버가 실행, 중지 또는 문제점 발생 중인지 여부를 나타냅니다.

이 관리 콘솔 페이지를 보려면 서버 > Application Servers를 클릭하십시오.

JMS 서버

각 JMS 서버는 관리 도메인의 노드에 JMS 프로바이더 기능을 제공합니다. 관리 도메인의 각 노드에 최대 하나의 JMS 서버가 있을 수 있고, 도메인 내의 모든 Application Server가 도메인의 노드에서 JMS 서버가 서비스를 제공하는 JMS 자원에 액세스할 수 있습니다.

이 관리 콘솔 페이지를 보려면 서버 > JMS 서버를 클릭하십시오.

이름
서버의 논리 이름을 지정합니다. 서버 이름은 노드 내에서 고유해야 합니다.
노드
Application Server에 대한 노드의 이름을 지정합니다.
상태  
Application Server가 시작되었는지 또는 중지되었는지를 표시합니다.(Network Deployment에만 해당)

상태가 사용 불가능일 경우, Node Agent가 노드에서 실행하고 있지 않으므로 서버를 시작하기 전에 Node Agent를 다시 시작해야 하는 것에 유의하십시오.
