관리 콘솔 단추

다음 단추 선택사항은 사용 가능하게 설정한 제품 기능에 따라, 관리 콘솔의 다양한 페이지에서 사용 가능합니다.

  • 중단. 아직 준비된 상태가 아닌 트랜잭션을 중단합니다. 트랜잭션이 완료된 모든 조작이 완료되지 않은 상태로 돌아갑니다.
  • 추가. 선택 또는 입력한 항목을 목록에 추가하거나, 항목을 목록에 추가하기 위한 대화 상자를 작성합니다.
  • 적용. 페이지를 종료하지 않고 변경사항을 페이지에 저장합니다.
  • 이전. 이전 페이지 또는 항목을 순서대로 표시합니다. 관리 콘솔은 브라우저의 이전 단추 및 다음 단추 사용을 지원하지 않습니다. 이 단추를 사용할 경우 종종 문제점이 발생합니다. 대신, 관리 콘솔 패널의 이전 단추 또는 취소 단추를 사용하십시오.
  • 찾아보기. 시스템의 파일을 검색할 수 있는 대화상자를 엽니다.
  • 취소. 저장되지 않은 변경사항을 버리고 현재 페이지 또는 대화 상자를 종료합니다. 관리 콘솔은 브라우저의 이전 단추 및 다음 단추 사용을 지원하지 않습니다. 이 단추를 사용할 경우 종종 문제점이 발생합니다. 대신, 관리 콘솔 패널의 취소 단추를 사용하십시오.
  • 변경. 보안 문맥에서, 실행할 응용프로그램의 사용자 ID에 대한 사용자 레지스트리를 검색할 수 있게 합니다. 컨테이너 등록 정보의 컨텍스트에서, 컨테이너가 사용 중인 데이터소스를 변경할 수 있게 합니다.
  • 지우기. 변경사항을 지우고, 최근에 저장된 값을 복원합니다.
  • 선택사항 지우기. 이 탭 처리된 페이지에 있는 테이블에서 선택된 모든 셀을 지웁니다.
  • 닫기. 대화 상자를 종료합니다.
  • 확약. 준비된 트랜잭션에 의해 지속되는 모든 잠금을 해제하고, 트랜잭션이 확약되도록 강제 실행합니다.
  • 복사. 선택된 응용프로그램 서버의 사본을 작성합니다.
  • 작성. 대화 상자의 모든 탭 처리된 페이지에 대한 변경사항을 저장하고, 대화 상자를 종료합니다.
  • 삭제. 선택된 인스턴스를 제거합니다.
  • 세부사항. 트랜잭션에 대한 세부사항을 보여줍니다.
  • 완료. 대화 상자의 모든 탭 처리된 페이지에 대한 변경사항을 저장하고, 대화 상자를 종료합니다.
  • 내리기. 목록의 아래로 이동합니다.
  • 덤프. 추적한 Application Server의 덤프를 활성화합니다.
  • 편집. 목록의 선택된 항목을 편집하거나, 항목을 편집하기 위한 대화 상자를 작성할 수 있게 합니다.
  • 내보내기. 엔터프라이즈 응용프로그램의 EAR 파일을 내보내기 위한 페이지에 액세스합니다.
  • DDL 내보내기. 엔터프라이즈 응용프로그램의 DDL 파일을 내보내기 위한 페이지에 액세스합니다.
  • 키 내보내기. LTPA 키를 다른 도메인으로 내보냅니다.
  • 필터. 이 탭 처리된 페이지에 있는 테이블에서 볼 자원을 지정하기 위해 대화 상자를 작성합니다.
  • 완료. 결과를 관여된 모든 응용프로그램에 보고했는지 여부와 무관하게, 완료할 트랜잭션을 강제 실행합니다.
  • 첫 번째. 일련의 레코드에서 첫 번째 레코드를 표시합니다.
  • 키 생성. 새 LTPA 키를 생성합니다. 보안이 LTPA에 대해 인증 메커니즘으로 처음 켜질 때, 패널에 입력된 암호로 LTPA 키가 자동 생성됩니다. 새 키를 생성해야 하는 경우, 보안이 켜진 상태에서 서버를 백업한 후 이 단추를 사용하십시오. 이 단추를 클릭하면 키가 생성되어 모든 활성 서버(셀, 노드 및 Application Server)에 전달됩니다. 새 키를 사용하여 LTPA 토큰을 암호화하거나 암호 해독할 수 있습니다. 콘솔 작업 표시줄에서 저장을 클릭하여 새 키와 암호를 저장소에 저장하십시오.
  • 즉시 중지. 서버를 중지합니다. 그러나 전체 서버 프로세스를 종료하기 앞서 처리 중인 요청을 완료하도록 정상 서버의 quiesce 프로세스는 생략합니다. 이같은 시스템 종료 모드는 정상 서버 중지 처리보다 빠르지만, 일부 응용프로그램 클라이언트는 예외 통지를 수신할 수도 있습니다.
  • 키 가져오기. 다른 도메인에서 새 LTPA 키를 가져옵니다. 다수의 WebSphere 도메인(셀)에 걸쳐 WebSphere에서 단일 사인온(SSO)을 지원하려면, 도메인 간에 LTPA 키와 암호를 공유해야 합니다. 하나 이상의 셀에서 파일로 키를 내보낸 후, 이 단추를 클릭하면 모든 활성 서버(셀, 노드 및 Application Server)로 키를 가져옵니다. 새 키를 사용하여 LTPA 토큰을 암호화하거나 암호 해독할 수 있습니다. 콘솔 작업 표시줄에서 저장을 클릭하여 새 키와 암호를 저장소에 저장하십시오.
  • 설치. 응용프로그램 설치 준비 페이지를 표시하는 데 이 페이지를 사용하여 응용프로그램이나 EJB 또는 웹 구성요소를 Application Server에 전개합니다.
  • RAR 설치.RAR 설치 단추를 누르면 JCA 커넥터를 설치하고 자원 어댑터를 작성하는 대화 상자가 열립니다.
  • 트랜잭션 관리. 서버에서 실행 중인 활성 트랜잭션의 목록을 표시합니다. 트랜잭션 자원을 사용할 수 없기 때문에 처리를 중단한 트랜잭션을 강제로 완료할 수 없습니다.
  • 수정. 스펙의 변경에 사용하는 대화 상자를 엽니다.
  • 이동. 선택된 응용프로그램 서버를 관리 셀의 다른 위치로 이동시킵니다. 프롬프트될 때, 대상 위치를 지정하십시오.
  • 새로 작성. 새 인스턴스를 정의하기 위해 사용하는 페이지를 표시합니다. 예를 들어, Application Server 페이지에서 새로 작성을 클릭하면 새 Application Server를 구성할 수 있는 페이지가 표시됩니다.
  • 다음. 다음 페이지, 프레임 또는 항목을 순서대로 표시합니다.
  • 확인. 변경사항을 저장하고 페이지를 종료합니다.
  • 핑. 선택된 Application Server에 접속하려 시도합니다.
  • 이전. 이전 페이지, 프레임 또는 항목을 순서대로 표시합니다.
  • 종료. 저장되지 않은 변경사항을 버리고 대화 상자를 종료합니다.
  • 새로 고치기. 이 탭 처리된 페이지에 현재 나열된 인스턴스에 대한 데이터의 보기를 새로 고칩니다.
  • 제거. 선택된 항목을 삭제합니다.
  • 재설정. 탭 또는 페이지에서 변경사항을 지우고, 최근에 저장된 값을 복원합니다.
  • 재시작. 선택된 오브젝트를 중지했다가 다시 시작합니다.
  • 새로 검색. 새 레코드를 검색합니다.
  • 저장. 로컬 구성의 변경사항을 마스터 구성으로 저장합니다.
  • 선택. 자원 분석을 위해, 자원을 모니터할 범위를 선택할 수 있게 합니다.
  • 설정. 대화 상자의 설정 변경사항을 저장합니다.
  • 설정값. Servlet 관련 자원 설정을 편집하기 위한 대화 상자를 표시합니다.
  • 사용 중인 설정값. 현재 사용 중인 설정값을 보여주는 대화 상자를 표시합니다.
  • 시작. Application Server의 문맥에서, 선택된 Application Server를 시작합니다. 데이터 콜렉션의 컨텍스트에서, 이 탭 처리된 페이지의 테이블에 대한 데이터 수집을 시작합니다.
  • 중지. Application Server와 같은 서버 구성요소의 문맥에서, 선택된 서버 구성요소를 중지합니다. 데이터 콜렉션의 컨텍스트에서, 탭 처리된 페이지의 테이블에 대한 데이터 수집을 중지합니다.
  • 설치 제거. WebSphere Application Server 구성 저장소에서 전개된 응용프로그램을 삭제합니다. 또한 파일 시스템에서 응용프로그램 아카이브 파일을 삭제합니다.
  • 갱신. 서버에 전개된 응용프로그램을 갱신된 응용프로그램으로 바꿉니다. 갱신의 일부로서, 응용프로그램 설치 준비 및 응용프로그램 갱신 페이지의 단계를 완료해야 합니다.
  • 자원 목록 갱신. 테이블의 데이터를 갱신합니다. 새 인스턴스를 발견하여 테이블에 추가합니다.
  • 보기. 파일 대화 상자를 엽니다.
