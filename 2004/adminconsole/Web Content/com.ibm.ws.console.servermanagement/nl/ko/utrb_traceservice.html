진단 추적 서비스 설정

이 페이지를 참조하여 진단 추적 서비스의 등록 정보를 검토하고 수정하십시오.

이 페이지를 보려면 문제점 해결 > 로그 및 추적 > server > 진단 추적을 클릭하십시오.

구성 탭

추적 사용 가능
추적 서비스를 사용 가능하게 합니다.

이 옵션이 선택되지 않은 경우 서버가 시작할 때 다음 구성 등록 정보가 Application Server 추적 서비스로 전달되지 않습니다.

추적 스펙
추적 세부사항을 지정합니다.

추적할 구성요소, 패키지 또는 구룹을 지정하는 추적 문자열을 입력하십시오. 추적 문자열은 아래 설명된 특정 문법을 준수해야 합니다. 추적 문자열을 직접 입력하거나 그래픽 추적 인터페이스를 사용하여 생성할 수 있습니다. 수정을 클릭하여 그래픽 추적 인터페이스를 시작하십시오.

구성 탭에서 그래픽 추적 인터페이스를 시작하는 경우 잘 알려진 구성요소, 패키지 및 그룹의 목록이 표시됩니다. 이 목록은 포괄적이지 않을 수 있습니다.

런타임 탭에서 그래픽 추적 인터페이스를 시작하는 경우 구성요소, 패키지 및 그룹의 목록이 현재 실행 중인 서버에 등록된 모든 구성요소를 표시합니다.

추적 스펙의 형식은 다음과 같습니다. 

<component> = <trace_type>= enabled | disabled
여기서 <component>는 추적을 사용 가능 또는 사용 불가능하게 할 구성요소이고 <trace_type>은 사용 가능 또는 사용 불가능하게 할 추적의 유형입니다. 복수 추적 스펙은 콜론(:)으로 구분하십시오.

구성요소는 Java 패키지 및 클래스에 해당하거나 Java 패키지의 콜렉션에 해당합니다. 지정된 구성요소가 포함하는 모든 패키지의 모든 클래스를 포함하는 구성요소를 표시하려면 *를 와일드 카드로 사용하십시오. 예:

*
WebSphere Application Server 시스템 코드 및 고객 코드를 포함하여 Application Server에서 실행 중인 모든 추적 가능한 코드를 지정합니다.
com.ibm.ws.*
패키지 이름이 com.ibm.ws로 시작하는 모든 클래스를 지정합니다.
com.ibm.ws.classloader.JarClassLoader
JarClassLoader 클래스만을 지정합니다.

추적 문자열 문법에 대한 자세한 정보는 WebSphere Application Server InfoCenter의 추적 사용 가능을 참조하십시오.

참고: 그룹 및 구성요소 목록 모두에서 선택할 경우 관리 콘솔에서 추적 스펙을 설정할 때 오류가 발생할 수 있습니다. 경우에 따라, 어떤 목록에서의 선택을 추가할 때 다른 목록에서의 선택이 소실되지 않는 경우가 있습니다. 이 문제점을 해결하려면, 원하는 추적 스펙을 직접 추적 스펙 입력 필드에 입력하십시오.

추적 출력  
추적 출력이 작성되어야 할 위치를 지정합니다.

추적 출력은 출력 파일에 직접 작성되거나 런타임 페이지의 덤프 단추를 사용하여 메모리에 저장되거나 요구되는 파일에 기록될 수 있습니다.

메모리 버퍼
추적 출력은 메모리내 순환 버퍼에 기록되도록 지정합니다. 이 옵션을 선택하는 경우 다음 매개변수를 지정해야 합니다.
최대 버퍼 크기
버퍼에서 캐시될 수 있는 항목 수(수천)를 지정합니다. 이 수를 초과하면 기존의 항목이 새 항목으로 겹쳐쓰기됩니다.
덤프 파일 이름
메모리 버퍼가 덤프될 때 기록되는 파일의 이름. 이 옵션은 런타임 탭에서만 사용할 수 있습니다.

파일
자체 관리 로그 파일에 추적 결과를 작성하도록 지정합니다.

자체 관리 로그 파일은 크기 기준에 도달할 때까지 파일에 메시지를 작성합니다. 파일이 지정된 크기에 도달할 경우 로깅이 임시로 일시중단되고 로그 파일이 닫히고 이름 변경됩니다. 새 이름은 파일의 원래 이름 및 이름 바꾸기가 발생할 시기를 표시하는 시간 소인 규정자를 기준으로 합니다. 이름 바꾸기가 완료되면 원래 이름을 가진 새로운 빈 로그 파일이 다시 열리며 로깅이 재개됩니다. 단일 메시지가 두 개의 파일로 분리되어도 롤오버의 결과로 유실되는 메시지는 없습니다.

이 옵션을 선택하는 경우 다음 매개변수를 지정해야 합니다.

최대 파일 크기
출력 파일이 증가할 수 있는 최대 크기(MB)를 지정합니다.

이 속성은 파일 크기 속성이 선택되어 있을 때에만 유효합니다. 파일이 이 크기에 도달하면 위에 설명된 대로 롤오버됩니다.

이력 파일의 최대 수
유지할 최대 롤오버 파일 수를 지정합니다.
파일 이름
추적 출력이 기록되는 파일의 이름을 지정합니다.

추적 출력 형식  
추적 출력의 형식을 지정합니다.

추적 출력에 대한 세 레벨 중 하나를 지정할 수 있습니다.

기본(호환 가능)
기본적인 추적 정보만 보관합니다. 이 옵션을 선택하여 추적 출력이 차지하는 공간의 양을 최대화하십시오.
고급
보다 특정적인 추적 정보를 보관합니다. 이 옵션을 선택하여 문제점 해결 및 문제점 판별에 사용할 자세한 추적 정보를 참조하십시오.
로그 분석기
로그 분석기 도구와 호환 가능한 형식으로 추적 정보를 보관합니다. 추적 출력을 로그 분석기 도구에 대한 입력으로 사용하려는 경우 이 옵션을 선택하십시오.

런타임 탭  
추적 출력의 형식을 지정합니다.
추적 저장

추적 출력은 출력 파일에 직접 작성되거나 런타임 페이지의 덤프 단추를 사용하여 메모리에 저장되거나 요구되는 파일에 기록될 수 있습니다.

메모리 버퍼
추적 출력은 메모리내 순환 버퍼에 기록되도록 지정합니다. 이 옵션을 선택하는 경우 다음 매개변수를 지정해야 합니다.
최대 버퍼 크기
버퍼에서 캐시될 수 있는 항목 수(수천)를 지정합니다. 이 수를 초과하면 기존의 항목이 새 항목으로 겹쳐쓰기됩니다.
덤프 파일 이름
메모리 버퍼가 덤프될 때 기록되는 파일의 이름. 이 옵션은 런타임 탭에서만 사용할 수 있습니다.

파일
자체 관리 로그 파일에 추적 결과를 작성하도록 지정합니다.

자체 관리 로그 파일은 크기 기준에 도달할 때까지 파일에 메시지를 작성합니다. 파일이 지정된 크기에 도달할 경우 로깅이 임시로 일시중단되고 로그 파일이 닫히고 이름 변경됩니다. 새 이름은 파일의 원래 이름 및 이름 바꾸기가 발생할 시기를 표시하는 시간 소인 규정자를 기준으로 합니다. 이름 바꾸기가 완료되면 원래 이름을 가진 새로운 빈 로그 파일이 다시 열리며 로깅이 재개됩니다. 단일 메시지가 두 개의 파일로 분리되어도 롤오버의 결과로 유실되는 메시지는 없습니다.

이 옵션을 선택하는 경우 다음 매개변수를 지정해야 합니다.

최대 파일 크기
출력 파일이 증가할 수 있는 최대 크기(MB)를 지정합니다.

이 속성은 파일 크기 속성이 선택되어 있을 때에만 유효합니다. 파일이 이 크기에 도달하면 위에 설명된 대로 롤오버됩니다.

이력 파일의 최대 수
유지할 최대 롤오버 파일 수를 지정합니다.
파일 이름
추적 출력이 기록되는 파일의 이름을 지정합니다.
