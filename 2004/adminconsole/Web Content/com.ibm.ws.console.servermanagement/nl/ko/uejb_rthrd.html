스레드 풀 설정

이 페이지를 참조하여 Application Server가 사용하는 스레드 그룹을 구성하십시오. 요청은 HTTP 전송을 통해 서버로 전송됩니다. 스레드 풀을 사용하면 서버의 구성요소가 스레드를 재사용하여 실행 시 새 스레드를 작성할 필요가 없어집니다. 새 스레드를 작성하면 시간 및 자원이 확장됩니다.

이 관리 콘솔 페이지를 보려면 서버 > Application Servers 관리 > server_name > ORB 서비스 > 스레드 풀을 클릭하십시오.(둘 이상의 탐색 라우트를 통해 이 페이지에 도달할 수 있습니다.)

구성 탭

최소 크기
풀에 허용할 스레드의 최소 수를 지정합니다.
데이터 유형 정수
기본값 10
최대 크기
풀에 허용할 스레드의 최대 수를 지정합니다.

Tivoli 성능 표시기가 표시하는 최대 퍼센트 측정값이 계속 두 자릿수에 머무르면, 최대 크기 증가에 대해 고려하십시오. 최대 퍼센트 측정값은 구성한 스레드를 사용하는 시간을 표시합니다. 서버측 ORB로 동시에 많은 클라이언트가 연결 중이면 크기를 늘려 클라이언트를 최대 1000까지지원하도록 하십시오.

데이터 유형 정수
기본값 50
권장 50(Linux 시스템에서는 25)
스레드 비활성화 제한시간
스레드 재이용 전에 경과해야 하는 비활성 시간(밀리초)을 지정합니다. 0 값은 대기하지 않는 것을 표시하며 음수 값(0 미만)은 영구 대기를 의미합니다.
데이터 유형 정수
단위 밀리초
기본값 3500
증가 가능
스레드 풀에 대해 구성된 최대 크기를 넘어 스레드 수가 증가할 수 있는지 여부를 지정합니다.
데이터 유형 부울
기본값 사용 불가능(false)
범위 유효한 값은 최대 스레드 크기를 넘는 스레드 할당 허용 또는 사용 불가능입니다.
