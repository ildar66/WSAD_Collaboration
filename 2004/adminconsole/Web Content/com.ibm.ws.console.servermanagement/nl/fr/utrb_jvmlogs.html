Paramètres des journaux de la JVM

Cette page permet d'afficher et de modifier les paramètres des journaux System.out et System.err de la JVM (Java Virtual Machine).

Pour afficher cette page de la console d'administration, cliquez sur Résolution des incidents > Journaux et trace >serveur > Journaux de la JVM.

Onglet Configuration

Nom de fichier
Indique le nom de l'un des fichiers journaux décrits dans cette page.

La zone du premier nom de fichier indique le nom du journal System.out. La zone du deuxième nom de fichier indique le nom du journal System.err.

Cliquez sur le bouton Afficher dans l'onglet Exécution pour visualiser le contenu du journal sélectionné.

Le nom de fichier spécifié pour le journal System.out ou System.err doit avoir l'une des valeurs suivantes :

nom de fichier
Nom d'un fichier du système de fichiers. Utilisez de préférence un nom de fichier complet. A défaut, le système considère que le chemin d''accès au fichier correspond au répertoire de travail en cours du serveur. Chaque flux doit être configuré avec un fichier dédié. Par exemple, vous ne pouvez pas réacheminer à la fois les fichiers System.out et System.err vers le même fichier physique.

Si le répertoire contenant le fichier existe déjà, l'ID utilisateur sous lequel est exécuté le serveur doit posséder des droits en lecture/écriture sur ce répertoire. Si le répertoire n'existe pas, il est créé avec les droits appropriés. L'ID utilisateur sous lequel le serveur est exécuté doit disposer des droits d'accès nécessaires à la création du répertoire.

console
Nom de fichier spécial permettant de réacheminer le flux vers le flux de processus correspondant. Si cette valeur est indiquée pour System.out, le fichier est réacheminé vers stdout. Si cette valeur est indiquée pour System.err, le fichier est réacheminé vers stderr.
none
Annule toutes les données écrites dans le flux. Le fait de spécifier none équivaut à réacheminer le flux vers dev/null sur un système Unix.

Formatage du fichier
Indique le format à utiliser pour la sauvegarde du fichier System.out.
Rotation des fichiers journaux

Cet ensemble d'attributs de configuration permet de configurer le fichier journal System.out ou System.err de manière à activer la gestion automatique.

Un fichier journal à gestion automatique enregistre des messages jusqu'à ce que le critère date/heure ou taille soit rempli. A la date indiquée ou lorsque la limite de taille spécifiée est atteinte, la consignation est temporairement suspendue et le fichier journal est fermé et renommé. Le nouveau nom du fichier est basé sur le nom d'origine auquel s'ajoute un qualifiant d'horodatage indiquant la date du changement de nom. Une fois que le fichier a été renommé, un nouveau fichier journal portant le nom d'origine s'ouvre et la consignation reprend. Tous les messages sont conservés une fois le fichier renommé mais il peut arriver qu'un message soit réparti entre l'ancien fichier et le fichier en cours.

Un journal ne peut être configuré pour une gestion automatique que si le flux correspondant est réacheminé vers un fichier.

Taille du fichier
Sélectionnez cet attribut pour activer la gestion automatique du fichier journal en fonction de sa taille. Le fichier est remplacé lorsqu'il atteint la taille que vous avez définie dans la zone Taille maximale.
Taille maximale
Indique la taille maximale du fichier (en Mo). Lorsque le fichier atteint cette taille, un autre se substitue à lui.

Cet attribut n'est accessible que si l'attribut Taille du fichier est défini.

Heure
Sélectionnez cet attribut pour activer la gestion automatique du fichier journal en fonction de l'heure de la journée. Le fichier est substitué à l'heure indiquée dans la zone Heure de début.
Heure de début
Indique l'heure du premier lancement de l'algorithme de substitution périodique (comprise entre 1 et 24) après le redémarrage d'un serveur d'applications. Cet algorithme se charge au démarrage du serveur d'applications. Une fois lancé à l'heure indiquée (heure de début), le fichier est substitué selon la fréquence définie par la zone Fréquence. Cette substitution continue ainsi jusqu'à l'arrêt du serveur d'applications.
Intervalle
Indique la fréquence de substitution d'un fichier (nombre d'heures compris entre 1 et 24).
Intervalle
Indique le nombre d'heures au-delà duquel un nouveau journal se substitue au journal existant. Cette valeur doit être comprise entre 1 et 24.

Vous pouvez configurer la substitution d'un fichier journal par un autre à la fois en fonction de l'heure, de la taille du fichier, ou de ces deux critères à la fois. Cliquez sur Taille du fichier et Heure pour substituer le fichier dès que l'un de ces critères est rempli. Par exemple, si la fréquence est de 5 heures et que la taille maximale du fichier est de 2 Mo, le fichier est substitué toutes les 5 heures à moins qu'il n'atteigne la taille maximale entre temps. Une fois le fichier substitué parce qu'il a atteint sa taille maximale, la substitution du fichier continue à la fréquence définie.

Nombre maximal des fichiers journaux historiques

Indique le nombre de fichiers historiques (fichiers remplacés par d'autres) à conserver. Les données sont consignées dans le fichier en cours jusqu'à ce qu'il soit remplacé. A ce moment, le fichier en cours est fermé et enregistré sous un nouveau nom. Ce nom reprend le nom du fichier en cours et un horodatage propre à la substitution lui est ajouté. Un nouveau fichier est alors rouvert avec le nom du fichier d'origine et les données y sont consignées. Le nombre de fichiers historiques passe de zéro à la valeur maximale définie pour le nombre de fichiers historiques. Lorsque le fichier est à nouveau substitué, le fichier historique le plus ancien est supprimé.

Sortie des applications installées

Indique si les instructions d'impression System.out ou System.err provenant du code de l'application sont consignées et formatées.

Affichage des instructions d'impression des applications
Cliquez sur cette zone pour afficher les messages que les applications consignent dans le flux à l'aide des méthodes de flux print et println. Les messages système WebSphere Application Server sont toujours visibles.
Formatage des instructions d'impression
Cliquez sur cette zone pour formater les instructions d'impression telles que les messages système de WebSphere Application Server.
