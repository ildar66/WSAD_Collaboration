Paramètres de l'environnement réparti

Cette page permet d'indiquer un type pour la sauvegarde d'une session dans un environnement réparti.

Pour afficher cette page de la console d'administration, cliquez sur Serveurs > Serveurs d'applications > nom_serveur > Conteneur Web > Gestion de session > Distributed Environment Settings.

Onglet Configuration

Sessions distribuées
Indique le type d'environnement distribué à utiliser pour sauvegarder les sessions.

Aucune Indique que la fonction de gestion de session ignore les données de la session lors de l'arrêt du serveur.
Base de données Indique que la fonction de gestion de la session enregistre les informations de session dans la source de données indiquée par les paramètres de connexion à la source de données. Cliquez sur Base de données pour modifier ces paramètres de source de données.
Réplication de mémoire à mémoire Indique que la fonction de gestion de la session enregistre les informations de session dans une source de données en mémoire. Les informations de session sont copiées vers les autres fonctions de gestion de session afin de permettre la reprise sur incident. Cliquez sur Réplication de mémoire à mémoire pour indiquer le réplicateur à utiliser et modifier les paramètres de mémoire à mémoire. (Uniquement pour WebSphere Application Server Network Deployment.)
