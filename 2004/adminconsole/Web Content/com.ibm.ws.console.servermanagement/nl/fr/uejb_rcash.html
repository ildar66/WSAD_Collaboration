Paramètres de la cache des EJB

Utilisez cette page pour configurer et gérer la mémoire cache pour un conteneur EJB spécifique. Pour déterminer la limite absolue de la cache (nombre maximal d'instances de beans enterprise qu'elle peut contenir), multipliez le nombre de beans actifs dans une transaction donnée par le nombre total de transactions concurrentes attendues. Ajoutez ensuite le nombre d'instances de bean session actives. Vous pouvez utiliser Tivoli Performance Viewer pour afficher les informations relatives aux performances des beans.

Pour afficher cette page de la console d'administration, cliquez sur Serveurs > Serveurs d'applications > nomServeur > Conteneur d'EJB > Paramètres de la cache des EJB.

Onglet Configuration

Intervalle entre les nettoyages
Indique la fréquence à laquelle le conteneur tente de supprimer les éléments inutilisés dans la mémoire cache afin que le nombre total d'éléments de la mémoire cache soit égal à la valeur de la taille de la mémoire cache.

Le gestionnaire de mémoire cache tente de maintenir des entrées non allouées qui pourront l'être rapidement en cas de besoin. Une unité d'exécution en tâche de fond tente de libérer des entrées tout en maintenant des entrées non allouées. Si l'unité d'exécution est exécutée alors que le serveur est inactif, lorsque le serveur d'applications doit allouer de nouvelles entrées de cache, il n'a pas besoin de supprimer d'entrées dans la mémoire, ce qui réduit les performances. D'une manière générale, augmentez la valeur de ce paramètre si la taille de la mémoire cache augmente.

Type de données Entier
Unités Millisecondes
Plage de valeurs Supérieur à 0
Valeur par défaut 3000
Taille de la cache
Indique le nombre d'emplacements dans la liste d'instances actives dans le conteneur EJB.

Un emplacement peut contenir plusieurs instances de bean enterprise actives mais les performances augmentent si un nombre restreint d'instances est affecté à chaque compartiment de la table. Lorsque le nombre d'instances actives dans le conteneur dépasse le nombre d'emplacements, c'est-à-dire la taille de la mémoire cache, le conteneur tente périodiquement de réduire le nombre d'instances actives dans la table en passivant certaines instances actives. Pour équilibrer au mieux les performances et la mémoire, définissez cette valeur en indiquant le nombre maximal d'instances actives attendues pour une charge de travail classique.

Type de données Entier
Unités Nombre d'emplacements dans la table de hachage
Plage de valeurs Supérieure à 0. Le conteneur sélectionne le nombre premier le plus élevé supérieur ou égal à la valeur spécifiée.
Valeur par défaut 2053
