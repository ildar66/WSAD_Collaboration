Paramètres du service de trace de diagnostic

Cette page permet de revoir et de modifier les propriétés du service de trace des diagnostics.

Pour afficher cette page, cliquez sur Résolution des incidents > Journaux et trace > serveur > Trace des diagnostics.

Onglet Configuration

Activer la trace
Active le service de trace.

Si cette option n'est pas sélectionnée, les propriétés de configuration suivantes ne sont pas transmises au service de trace du serveur d'applications lors du démarrage du serveur.

Spécification de trace
Indique les détails du traçage.

Entrez une chaîne de trace définissant les composants, les packages ou les groupes à tracer. La chaîne de trace doit respecter la syntaxe présentée ci-dessous. Vous pouvez entrer la chaîne de trace directement ou générer celle-ci à l'aide de l'interface graphique de trace. Cliquez sur Modifier pour lancer l'interface graphique de trace.

Si vous lancez l'interface graphique de trace à partir de l'onglet de configuration, une liste des composants, packages et groupes reconnus s'affiche. Cette liste n'est peut-être pas exhaustive.

Si vous lancez l'interface graphique de trace à partir de l'onglet d'exécution, la liste des composants, packages et groupes contient tous les éléments de ce type actuellement enregistrés sur le serveur actif.

Le format de la spécification de trace est le suivant :

<composant> = <type_trace>= enabled | disabled
où : <composant> est le composant pour lequel le traçage est activé ou désactivé et <type_trace> est le type de traçage à activer ou désactiver. Plusieurs spécifications de traçage doivent être séparés par deux points (:).

Les composants correspondent aux packages et classes Java ou aux collections de packages Java. Utilisez un caractère * comme caractère générique pour indiquer les composants incluant toutes les classes dans tous les packages contenus par le composant spécifié. Par exemple,

*
Indique tous les éléments de code traçables exécutés sur le serveur d'applications, y compris le code système et le code client WebSphere Application Server.
com.ibm.ws.*
Spécifie toutes les classes dont le nom de package commence par com.ibm.ws.
com.ibm.ws.classloader.JarClassLoader
Spécifie uniquement la classe JarClassLoader.

Pour plus d'informations sur la syntaxe de la chaîne de trace, reportez-vous à l'article Activation de la trace dans l'InfoCenter de WebSphere Application Server.

Important: Une erreur peut se produire lors de la définition d'une spécification de trace à partir de la console d'administration si des éléments sont sélectionnés à la fois dans les listes Groupes et Composants. Dans certains cas, les éléments sélectionnés dans une liste ne sont pas perdus lors de l'ajout d'une sélection dans l'autre liste. Pour éviter cet incident, tapez la spécification de trace directement dans la zone d'entrée Spécification de trace.

Sortie de trace  
Indique l'emplacement où la sortie de trace doit être enregistrée.

La sortie de trace peut être enregistrée directement dans un fichier de sortie ou stockée en mémoire et enregistrée dans un fichier sur demande par le biais du bouton Vidage disponible dans la page d'exécution.

Mémoire tampon
Indique que la sortie de trace doit être enregistrée dans une mémoire tampon circulaire. Si vous sélectionnez cette option, vous devez indiquer les paramètres suivants :
Taille maximale de la mémoire tampon
Indique le nombre d'entrées, en milliers, pouvant être mises en mémoire tampon. En cas de dépassement, les nouvelles entrées remplacent les anciennes.
Nom du fichier de vidage
Nom du fichier dans lequel sont transférées les données stockées en mémoire tampon lors du vidage. Cette option est disponible uniquement à partir de l'onglet Contexte d'exécution.

Fichier
Indique que la sortie de trace doit être enregistrée dans un fichier journal à gestion automatique.

Le fichier journal à gestion automatique enregistre des messages jusqu'à ce qu'il atteigne sa taille maximale. Lorsque cette limite est atteinte, la consignation est temporairement suspendue et le fichier journal est fermé et renommé. Le nouveau nom est basé sur le nom d'origine auquel s'ajoute un qualifiant d'horodatage indiquant la date du changement de nom. Une fois que le fichier a été renommé, un nouveau fichier journal portant le nom d'origine s'ouvre et la consignation reprend. Aucun message n'est perdu à la suite de cette substitution, bien qu'un seul message puisse être scindé en deux fichiers.

Si vous sélectionnez cette option, vous devez indiquer les paramètres suivants :

Taille maximale du fichier
Indique la taille maximale, en méga-octets, du fichier de sortie.

Cet attribut est valide uniquement si l'attribut Taille du fichier est sélectionné. Lorsque le fichier atteint cette taille, un autre se substitue à lui comme indiqué plus haut.

Nombre maximal de fichiers historiques
Indique combien de fichiers remplacés par d'autres doivent être conservés.
Nom de fichier
Indique le nom du fichier dans lequel est enregistrée la sortie de trace.

Fichier de sortie de trace  
Indique le format de la sortie de trace.

Vous pouvez spécifier l'un des trois niveaux suivants pour la sortie de trace :

De base (Compatible)
Conserve uniquement les données de trace de base. Sélectionnez cette option pour réduire la quantité d'espace occupée par la sortie de trace.
Avancé
Conserve des données de trace plus spécifiques. Sélectionnez cette option pour obtenir des données de trace détaillées utiles pour l'identification et la résolution des incidents.
Analyseur de journaux
Conserve les données de trace dans un format compatible avec l'outil du même nom. Sélectionnez cette option si vous souhaitez utiliser la sortie de trace comme entrée de l'Analyseur de journaux.

Indique le format de la sortie de trace.
Enregistrer la trace
Enregistre les modifications apportées dans l'onglet de l'environnement d'exécution dans la configuration de trace également.

Cochez cette case pour copier également les modifications de trace d'exécution dans les paramètres de configuration de trace. La sauvegarde de ces modifications dans la configuration de trace conserve les modifications même si l'application est redémarrée.
