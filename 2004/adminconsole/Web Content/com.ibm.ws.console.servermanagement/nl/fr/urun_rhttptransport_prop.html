Paramètres de transport HTTP

Utilisez cette page pour afficher et configurer un transport HTTP. Le nom de la page peut être celui d'un paramètre SSL tel que Paramètres SSL par défaut.

Pour afficher la page de la console d'administration, sélectionnez Serveurs > Serveurs d'applications > nom_serveur > Conteneur Web > Transports HTTP > nom_hôte.

Onglet Configuration

Hôte
Indique l'adresse IP de l'hôte auquel se connecter pour le transport.

Si le serveur d'applications réside sur une machine locale, le nom d'hôte peut être localhost.

Type de données Chaîne
Port
Indique le port auquel se connecter pour le transport. Spécifiez un numéro de port compris entre 1 et 65535. Il doit être unique pour chaque serveur d'applications sur une machine donnée.
Type de données Entier
Plage de valeurs 1 à 65535
SSL activé
Indique s'il est nécessaire de protéger les connexions entre le plug-in WebSphere et le serveur d'applications à l'aide du protocole SSL (Secure Sockets Layer). Par défaut, le protocole SSL n'est pas utilisé.
Type de données Booléen
Valeur par défaut false
SSL
Indique le type de paramètre SSL (Secure Sockets Layer) pour les connexions entre le plug-in WebSphere et le serveur d'applications. Les options incluent un ou plusieurs paramètres SSL définis dans le Centre de sécurité. Par exemple, Paramètres SSL par défaut, Paramètres SSL de l'ORB ou Paramètres SSL LDAP.
Type de données Chaîne
Valeur par défaut Paramètre SSL défini dans le Centre de sécurité.
