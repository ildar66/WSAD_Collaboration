Paramètres du conteneur EJB

Utilisez cette page pour configurer et gérer un conteneur EJB spécifique.

Pour afficher cette page de la console d'administration, cliquez sur Serveurs > Serveurs d'applications > nomServeur > Conteneur d'EJB.

Onglet Configuration

Répertoire de passivation
Indique le répertoire dans lequel le conteneur enregistre l'état persistant des beans session à l'état passif.

Les beans sont mis à l'état passif lorsque le nombre d'instances de beans actifs est supérieur à la taille de la mémoire cache indiquée lors de la configuration du conteneur. Lorsqu'un bean avec état est mis à l'état passif, le conteneur sérialise l'instance de bean dans un fichier dans le répertoire de passivation et supprime l'instance dans la mémoire cache du bean. Si par la suite une demande arrive pour l'instance de bean à l'état passif, le conteneur l'extrait du répertoire de passivation, la désérialise, la renvoie dans la mémoire cache et répartit la demande. Si l'une de ces étapes ne peut être effectuée (par exemple, si l'instance de bean concernée ne se trouve plus dans le répertoire de passivation), l'appel de la méthode échoue.

Intervalle de nettoyage de groupes inactifs
Indique l'intervalle selon lequel le conteneur examine les groupes d'instances de beans disponibles afin de déterminer les instances qui peuvent être supprimées pour réduire la mémoire utilisée.
Type de données Entier
Unités Millisecondes
Plage de valeurs Supérieur à 0
Nom JNDI de la source de données par défaut
Indique le nom JNDI d'une source de données à utiliser si aucune source de données n'est spécifiée lors du déploiement d'une application. Ce paramètre ne s'applique pas aux beans CMP compatibles avec la spécification EJB 2.x.

Les servlets et les beans enterprise utilisent des sources de données pour obtenir ces connexions. Lors de la configuration d'un conteneur, vous pouvez spécifier une source de données par défaut pour le conteneur. Cette source de données devient la source de données par défaut utilisée par les beans entity dans le conteneur qui utilise la fonction de persistance gérée par le conteneur (CMP).

La source de données par défaut d'un conteneur est sécurisée. Lorsque vous spécifiez la source, vous devez indiquer un ID utilisateur et un mot de passe pour accéder à la source de données.

Il n'est pas obligatoire d'indiquer une source de données par défaut si chaque bean entity CMP du conteneur possède une source de données spécifiée dans sa configuration. Si aucune source de données par défaut n'est indiquée et qu'un bean entity CMP pour lequel aucune source de données n'est indiquée est installé dans le conteneur, les applications ne peuvent pas utiliser ce bean.

Etat initial
Indique l'état d'exécution demandé lors du premier démarrage du serveur.
Type de données Chaîne
Valeur par défaut Démarré
Plage de valeurs Les valeurs valides sont Démarré et Arrêté
