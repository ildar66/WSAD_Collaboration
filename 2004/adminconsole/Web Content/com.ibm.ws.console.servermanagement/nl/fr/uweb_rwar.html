Paramètres de déploiement du module Web

Cette page permet de configurer une instance de déploiement du module Web.

Pour afficher cette page de la console d'administration, cliquez sur Applications > Applications d'entreprise > instance_application > Web Modules > instance_module_Web.

URI
Spécifie un URI qui, une fois résolu par rapport à l'URL de l'application, indique l'emplacement du contenu de l'archive du module dans un système de fichiers. Si ce module constitue l'un des composants intégrés dans l'archive d'une application déployée (fichier EAR), l'URI indiqué ici doit correspondre à une référence de module (ModuleRef) spécifiée dans le descripteur de déploiement de cette application.
Descripteur de déploiement secondaire
Spécifie le nom de fichier d'un autre descripteur de déploiement à utiliser à la place du descripteur original se trouvant dans le fichier JAR du module.

Ce fichier est la version post-assemblage du fichier de descripteur de déploiement. Vous pouvez éditer le fichier descripteur de déploiement original pour résoudre les interdépendance et les informations de sécurité. L'utilisation d'un autre descripteur vous permet de conserver intact le descripteur d'origine.

La valeur de la propriété Autre DD doit être le nom complet du fichier descripteur de déploiement, spécifié par rapport au répertoire racine du module. Par convention, ce fichier se trouve dans le répertoire ALT-INF. Si cette propriété n'est pas spécifiée, le fichier descripteur de déploiement est lu directement dans le fichier JAR du module.

Pondération de démarrage
Détermine l'ordre de démarrage des différents modules de l'application. Les modules sont démarrés par ordre croissant de poids.
Mode chargeur de classe
Indique si le chargeur de classe doit d'abord chercher dans le chargeur de classe parent ou dans le chargeur de classe de l'application pour charger une classe. La norme pour les chargeurs de classe JDK et les chargeurs de classe WebSphere est PARENT_FIRST. Si vous indiquez PARENT_LAST, votre application peut remplacer des classes contenues dans le chargeur de classe parent, mais cette action peut éventuellement aboutir à l'exception ClassCastException ou à des erreurs LinkageErrors si vous avez utilisé aussi bien des classes remplacées que des classes non remplacées.

Les options possibles sont PARENT_FIRST et PARENT_LAST. Par défaut, la recherche s'effectue d'abord dans le chargeur de classe parent, puis dans le chargeur de classe d'application pour charger une classe.

Type de données Chaîne
Valeur par défaut PARENT_FIRST
