Paramètres de base de données

Cette page permet d'indiquer les paramètres de support de la session de base de données.

Pour afficher cette page de la console d'administration, cliquez sur Serveurs > Serveurs d'applications > nom_serveur > > Conteneur Web > Gestion de session > Paramètres de l'environnement distribué > Base de données.

Onglet Configuration

Nom JNDI de la source de données
Indique la description des la source de données

Nom JNDI de la source de données non XA à partir de laquelle l'utilitaire de gestion de session obtient des connexions à la base de données. Par exemple, si le nom JNDI de la source de données est "jdbc/sessions", spécifiez "jdbc/sessions". La source de données représente à la fois un pool de connexions à une base de données et les paramètres de configuration de ce pool (par exemple, sa taille). La source de données doit déjà exister en tant que ressource configurée dans l'environnement.

ID utilisateur  
Indique l'ID utilisateur pour l'accès à la base de données.
Mot de passe  
Indique le mot de passe permettant d'accéder à la base de données.
Confirmation du mot de passe  
Indique le mot de passe une seconde fois pour vous garantir qu'il est correctement enregistré.
Taille de ligne DB2  
Indique la taille de page d'espace table configurée pour la table des sessions, si vous utilisez une base de données DB2. Valeurs possibles: 4, 8, 16 et 32 kilo-octets (ko). La taille de ligne par défaut est 4 ko.

La taille de ligne par défaut est 4 ko. Dans DB2, elle peut être modifiée pour une valeur supérieure. Cela permet d'améliorer les performances de la base de données dans certains environnements. Si vous choisissez une valeur autre que 4, vous devez spécifier le nom de l'espace table à utiliser. Pour les pages de 4 ko, l'indication d'un nom d'espace table est facultative.

Nom de l'espace table  
Indique l'espace table à utiliser pour la table des sessions.

Cette valeur est requise lorsque la taille des pages DB2 est différente de 4 ko.

Utiliser des sessions multilignes
Indique que chaque instance de données d'application doit être placée dans une ligne distincte de la base de données, autorisant ainsi le stockage d'une plus grande quantité de données pour chaque session. Cette opération peut améliorer les performances dans certains cas. Si l'utilisation d'un schéma multiligne n'est pas activée, les différentes instances de données d'application pourront être placées sur la même ligne.
