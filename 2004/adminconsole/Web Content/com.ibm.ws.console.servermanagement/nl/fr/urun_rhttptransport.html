Collection de transports HTTP

Cette page permet d'afficher ou de gérer les transports HTTP. Les transports génèrent des files d'attente de demandes entre les plug-ins WebSphere pour les serveurs Web et les conteneurs Web dans lesquels résident les modules Web des applications. Lorsque vous appelez une application à partir d'un navigateur Web, la demande est transmise au serveur Web, puis via le transport au conteneur Web.

Pour afficher la page de la console d'administration, sélectionnez Serveurs > Serveurs d'applications > nom_serveur > Conteneur Web > Transports HTTP.

Hôte
Indique l'adresse IP de l'hôte auquel se connecter pour le transport. Si le serveur d'applications réside sur une machine locale, le nom d'hôte peut être localhost.
Port
Indique le port auquel se connecter pour le transport. Le numéro du port peut correspondre à n'importe quel port en cours d'utilisation sur le système. Il doit être unique pour chaque instance de serveur d'applications sur une machine donnée.
SSL activé
Indique s'il est nécessaire de protéger les connexions entre le plug-in WebSphere et le serveur d'applications à l'aide du protocole SSL (Secure Sockets Layer). Par défaut, le protocole SSL n'est pas utilisé.
