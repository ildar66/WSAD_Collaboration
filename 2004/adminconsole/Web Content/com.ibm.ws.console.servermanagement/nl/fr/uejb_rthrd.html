Paramètres du pool d'unités d'exécution

Utilisez cette page pour configurer un groupe d'unités d'exécution utilisé par le serveur d'applications. Les demandes sont envoyées au serveur via l'un des transports HTTP. Un groupe d'unités d'exécution permet à des composants du serveur de réutiliser des unités d'exécution afin de ne pas avoir à recréer d'autres unités lors de l'exécution. La création d'unités d'exécution contribue à augmenter les délais et les ressources.

Pour afficher cette page de la console administrative, cliquez sur Serveurs > Gérer les serveurs d'applications > nom_serveur > Service ORB > Pool d'unités. (Il existe d'autres chemins de navigation pour accéder à cette page.)

Onglet Configuration

Taille minimale
Indique le nombre minimal d'unités d'exécution autorisées dans le groupe.
Type de données Entier
Valeur par défaut 10
Taille maximale
Spécifie le nombre maximal d'unités d'exécution à allouer dans le pool.

Si Tivoli Performance Viewer affiche la plupart du temps un pourcentage maximal à deux chiffres, pensez à augmenter la taille maximale. Ce pourcentage indique la durée d'utilisation des unités d'exécution. Si plusieurs clients sont connectés simultanément à l'ORB côté serveur, la valeur de ce paramètre peut être augmentée afin d'accepter une charge élevée (jusqu'à 1000 clients).

Type de données Entier
Valeur par défaut 50
Valeur conseillée 50 (25 sous Linux)
Délai d'inactivité de l'unité
Indique le nombre de millisecondes d'inactivité à l'issue desquelles une unité d'exécution est récupérée. La valeur 0 correspond à un délai nul et une valeur négative (inférieure à 0) correspond à un délai illimité.
Type de données Entier
Unités Millisecondes
Valeur par défaut 3 500
Evolutif
Indique si le nombre d'unités d'exécution peut être augmenté au-delà de la taille maximale configurée pour le groupe d'unités d'exécution.
Type de données Elément booléen
Valeur par défaut Non activé (false)
Plage de valeurs Les valeurs valides sont Permet d'allouer un nombre d'unités d'exécution dépassant la taille maximale autorisée et Non activé.
