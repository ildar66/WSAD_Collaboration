「一般 JMS 目的地」設定

請使用這個畫面來檢視或變更選取要搭配相關 JMS 提供者使用之 JMS 目的地的配置內容。

JMS 目的地用來配置相關聯的一般 JMS 提供者之 JMS 目的地的內容。 通往 JMS 目的地的連線是由相關聯的 JMS Connection Factory 來建立的。 要搭配一般 JMS 提供者（不是內嵌 WebSphere JMS Provider 或 WebSphere MQ JMS Provider）使用之 JMS 目的地有下列內容。

如果要檢視這個管理主控台頁面，請按一下資源 -> 一般 JMS 提供者 -> provider_name -> JMS 目的地 -> destination。

名稱
用來識別 IBM WebSphere Application Server 內管理作業所用之佇列的名稱。
資料類型 字串
類型
這個 JMS 目的地是佇列（用於點對點）或主題（用於發佈/訂閱）。

請選取下列選項之一：

佇列
點對點傳訊所用的 JMS Queue Destination。
主題
發佈/訂閱傳訊所用的 JMS Topic Destination。

JNDI 名稱
將佇列連結至應用程式伺服器名稱空間的 JNDI 名稱。

請依照慣例使用完整的 JNDI 名稱；比方說，在 jms/Name 格式中，Name 是資源的邏輯名稱。

這個名稱用來鏈結平台的連結資訊。 連結會將模組部署描述子所定義的資源關聯到由平台連結至 JNDI 內的實際（實體）資源。

資料類型 字串
說明
佇列的說明，供管理使用。
種類
就您的 IBM WebSphere Application Server 管理記錄，用來分類或分組這個佇列的種類。
資料類型 字串
外部 JNDI 名稱
將佇列連結至應用程式伺服器名稱空間的 JNDI 名稱。

請依照慣例使用完整的 JNDI 名稱；比方說，在 jms/Name 格式中，Name 是資源的邏輯名稱。

這個名稱用來鏈結平台的連結資訊。 連結會將模組部署描述子所定義的資源關聯到由平台連結至 JNDI 內的實際（實體）資源。
