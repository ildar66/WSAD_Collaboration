「WebSphere MQ 佇列」設定

請使用這個畫面，來檢視或變更選擇要搭配 WebSphere MQ JMS Provider 使用之 Queue Destination 的配置內容。

Queue Destination 用來配置 JMS 佇列的內容。 佇列的連線是由相關的 Queue Connection Factory 針對 WebSphere MQ JMS Provider 所建立。 與 WebSphere MQ JMS Provider 搭配使用的佇列，其內容如下。

附註:

  • 您指定的內容值必須符合在配置 WebSphere MQ JMS 資源時所指定的值。 如果需要配置 WebSphere MQ JMS 資源的詳細資訊，請參閱 WebSphere MQ 使用 Java 一書。
  • 在 WebSphere MQ 中，名稱最多可有 48 個字元，但通道例外，它最多只能有 20 個字元。

如果要檢視這個管理主控台頁面，請按一下資源 -> WebSphere MQ JMS Provider->（在內容窗格的「其他內容」下）WebSphere MQ Queue Destination -> destination_name

名稱
用來識別佇列供管理使用 IBM WebSphere Application Server 的名稱。
資料類型 字串
JNDI 名稱
將佇列連結至應用程式伺服器名稱空間的 JNDI 名稱。

請依照慣例使用完整的 JNDI 名稱；比方說，在 jms/Name 格式中，Name 是資源的邏輯名稱。

這個名稱用來鏈結平台的連結資訊。 連結會將模組部署描述子所定義的資源關聯到由平台連結至 JNDI 內的實際（實體）資源。

資料類型 字串
說明
佇列的說明，供管理使用。
資料類型 字串
預設值 空值
種類

資料類型 字串
持續性
傳送至目的地的所有訊息是不是全部具持續性、不具持續性，或是由應用程式來定義其持續性

資料類型 列舉

範圍
定義的應用程式
在這個目的地之訊息，其優先順序由將它們放入目的地的應用程式來定義。
已定義的佇列
[只適用於 WebSphere MQ 目的地] 在目的地的訊息，其持續性由 WebSphere MQ 佇列定義內容來定義。
指定的
在這個目的地之訊息，其優先順序由指定的優先順序內容來定義。 如果您選取這個選項，您必須在指定的優先順序內容中定義一個優先順序。
指定的優先順序
如果優先順序內容設為指定的，請在這裡輸入這個佇列的訊息優先順序，範圍是 0（最低）至 9（最高）。

如果優先順序內容設為指定的，送往這個佇列的訊息，其優先順序值由這個內容來指定。

預設值 空值
範圍 0（最低優先順序）至 9（最高優先順序）
期限
這個佇列的期限逾時值是由應用程式來定義，或由指定的期限內容來定義，或是佇列中的訊息永遠不會到期（期限逾時值無限）
資料類型 列舉
單位 不適用
預設值 APPLICATION_DEFINED
範圍
定義的應用程式
在這個佇列之訊息，其期限逾時值由將它們放入佇列的應用程式來定義。
指定的
在這個佇列之訊息，其期限逾時值由指定的期限內容來定義。 如果您選取這個選項，您必須在指定的期限內容中定義一個逾時值。
無限
這個佇列中的訊息沒有期限逾時值，因此，訊息永遠不會到期。
指定的期限
如果期限逾時值內容設為指定的，請在這裡輸入這個佇列中的訊息到期之間的毫秒數（大於 0）。
資料類型 整數
單位 毫秒
預設值 空值
範圍 大於或等於 0
  • 0 表示訊息永遠不會逾時
  • 其他值是整數的毫秒數
