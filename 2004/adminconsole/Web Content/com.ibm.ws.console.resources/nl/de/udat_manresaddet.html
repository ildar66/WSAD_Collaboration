Einstellungen für Ressourcenadapter

Auf dieser Seite können Sie Einstellungen für einen Ressourcenadapter angeben.

Es besteht die Möglichkeit, den Ressourcenadapter eines Fremdanbieters zu verwenden. Normalerweise kann ein einzelner Ressourcenadapter nur zu einem Back-End-Systemtyp (EIS) eine Verbindung herstellen, jedoch viele verschiedene Konfigurationen für Verbindungen zu diesem EIS unterstützen. Der Ressourcenadapter hat viele Konfigurationsmerkmale, die in der J2C-Spezifikation definiert sind und vom Hersteller des Codes festgelegt werden.

Klicken Sie, um diese Seite der Administrationskonsole anzuzeigen, auf Ressourcen > Ressourcenadapter > Ressourcenadapter.

Register "Konfiguration"

Bereich
Gibt die Ebene an, bis zu der diese Ressourcendefinition sichtbar ist - Zelle, Knoten oder Server.

WebSphere 5.0 lässt Ressourcen, wie z. B. JDBC-Provider, Namespace-Bindungen oder gemeinsam benutzte Bibliotheken in mehreren Bereichen zu. Dabei überschreiben Ressourcen, die in spezifischeren Bereichen definiert wurden, Ressourcen, die in allgemeineren Bereichen definiert wurden.

Zelle
Der allgemeinste Bereich. Ressourcen, die im Zellenbereich definiert werden, sind von allen Knoten und Servern sichtbar, vorausgesetzt, sie wurden nicht überschrieben. Möchten Sie Ressourcen anzeigen, die im Zellenbereich definiert wurden, dürfen Sie keinen Server oder Knotennamen im Formular für die Bereichsauswahl angeben.
Knoten
Der Standardbereich für die meisten Ressourcentypen. Ressourcen, die im Knotenbereich definiert werden, überschreiben alle Duplikate im Zellenbereich und sind auf allen Servern, die sich auf demselben Knoten befinden, sichtbar, vorausgesetzt, sie wurden nicht in einem Serverbereich auf diesem Knoten überschrieben. Möchten Sie Ressourcen anzeigen, die in einem Knotenbereich definiert wurden, dürfen Sie keinen Server angeben, sondern müssen einen Knotennamen im Formular für die Bereichsauswahl auswählen.
Server
Der spezifischste Bereich für die Definition von Ressourcen. Ressourcen, die im Serverbereich definiert werden, überschreiben alle Duplikate von Ressourcendefinitionen, die im Zellenbereich oder im übergeordneten Knotenbereich definiert wurden, und können nur von einem spezifischen Server erkannt werden. Möchten Sie Ressourcen anzeigen, die in einem Serverbereich definiert wurden, müssen Sie im Formular für die Bereichsauswahl einen Servernamen und einen Knotennamen auswählen.

Wenn Ressourcen erstellt werden, werden sie immer im aktuellen, in der Anzeige ausgewählten Bereich erstellt. Möchten Sie Ressourcen in anderen Bereichen anzeigen, geben Sie einen anderen Knoten oder Server im Formular für die Bereichsauswahl an.

Datentyp String
Name
Gibt den Namen des Ressourcenproviders an.

Eine Textzeichenfolge ohne Leerzeichen, die den Ressourcen-Provider sinnvoll bezeichnet.

Datentyp String
Beschreibung
Gibt eine Textbeschreibung der Ressourcen-Factory an.

Eine unformatierte Zeichenfolge zur Beschreibung der Ressourcen-Factory und ihres Zwecks.

Datentyp String
Pfad des Archivs
Gibt den Pfad zur .RAR-Datei an, die das Modul für diesen Ressourcenadapter enthält.
Datentyp String
Klassenpfad
Eine Liste von Pfaden bzw. Namen von Jar-Dateien, die zusammen die Speicherposition der Ressourcen-Provider-Klassen angeben.

Dies umfasst alle zusätzlichen Bibliotheken, die vom Ressourcenadapter benötigt werden. Die Codebasis des Ressourcenadapters wird dem Klassenpfad automatisch hinzugefügt. Wenn jedoch Daten außerhalb der .rar-Datei benötigt werden, können diese hier angegeben werden.

Datentyp String
Nativer Pfad
Eine Liste von Pfaden, die das Verzeichnis der nativen Bibliotheken des Ressourcenproviders angibt.

Die Codebasis des Ressourcenadapters wird dem Klassenpfad automatisch hinzugefügt. Wenn jedoch Daten außerhalb der .RAR-Datei benötigt werden, können diese hier angegeben werden.
