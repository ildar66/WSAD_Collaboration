URL-Provider

Auf dieser Seite können Sie neue URL-Provider erstellen, um URL-Protokolle, die vom JDK (Java Developer Kit) nicht verarbeitet werden, verarbeiten zu können, z. B. andere Protokolle als "http", "ftp" und "file". Ein URL-Provider implementiert die Funktionalität für ein bestimmtes URL-Protokoll. Dieser Provider besteht aus zwei Klassen, die java.net.URLStreamHandler und java.net.URLConnection erweitern. Ein Standard-URL-Provider ist in der Produktkonfiguration bei der Initialisierung enthalten. Dieser Provider verwendet die vom JDK bereitgestellte URL-Unterstützung. Alle URL-Ressourcen mit Protokollen, die auf der Java 2 Standard Edition 1.3.1 basieren, wie z. B. HTTP oder FTP, können den Standard-URL-Provider verwenden.

Klicken Sie, um diese Seite der Administrationskonsole anzuzeigen, auf Ressourcen > URL-Provider.

Name
Gibt den Administrationsnamen für den URL-Provider an.
Beschreibung
Beschreibt den URL-Provider für Ihre Administrationsdatensätze.
