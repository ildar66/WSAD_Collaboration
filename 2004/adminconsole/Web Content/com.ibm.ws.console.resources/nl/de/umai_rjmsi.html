Einstellungen für Mail-Sitzungen

Auf dieser Seite können Sie spezifische Mail-Provider konfigurieren.

Klicken Sie, um diese Seite der Administrationskonsole anzuzeigen, auf Ressourcen > Mail-Provider > Mail-Provider > Mail-Sitzungen > Mail-Sitzung.

Register "Konfiguration"

Name
Gibt den Administrationsnamen des JavaMail-Sitzungsobjekts an.
JNDI-Name
Gibt den JNDI-Namen (Java Naming and Directory Interface) für die Ressource einschließlich aller Subkontexte an.

Dieser Name wird als Verbindung zwischen den Bindungsinformationen der Plattform für die Ressourcen, die im Deployment-Deskriptor der Clientanwendung definiert sind, und den Ressourcen verwendet, die tatsächlich von der Plattform an die JNDI gebunden wurden.

Beschreibung
Eine optionale Beschreibung der Datensätze für die Administration.
Kategorie
Gibt eine optionale Gruppe für das Klassifizieren oder das Zusammenfassen von Sitzungen an.
Host für Mail-Transport
Gibt den Server an, auf den beim Senden von Mails zugegriffen wird.
Protokoll für Mail-Transport
Gibt das Transportprotokoll an, das beim Senden von Mails verwendet wird.
Benutzer-ID für Mail-Transport
Gibt die Benutzer-ID an, die verwendet werden soll, wenn der Host für Mail-Übertragung die Authentifizierung anfordert.

Diese Einstellung wird für die meisten Mail-Server nicht verwendet. Wenn Sie keinen Server verwenden möchten, für den eine Benutzer-ID und ein Kennwort erforderlich sind, lassen Sie dieses Feld leer.

Kennwort für Mail-Transport
Gibt das Kennwort an, das verwendet werden soll, wenn der Host für Mail-Transport die Authentifizierung anfordert.

Diese Einstellung wird für die meisten Mail-Server nicht verwendet. Wenn Sie keinen Server verwenden möchten, für den eine Benutzer-ID und ein Kennwort erforderlich sind, lassen Sie dieses Feld leer.

Mail von
Gibt den Absender der Mail an.

Dieser Wert bezeichnet die Internet-E-Mail-Adresse, die standardmäßig in der empfangenen Nachricht entweder im Feld Von oder Antwort erscheint. Die Antwort des Empfängers wird an diese Adresse gesendet.

Host für Mail-Speicher
Gibt den Server an, auf den beim Empfangen von Mails zugegriffen wird.

Diese Einstellung in Kombination mit der Benutzer-ID und dem Kennwort für den Mail-Speicher stellt einen gültigen Mail-Account dar. Lautet ein Mail-Account z. B. john_william@my.company.com, dann ist der Host für den Mail-Speicher my.company.com.

Protokoll für Mail-Speicher
Gibt das Protokoll an, das beim Empfangen von Mails verwendet wird, z. B. IMAP, POP3 oder ein Speicherprotokoll, für das der Benutzer einen Provider installiert hat.
Benutzer für Mail-Speicher
Gibt die Benutzer-ID für einen bestimmten Mail-Account an.

Lautet z. B. ein Mail-Account john_william@my.company.com, dann ist der Benutzer john_william.

Kennwort für Mail-Speicher
Gibt das Kennwort für einen bestimmten Mail-Account an.

Lautet z. B. ein Mail-Account john_william@my.company.com, dann müssen Sie das Kennwort für die Benutzer-ID john_william eingeben.

Debug
Schaltet den Debug-Modus für diese Mail-Sitzung an oder aus.
