Einstellungen des JMS-Providers

Wenn Sie einen anderen JMS-Provider als den eingebetteten WebSphere-JMS-Provider oder den WebSphere-MQ-JMS-Provider verwenden möchten, konfigurieren Sie in dieser Anzeige die Merkmale des JMS-Providers:

Klicken Sie zum Anzeigen dieser Seite der Administrationskonsole auf Ressourcen-> Generische JMS-Provider-> Providername.

Name
Der Name, mit dem der JMS-Provider für die Administration bezeichnet wird.
Datentyp String
Standardwert Keiner
Beschreibung
Eine Beschreibung des JMS-Providers für die Administration.
Datentyp String
Standardwert Keiner
Klassenpfad
Der Java-Klassenpfad des JMS-Providers. Die Liste der Pfade bzw. Name von JAR-Dateien, die zusammen die Speicherposition der JMS-Provider-Klassen angeben.
Datentyp String
Standardwert Keiner
Pfad der nativen Bibliothek
Der native Bibliothekspfad für den JMS-Provider. Ein optionaler Pfad zu den nativen Bibliotheken, die der JMS-Provider benötigt.
Datentyp String
Standardwert Keiner
Externe Kontext-Factory bei Initialisierung
Der Java-Klassenname der Ausgangskontext-Factory für den JMS-Provider.

Für einen LDAP-Service-Provider hat der Wert folgendes Format: com.sun.jndi.ldap.LdapCtxFactory.

Datentyp String
Standardwert Keiner
URL für externen Provider
Der URL des JMS-Providers für externe JNDI-Lookup-Operationen.

Beispielsweise hat ein LDAP-URL für einen JMS-Provider das Format: ldap://Hostname.Unternehmen.com/Kontextname.

Datentyp String
Standardwert Keiner
