Valores de cola de WebSphere MQ (Configuración de MQ)

Utilice este panel para ver o modificar las propiedades de configuración definidas en WebSphere MQ para el destino de cola seleccionado.

Un destino de cola se utiliza para configurar las propiedades de una cola JMS. Una cola que se vaya a utilizar con el proveedor de JMS de WebSphere MQ tiene las propiedades adicionales siguientes definidas para WebSphere MQ.

Notas:

  • Algunas propiedades mostradas son de sólo lectura y no se pueden modificar.
  • Para poder ver o modificar propiedades, el Queue Manager de WebSphere MQ en el que reside la cola tiene que estar configurado para la administración remota y debe estar ejecutándose.
  • Los valores de las propiedades que especifique deben coincidir con los valores especificados al configurar los recursos JMS de WebSphere MQ. Para obtener más información sobre la configuración de recursos JMS de WebSphere MQ, consulte la publicación WebSphere MQ Utilización de Java; por ejemplo en la página Web de la biblioteca multiplataforma de WebSphere MQ en    http://www.ibm.com/software/ts/mqseries/library/manualsa/manuals/crosslatest.html .
  • En WebSphere MQ, los nombres pueden tener un máximo de 48 caracteres, a excepción de los canales, que tienen un máximo de 20 caracteres.

Para ver esta página de la consola administrativa, pulse Recursos-> Proveedor de JMS de WebSphere MQ-> (en el panel de contenido, en Propiedades adicionales) Destinos de cola de WebSphere MQ-> nombre_destino-> (en el panel de contenido, en Propiedades adicionales) Configuración de MQ

Número máximo de mensajes permitidos en la cola.
Tipo de datos Integer
Unidades Mensajes
Valor por omisión
Rango Un valor mayor o igual a 0 y menor o igual a:

Para obtener más información sobre el valor máximo permitido, consulte WebSphere MQ MQSC Command Reference.

Umbral de retroceso
Número de veces máximo que un mensaje puede someterse a una operación de retroceso. Si se alcanza este umbral, el mensaje se vuelve a colocar en la cola de retroceso especificada mediante la propiedad Backout Requeue name (nombre de cola de retroceso).

WebSphere MQ Queue Manager mantiene un registro del número de veces que cada mensaje se somete a una operación de retroceso. Cuando este número alcanza un umbral configurable, el cliente de conexiones vuelve a colocar el mensaje en una cola de retroceso con nombre. Si esta operación no se realiza correctamente por alguna razón, el mensaje se elimina de la cola y se vuelve a colocar en la cola de elementos no reclamados o se descarta.
