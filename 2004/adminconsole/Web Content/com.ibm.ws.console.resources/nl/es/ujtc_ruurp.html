Colección de proveedores de URL

Utilice esta página para crear nuevos proveedores de URL para manejar los protocolos de URL que JDK (Java Developer Kit) no maneje, como por ejemplo, los protocolos que no sean http, ftp y file. Un proveedor de URL implementa la funcionalidad de un protocolo de URL determinado. Este proveedor consta de dos clases que amplían java.net.URLStreamHandler y java.net.URLConnection. En la configuración inicial del producto se incluye un proveedor de URL por omisión. Este proveedor utiliza el soporte de URL facilitado por JDK. Cualquier recurso de URL con protocolos basados en Java 2 Standard Edition 1.3.1, como por ejemplo, HTTP o FTP, puede utilizar el proveedor de URL por omisión.

Para ver esta página de la consola administrativa, pulse Recursos > Proveedores de URL.
