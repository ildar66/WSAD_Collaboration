Valores de fábrica de conexiones de cola de WebSphere MQ

Utilice este panel para ver o modificar las propiedades de configuración de la fábrica de conexiones de cola seleccionada que se va a utilizar con el proveedor de JMS de WebSphere MQ. Estas propiedades de configuración controlan cómo se crean las conexiones para el destino de cola JMS asociado.

Una fábrica de conexiones de cola sirve para crear conexiones de JMS con destinos de cola. El proveedor de JMS de WebSphere MQ crea la fábrica de conexiones de cola. Una fábrica de conexiones de cola del proveedor de JMS de WebSphere MQ tiene las propiedades siguientes.

Nota:

  • Los valores de las propiedades que especifique deben coincidir con los valores especificados al configurar los recursos JMS de WebSphere MQ. Para obtener más información sobre la configuración de recursos JMS de WebSphere MQ, consulte la publicación WebSphere MQ Utilización de Java y la publicación Websphere MQ Administración de sistemas, SC10-3802, que están disponibles en la página Web de publicaciones específicas de la plataforma de mensajería de WebSphere MQ en la dirección    http://www-3.ibm.com/software/ts/mqseries/library/manualsa/manuals/platspecific.html , en    IBM Publications Center o en WebSphere MQ collection kit, SK2T-0730.
  • En WebSphere MQ, los nombres pueden tener un máximo de 48 caracteres, a excepción de los canales, que tienen un máximo de 20 caracteres.
  • Puede utilizar la página Propiedades personalizadas para definir las propiedades SSL (Secure Sockets Layer) de WebSphere MQ. Para obtener más información sobre el establecimiento de estas propiedades, consulte Propiedades personalizadas.

Para ver esta página de la consola administrativa, pulse Recursos-> Proveedor de JMS de WebSphere MQ-> (en el panel de contenido, en Propiedades adicionales) Fábricas de conexiones de cola de WebSphere MQ-> fábrica_conexiones

Nombre
Nombre por el que se conoce esta fábrica de conexiones de cola para fines administrativos dentro de IBM WebSphere Application Server. El nombre debe ser exclusivo dentro de las fábricas de conexiones de JMS en el dominio administrativo de WebSphere.
Tipo de datos String
Nombre JNDI
Nombre JNDI utilizado para enlazar la fábrica de conexiones con el espacio de nombres del servidor de aplicaciones.

Como convención, utilice el nombre JNDI plenamente cualificado; por ejemplo, con el formato jms/Nombre, donde Nombre es el nombre lógico del recurso.

Este nombre se utiliza para enlazar la información de enlace de la plataforma. El enlace asocia los recursos definidos en el descriptor de despliegue del módulo con los recursos reales (físicos) enlazados con JNDI por la plataforma.

Tipo de datos String
Descripción
Descripción de esta fábrica de conexiones, para fines administrativos dentro de IBM WebSphere Application Server.
Tipo de datos String
Valor por omisión Null
Categoría
Categoría utilizada para clasificar o agrupar esta fábrica de conexiones, para sus registros administrativos de IBM WebSphere Application Server.
Tipo de datos String
Alias de autenticación gestionado por componentes
Este alias especifica un ID de usuario y una contraseña que sirven para autenticar la conexión con un proveedor de JMS para la autenticación gestionada por aplicación.

Esta propiedad proporciona una lista de los alias de entradas de datos de autenticación de J2C que se hayan definido en WebSphere Application Server. Puede seleccionar un alias de entrada de datos que sirva para autenticar la creación de una nueva conexión con el proveedor de JMS.

Si ha habilitado la seguridad global para WebSphere Application Server, seleccione el alias que especifica el ID de usuario y la contraseña que sirven para autenticar la creación de una nueva conexión con el proveedor de JMS. La utilización del alias depende del valor de la autenticación de recursos (res-auth) declarado en la referencia de recursos de la fábrica de conexiones de los descriptores de despliegue de un componente de aplicación.

Nota:

  1. No se pueden utilizar ID de usuario de más de 12 caracteres para la autenticación con el proveedor de JMS incorporado de WebSphere. Por ejemplo, el ID de usuario por omisión de Windows NT, Administrator, no se puede utilizar en WebSphere Embedded Messaging, ya que contiene 13 caracteres. Por lo tanto, un alias de autenticación para la fábrica de conexiones del proveedor de JMS de WebSphere debe especificar un ID de usuario que no contenga más de 12 caracteres.
  2. Si desea utilizar una conexión de proveedor de JMS de WebSphere MQ al utilizar la modalidad de transporte de enlaces, debe establecer la propiedad Tipo de transporte=ENLACES en la fábrica de conexiones de cola de WebSphere MQ. Asimismo, debe elegir una de las siguientes opciones:
    • Para utilizar la credenciales de seguridad, asegúrese de que el usuario especificado es el usuario que ha iniciado la sesión actualmente para el proceso de WebSphere Application Server. Si el usuario especificado no es el usuario que ha iniciado la sesión actualmente para el proceso de WebSphere Application Server, la autenticación de enlaces JMS de WebSphere MQ genera un error "MQJMS2013 invalid security authentication supplied for MQQueueManager" (MQJMS2013 Autenticación de seguridad inválida proporcionada por MQQueueManager.)
    • No especifique la credenciales de seguridad. En la fábrica de conexiones de WebSphere MQ, asegúrese de que tanto la propiedad Alias de autenticación gestionado por componentes como la propiedad Alias de autenticación gestionado por contenedor no están establecidas.

Tipo de transporte
Especifica si se utilizan la conexión del cliente de WebSphere MQ o enlaces JNI para la conexión con WebSphere MQ Queue Manager. El proveedor de JMS externo controla los protocolos de comunicación entre clientes y servidores JMS. Ajuste el tipo de transporte cuando utilice un servicio de mensajes que no sea ASF, no persistente, no perdurable y no transaccional o cuando desea satisfacer asuntos de seguridad y el cliente es local al nodo de Queue Manager.
Tipo de datos Enum
Unidades No se aplica
Valor por omisión ENLACES
Rango
ENLACES
Los enlaces JNI se utilizan para conectar con Queue Manager. ENLACES (bindings) es un protocolo de memoria compartida y sólo se puede utilizar cuando Queue Manager se encuentra en el mismo nodo que el cliente de JMS y viene con algunos riesgos de seguridad que se deben tratar mediante el uso de roles de EJB.

Si desea utilizar una conexión de proveedor de JMS de WebSphere MQ al utilizar la modalidad de transporte de enlaces, debe establecer la propiedad Tipo de transporte=ENLACES en la fábrica de conexiones de cola de WebSphere MQ. Asimismo, debe elegir una de las siguientes opciones:

  • Para utilizar la credenciales de seguridad, asegúrese de que el usuario especificado es el usuario que ha iniciado la sesión actualmente para el proceso de WebSphere Application Server. Si el usuario especificado no es el usuario que ha iniciado la sesión actualmente para el proceso de WebSphere Application Server, la autenticación de enlaces JMS de WebSphere MQ genera un error "MQJMS2013 invalid security authentication supplied for MQQueueManager".
  • No especifique la credenciales de seguridad. En la fábrica de conexiones de WebSphere MQ, asegúrese de que tanto la propiedad Alias de autenticación gestionado por componentes como la propiedad Alias de autenticación gestionado por contenedor no están establecidas.

CLIENTE
La conexión del cliente de WebSphere MQ se utiliza para conectar con Queue Manager. CLIENTE es un protocolo TCP típico.
DIRECTO
Para WebSphere MQ Event Broker mediante la modalidad DIRECTO. DIRECTO es un protocolo Lightweight Sockets que se utiliza en servicios de mensajes de publicación/suscripción no transaccionales, no perdurables y no persistentes. DIRECTO es sólo para que funcione en clientes y beans controlados por mensajes mediante el protocolo que no es ASF.
EN COLA
En COLA es un protocolo TCP sin estado.
Recomendado
Tipo de transporte de fábrica de conexiones de cola
ENLACES es un 30% o más rápido, pero carece de seguridad. Cuando le atañe la seguridad, es preferible ENLACES que CLIENTE.
Tipo de transporte de fábrica de conexiones de tema
DIRECTO es lo más rápido y debe utilizarse cuando sea posible. Utilice ENLACES cuando desee satisfacer tareas de seguridad adicionales y el Queue Manager sea local para el cliente JMS. EN COLA es lo que se utiliza para todos los demás casos. Recuerde, WebSphere MQ 5.3 antes de CSD2 con el valor DIRECTO puede perder mensajes cuando se utiliza con beans controlados por mensajes y bajo carga. Esto también sucede con aplicaciones en el cliente a no ser que maxClientQueueSize del broker esté establecido en 0. Puede establecer este valor en 0 con el mandato #wempschangeproperties WAS_nodeName_server1 -e default -o DynamicSubscriptionEngine -n maxClientQueueSize -v 0 -x executionGroupUUID, donde executionGroupUUID se puede encontrar iniciando el broker y buscando en el archivo de anotaciones cronológicas o aplicaciones de sucesos del suceso 2201. Este valor suele ser ffffffff-0000-0000-000000000000.

Habilitado para XA
Especifica si la fábrica de conexiones es para coordinación XA y no XA de los mensajes y controla si el servidor de aplicaciones utilizará XA QCF/TCF. Habilite XA cuando no se utilizan en la misma transacción varios recursos.

Si establece esta propiedad en NON_XA, la sesión JMS está todavía incluida en una transacción, pero utiliza las llamadas de transacción local del gestor de recursos (session.commit y session.rollback) en lugar de las llamadas de XA. Esto puede producir una mejora en el rendimiento. Sin embargo, esto significa que sólo un único recurso puede incluirse en una transacción en WebSphere Application Server. En WebSphere Application Server Enterprise, el soporte de último participante le permite incluir un recurso que no sea XA con otros recursos compatibles con XA.

Esta propiedad no se aplica para una fábrica de conexiones de tema WebSphere con la propiedad Port establecida en DIRECT y siempre toma el valor NON_XA.

Tipo de datos Enum
Unidades No se aplica
Valor por omisión Habilitado para XA
Rango
XA
La fábrica de conexiones es para coordinación XA de mensajes
NON_XA
La fábrica de conexiones es para coordinación no XA de mensajes
Recomendado No habilite XA cuando la cola de mensajes o el tema recibido sea el único recurso de la transacción. Habilite XA cuando otros recursos, incluidas otras colas o temas, estén implicados.
Agrupación de conexiones
Especifica un conjunto opcional de valores de agrupación de conexiones.

Las propiedades de agrupación de conexiones son comunes a todos los conectores J2C.

El servidor de aplicaciones agrupa conexiones y sesiones con el proveedor de JMS para mejorar el rendimiento. Esta acción es independiente de cualquier agrupación de conexiones de WebSphere MQ. Es necesario que configure las propiedades de agrupación de sesiones y conexiones para las aplicaciones adecuadamente; de lo contrario, es posible que no obtenga el comportamiento de sesión o conexión deseado.

Cambie el tamaño de la agrupación de conexiones si el acceso simultáneo en el servidor al recurso JMS supera el valor por omisión. El tamaño de la agrupación de conexiones se establece por cola o por tema. Para obtener más detalles, consulte la tabla siguiente.

Propiedades personalizadas
Un conjunto opcional de pares de nombre y valor para propiedades personalizadas pasadas a WebSphere MQ.

Puede utilizar la página Propiedades personalizadas para definir las siguientes propiedades SSL (Secure Sockets Layer) de WebSphere MQ. Estas propiedades sólo se aplican, si establece la propiedad Tipo de transporte=CLIENTE.

  • SSLCIPHERSUITE
  • SSLCRL
  • SSLPEERNAME

Para obtener más información sobre cómo establecer estas propiedades, consulte la sección SSL properties en el manual WebSphere MQ Using Java.
