Valores de sesión de correo

Utilice esta página para configurar proveedores de correo específicos.

Para ver esta página de la consola administrativa, pulse Recursos > Proveedores de correo > proveedor_correo > Sesiones de correo > sesión_correo.

Pestaña Configuración

Nombre
Especifica el nombre administrativo del objeto de sesión de JavaMail.
Nombre JNDI
Especifica el nombre de la interfaz JNDI (Java Naming and Directory Interface) correspondiente al recurso, incluido cualquier subcontexto de denominación.

Este nombre proporciona el enlace entre la información de enlace de la plataforma para los recursos definidos en el descriptor de despliegue de la aplicación cliente y los recursos reales unidos a JNDI por la plataforma.

Descripción
Especifica una descripción opcional de los registros administrativos.
Categoría
Especifica una colección opcional para clasificar o agrupar sesiones.
Sistema principal de transporte de correo
Especifica el servidor al que se accede al enviar correo.
Protocolo de transporte de correo
Indica el protocolo de transporte utilizado al enviar correo.
Usuario de transporte de correo
Especifica el ID de usuario cuando el sistema principal de transporte de correo requiere autenticación.

Este valor no se suele utilizar con la mayoría de los servidores de correo. No rellene este campo a no ser que el servidor de correo utilizado necesite un ID de usuario y una contraseña.

Contraseña de transporte de correo
Especifica la contraseña cuando el sistema principal de transporte de correo requiere autenticación.

Este valor no se suele utilizar con la mayoría de los servidores de correo. No rellene este campo a no ser que el servidor de correo utilizado necesite un ID de usuario y una contraseña.

Correo de
Especifique el emisor de correo.

Este valor representa la dirección de correo electrónico de Internet que, por omisión, se muestra en el mensaje recibido, como la dirección De o Responder a. La respuesta del destinatario llega a esta dirección.

Sistema principal de almacenamiento de correo
Especifica el servidor al que se accede al recibir correo.

Este valor, combinado con el ID de usuario y la contraseña del almacenamiento de correo, representa una cuenta de correo válida. Por ejemplo, si la cuenta de correo es john_william@my.company.com, el sistema principal de almacenamiento de correo es my.company.com.

Protocolo de almacenamiento de correo
Especifica el protocolo que se utiliza al recibir correo y puede ser IMAP, POP3 o cualquier protocolo de almacenamiento para el que el usuario ha instalado un proveedor.
Usuario de almacenamiento de correo
Especifica el ID de usuario para la cuenta de correo específica.

Por ejemplo, si la cuenta de correo es john_william@my.company.com, el usuario es john.william.com.

Contraseña de almacenamiento de correo
Especifica la contraseña para la cuenta de correo específica.

Por ejemplo, si la cuenta de correo es john_william@my.company.com, deberá especificar la contraseña del usuario john.william.com.

Depuración
Activa o desactiva la modalidad de depuración para esta sesión de correo.
