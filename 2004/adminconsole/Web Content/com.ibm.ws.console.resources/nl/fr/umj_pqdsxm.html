Paramètres de file d'attente WebSphere MQ (MQ Config)

Utilisez cet écran pour afficher ou modifier les propriétés de configuration définies dans Websphere MQ pour la destination de file d'attente sélectionnée.

Les destinations de file d'attente permettent de configurer les propriétés des files d'attente JMS. Une file d'attente à utiliser avec le fournisseur WebSphere MQ JMS est dotée des propriétés supplémentaires suivantes définies pour WebSphere MQ.

Remarques:

  • Certaines propriétés affichées sont en lecture seule et ne peuvent pas être modifiées.
  • Pour afficher ou modifier les propriétés, le gestionnaire de files d'attente de WebSphere MQ dans lequel se trouve la file d'attente doit être configuré pour une administration à distance et être actif.
  • Les valeurs de propriété que vous spécifiez doivent correspondre aux valeurs que vous avez spécifiées lors de la configuration des ressources WebSphere MQ JMS. Pour plus d'informations sur la configuration des ressources WebSphere MQ JMS, reportez-vous au document WebSphere MQ Using Java dans la page Web de la bibliothèque WebSphere MQ multi-plateforme, à l'adresse    http://www.ibm.com/software/ts/mqseries/library/manualsa/manuals/crosslatest.html .
  • Dans WebSphere MQ, les noms peuvent comporter un maximum de 48 caractères, à l'exception des canaux dont la longueur maximum est de 20 caractères.

Pour afficher cette page de la console d'administration, cliquez sur Ressources-> Fournisseur WebSphere MQ JMS-> (Dans la sous-fenêtre de contenu, sous les propriétés additionnelles) Destinations de file d'attente WebSphere MQ-> nom_destination-> (Dans la sous-fenêtre de contenu, sous Propriétés supplémentaires) Configuration MQ

Seuil d'annulation
Le nombre maximum d'annulations d'un message. Si ce seuil est atteint, le message est replacé dans la file d'attente d'annulation indiquée par la propriété Nom de file d'attente d'annulation.

Le gestionnaire de files d'attente WebSphere MQ conserve un enregistrement du nombre d'annulations de chaque message. Lorsque ce nombre atteint un certain seuil (configurable), le client de connexions replace le message dans une file d'attente d'annulation nommée. Si cette remise en file d'attente échoue, le message est supprimé de la file et replacé dans la file d'attente de rebut ou supprimé.
