Paramètres d'adaptateur de ressources

Cette page permet de spécifier les paramètres d'un adaptateur de ressources.

Cet adaptateur de ressources peut être fourni par une société tierce (autre qu'IBM). En général, un adaptateur donné ne peut se connecter qu'à un seul type de système dorsal (EIS), mais il peut accepter de nombreuses configurations différentes pour les connexions à ce système. L'adaptateur de ressources possède de nombreuses propriétés de configuration qui sont définies dans la spécification J2C. Les valeurs de ces propriétés sont configurées par la société fournissant l'adaptateur.

Pour afficher cette page de la console d'administration, cliquez sur Ressources >Adaptateurs de ressources > adaptateur_ressources.

Onglet Configuration

Portée
Indique à quel niveau cette définition de ressource est visible : cellule, noeud ou serveur.

WebSphere 5.0 permet à des ressources telles que fournisseurs JDBC, liaisons d'espace de noms et bibliothèques partagées d'être définies à plusieurs niveaux. Dans ce cas, pour une ressource donnée, la définition figurant au niveau le plus spécifique (par exemple, le serveur) l'emporte sur l'éventuelle définition homologue figurant à un niveau plus général (par exemple, la cellule).

Cellule
Niveau de portée le plus général. Les ressources définies au niveau cellule sont visibles depuis tous les noeuds et serveurs de cette cellule, sauf si elles sont redéfinies à un niveau inférieur. Pour visualiser les ressources définies au niveau cellule, ne spécifiez pas de nom de serveur ni de nom de noeud dans le formulaire de sélection de portée.
Noeud
Portée par défaut de la plupart des types de ressources. Les ressources définies au niveau noeud l'emportent sur les éventuelles définitions homologues figurant au niveau cellule et sont visibles depuis tous les serveurs du noeud considéré, sauf si elles sont redéfinies à un niveau serveur sur ce noeud. Pour visualiser les ressources définies au niveau d'un noeud, ne spécifiez pas de serveur dans le formulaire de sélection de portée ; spécifiez simplement le nom du noeud concerné.
Serveur
Niveau de portée le plus spécifique pour la définition des ressources. Les ressources définies au niveau d'un serveur l'emportent sur les éventuelles définitions homologues apparaissant au niveau de la cellule parente ou du noeud parent ; elles sont visibles uniquement depuis ce serveur. Pour visualiser les ressources définies au niveau d'un serveur, spécifiez le nom de ce serveur ainsi que celui de son noeud parent dans le formulaire de sélection de portée.

Lorsque des ressources sont créées, elles le sont toujours dans la portée en cours, c'est-à-dire celle qui est sélectionnée dans le formulaire. Pour afficher les ressources d'autres portées, spécifiez un noeud ou un serveur différent dans le formulaire de sélection de portée.

Type de données String (chaîne)
Nom
Indique le nom du fournisseur de ressources.

Chaîne sans espace devant identifier clairement le fournisseur de ressources.

Type de données String (chaîne)
Description
Texte décrivant le fournisseur de ressources.

Chaîne de texte libre destinée à décrire la fabrique de ressources et son objectif.

Type de données String (chaîne)
Chemin de l'archive
Chemin d'accès au fichier archive .RAR qui contient le module de cet adaptateur de ressources.
Type de données String (chaîne)
Chemin d'accès aux classes
Liste de chemins ou de noms de fichiers JAR qui, ensemble, spécifient l'emplacement des classes du fournisseur de ressources.

Ce chemin inclut toutes les bibliothèques supplémentaires requises par l'adaptateur de ressources. La base du code de l'adaptateur lui-même est automatiquement ajoutée à ce chemin de classes, mais si une ressource étrangère au fichier .RAR est nécessaire, elle peut être spécifiée ici.

Type de données String (chaîne)
Chemin d'accès natif
Liste de chemins d'accès qui, ensemble, spécifient l'emplacement des bibliothèques natives du fournisseur de ressources.

La base du code de l'adaptateur lui-même est automatiquement ajoutée à ce chemin de classes, mais si une ressource étrangère au fichier .RAR est nécessaire, elle peut être spécifiée ici.
