Paramètres de destinations JMS génériques

Utilisez cet écran pour afficher ou modifier les propriétés de configuration de la destination JMS sélectionnée pour utilisation avec le fournisseur JMS associé.

Une destination JMS est utilisée pour configurer les propriétés d'une destination JMS pour le fournisseur JMS générique associé. Les connexions vers la destination JMS sont créées par la fabrique de connexions JMS associée. Une destination JMS à utiliser avec un fournisseur JMS générique (et non le fournisseur JMS WebSphere intégré ou le fournisseur WebSphere MQ JMS) comporte les propriétés suivantes.

Pour afficher la page de la console d'administration, cliquez sur Ressources-> Fournisseurs JMS génériques-> nom_fournisseur-> Destinations JMS-> destination

Nom
Nom sous lequel cette file d'attente est connue pour l'administration d'IBM WebSphere Application Server.
Type de données Chaîne
Type
Indique si cette destination JMS est une file d'attente (pour la messagerie point à point) ou une rubrique (pour la messagerie par publication/abonnement).

Sélectionnez l'une des options suivantes :

File d'attente
Une destination de file d'attente JMS pour la messagerie point à point.
Rubrique
Une destination de rubriques JMS pour la messagerie par publication/abonnement.

Nom JNDI
Nom JNDI utilisé pour lier la file d'attente à l'espace nom du serveur d'applications.

Par convention, utilisez le nom JNDI complet, par exemple sous la forme jms/Nom, Nom correspondant au nom logique de la ressource.

Ce nom est utilisé pour lier les informations relatives à la liaison des plateformes. La liaison associe les ressources définies dans le descripteur de déploiement du module aux ressources réelles (physiques) liées sous JNDI par la plateforme.

Description de la file d'attente, pour utilisation administrative
Catégorie
Catégorie utilisée pour classer ou grouper cette file d'attente dans le cadre de l'administration d'IBM WebSphere Application Server.
