Impostazioni dell'adattatore risorse

Utilizzare questa pagina per specificare le impostazioni di un adattatore risorse.

Tale adattatore risorse può essere offerto da fornitori terzi diversi da IBM. In genere, un singolo adattatore risorse può collegarsi a un tipo di sistema back-end (EIS) ma può supportare molte configurazioni diverse per le connessioni a quel sistema EIS. L'adattatore di risorse ha molte proprietà di configurazione che sono definite nelle specifiche J2C e impostate dal fornitore del codice.

Per visualizzare questa pagina della console di gestione, fare clic su Risorse >Adattatori risorse > adattatore_risorsa.

Scheda Configurazione

Ambito
Specifica il livello in cui è visibile questa definizione di risorsa -- il livello cella, nodo o server.

WebSphere 5.0 rende disponibili risorse quali JDBCProvider, collegamenti spazio nome o librerie condivise a più ambiti, insieme a risorse definite in ambiti più specifici che sovrascrivono i duplicati definiti in ambiti più generali.

Cella
L'ambito più generale. Le risorse definite nell'ambito cella sono visibili da tutti i nodi e i server a meno che non siano state sovrascritte. Per visualizzare le risorse definite nell'ambito cella, non specificare il nome di un server o di un nodo nel modulo di selezione dell'ambito.
Nodo
L'ambito predefinito per la maggior parte dei tipi di risorse. Le risorse definite nell'ambito nodo sovrascrivono gli eventuali duplicati definiti nell'ambito cella e sono visibili a tutti i server sullo stesso nodo a meno che non siano state sovrascritte in un ambito server sul quel nodo. Per visualizzare le risorse definite in un ambito nodo, non specificare un server ma selezionare il nome di un nodo nel modulo di selezione dell'ambito.
Server
L'ambito più specifico per la definizione delle risorse. Le risorse definite nell'ambito server sovrascrivono eventuali definizioni di risorse duplicate specificate nell'ambito cella o nell'ambito nodo parent e sono visibili solo in un determinato server. Per visualizzare le risorse definite in un ambito server, specificare il nome di un server e il nome di un nodo nel modulo di selezione dell'ambito.

Quando le risorse vengono generate, vengono sempre create nell'ambito corrente selezionato nel pannello. Per visualizzare le risorse in altri ambiti, specificare un nodo o un server diverso nel modulo di selezione dell'ambito.

Tipo dati Stringa
Name
Specifica il nome del provider di risorse.

Una stringa senza spazi indica che si tratta di un identificativo di testo significativo per il provider di risorse.

Tipo dati Stringa
Descrizione
Specifica una descrizione della produzione risorse.

Una stringa di testo libera per descrivere la produzione di risorse e il relativo scopo.

Tipo dati Stringa
Percorso archivio
Specifica il percorso al file .RAR contenente il modulo per l'adattatore risorse specificato.
Tipo dati Stringa
Classpath
Specifica un elenco di percorsi o nomi di file JAR che insieme costituiscono l'ubicazione delle classi dei provider di risorse.

Sono incluse tutte le ulteriori librerie necessarie per l'adattatore risorse. Il codice base dell'adattatore risorse viene automaticamente aggiunto al percorso classe ma se sono necessari altri elementi oltre al file .RAR, possono essere specificati in questo punto.

Tipo dati Stringa
Percorso nativo
Specifica un elenco di percorsi che costituiscono l'ubicazione delle librerie native del provider di risorse.

Il codice base dell'adattatore risorse viene automaticamente aggiunto al percorso classe ma se sono necessari altri elementi oltre al file .RAR, possono essere specificati in questo punto.
