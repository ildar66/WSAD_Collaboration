Impostazioni della coda di WebSphere MQ

Utilizzare questo pannello per visualizzare o modificare le proprietà di configurazione della destinazione coda selezionata da utilizzare con il provider JMS di WebSphere MQ.

Una destinazione coda viene utilizzata per configurare le proprietà di una coda JMS. Le connessioni alla coda vengono create dalla produzione connessioni code associata per il provider incorporato di JMS di WebSphere MQ. Una coda da utilizzare con il provider JMS di WebSphere MQ ha le seguenti proprietà.

Nota:

  • I valori delle proprietà specificati devono corrispondere ai valori specificati durante la configurazione di WebSphere MQ per le risorse JMS. Per ulteriori informazioni sulla configurazione delle risorse JMS di WebSphere MQ, consultare il manuale Using Java di WebSphere MQ.
  • In WebSphere MQ, i nomi possono avere un massimo di 48 caratteri, ad eccezione dei canali, che possono contenere un massimo di 20 caratteri.

Per visualizzare questa pagina della console di gestione, fare clic su Risorse-> Provider JMS di WebSphere MQ-> (Nel pannello contenuto, in Ulteriori proprietà) Destinazioni code WebSphere MQ-> nome_destinazione

Nome
Il nome attribuito alla coda per scopi di gestione all'interno di IBM WebSphere Application Server.
Tipo dati Stringa
Nome JNDI
Il nome JNDI utilizzato per collegare la coda allo spazio nome del server delle applicazioni.

Per convenzione, utilizzare il nome completo JNDI; ad esempio, nel formato jms/Nome, dove Nome è il nome logico della risorsa.

Tale nome viene utilizzato per il collegamento alle informazioni specifiche della piattaforma. Il collegamento associa le risorse definite dal descrittore di distribuzione del modulo alle risorse effettive (fisiche) collegate a JNDI dalla piattaforma.

Tipo dati Stringa
Descrizione
Una descrizione della coda, per scopi di gestione
Tipo dati Stringa
Predefinito Nullo
Categoria
Una categoria utilizzata per classificare o raggruppare questa coda per i record di gestione IBM WebSphere Application Server.
Tipo dati Stringa
Persistenza
Se tutti i messaggi inviati alla destinazione sono o non sono persistenti o se la persistenza è definita dall'applicazione
Tipo dati Enum
Unità Non applicabile
Predefinito APPLICAZIONE_DEFINITA
Intervallo
Applicazione definita
I messaggi sulla destinazione hanno la persistenza definita dall'applicazione che li ha inseriti nella coda.
Coda definita
[Solo una destinazione WebSphere MQ] I messaggi sulla destinazione hanno la persistenza definita dalle proprietà di definizione delle code di WebSphere MQ.
Persistente
I messaggi sulla destinazione sono persistenti.
Non persistente
I messaggi sulla destinazione non sono persistenti.
Priorità
Se la priorità messaggio di questa destinazione viene definita dall'applicazione o dalla proprietà Priorità specificata
Tipo dati Enum
Unità Non applicabile
Predefinito APPLICAZIONE_DEFINITA
Intervallo
Applicazione definita
La priorità dei messaggi su questa destinazione viene definita dall'applicazione che li ha inseriti nella destinazione.
Coda definita
[Solo una destinazione WebSphere MQ] I messaggi sulla destinazione hanno la persistenza definita dalle proprietà di definizione delle code di WebSphere MQ.
Specificato
La priorità dei messaggi su questa destinazione viene definita dalla proprietà Priorità specificata. Se viene selezionata quest'opzione, è necessario definire una priorità sulla proprietà Priorità specificata.
Priorità specificata
Se la proprietà Priorità è impostata su Specified, immettere qui la priorità da applicare ai messaggi di questa coda nell'intervallo compreso tra 0 (il valore più basso) a 9 (il valore più alto)

Se la proprietà Priorità è impostata su Specificata, i messaggi inviati a questa coda hanno il valore di priorità specificato da questa proprietà.

Tipo dati Numero intero
Unità Livello di priorità messaggio
Predefinito Nullo
Intervallo Da 0 (priorità minore) a 9 (priorità maggiore)
Scadenza
Se il timeout di scadenza della coda prescelta è definito dall'applicazione o dalla proprietà Scadenza specificata, oppure se il messaggio sulla coda è permanente (cioè ha un timeout illimitato)
Tipo dati Enum
Unità Non applicabile
Predefinito APPLICAZIONE_DEFINITA
Intervallo
Applicazione definita
Il timeout di scadenza dei messaggi presenti in questa coda viene definito dall'applicazione che li ha inseriti nella coda.
Specificato
Il timeout di scadenza dei messaggi su questa coda viene definito dalla proprietà Scadenza specificata.Se viene selezionata questa opzione, è necessario definire un timeout sulla proprietà Scadenza specificata.
Illimitato
I messaggi presenti in questa coda non hanno un timeout di scadenza, dunque tali messaggi non scadono mai.
Scadenza specificata
Se la proprietà Timeout di scadenza viene impostata su Specificato, digitare qui il numero di millisecondi (maggiore di 0) trascorsi i quali i messaggi presenti in questa coda scadono
Tipo dati Numero intero
Unità Millisecondi
Predefinito Nullo
Intervallo Maggiore o pari a 0
  • 0 indica i messaggi che non scadono mai
  • Gli altri valori sono un numero intero di millisecondi
Nome coda di base
Il nome della coda in cui vengono inviati i messaggi, sul gestore code specificato dalla proprietà Nome gestore code base
Tipo dati Stringa
Predefinito Nullo
Intervallo Da 1 a 48 caratteri ASCII
Nome gestore code base
Il nome del gestore code WebSphere MQ a cui vengono inviati i messaggi

Tale gestore code fornisce la coda specificata dalla proprietà Nome coda di base.

Tipo dati Stringa
Unità Caratteri ASCII En_US
Predefinito Nullo
Intervallo Un nome del gestore code WebSphere MQ valido, con un massimo di 48 caratteri ASCII
CCSID
L'identificatore di un insieme di caratteri codificati da utilizzare con il gestore code WebSphere MQ.

È necessario che questo identificatore di un insieme di caratteri codificati (CCSID) sia del tipo supportato da WebSphere.

Tipo dati Stringa
Unità Numero intero
Predefinito Nullo
Intervallo Da 1 a 65535

Per ulteriori informazioni sui CCSID supportati e sulla conversione dei dati dei messaggi da un insieme di caratteri codificati a un altro, consultare i manuali WebSphere MQ System Administration e WebSphere MQ Application Programming Reference. Questi sono disponibili sulle pagine Web dei manuali specifici della piattaforma e della multipiattaforma di messaggistica WebSphere MQ; ad esempio all'indirizzo   http://www-3.ibm.com/software/ts/mqseries/library/manualsa/manuals/platspecific.html ,    IBM Publications Center o dal Kit di raccolta WebSphere MQ, SK2T-0730.
