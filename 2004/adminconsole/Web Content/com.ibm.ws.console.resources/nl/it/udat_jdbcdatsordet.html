Impostazioni dell'origine dati

Utilizzare questa pagina per creare un'origine dati in un provider di JDBC che fornisce la specifica classe di implementazione del driver JDBC.

Accertarsi di voler utilizzare un'origine dati Versione 5.0. Se si utilizza il modello del componente Enterprise JavaBean versione 1.0 e Servlet 2.2, è necessario utilizzare un'origine dati Versione 4.0.

Per visualizzare questa pagina della console di gestione, fare clic su Risorse > Provider JDBC > provider_JDBC > Origini dati > origine_dati.

Scheda Configurazione

Ambito
Specifica il livello in cui è visibile questa definizione di risorsa -- il livello cella, nodo o server.

È possibile definire in più ambiti, risorse quali provider di JDBC, collegamenti spazio nome o librerie condivise, insieme a risorse definite in ambiti più specifici che sovrascrivono i duplicati definiti in ambiti più generali.

Notare che non è importante l'ambito di una risorsa definita, le proprietà della risorsa si applicano solo ad un singolo livello del server. Ad esempio, se si definisce l'ambito di un'origine dati al livello Cella, tutti gli utenti possono ricercare e utilizzare quella origine dati che risulta univoca all'interno di quella cella. Tuttavia, le impostazioni delle proprietà di risorsa sono locali per ogni server all'interno della Cella. Ad esempio, se si imposta numero massimo connessioni su 10, ogni server in quella cella può disporre di 10 connessioni.

Cella
L'ambito più generale. Le risorse definite nell'ambito cella sono visibili da tutti i nodi e i server, a meno che non siano state sovrascritte. Per visualizzare le risorse definite nell'ambito cella, non specificare il nome di un server o di un nodo nel modulo di selezione dell'ambito.
Nodo
L'ambito predefinito per la maggior parte dei tipi di risorse. Le risorse definite nell'ambito nodo sovrascrivono gli eventuali duplicati definiti nell'ambito cella e sono visibili a tutti i server sullo stesso nodo, a meno che non siano state sovrascritte in un ambito server sul quel nodo. Per visualizzare le risorse definite in un ambito nodo, non specificare un server ma selezionare il nome di un nodo nel modulo di selezione dell'ambito.
Server
L'ambito più specifico per la definizione delle risorse. Le risorse definite nell'ambito server sovrascrivono eventuali definizioni di risorse duplicate specificate nell'ambito cella o nell'ambito nodo parent e sono visibili solo in un determinato server. Per visualizzare le risorse definite in un ambito server, specificare il nome di un server e il nome di un nodo nel modulo di selezione dell'ambito.

Quando le risorse vengono generate, vengono sempre create nell'ambito corrente selezionato nel pannello. Per visualizzare le risorse in altri ambiti, specificare un nodo o un server diverso nel modulo di selezione dell'ambito.

Tipo dati Stringa
Nome
Specifica il nome visualizzato dell'origine dati.

Ad esempio è possibile impostare questa campo su Test origini dati.

Tipo dati Stringa
Nome JNDI
Specifica il nome JNDI (Java Naming and Directory Interface).

Gli ambienti informatici distribuiti utilizzano spesso i servizi directory e di denominazione per ottenere dei componenti e delle risorse condivise. Questi servizi associano i nomi con le ubicazioni, i servizi, le informazioni e le risorse.

I servizi di denominazione forniscono le associazioni nome-oggetto. I servizi directory, invece, forniscono le informazioni relative agli oggetti e agli strumenti di ricerca necessari per localizzare tali oggetti.

Esistono molte implementazioni di servizi directory e di denominazione e interfacce per modificarle. JNDI fornisce un'interfaccia comune utilizzata per accedere ai diversi servizi di directory e denominazione.

È possibile, ad esempio, utilizzare il nome jdbc/markSection.

Se si lascia questo campo vuoto, viene generato un nome JNDI dal nome dell'origine dati. Ad esempio, il nome markSection di un'origine dati genera un nome JNDI jdbc/markSection.

Una volta impostato e salvato questo valore e riavviato il server, è possibile visualizzare questa stringa quando viene eseguito dumpnamespace.

Tipo dati Stringa
Dimensione cache istruzioni
Specifica il numero di istruzioni libere memorizzate nella cache per connessione.

L'origine dati di WebSphere Application Server ottimizza l'elaborazione delle istruzioni preparate. Un'istruzione preparata è un'istruzione SQL precompilata memorizzata in un oggetto istruzioni preparate. Questo oggetto viene utilizzato per eseguire in modo efficace l'istruzione SQL specificata più volte.

Se la cache non è sufficientemente grande, vengono eliminate voci utili per liberare spazio su disco per le nuovi voci. Per impostare la dimensione cache sul valore più alto, evitando che vengano scaricate istruzioni dalla cache, aggiungere, per ciascuna applicazione che utilizza quest'origine dati su un particolare server, il numero di istruzioni univoche preparate, istruzioni richiamabili (come determinato dalla stringa sql, e, contemporaneamente, dal tipo di scorrimento). Questo valore rappresenta il numero massimo di possibili istruzioni preparate memorizzate nella cache su una connessione specificata durante il ciclo di vita del server. Se si imposta la dimensione della cache su questo valore, non ci saranno mai istruzioni eliminate dalla cache. In generale, maggiori sono le istruzioni di cui dispone l'applicazione, più grande è la cache. Ad esempio, se l'applicazione dispone di 5 istruzioni SQL, impostare la dimensione della cache istruzioni su 5, in modo che ciascuna connessione disponga di 5 istruzioni.

Inoltre, è possibile utilizzare il Visualizzatore prestazioni Tivoli per ridurre il numero delle istruzioni eliminate dalla cache. Utilizzare un carico di lavoro standard che rappresenti un numero tipico di richieste client in entrata, un numero stabilito di iterazioni e un gruppo standard di impostazioni di configurazione. Nota: più è grande la cache istruzioni, maggiori sono le risorse di sistema che vengono eliminate. Pertanto, se si imposta il numero su un valore troppo alto, è possibile non disporre di risorse poiché il sistema non è in grado di aprire le molte istruzioni preparate.

Nelle applicazioni di verifica, l'ottimizzazione della cache istruzione ha migliorato la velocità di trasmissione del 10-20%. Tuttavia, a causa di potenziali limiti delle risorse ciò non è sempre possibile.

Tipo dati Numero intero
Predefinito A seconda del database. La maggior parte sono 10. Informix Versione 7.3, 9.2 o 9.3 senza le ultime correzioni, deve essere impostato su 0. Un valore predefinito 0 indica che non esistono istruzioni cache.
Datasource Helper Classname
Specifica il datastore Helper utilizzato per eseguire funzioni specifiche del database.

È utilizzato dall'Adattatore risorse relazionali durante il runtime. La classe di implementazione DataStoreHelper predefinita è impostata in base alla classe di implementazione del driver JDBC utilizzando la struttura: com.ibm.websphere.rsadapter.<database>DataStoreHelper. Ad esempio, se il provider di JDBC è DB2, la classe DataStoreHelper predefinita è com.ibm.websphere.rsadapter.DB2DataStoreHelper. È possibile modificare la sottoclasse di questa classe DataStoreHelper, se necessario.

Tipo dati Stringa
Predefinito Dipende dalla classe di implementazione del driver JDBC
Alias dell'autenticazione gestita dal componente
Questo alias viene utilizzato per l'autenticazione del database nel runtime.

Se l'autenticazione di risorse (res-auth) è impostato su Applicazione, impostare l'alias nell'Alias di autenticazione gestito dal componente.

Se questo campo non è impostato e il database richiede l'ID utente e la password per richiamare una connessione, si riceverà un'eccezione durante il runtime.

Se il database (ad esempio, Cloudscape) non supporta l'ID utente e la password, non impostare l'alias nel campi Alias di autenticazione gestito dal componente o Alias di autenticazione gestito dal contenitore. Altrimenti, verrà visualizzato il messaggio di avviso nel log di sistema che indica che l'utente e la password non sono proprietà valide. Questo messaggio è solo un avviso, per cui l'origine dati verrà creata con esito positivo.
