WebSphere MQ 队列设置（MQ 配置）

使用此面板查看或更改为所选队列目标而定义到 WebSphere MQ 的配置属性。

队列目标用于配置 JMS 队列的属性。 与 WebSphere MQ JMS 提供程序一起使用的队列具有以下额外定义到 WebSphere MQ 的属性。

注意事项:

  • 显示的一些属性是只读的，而且无法更改。
  • 要能够查看或更改属性，队列驻留的 WebSphere MQ 队列管理器必须配置为远程管理，并正在运行。
  • 您指定的属性值必须与配置 WebSphere MQ JMS 资源时指定的值相匹配。要获得有关配置 WebSphere MQ JMS 资源的更多信息，请参阅 WebSphere MQ 使用 Java 一书；例如，从 WebSphere MQ 多平台库 Web 页面（位于    http://www.ibm.com/software/ts/mqseries/library/manualsa/manuals/crosslatest.html ）。
  • 在 WebSphere MQ 中，名称最长可以是 48 个字符，但例外是通道最长为 20 个字符。

要查看此管理控制台页面，单击资源 -> WebSphere MQ JMS 提供程序 ->（在内容窗格中，“其它属性”下）WebSphere MQ 队列目标 -> destination_name ->（在内容窗格中，“其它属性”下）MQ 配置

最大队列深度

数据类型 整型

缺省
范围 一个大于或等于零的值，并小于或等于：
  • 如果队列在 OS/390 上，则是 999 999 999
  • 640 000（如果队列在任何其它 WebSphere MQ 平台上）

要获取有关允许的最大值的更多信息，请参阅《WebSphere MQ MQSC 命令参考》。

如果减小这个值，则已经在队列上的任何消息不受影响，即使消息数超过新的最大值。

逆序恢复阈值
消息可被逆序恢复的最多次数。如果达到这个阈值，则在逆序恢复重新排队名属性指定的逆序恢复队列上重新排队消息。

WebSphere MQ Queue Manager 保留每个消息被逆序恢复的次数记录。当这个数目达到可配置阈值时，连接消费者对指定的逆序恢复队列上的消息重新排队。如果 出于某个原因这个重新排队失败，则从队列除去这个消息并且重新排队死信队列或废弃。
