자원 어댑터 설정

이 페이지를 참조하여 자원 어댑터에 대한 설정을 지정하십시오.

이 자원 어댑터는 IBM 이외의 타사 벤더에서 공급할 수 있습니다. 일반적으로 단일 자원 어댑터는 한 가지 유형의 백엔드 시스템(EIS)에만 연결할 수 있지만 해당 EIS에 대한 연결에 대해 여러 가지 구성을 지원할 수 있습니다. 자원 어댑터에는 J2C 스펙에서 정의하고 코드를 공급하는 벤더가 설정한 여러 구성 등록 정보가 있습니다.

이 관리 콘솔 페이지를 보려면 자원 >자원 어댑터 > resource_adapter를 클릭하십시오.

구성 탭

범위
이 자원 정의가 표시될 수 있는 레벨(셀, 노드 또는 서버 레벨)을 지정합니다.

WebSphere 5.0은 보다 일반적인 범위에서 정의되는 중복을 대체하는 더욱 특정한 범위에서 정의되는 자원과 함께 복수 범위에서 JDBCProviders, Namespace Bindings 또는 공유 라이브러리 같은 자원을 허용합니다.

셀
가장 일반적인 범위. 셀 범위에서 정의되는 자원은 모든 노드 및 서버에서 볼 수 있습니다(대체되지 않은 경우). 셀 범위에서 정의되는 자원을 보려면 범위 선택사항 양식에서 서버 또는 노드 이름을 지정하지 마십시오.
Node
대부분의 자원 유형에 대한 기본 범위. 노드 범위에서 정의되는 자원은 셀 범위에서 정의되는 모든 중복을 대체하고 동일한 노드에 있는 모든 서버에서 볼 수 있습니다(해당 노드의 서버 범위에서 대체되지 않은 경우). 노드 범위에서 정의되는 자원을 보려면 서버를 지정하지 말고 범위 선택사항 양식에서 노드 이름을 선택하십시오.
서버
자원 정의에 대한 가장 특정한 범위. 서버 범위에서 정의되는 자원은 셀 범위 및 상위 노드 범위에서 정의되는 모든 중복 자원 정의를 대체하고 특정 서버에서만 볼 수 있습니다. 서버 범위에서 정의되는 자원을 보려면 범위 선택사항 양식에서 서버 이름뿐 아니라 노드 이름을 선택하십시오.

자원은 작성될 때 항상 패널에서 선택되는 현재 범위에 작성됩니다. 다른 범위에서 자원을 보려면 범위 선택사항 양식에 다른 노드 또는 서버를 지정하십시오.

데이터 유형 문자열
이름
자원 프로바이더의 이름을 지정합니다.

자원 프로바이더에 대해 유효한 텍스트 ID인 공백을 갖지 않는 문자열.

데이터 유형 문자열
설명
자원 팩토리의 텍스트 설명을 지정합니다.

자원 팩토리 및 그의 목적을 설명하는 자유 양식의 텍스트 문자열.

데이터 유형 문자열
아카이브 경로
이 자원 어댑터에 대한 모듈이 들어 있는 .RAR 파일에 대한 경로를 지정합니다.
데이터 유형 문자열
클래스 경로
함께 자원 프로바이더 클래스에 대한 위치를 형성하는 경로 또는 JAR 파일 이름의 목록을 지정합니다.

여기에는 자원 어댑터에 필요한 모든 추가 라이브러리가 포함됩니다. 자원 어댑터 코드 베이스 자체는 자동으로 클래스 경로에 추가되지만 .RAR 외부에 어떤 것이 필요한 경우 여기에 지정할 수 있습니다.

데이터 유형 문자열
기본 경로
자원 프로바이더 기본 라이브러리에 대한 위치를 형성하는 경로의 목록을 지정합니다.

자원 어댑터 코드 베이스 자체는 자동으로 클래스 경로에 추가되지만 .RAR 외부에 어떤 것이 필요한 경우 여기에 지정할 수 있습니다.
