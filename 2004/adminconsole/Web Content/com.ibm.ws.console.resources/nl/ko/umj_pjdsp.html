일반 JMS 목적지 설정

연관된 JMS 프로바이더에 사용하려고 선택한 JMS 목적지의 구성 등록 정보를 보거나 변경하려면 이 패널을 사용하십시오.

JMS 목적지는 연관된 일반 JMS 프로바이더에 대한 JMS 목적지 등록 정보를 구성하는 데 사용됩니다. 연관된 JMS 연결 팩토리에서 JMS 목적지에 대한 연결을 작성합니다. 일반 JMS 프로바이더(임베드된 WebSphere JMS 프로바이더나 WebSphere MQ JMS 프로바이더가 아님)에서 사용할 JMS 목적지는 다음 등록 정보를 갖습니다.

이 관리 콘솔 페이지를 보려면 자원-> 일반 JMS 프로바이더-> provider_name-> JMS 목적지-> destination을 클릭하십시오.

이름
IBM WebSphere Application Server 내에서 대기열이 관리 목적으로 알려진 이름.
데이터 유형 문자열
유형
이 JMS 목적지가 대기열(지점간의 경우) 또는 주제(공개/등록의 경우)인지 여부.

다음 옵션 중 하나를 선택하십시오.

대기열
지점간 메시지에 대한 JMS 대기열 목적지
주제
공개/등록 메시징을 위한 JMS 주제 목적지.

JNDI 이름
대기열을 Application Server의 네임 스페이스에 바인드하는 데 사용되는 JNDI 이름.

규칙에 따라, 완전한 JNDI 이름을 사용하십시오(예: jms/Name 양식). 여기서 Name은 자원의 논리 이름입니다.

이 이름은 플랫폼의 바인딩 정보를 연결하는 데 사용됩니다. 바인딩은 모듈 전개 설명자에서 정의한 자원을 플랫폼에서 JNDI에 바인드한 실제(물리적) 자원에 연관시킵니다.
