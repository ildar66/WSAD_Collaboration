WebSphere MQ 대기열 설정

이 패널을 사용하여 WebSphere MQ JMS 프로바이더에서 사용하기 위해 선택된 대기열 목적지의 구성 등록 정보를 보거나 변경하십시오.

대기열 목적지는 JMS 대기열의 등록 정보를 구성하기 위해 사용됩니다. WebSphere JMS 프로바이더에 대해 연관된 대기열 연결 팩토리에서 대기열에 대한 연결을 작성합니다. WebSphere JMS 프로바이더에서 사용하는 대기열의 등록 정보는 다음과 같습니다.

참고:

  • 사용자가 지정하는 등록 정보 값은 WebSphere MQ JMS 자원 구성 시 지정한 값과 일치해야 합니다. WebSphere MQ JMS 자원 구성에 대한 자세한 정보는 WebSphere MQ Using Java 서적을 참조하십시오.
  • WebSphere MQ에서 이름은 최대 48자이며, 최대 20자의 채널 예외를 갖습니다.

이 관리 콘솔 페이지를 보려면 자원-> WebSphere MQ JMS 프로바이더->(내용 분할창의 추가 등록 정보 아래) WebSphere MQ 대기열 목적지-> destination_name을 클릭하십시오.

이름

데이터 유형 문자열
JNDI 이름
대기열을 Application Server의 네임 스페이스에 바인드하는 데 사용되는 JNDI 이름.

규칙에 따라, 완전한 JNDI 이름을 사용하십시오(예: jms/Name 양식). 여기서 Name은 자원의 논리 이름입니다.

이 이름은 플랫폼의 바인딩 정보를 연결하는 데 사용됩니다. 바인딩은 모듈 전개 설명자에서 정의한 자원을 플랫폼에서 JNDI에 바인드한 실제(물리적) 자원에 연관시킵니다.

지정된 우선순위
우선순위 등록 정보가 지정된으로 설정되어 있으면 여기에 이 대기열의 메시지 우선순위를 입력하십시오(범위: 0 - 9).

우선순위 등록 정보가 지정된으로 설정되어 있으면 이 대기열로 전송된 메시지가 이 등록 정보로 지정된 우선순위 값을 가집니다.
