메일 세션 설정

이 페이지를 참조하여 특정 메일 프로바이더를 구성하십시오.

이 관리 콘솔 페이지를 보려면 자원> 메일 프로바이더 > mail_provider > 메일 세션 > mail_session을 클릭하십시오.

구성 탭

이름
JavaMail 세션 오브젝트의 관리 이름을 지정합니다.
JNDI 이름
모든 네이밍 서브컨텍스트를 포함하는 JNDI(Java Naming and Directory Interface) 이름을 지정합니다.

이 이름은 플랫폼에서 JNDI에 바인드한 실제 자원과 클라이언트 응용프로그램 전개 설명자에 정의된 자원에 대한 플랫폼 바인딩 정보 간의 링크를 제공합니다.

설명
관리 레코드에 대한 선택적 설명을 지정합니다.
카테고리
세션을 분류하거나 그룹화하기 위한 선택적 콜렉션을 지정합니다.
메일 전송 호스트
메일을 전송할 때 액세스한 서버를 지정합니다.
메일 전송 프로토콜
메일을 전송할 때 사용되는 전송 프로토콜을 지정합니다.
메일 전송 사용자
메일 전송 호스트가 인증을 요구할 때 사용자 ID를 지정합니다.

이 설정은 일반적으로 대부분의 메일 서버에는 사용되지 않습니다. 사용자 ID 및 암호가 필요한 메일 서버를 사용하지 않으면 이 필드를 공백으로 두십시오.

메일 전송 암호
메일 전송 호스트가 인증을 요구할 때 암호를 지정합니다.

이 설정은 일반적으로 대부분의 메일 서버에는 사용되지 않습니다. 사용자 ID 및 암호가 필요한 메일 서버를 사용하지 않으면 이 필드를 공백으로 두십시오.

보낸 사람
메일 송신자를 지정합니다.

이 값은 기본적으로 수신된 메시지에 발신 또는 회신 주소로 표시되는 인터텟 전자 우편 주소를 표시합니다. 받는 사람의 응답이 이 주소로 옵니다.

메일 저장 호스트
메일을 수신할 때 액세스하는 서버를 지정합니다.

이 설정은 메일 저장 사용자 ID 및 암호와 함께 올바른 메일 계정을 나타냅니다. 예를 들어 메일 계정이 john_william@my.company.com이면 메일 저장 호스트는 my.company.com입니다.

메일 저장 프로토콜
메일을 수신할 때 사용하는 프로토콜을 지정합니다. IMAP, POP3 또는 사용자가 프로바이더를 설치한 임의의 저장 프로토콜일 수 있습니다.
메일 저장 사용자
주어진 메일 계정에 대한 사용자 ID를 지정합니다.

예를 들어 메일 계정이 john_william@my.company.com이면 사용자는 john_william입니다.

메일 저장 암호
주어진 메일 계정에 대한 암호를 지정합니다.

예를 들어 메일 계정이 john_william@my.company.com이면 ID john_william에 대한 암호를 입력하십시오.

디버그
이 메일 세션에 대해 디버그 모드를 On/Off로 토글합니다.
