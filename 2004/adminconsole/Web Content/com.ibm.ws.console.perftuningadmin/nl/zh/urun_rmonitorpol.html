监控策略设置

使用此页面查看或更改控制 Node Agent 如何监控和重新启动流程的设置。

要查看此管理控制台页面，单击服务器 > 应用程序服务器 > server_name > 流程定义 > 监控策略。

“配置”选项卡

最大启动尝试次数
指定在放弃之前尝试启动应用程序服务器的最大次数。
数据类型 整型
ping 间隔
指定父进程（如 Node Agent）和它产生的进程（如应用程序服务器）间尝试通信的频率。可根据您重新启动出现故障的服务器的要求来调整此值。降低此值将很快检测到发生的故障；而增大此值可减少 ping 的频率并因此减少系统开销。
数据类型 整型
ping 超时
当父进程产生子进程时（如当进程管理器产生服务器时），父进程会 ping 子进程以查看是否成功产生子进程。此值指定（在 ping 子进程之后）在假设子进程失败之前，父进程应该等待的秒数。
数据类型 整型
单位 秒
自动重新启动
指定进程是否在失败的情况下自动重新启动。缺省为自动重新启动进程。
数据类型 Boolean
缺省 true
节点启动状态
指定节点完全关闭并重新启动后期待的进程状态。选项为已停止、正在运行和先前。缺省为已停止。
数据类型 String
缺省 已停止
范围 有效值为“已停止”、“正在运行”或“先前”。
