Paramètres d'exécution des processus

Cette page permet d'afficher ou de modifier les informations de ligne de commande relatives au démarrage ou à l'initialisation d'un processus UNIX.

Pour afficher la page de la console d'administration, sélectionnez Serveurs > Serveurs d'applications > nom_serveur > Définition des processus > Exécution du processus.

Onglet Configuration

Priorité du processus
Indique la priorité du système d'exploitation pour le processus. Seuls les utilisateurs root peuvent modifier cette valeur.
Type de données Entier
Valeur par défaut 1000 pour WebSphere Application Server sur la plupart des systèmes d'exploitation. Sous OS/400, la valeur par défaut est 25.
UMASK
Indique le masque utilisateur sous lequel s'exécute le processus (masque de droit d'accès du mode de fichier).
Type de données Entier
Exécuter en tant qu'utilisateur
Indique l'utilisateur pour lequel s'exécute le processus (en tant que tel ou tel utilisateur).
Type de données Chaîne
Exécuter en tant que groupe
Indique le groupe dont le processus est membre et pour lequel il s'exécute (en tant que tel ou tel groupe).

Sous OS/400, le paramètre Exécuter en tant que groupe est ignoré.

Type de données Chaîne
Groupe de processus d'exécution
Indique un groupe de processus spécifique pour le processus. Ce groupe de processus est utile pour le partitionnement de processeur, par exemple. Un administrateur système peut attribuer l'exécution d'un groupe de processeurs à 6 des 12 processeurs, par exemple. La valeur par défaut (0) est d'affecter le processus à aucun groupe spécifique.

Sous OS/400, le paramètre Groupe de processus d'exécution est ignoré.

Type de données Entier
Valeur par défaut 0
