Options personnalisées d'optimisation des paramètres

Cette page permet de personnaliser des paramètres d'optimisation pour les sessions réparties.

Pour afficher cette page de la console d'administration, cliquez sur Serveurs > Serveurs d'applications > nom_serveur > Conteneur Web > Gestion de session > Paramètres d'environnement répartis > Paramètres d'optimisation personnalisés > Paramètres personnalisés.

Onglet Configuration

Fréquence d'écriture
Détermine à quel moment les données d'une session sont écrites dans le système de stockage permanent.

Fin du service de servlet Une session écrit dans une base de données ou dans une autre instance de WebSphere Application Server une fois l'exécution du servlet terminée.
Mise à jour manuelle L'écriture des données de la session dans la base de données ou dans une autre instance de WebSphere Application Server n'a lieu que sur appel de la méthode sync() de l'objet IBMSession (appel qui doit être programmé par le concepteur du servlet).
Périodique Les données de session sont écrites dans la base de données ou dans une autre instance de WebSphere Application Server en fonction de la valeur indiquée de l'intervalle d'écriture. Valeur par défaut : 10 secondes.
Ecriture sélective
Indique si seuls les attributs mis à jour ou tous les attributs de la session sont écrits dans l'emplacement externe, qu'ils aient été modifiés ou non. L'emplacement externe peut correspondre à une base de données ou à une autre instance du serveur d'applications.
Seuls les attributs mis à jour Seuls les attributs mis à jour sont écrits dans le système de stockage permanent.
Tous les attributs de session Tous les attributs sont écrits dans le système de stockage permanent.
Planification du nettoyage des sessions
Indique quand les sessions incorrectes doivent être supprimées d'une base de données ou d'une autre instance du serveur d'applications.
Spécifier la planification du nettoyage des sessions réparties Active le processus d'invalidation planifiée qui vise à supprimer les sessions HTTP invalidées de l'emplacement externe. Activez cette option pour réduire le nombre de mises à jour d'une base de données ou d'une autre instance du serveur d'applications, qui sont nécessaires au maintien en vie des sessions HTTP. Si cette option est désactivée, le processus d'invalidation est périodique et s'exécute à intervalles de quelques minutes pour éliminer les sessions HTTP invalidées.

Lorsque cette option est activée, spécifiez les deux heures de la journée au cours desquelles le processus débarrasse l'emplacement externe des sessions persistantes. Spécifiez les heures d'activité minimale au niveau des serveurs d'applications. Un emplacement externe peut correspondre à une base de données ou à une autre instance du serveur d'applications.

Première heure du jour (0 - 23) Indique la première heure de la journée (heure GMT) au cours de laquelle les sessions invalidées sont éliminées de l'emplacement externe. Cette valeur doit correspondre à un entier positif compris entre 0 et 23. Elle n'est valable que lorsque l'invalidation planifiée est activée.
Seconde heure du jour (0 - 23) Indique la seconde heure de la journée (heure GMT) au cours de laquelle les sessions invalidées sont éliminées de l'emplacement externe. Cette valeur doit correspondre à un entier positif compris entre 0 et 23. Elle n'est valable que lorsque l'invalidation planifiée est activée.
