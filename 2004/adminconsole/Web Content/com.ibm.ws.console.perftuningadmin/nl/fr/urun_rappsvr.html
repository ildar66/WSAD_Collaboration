Paramètres du serveur d'applications

Cette page permet d'afficher ou de modifier les paramètres d'instance d'un serveur d'applications.

Pour afficher cette page de la console d'administration, cliquez sur Serveurs > Serveurs d'applications > nom_serveur.

L'onglet Configuration contient des zones que vous pouvez modifier et l'onglet Exécution fournit des informations en lecture seule. L'onglet Exécution est disponible uniquement lorsque le serveur est en cours de fonctionnement.

Onglet Configuration

Nom
Indique le nom logique du serveur. Les noms de serveur doivent être uniques dans un noeud.
Type de données Chaîne
Valeur par défaut server1
Etat initial
Indique l'état d'exécution du composant demandé lors du premier démarrage du serveur.
Type de données Chaîne
Valeur par défaut Démarré
Règle de chargeur de classe d'application
Indique s'il faut utiliser un chargeur de classe unique pour charger toutes les applications ou un chargeur de classe différent pour chaque application.

Les options possibles sont SINGLE et MULTIPLE. Par défaut, un chargeur de classe différent est utilisé pour chaque application (MULTIPLE).

Type de données Chaîne
Valeur par défaut MULTIPLE
Mode de chargement des classes de l'application
Indique si le chargeur de classe doit d'abord effectuer la recherche dans le chargeur de classe parent ou dans le chargeur de classe de l'application afin de charger une classe. PARENT_FIRST est la norme pour les chargeurs de clase JDK et les chargeurs de classe WebSphere. Si vous indiquez PARENT_LAST, votre application peut remplacer des classes contenues dans le chargeur de classe parent, mais cette action peut éventuellement aboutir à l'exception ClassCastException ou à des erreurs LinkageErrors si vous avez utilisé aussi bien des classes remplacées que des classes non remplacées.

Les options possibles sont PARENT_FIRST et PARENT_LAST. Par défaut, la recherche s'effectue d'abord dans le chargeur de classe parent, puis dans le chargeur de classe d'application pour charger une classe.

Type de données Chaîne
Valeur par défaut PARENT_FIRST
