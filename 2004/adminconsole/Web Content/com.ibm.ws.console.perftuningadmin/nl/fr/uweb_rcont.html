Paramètres des conteneurs Web

Utilisez cette page pour configurer les paramètres des conteneurs.

Pour accéder à cette page, recherchez le chemin d'accès suivant dans la console d'administration :

Serveurs > Serveurs d'applications > instance_serveur > conteneur Web

Configuration - Propriétés générales

Hôte virtuel par défaut
Spécifie un hôte virtuel, c'est-à-dire une configuration permettant à une seule machine hôte de se comporter comme plusieurs machines hôtes. Les ressources associées à un hôte virtuel ne peuvent pas partager de données avec les ressources associées à un autre hôte virtuel, même si les deux hôtes sont sur la même machine physique.

Sélectionnez une option d'hôte virtuel :

Hôte par défaut
Le produit fournit un hôte virtuel par défaut comportant des alias courants, tels que l'adresse IP de la machine, un nom d'hôte court et un nom d'hôte complet. L'alias constitue la première partie du chemin d'accès d'une ressource, comme un servlet. Par exemple, il s'agit de localhost:9080 dans la demande http://localhost:9080/monServlet.
Hôte d'administration
Autre nom du serveur d'applications ; désigné également sous le nom server1 dans l'installation de base. Ce processus supporte l'utilisation de la console d'administration.
Mise en cache du servlet
Lorsqu'un servlet est appelé et génère une sortie destinée à être enregistrée en mémoire cache, l'entrée créée en mémoire cache contient non seulement la sortie mais aussi des éléments découlant de l'appel. Ces éléments sont constitués par les appels à d'autres servlets ou fichiers JSP (Java Server Pages), ainsi que par les métadonnées relatives à l'entrée, y compris les informations concernant les délais et la priorité des entrées.
