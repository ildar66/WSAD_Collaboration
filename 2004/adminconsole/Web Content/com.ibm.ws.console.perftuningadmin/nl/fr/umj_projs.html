Paramètres de serveur JMS WebSphere interne

les fonctions JMS sur un noeud situé à l'intérieur du domaine d'administration de WebSphere Application Server sont assurées par le serveur JMS de ce noeud. Utilisez cet écran pour afficher ou modifier les propriétés de configuration du fournisseur JMS sélectionné.

Vous pouvez utiliser cet écran pour configurer un ensemble général de propriétés de serveur JMS, qui s'ajoute aux valeurs par défaut des propriétés configurées automatiquement pour le fournisseur JMS WebSphere imbriqué.

Important: En général, les valeurs par défaut des propriétés WebSphere MQ conviennent pour les serveurs JMS internes de WebSphere. Cependant, si vous avez de fortes charges de messagerie, vous pouvez modifier certaines propriétés WebSphere MQ ; par exemple, les propriétés pour les emplacements des fichiers journaux, les pages de fichiers et les pages de mémoire intermédiaire. Pour davantage d'informations sur la configuration des propriétés WebSphere MQ, consultez le manuel WebSphere MQ System Administration, SC33-1873, disponible sur le site    IBM Publications Center ou dans le kit de collection MQ WebSphere, SK2T-0730.

Nom
Nom sous lequel le serveur JMS est connu pour l'administration d'IBM WebSphere Application Server.

Ce nom ne doit pas être modifié.

Type de données Chaîne
Unité Non applicable
Valeur par défaut Serveur JMS interne WebSphere
Plage Non applicable
Description
Description du serveur JMS dans le cadre de l'administration d'IBM WebSphere Application Server.

Cette chaîne ne doit pas être modifiée.

Type de données Chaîne
Valeur par défaut Serveur JMS interne WebSphere
Nombre d'unités d'exécution  
Le nombre d'unités d'exécution simultanées à utiliser par le moteur de correspondance de Publication/Abonnement

Le nombre d'unités d'exécution simultanées défini doit être un petit nombre.

Type de données Entier
Unité unités d'exécution
Valeur par défaut 1
Plage Supérieur ou égal à 1
Noms de files d'attente
Les noms des files d'attente hébergées par ce serveur JMS. Chaque nom de file d'attente doit être ajouté sur une ligne distincte.

Chaque file d'attente répertoriée dans cette zone doit avoir un objet administratif de file d'attente séparé avec le même nom administratif. Pour qu'une file d'attente soit disponible pour les applications, définissez une file d'attente WebSphere et ajoutez son nom dans cette zone sur l'écran du serveur JMS pour l'hôte devant héberger la file d'attente.

Type de données ASCII
Unité Nom de la file d'attente
Valeur par défaut Non applicable
Plage Chaque entrée de cette zone est un nom de file d'attente d'un maximum de 45 caractères, qui doit correspondre exactement (y compris les majuscules et les minuscules) à l'objet administratif de file d'attente WebSphere défini pour cette file d'attente.
Etat initial
L'état souhaité pour le serveur JMS lors de son prochain redémarrage.
Type de données Enumération
Unité Non applicable
Valeur par défaut Démarré
Plage
Démarré
Le serveur JMS est démarré automatiquement.
Arrêté
Le serveur JMS n'est pas démarré automatiquement. Si des applications d'entreprise doivent utiliser les fonctions de serveur JMS fournies par le serveur JMS, l'administrateur système doit démarrer manuellement le serveur JMS ou sélectionner la valeur Démarré de cette propriété puis relancer le serveur JMS.

Pour relancer le serveur JMS pour un serveur d'applications, arrêtez puis relancez ce serveur d'applications.
