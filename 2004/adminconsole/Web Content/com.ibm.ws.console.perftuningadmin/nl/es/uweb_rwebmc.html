Valores de módulo Web

Utilice esta página para configurar los valores de módulos Web.

Para acceder a esta página, recorra la siguiente vía de acceso en la consola administrativa:

Aplicaciones > Aplicación de empresa >
instancia_aplicación > Módulo Web
URI
Especifica un URI que, cuando resuelve con respecto al URL de la aplicación, especifica la ubicación del contenido del archivador de módulos de un sistema de archivos. Si el módulo se empaqueta como parte de una aplicación desplegada o de un archivo EAR (Enterprise Archive), el URI debe coincidir con el URI de un URI ModuleRef en el descriptor de despliegue de la aplicación.
Nombre
Especifica el nombre de visualización exclusivo del módulo.
ADD alternativo
Especifica el nombre de archivo de un archivo descriptor de despliegue alternativo que se debe utilizar en lugar del archivo descriptor de despliegue original en el archivo JAR del módulo.

Este archivo es la versión postensamblaje del archivo descriptor de despliegue. Puede editar el archivo descriptor de despliegue original para que resuelva las dependencias y la información de seguridad. Si especifica el uso del descriptor de despliegue alternativo mantendrá intacto el archivo descriptor de despliegue.

El valor de la propiedad ADD alternativo debe ser el nombre completo de la vía de acceso del archivo descriptor de despliegue relativo al directorio raíz del módulo. Por convenio, el archivo se encuentra en el directorio ALT-INF. Si no se especifica esta propiedad, el archivo descriptor de despliegue se lee desde el archivo JAR del módulo.

Peso inicial
Especifica el orden en que se inician los módulos. Los módulos de menor peso se inician antes que los de más peso.
Preferencia de clases WEB-INF
Especifica las clases que deben cargarse en WEB-INF antes que ninguna otra. Se recomienda implantar el cargador de clases de aplicaciones para que las clases y los recursos empaquetados en el archivo WAR se carguen antes que las clases y los recursos que residan en los archivos JAR de la biblioteca de contenedores.
Estado inicial
Especifica el estado por omisión de esta aplicación al arrancar el servidor.
