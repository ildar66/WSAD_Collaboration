Valores de base de datos

Utilice esta página para especificar los valores del soporte de sesión de base de datos.

Para ver esta página de la consola administrativa, pulse Servidores > Servidores de aplicaciones > nombre_servidor > Contenedor Web > Gestión de sesiones > Valores de entorno distribuido > Base de datos.

Pestaña Configuración

Nombre JNDI de origen de datos
Especifica la descripción del origen de datos.

El nombre JNDI del origen de datos no habilitado para XA desde donde la gestión de sesiones obtiene las conexiones de base de datos. Por ejemplo, si el nombre JNDI del origen de datos es "jdbc/sessions", especifique "jdbc/sessions." El origen de datos representa una agrupación de conexiones de bases de datos y una configuración para dicha agrupación (como el tamaño de la agrupación). El origen de datos ya debe existir como un recurso configurado en el entorno.

ID de usuario  
Especifica el ID de usuario para el acceso a bases de datos
Contraseña  
Especifica la contraseña para el acceso a bases de datos
Confirmar contraseña  
Especifica la contraseña una segunda vez para confirmar que se entró correctamente.
Tamaño de fila de DB2  
Especifica el tamaño de página del espacio de tablas configurado para la tabla de sesiones, si utiliza una base de datos DB2. Los valores posibles son 4, 8, 16 y 32 kilobytes (K). El tamaño de fila por omisión es 4K.

El tamaño de fila por omisión es 4K. En DB2, puede actualizarse en un valor superior. Esto puede mejorar el rendimiento de la base de datos en algunos entornos. Cuando este valor es distinto de 4, debe especificar el Nombre de espacio de tablas que va a utilizar. Para páginas de 4 K, el Nombre de espacio de tablas es opcional.

Nombre de espacio de tablas  
Especifica el espacio de tablas que se va a utilizar para la tabla de sesiones.

Este valor es necesario cuando el tamaño de página de DB2 es distinto de 4K.

Utilizar sesiones de varias filas
Especifica que debe ubicarse cada instancia de los datos de la aplicación en una fila por separado en la base de datos, permitiendo que en cada sesión se almacenen grandes cantidades de datos. Con ello puede obtenerse un mejor rendimiento en determinados ejemplos de uso. Si la utilización del esquema de varias filas no está habilitado. se pueden ubicar instancias de datos de aplicación en la misma fila.
