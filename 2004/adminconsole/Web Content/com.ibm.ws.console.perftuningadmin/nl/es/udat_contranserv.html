Valores del servicio de transacciones

Utilice esta página para modificar los valores del servicio de transacciones.

Para ver esta página de la consola administrativa, pulse Servidores > Servidores de aplicaciones > servidor > Servicio de transacciones.

Pestaña Configuración

Directorio de archivo de anotaciones cronológicas de transacciones
Especifica el nombre de un directorio de este servidor donde el servicio de transacciones almacena los archivos de anotaciones cronológicas para la recuperación.

Al arrancar, el archivo de anotaciones cronológicas de transacciones expande un valor en blanco de la configuración del servidor como directorio (raíz_instalación)/tranlog/(nombre_servidor).

Cuando la aplicación que se ejecuta en el producto WebSphere accede a más de un recurso, el producto WebSphere almacena información de la transacción para coordinar y gestionar correctamente la transacción distribuida. En cargas de transacción mayores, esta persistencia disminuye el rendimiento del servidor de aplicaciones por su dependencia del sistema operativo y de los sistemas de almacenamiento subyacentes.

Para aumentar el rendimiento, desplace los archivos de anotaciones cronológicas de transacción a dispositivos de almacenamiento con más unidades de disco físicas o, preferiblemente unidades de disco RAID. Cuando se mueven los archivos de anotaciones cronológicas a sistemas de archivos en los discos RAID, la tarea de grabar datos en el soporte físico se comparte en todas las distintas unidades de disco. Esto permite un acceso más simultáneo a información de transacciones persistente y un acceso más rápido a esos datos de los archivos de anotaciones cronológicas. En función del diseño del subsistema de aplicación y almacenamiento, la obtención de rendimiento puede variar del 10% al 100% o incluso más en algunos casos.

Este cambio sólo se aplica a la configuración donde la aplicación utiliza recursos distribuidos o transacciones XA, por ejemplo, se accede a varias bases de datos y recursos dentro de una sola transacción. Considere establecer esta propiedad cuando el servidor de aplicaciones muestra uno o más signos siguientes:

  • Sigue habiendo poca utilización de la CPU a pesar de que aumentan las transacciones
  • No se pueden efectuar las transacciones con tiempos de espera excedidos en varias ocasiones
  • Se producen retrotraimientos de transacciones con la excepción no se puede incluir la transacción
  • El servidor de aplicaciones se inhabilita en mitad de una ejecución y se requiere que se reinicie el servidor
  • El disco en que se está ejecutando el servidor de aplicaciones muestra mayor utilización

Tipo de datos String
Valor por omisión El valor inicial es el directorio %WAS_HOME%\tranlog/(nombre_servidor) y un tamaño por omisión de 1 MB.
Recomendado Cree un sistema de archivos con al menos 3 ó 4 unidades de disco RAID combinadas en una configuración RAID-0. A continuación, cree el archivo de anotaciones cronológicas de transacciones en este sistema de archivos con el tamaño por omisión. Cuando el servidor se ejecuta bajo carga, compruebe la entrada y la salida de disco. Si el tiempo de entrada y salida de disco supera el 5%, considere añadir más discos físicos para disminuir el valor. Si se produce poca entrada y salida de disco, pero el servidor sigue con alto volumen, considere aumentar el tamaño de los archivos de anotaciones cronológicas.
Tiempo de espera de actividad total de transacción
Especifica la duración máxima en segundos de las transacciones en este servidor de aplicaciones.

Cualquier transacción a la que no se solicite que finalice antes de dicho tiempo de espera se retrotraerá. Si está establecido en 0, no existe límite de tiempo de espera.

Tipo de datos Integer
Unidades Segundos
Valor por omisión 120
Rango 0 a 2 147 483 647
Tiempo de espera de inactividad de cliente
Especifica el intervalo máximo de tiempo en segundos entre peticiones de transacciones de un cliente remoto.

Cualquier periodo de inactividad del cliente que exceda este tiempo de espera, provoca que la transacción se retrotraiga en este servidor de aplicaciones. Si está establecido en 0, no existe límite de tiempo de espera.

Tipo de datos Integer
Unidades Segundos
Valor por omisión 60
Rango 0 a 2 147 483 647
