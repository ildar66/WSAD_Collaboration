Valores personalizados de parámetros de ajuste

Utilice esta página para personalizar los parámetro de ajuste para sesiones distribuidas.

Para ver esta página de la consola administrativa, pulse Servidores > Servidores de aplicaciones > nombre_servidor > Contenedor Web > Gestión de sesiones > Valores de entorno distribuido > Parámetros de ajuste personalizado > Valores personalizados.

Pestaña Configuración

Frecuencia de escritura
Especifica cuando se escribe la sesión en el almacén persistente.

Finalización del servicio de servlet Una sesión escribe en la base de datos o se produce otra instancia de WebSphere Application Server cuando finaliza la ejecución del servlet.
Actualización manual Se necesita una sincronización programada del objeto IBMSession para poder escribir los datos de la sesión en la base de datos o se necesita otra instancia de WebSphere Application Server.
Basado en la hora Los datos de la sesión se escriben en la base de datos o se genera otra instancia de WebSphere Application Server según el valor especificado para Intervalo de escritura. Valor por omisión: 10 segundos.
Contenido de escritura
Especifica si sólo los atributos actualizados deben escribirse en una ubicación externa o si todos los atributos de sesiones deben escribirse en la ubicación externa, se hayan modificado o no. La ubicación externa puede ser una base de datos u otra instancia del servidor de aplicaciones.
Sólo atributos actualizados Sólo se escriben atributos actualizados en el almacén persistente.
Todos los atributos de sesión Todos los atributos se escriben en el almacén persistente.
Planificación de limpieza de sesiones
Especifica cuándo se deben limpiar las sesiones no válidas de una base de datos o de otra instancia del servidor de aplicaciones.
Especificar planificación de limpieza de sesiones distribuidas Habilita el proceso de invalidación de planificación para borrar las sesiones HTTP invalidadas de la ubicación externa. Habilite esta opción para reducir el número de actualizaciones de una base de datos u otra instancia del servidor de aplicaciones necesaria para mantener activas las sesiones HTTP. Si esta opción no está inhabilitada, el proceso de invalidación se ejecuta cada vez que transcurren algunos minutos para eliminar las sesiones HTTP invalidadas.

Cuando esta opción está habilitada, debe especificar las dos horas del día para el proceso de limpieza de sesiones invalidadas de la ubicación externa. Especifique las horas en que haya menos actividad en los servidores de aplicaciones. Una ubicación externa puede ser una base de datos u otra instancia del servidor de aplicaciones.

Primera hora del día (0 - 23) Indica la primera hora según la hora media de Greeenwich (GMT), durante la cual las sesiones invalidadas se borran de la ubicación externa. Especifique este valor como un entero positivo entre 0 y 23. Este valor sólo es válido si está habilitada la invalidación de planificación.
Segunda hora del día (0 - 23) Indica la segunda hora según la hora media de Greeenwich (GMT), durante la cual las sesiones invalidadas se borran de la ubicación externa. Especifique este valor como un entero positivo entre 0 y 23. Este valor sólo es válido si está habilitada la invalidación de planificación.
