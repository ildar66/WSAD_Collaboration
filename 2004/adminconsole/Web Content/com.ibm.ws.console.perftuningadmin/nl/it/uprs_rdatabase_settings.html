Impostazioni database

Utilizzare questa pagina per specificare le impostazioni per il supporto della sessione del database.

Per visualizzare questa pagina della console di gestione, fare clic su Server > Server delle applicazioni > nome_server > Contenitore Web > Gestione sessioni > Impostazioni ambiente distribuito > Database.

Scheda Configurazione

Nome JNDI origine dati
Specifica la descrizione dell'origine dati

Il nome JNDI dell'origine dati abilitata non XA da cui Gestione sessioni ottiene i collegamenti database. Ad esempio, se il nome JNDI dell'origine dati è "jdbc/sessions", specificare "jdbc/sessions." L'origine dati rappresenta un lotto di connessioni database e una configurazione per tale lotto (come la dimensione lotto). L'origine dati deve già essere stata configurata come risorsa nell'ambiente.

ID utente  
Specifica l'ID utente per l'accesso al database
Password  
Specifica la password per l'accesso al database
Conferma password  
Specifica di nuovo la password per verificare che venga registrata correttamente.
Dimensione riga DB2  
Specifica la dimensione di pagina dello spazio tabella configurata per la tabella sessioni, se si utilizza un database DB2. I valori possibili sono 4, 8, 16 e 32 kilobyte (K). La dimensione di riga predefinita è 4K.

La dimensione riga predefinita è 4K. In DB2, è possibile aggiornarla a un valore maggiore. Ciò può supportare le prestazioni del database in alcuni ambienti. Se il valore è diverso da 4, è necessario specificare il Nome spazio tabella da utilizzare. Per pagine da 4K, il Nome spazio tabella è facoltativo.

Nome tablespace  
Specifica lo spazio tabella da utilizzare per la tabella sessioni.

Questo valore è richiesto quando la Dimensione pagina DB2 è diversa da 4K.

Usa sessioni a più righe
Specifica che ciascuna istanza dei dati dell'applicazione deve essere posta in una riga separata nel database, consentendo di memorizzare una quantità elevate di dati per ogni sessione. Quest'operazione può offrire prestazioni migliori in alcuni scenari d'uso. Se l'uso dello schema a più righe non è abilitato, le istanze dei dati dell'applicazione possono essere collocate sulla stessa riga.
