Impostazioni di Distribuzione del modulo Web

Utilizzare questa pagina per configurare un'istanza di distribuzione del modulo Web.

Per visualizzare questa pagina della console di gestione, fare clic su Applicazioni > Applicazioni Enterprise > istanza_applicazione > Moduli Web > istanza_modulo Web.

URI
Specifica un URI che, quando viene determinato come relativo all'URL dell'applicazione, specifica la posizione dei contenuti dell'archivio di un modulo su un file system. L'URI deve corrispondere all'URI di ModuleRef nel descrittore di distribuzione di un'applicazione, se il modulo è stato incluso come parte di un'applicazione sviluppata o di file EAR (enterprise archive).
DD alternato
Specifica il nome di un file alternativo del descrittore di distribuzione da utilizzare al posto di quello originale nel file JAR del modulo.

Questo file è la versione post-assemblaggio del file del descrittore di distribuzione. È possibile modificare il file del descrittore di distribuzione originale per risolvere le informazioni sulle dipendenze e sulla sicurezza. La specifica dell'uso del descrittore di distribuzione alternativo mantiene intatto il file originale del descrittore di distribuzione.

Il valore della proprietà del DD alternativo deve corrispondere al nome percorso completo del file del descrittore di distribuzione, relativo alla directory root del modulo. Per convenzione, il file si trova nella directory ALT-INF. Se questa proprietà non è specificata, il file del descrittore di distribuzione viene letto dal file JAR del modulo.

Peso di avvio
Specifica l'ordine in cui sono avviati i moduli. I moduli con relazione minore vengono avviati prima di quelli con relazione maggiore.
Modalità del programma di caricamento classi
Specifica se il programma di caricamento classi deve eseguire una ricerca prima nel programma di caricamento classi parent o in quello dell'applicazione per caricare una classe. Lo standard per i programmi di caricamento classi JDK e WebSphere è PARENT_FIRST. Se si specifica PARENT_LAST, l'applicazione può ignorare le classi contenute nel programma di caricamento classi parent, ma quest'azione può potenzialmente generare un'eccezione ClassCastException o un errore LinkageErrors se si utilizzano insieme le classi sovrascritte e le classi non sovrascritte.

Le opzioni sono PARENT_FIRST e PARENT_LAST. Per caricare una classe, il valore predefinito è quello di eseguire una ricerca nel programma di caricamento classi parent prima di effettuare tale operazione in quello dell'applicazione.

Tipo dati Stringa
Predefinito PARENT_FIRST
