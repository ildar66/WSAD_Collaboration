Impostazioni del servizio di traccia diagnostica

Utilizzare questa pagina per visualizzare di nuovo e modificare le proprietà del servizio di traccia diagnostica.

Per visualizzare questa pagina, fare clic su Risoluzione dei problemi > Log e Traccia > server > Traccia diagnostica.

Scheda Configurazione

Abilita traccia
Abilita il servizio di traccia.

Se questa opzione non è selezionata, le seguenti proprietà di configurazione non vengono inviate al servizio di traccia del server delle applicazioni all'avvio del server.

Specifica di traccia
Specifica i dettagli di traccia.

Immettere una stringa di traccia che specifica i componenti, i pacchetti o i gruppi da tracciare. Tale stringa deve essere conforme alla grammatica descritta di seguito. È possibile immettere direttamente la stringa, oppure generarla utilizzando un'interfaccia di traccia grafica. Fare clic su Modifica per avviare l'interfaccia di traccia grafica.

Se si avvia questa interfaccia dalla scheda di configurazione, viene visualizzato un elenco di componenti, pacchetti e gruppi ben noto. Tale elenco potrebbe non essere esaustivo.

Se si avvia questa interfaccia di traccia grafica dalla scheda runtime, l'elenco di componenti, pacchetti e gruppi visualizza tutti i componenti registrati sul server in esecuzione.

Il formato della specifica della traccia è:

<componente> =
<tipo_traccia>= enabled | disabled
dove: <componente> è il componente per cui abilitare o disabilitare la traccia e <tipo_traccia> è il tipo di traccia da abilitare o disabilitare. Separare più specifiche di traccia diverse con due punti (:).

I componenti corrispondono ai pacchetti Java e alle classi o alle raccolte di pacchetti Java. Utilizzare * come carattere jolly per indicare i componenti che includono tutte le classi in tutti i pacchetti contenuti dal componente specificato. Ad esempio:

*
Specifica tutti i codici tracciabili in esecuzione sul server delle applicazioni, compresi il codice di sistema di WebSphere Application Server e il codice del cliente.
com.ibm.ws.*
specifica tutte le classi il cui nome del pacchetto inizia per com.ibm.ws.
com.ibm.ws.classloader.JarClassLoader
Specifica solo la classe JarClassLoader.

Per ulteriori informazioni sulla grammatica della stringa di traccia, consultare la sezione Abilitazione della traccia nell'InfoCenter di WebSphere Application Server.

Nota: Può verificarsi un errore durante l'impostazione di una specifica di traccia dalla console di gestione se le selezioni vengono effettuate dagli elenchi Gruppi e Componenti. In alcuni casi, la selezione effettuata da un elenco non va perduta durante l'aggiunta di una selezione dall'altro elenco. Per risolvere questo problema, immettere la specifica di traccia desiderata direttamente nel campo della voce Specifica di traccia.

Output di traccia  
Indica dove andrebbe l'output di traccia.

È possibile scrivere l'output di traccia direttamente in un file di output, oppure memorizzarlo e scriverlo in un file su richiesta utilizzando il pulsante Dump, che si trova sulla pagina runtime.

Buffer di memoria
Specifica che l'output di traccia deve essere scritto su un buffer circolare in memoria. Se si seleziona questa opzione, è necessario specificare i seguenti parametri:
Dimensione massima buffer
Specifica il numero di voci, in migliaia, che può essere memorizzato nel buffer. Una volta superato questo numero, le vecchie voci vengono sovrascritte dalle nuove.
Nome file dump
Il nome del file su cui verrà scritta la memoria buffer una volta eseguito il dump. Questa opzione non è disponibile dalla scheda Runtime.

File
Specifica la scrittura dell'output di traccia su un file di log gestito automaticamente.

Questo file scrive i messaggi nel file finché non viene raggiunta la dimensione massima. Quando il file raggiunge la dimensione specificata, il collegamento è temporaneamente sospeso e il file di log viene chiuso e ridenominato. Il nuovo nome si basa su quello originale del file, più un qualificativo data/ora che indica, appunto, quando si è verificata la ridenominazione. Una volta completata l'operazione, viene riaperto un nuovo file di log vuoto con il nome originale e il collegamento viene ripristinato. Come risultato del rollover, non è stato perso alcun messaggio, sebbene un singolo messaggio può essere diviso tra i due file.

Se si seleziona questa opzione, è necessario specificare i seguenti parametri:

Dimensione massima file
Specifica la dimensione massima, in megabyte, che un file di output può raggiungere.

Questo attributo è valido solo se viene selezionato l'attributo Dimensione file. Quando un file raggiunge questa dimensione, viene eseguito il rollover, come descritto in precedenza.

Numero massimo di file di cronologia
Specifica il numero massimo di file, sottoposto a rollover, da conservare.
Nome file
Specifica il nome del file sul quale è scritto l'output di traccia.

Formato output di traccia  
Specifica il formato dell'output di traccia.

È possibile specificare uno dei tre livelli per l'output di traccia:

Base (Compatibile)
Conserva solo le informazioni di traccia di base. Selezionare quest'opzione per ridurre lo spazio occupato dell'output di traccia.
Advanced
Conserva le informazioni di traccia più specifiche. Selezionare quest'opzione per visualizzare ulteriori informazioni sulla traccia da utilizzare per l'individuazione e la risoluzione dei problemi.
Log Analyzer
Conserva le informazioni di traccia in un formato compatibile con lo strumento Log Analyzer. Selezionare quest'opzione se si desidera utilizzare l'output di traccia come input allo strumento Log Analyzer.
