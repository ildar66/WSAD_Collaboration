Impostazioni applicazione enterprise

Utilizzare questa pagina per configurare un'applicazione enterprise.

Per visualizzare questa pagina della console di gestione, fare clic su Applicazioni > Applicazioni enterprise > nome_applicazione.

Scheda Configurazione

Nome
Specifica un nome logico dell'applicazione. I nomi dell'applicazione devono essere univoci all'interno di una cella e non possono contenere caratteri non validi per i nomi oggetto.
Tipo dati Stringa
Relazione di avvio
Specifica l'ordine in cui le applicazioni sono state avviate all'attivazione del server. L'applicazione con la relazione di avvio più bassa viene avviata per prima.
Tipo dati Numero intero
Predefinito 1
Intervallo Da 0 a 100
File binari dell'applicazione
Specifica la directory in cui verrà installato il file EAR dell'applicazione. Il valore predefinito è quello di APP_INSTALL_ROOT/nome_cella, dove la variabile APP_INSTALL_ROOT è root_installazione/installedApps; ad esempio, C:\WebSphere\AppServer\installedApps\nome_cella.

È possibile specificare un percorso assoluto oppure utilizzare una variabile pathmap, come ad esempio ${MY_APPS}. Si può utilizzare una variabile pathmap in qualunque installazione . Una variabile ${CELL} di WebSphere Application Server, che denota il nome cella corrente, può trovarsi anche nella variabile pathmap; ad esempio, ${MY_APP}/${CELL}.

Tipo dati Stringa
Unità Nome percorso completo
Utilizzo dei metadati dei file binari
Specifica se il server delle applicazioni utilizzerà il collegamento, le estensioni e i descrittori di distribuzione individuati con il documento di distribuzione dell'applicazione, il file deployment.xml (predefinito), o quelli individuati nel file dell'applicazione enterprise (EAR).
Tipo dati Valore booleano
Predefinito vero
Abilitazione distribuzione
Specifica se WebSphere Application Server espande o cancella i file binari dell'applicazione nella destinazione di installazione. Il valore predefinito è impostato sull'abilitazione della distribuzione dell'applicazione. I file binari delle applicazioni installate sono estesi alla directory specificata. I file binari, inoltre, vengono cancellati con la disinstallazione e quando si salvano le modifiche sulla configurazione. Se si disattiva questa opzione, bisogna verificare che i file binari dell'applicazione siano estesi in modo appropriato nelle directory di destinazione di tutti i nodi, dove è in esecuzione l'applicazione.
Tipo dati Valore booleano
Predefinito vero
Modalità del programma di caricamento classi
Specifica se il programma di caricamento classi deve eseguire una ricerca nel programma di caricamento classi parent o in quello dell'applicazione, prima di caricare una classe. Il valore standard per i programmi di caricamento classi di JDK e di WebSphere Application Server è PARENT_FIRST. Se si specifica PARENT_LAST, l'applicazione può sovrapporsi alle classi contenute nel programma di caricamento classi parent ma questa azione può, potenzialmente, causare un'eccezione ClassCastException o un errore LinkageErrors, se si utilizzano insieme classi sovrapposte e non.

Le opzioni sono PARENT_FIRST e PARENT_LAST. Per caricare una classe, il valore predefinito è quello di eseguire una ricerca nel programma di caricamento classi parent prima di effettuare tale operazione in quello dell'applicazione.

Tipo dati Stringa
Predefinito PARENT_FIRST
Politiche del programma di caricamento classi WAR
Specifica se utilizzare un solo programma di caricamento classi per caricare tutti i file WAR dell'applicazione o utilizzare un programma di caricamento classi diverso per ogni file WAR.

Le opzioni sono APPLICATION e MODULE. Il valore predefinito è quello di utilizzare un programma di caricamento classi per caricare ogni file WAR.

Tipo dati Stringa
Predefinito MODULE
Creazione di MBeans per le risorse
Specifica se creare file MBean per varie risorse (come servlet o file JSP) all'interno di un'applicazione.
Tipo dati Valore booleano
Predefinito vero
Ricaricamento abilitato
Specifica se abilitare o meno il ricaricamento delle classi quando i file dell'applicazione vengono aggiornati.
Tipo dati Valore booleano
Predefinito vero
Intervallo di ricaricamento
Specifica il numero di secondi impiegato per eseguire la scansione del file system dell'applicazione per ricercare i file aggiornati. Il valore predefinito è il valore dell'attributo di intervallo di caricamento nel file (META-INF/ibm-application-ext.xmi) dell'estensione IBM del file EAR. Questa impostazione diventa attiva solo se è abilitato il caricamento classi.

L'intervallo di caricamento specificato qui sovrascrive il valore indicato nelle estensioni IBM per ogni modulo Web nel file EAR (che, in risposta, sovrascrive l'intervallo di caricamento specificato nelle estensioni IBM dell'applicazione nel file EAR).

Tipo dati Numero intero
