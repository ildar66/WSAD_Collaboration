Impostazioni personalizzate di parametri di ottimizzazione

Utilizzare questa pagina per personalizzare i parametri di ottimizzazione per le sessioni distribuite.

Per visualizzare questa pagina della console di gestione, fare clic su Server > Server delle applicazioni > nome_server > Contenitore Web > Gestione sessioni > Impostazioni ambiente distribuito > Parametri di ottimizzazione personalizzati > Impostazioni personalizzate.

Scheda Configurazione

Frequenza di scrittura
Specifica quando la sessione viene scritta nella memoria persistente.

Fine del servizio servlet Una sessione viene scritta sul database o su un'altra istanza di WebSphere Application Server dopo l'esecuzione del server.
Aggiornamento manuale È necessaria una sincronizzazione programmatica sull'oggetto IBMSession per scrivere i dati della sessione sul database o su un'altra istanza di WebSphere Application Server
In base all'ora I dati della sessione vengono scritti sul database o su un'altra istanza di WebSphere Application Server basata sul valore Intervallo scrittura specificato. Valore predefinito: 10 secondi
Contenuti di scrittura
Specifica se sull'ubicazione esterna devono essere scritti solo gli attributi aggiornati o tutti gli attributi di sessione, che siano stati modificati o meno. L'ubicazione esterna può essere un database o un'altra istanza del server delle applicazioni.
Solo attributi aggiornati Nella memoria persistente vengono scritti solo gli attributi aggiornati.
Tutti gli attributi sessione Nella memoria persistente vengono scritti tutti gli attributi.
Pianificazione ripulitura sessioni
Specifica quando ripulire le sessioni non valide dal database o dall'istanza del server delle applicazioni.
Specifica pianificazione di ripulitura sessioni distribuite Abilita il processo di scadenza validità pianificato per ripulire le sessioni HTTP scadute dall'ubicazione esterna. Abilitare questa opzione per ridurre il numero di aggiornamenti su un database o su un'altra istanza del server delle applicazione richiesti per mantenere attive le sessioni HTTP. Se l'opzione non è attivata, le sessioni HTTP scadute vengono rimosse dopo alcuni minuti.

Se invece è abilitata, specificare le due ore al giorno in cui il processo ripulisce le sessioni scadute nell'ubicazione esterna. Specificare le ore di minore attività nei server delle applicazioni. Un'ubicazione esterna può essere un database o un'altra istanza del server delle applicazioni.

Prima ora del giorno (0 - 23) Indica la prima ora, in GMT (Greenwich Mean Time), durante la quale le sessioni non convalidate verranno eliminate dall'ubicazione esterna. Specificare questo valore con un numero intero positivo compreso tra 0 e 23. Questo valore è valido solo quando la scadenza convalida pianificata viene attivata.
Seconda ora del giorno (0 - 23) Indica la seconda ora, in GMT (Greenwich Mean Time), durante la quale le sessioni non convalidate verranno eliminate dall'ubicazione esterna. Specificare questo valore con un numero intero positivo compreso tra 0 e 23. Questo valore è valido solo quando la scadenza convalida pianificata viene attivata.
