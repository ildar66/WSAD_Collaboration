Impostazioni del lotto thread

Utilizzare questa pagina per configurare un gruppo di thread utilizzato da un server delle applicazioni. Le richieste vengono inviate al server attraverso un qualsiasi trasporto HTTP. Un lotto di thread consente ai componenti server di riutilizzare i thread per eliminare la necessità di creare nuovi thread durante il run time. La creazione di nuovi thread richiede tempo e risorse.

Per visualizzare questa pagina della console di gestione, fare clic su Server > Gestisci server delle applicazioni > nome_server > Servizio ORB > Lotto thread. (È possibile accedere a questa pagina tramite più percorsi di navigazione).

Scheda Configurazione

Dimensione minima
Specifica il numero minimo di thread consentiti nel lotto.
Tipo dati Numero intero
Predefinito 10
Dimensione massima
Specifica il numero massimo di thread consentiti nel lotto.

Se il Visualizzatore di prestazioni Tivoli visualizza la metrica Raggiunta percentuale massima con una cifra doppia, aumentare la dimensione Massima. La metrica Raggiunta percentuale massima indica l'intervallo di tempo utilizzato dai thread configurati. Se sono presenti diversi client simultanei collegati all'ORB lato server, aumentare le dimensioni per supportare fino a 1000 client.

Tipo dati Numero intero
Predefinito 50
Consigliato 50 (25 sui sistemi Linux)
Timeout di inattività thread
Specifica il numero di millisecondi di inattività che devono scadere prima che venga recuperato un thread. Un valore 0 vuol dire che non è necessario attendere, mentre un valore negativo (inferiore a 0) indica un'attesa indefinita.
Tipo dati Numero intero
Unità Millisecondi
Predefinito 3500
Incrementabile
Specifica se il numero di thread può aumentare oltre la dimensione massima configurata per il lotto di thread.
Tipo dati Valore booleano
Predefinito Non abilitato (false)
Intervallo I valori validi sono Consente l'assegnazione thread oltre la dimensione massima del thread o Non abilitato.
