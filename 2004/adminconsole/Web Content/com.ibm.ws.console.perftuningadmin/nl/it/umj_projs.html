Impostazioni del server JMS interno di WebSphere

Le funzioni JMS su un nodo all'interno del dominio di gestione WebSphere Application Server sono supportate dal server JMS sul nodo stesso. Utilizzare questo pannello per visualizzare o modificare le proprietà di configurazione del server JMS selezionato.

È possibile utilizzare questo pannello per configurare un insieme generale di proprietà del server JMS, che vengono aggiunte ai valori predefiniti delle proprietà configurate automaticamente per il provider JMS incorporato di WebSphere.

Nota: in generale, i valori predefiniti delle proprietà WebSphere MQ sono adatte ai server JMS interni di WebSphere. Tuttavia, in presenza di carichi elevati di messaggi, potrebbe essere necessario modificare alcune proprietà WebSphere MQ; ad esempio, le proprietà per le ubicazioni dei file di log, le pagine di file e le pagine buffer. Per ulteriori informazioni sulla configurazione delle proprietà di WebSphere MQ, consultare il manuale WebSphere MQ System Administration, SC33-1873, disponibile in    IBM Publications Center o nel kit di raccolta WebSphere MQ, SK2T-0730.

Nome
Il nome attribuito al server JMS per scopi di gestione all'interno di IBM WebSphere Application Server.

Questo nome non deve essere modificato.

Tipo dati Stringa
Unità Non applicabile
Predefinito Server JMS interno di WebSphere
Intervallo Non applicabile
Descrizione
Una descrizione del server JMS, per scopi di gestione all'interno di IBM WebSphere Application Server.

Questa stringa non deve essere modificata.

Tipo dati Stringa
Predefinito Server JMS interno di WebSphere
Numero di thread  
Il numero di thread simultanei utilizzati dal motore corrispondente pubblicazione/sottoscrizione

Il numero di thread simultanei deve essere impostato solo su un numero basso.

Tipo dati Numero intero
Unità Thread
Predefinito 1
Intervallo Maggiore o pari a 1.
Nomi coda
I nomi delle code ospitate da questo server JMS. Ciascun nome coda deve essere aggiunto su una riga diversa.

Ciascuna coda elencata in questo campo deve disporre di un diverso oggetto di gestione code con lo stesso nome di gestione. Per rendere disponibile una coda alle applicazioni, definire una coda WebSphere e aggiungere il nome in questo campo sul pannello del Server JMS dell'host su cui si desidera ospitare la coda.

Tipo dati ASCII
Unità Nome coda
Predefinito Non applicabile
Intervallo Ciascuna voce in questo campo è un nome di coda lungo al massimo 45 caratteri, che deve corrispondere esattamente (anche per i caratteri maiuscoli e minuscoli) all'oggetto di gestione code di WebSphere definito per quella coda.
Stato iniziale
Lo stato che si desidera per il server JMS al successivo riavvio.
Tipo dati Enum
Unità Non applicabile
Predefinito Avviato
Intervallo
Avviato
Il server JMS viene avviato automaticamente.
Arrestato
Il server JMS non viene avviato automaticamente. Se un'applicazione enterprise distribuita deve utilizzare le funzioni del server JMS fornite dallo stesso server, l'amministratore del sistema deve avviare il server JMS manualmente o selezionare il valore Avviato di questa proprietà, quindi riavviare il server JMS.

Per riavviare il server JMS per un server delle applicazioni, arrestare quindi riavviare il server delle applicazioni.
