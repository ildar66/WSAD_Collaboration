Per visualizzare questa pagina della console di gestione, fare clic su Server > Server delle applicazioni > serverName > Contenitore EJB.

Directory di modalità passiva
Specifica la directory in cui il contenitore salva lo stato persistente dei bean di sessione in modalità stateful passiva.

I bean sono in modalità passiva quando il numero di istanze bean attive diventa maggiore della dimensione di cache specificata nella configurazione del contenitore. Quando un bean stateful è stato reso passivo, il contenitore serializza l'istanza bean in un file nella directory di modalità passiva e scarica l'istanza dalla cache del bean. Se, in un secondo momento, arriva una richiesta per un'istanza del bean in modalità passiva, il contenitore la richiama dalla relativa directory, la deserializza, la restituisce alla cache e le invia la richiesta. Se un'operazione non riesce (ad esempio, se l'istanza bean non si trova più nella directory di passivazione), il metodo di richiamo non riesce.

Intervallo di eliminazione dei lotti non attivi
Specifica l'intervallo in cui il contenitore esamina i lotti delle istanze bean disponibili per stabilire se è possibile eliminare alcune istanze e limitare l'uso della memoria.
Tipo dati Numero intero
Unità Millisecondi
Intervallo Maggiore di 0
Nome JNDI dell'origine dati predefinito
Specifica il nome JNDI di un'origine dati da utilizzare se non ne è stata specificata una durante la distribuzione dell'applicazione. Questa impostazione non è applicabile per i bean CMP compatibili con EJB 2.x.

I servlet e i bean enterprise utilizzano le origini dati per ottenere queste connessioni. Durante la configurazione di un contenitore, è possibile specificare un'origine dati predefinita per il contenitore. Questa origine dati diventa quella predefinita, utilizzata poi da qualunque bean di entità installato nel contenitore che utilizza CMP (container-managed persistence).

L'origine dati predefinita di un contenitore è protetta. Specificandola, è necessario fornire anche un ID utente e la password per accedervi.

La specifica di un'origine dati predefinita è facoltativa qualora ogni bean di entità CMP, presente nel contenitore, disponga di un'origine dati specificata nella configurazione. Se non viene specificata alcuna origine dati predefinita, e un bean di entità CMP viene installato nel contenitore senza specificare un'origine dati per quel bean, le applicazioni non possono utilizzare quel bean di entità CMP.
