診断トレース・サービス設定

このページを使用して、診断トレース・サービスのプロパティーを検討および変更します。

このページを表示するには、「トラブルシューティング」>「Logs and Trace」>「server」>「診断トレース」をクリックします。

「構成」タブ

トレースの使用可能化
トレース・サービスを使用可能にします。

このオプションが選択されていない場合は、以下の構成プロパティーは、 サーバー始動時にアプリケーション・サーバーのトレース・サービスに渡されません。

トレース仕様
トレースの詳細を指定します。

トレースするコンポーネント、パッケージ、またはグループを指定するトレース・ストリングを入力します。 トレース・ストリングは、以下に記載されている特定の文法に準拠していなければなりません。 トレース・ストリングを直接入力するか、 またはグラフィカル・トレース・インターフェースを使用して生成することができます。 グラフィカル・トレース・インターフェースを開始するには、「変更」をクリックします。

グラフィカル・トレース・インターフェースを構成タブから開始する場合は、 既知のコンポーネント、パッケージ、およびグループのリストが表示されます。 このリストはすべてを網羅していない場合があります。

グラフィカル・トレース・インターフェースをランタイム・タブから開始する場合は、 コンポーネント、パッケージ、およびグループのリストに、 稼働中のサーバー上に現在登録されているすべての当該コンポーネントが表示されます。

トレース仕様の形式は、次のとおりです。 

<component> = <trace_type>= enabled | disabled
ここで、<component > は、トレースを使用可能または使用不可にするコンポーネントで、<trace_type > は、使用可能または使用不可にするトレースのタイプです。複数のトレース仕様は、コロン (:) で区切ります。

コンポーネントは、Java パッケージおよびクラス、または Java パッケージの集合に対応しています。 ワイルドカードとして * を使用して、 指定されたコンポーネントに含まれているすべてのパッケージ内のすべてのクラスが含まれているコンポーネントを示します。 以下に例を示します。

*
WebSphere Application Server システム・コードおよびカスタマー・コードを含め、 アプリケーション・サーバーで稼働しているすべてのトレース可能なコードを指定します。
com.ibm.ws.*
パッケージ名が com.ibm.ws で始まるすべてのクラスを指定します。
com.ibm.ws.classloader.JarClassLoader
JarClassLoader クラスのみを指定します。

トレース・ストリングの文法について、詳しくは、 WebSphere Application Server インフォセンター内の『Enabling trace』の項目を参照してください。

注: 「グループ」および「コンポーネント」の両方のリストから選択した場合、 トレース仕様を管理コンソールから設定する際に、エラーが起こる可能性があります。 場合によっては、あるリストから行われた選択が、 別のリストからの選択を追加する際に失われない場合があります。 この問題に対処するには、必要なトレース仕様を直接「トレース仕様」入力フィールドに入力します。

トレース出力  
トレース出力をどこへ書き込むかを指定します。

トレース出力は、出力ファイルに直接書き込むか、またはメモリーに保管し、ランタイム・ページにある「ダンプ」ボタンを使用して、要求に応じてファイルに書き込むことができます。

メモリー・バッファー
トレース出力をメモリー内の循環バッファーに書き込むように指定します。 このオプションを選択した場合は、以下のパラメーターを指定しなければなりません。
最大バッファー・サイズ
バッファー内のキャッシュに入れられるエントリー数を千単位で指定します。 この数値を超過すると、より古いエントリーは、新しいエントリーで上書きされます。
ダンプ・ファイル名
メモリー・バッファーがダンプされた場合に書き込まれるファイルの名前。 このオプションは、ランタイム・タブからのみ使用可能です。

ファイル
自己管理ログ・ファイルにトレース出力を書き込むように指定します。

自己管理ログ・ファイルは、サイズの基準に達するまで、ファイルにメッセージを書き込みます。 ファイルが指定したサイズに達すると、ロギングは一時的に中断され、ログ・ファイルがクローズされて、 名前が変更されます。 新しい名前は、そのファイルのオリジナル名が基になり、 それに名前変更の発生した時間を示すタイム・スタンプ修飾子が付きます。 名前変更が完了すると、オリジナル名を持った新規の空のログ・ファイルが再度開かれ、ロギングが再開します。 ロールオーバーの結果、単一メッセージが 2 行に分割されるような場合はあっても、なくなることはありません。

このオプションを選択した場合は、以下のパラメーターを指定しなければなりません。

最大ファイル・サイズ
出力ファイルを拡大できる最大サイズを MB 単位で指定します。

この属性が有効なのは、「File Size」属性が選択されている場合だけです。 ファイルは、このサイズに達すると、前述の説明のように、ロールオーバーされます。

ヒストリカル・ファイルの最大数
保持するロールオーバー・ファイルの最大数を指定します。
ファイル名
トレース出力を書き込むファイルの名前を指定します。

レース出力フォーマット  
トレース出力の形式を指定します。

トレース出力の以下の 3 つのレベルのいずれかを指定することができます。

基本 (互換)
基本トレース情報のみを保存します。 このオプションを選択すると、トレース出力に取られるスペースの量を最小化します。
拡張
より多くの特定のトレース情報を保存します。 このオプションを選択すると、トラブルシューティングおよび問題判別で使用する詳細なトレース情報が表示されます。
ログ・アナライザー
ログ・アナライザー・ツールと互換性のある形式でトレース情報を保存します。 トレース出力をログ・アナライザー・ツールへの入力として使用する場合は、このオプションを選択します。

「ランタイム」タブ  
トレース出力の形式を指定します。
トレースの保管
ランタイム・タブ上で行われた変更を、トレース構成にも保管します。

このボックスを選択すると、ランタイム・トレースの変更をトレース構成設定にもコピーします。これらの変更をトレース構成に保管すると、アプリケーションが再始動されても、変更は存続します。
