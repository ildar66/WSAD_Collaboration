Application Server 설정

이 페이지를 참조하여 Application Server 인스턴스에 대한 설정을 보거나 변경하십시오.

이 관리 콘솔 페이지를 보려면 서버 > Application Servers > server_name을 클릭하십시오.

구성 탭은 편집 가능한 필드를 제공하고 런타임 탭은 읽기 전용 정보를 제공합니다. 런타임 탭은 서버가 실행 중인 경우에만 사용 가능합니다.

구성 탭

이름
서버의 논리 이름을 지정합니다. 서버 이름은 노드 내에서 고유해야 합니다.
데이터 유형 문자열
기본값 server1
초기 상태
서버를 맨 처음 시작할 때 요청된 구성요소 실행 상태를 지정합니다.
데이터 유형 문자열
기본값 시작
응용프로그램 클래스 로더 정책
단일 클래스 로더를 사용하여 모든 응용프로그램을 로드하는지 또는 각각의 응용프로그램에 대해 상이한 클래스 로더를 사용하는지 지정합니다.

옵션은 SINGLE 및 MULTIPLE입니다. 기본값은 각각의 응용프로그램(MULTIPLE)에 대해 별도의 클래스 로더를 사용합니다.

데이터 유형 문자열
기본값 MULTIPLE
응용프로그램 클래스 로딩 모드
클래스를 로드하기 위해 클래스 로더가 첫 번째로 상위 클래스 로더 또는 응용프로그램 클래스 로더에서 검색하는지 여부를 지정합니다. JDK 클래스 로더 및 WebSphere 클래스 로더에 대한 표준은 PARENT_FIRST입니다. PARENT_LAST를 지정하면 응용프로그램이 상위 클래스 로더에 들어 있는 클래스를 대체할 수 있지만 이 조치는 대체 클래스와 비 대체 클래스를 혼합하여 사용한 경우 ClassCastException 또는 LinkageErrors를 초래할 수 있습니다.

옵션은 PARENT_FIRST 및 PARENT_LAST입니다.기본값은 클래스를 로드하기 위해 응용프로그램 클래스 로더를 검색하기 전에 상위 클래스 로더에서 검색하는 것입니다.

데이터 유형 문자열
기본값 PARENT_FIRST
