메시지 리스너 서비스

메시지 리스너 서비스는 JMS 프로바이더의 JMS 함수에 대한 확장자입니다. 하나 이상의 JMS 리스너를 제어하고 모니터하는 리스너 관리자를 제공하고 각 JMS 리스너는 전개된 메시지 구동 Bean 대신 JMS 목적지를 모니터합니다.

이 패널은 메시지 리스너 서비스의 사용자 정의 등록 정보, 스레드 풀 및 리스너 포트의 추가 등록 정보 페이지로의 링크를 표시합니다.

이 관리 콘솔 페이지를 보려면 서버-> application_server-> 메시지 리스너 서비스를 클릭하십시오.

관리를 위한 메시지 리스너 서비스의 이름

스레드 풀  
메시지 리스너 서비스가 실행하도록 허용받은 스레드의 최대 수를 제어합니다. 서비스 스레드 풀 등록 정보를 표시하려면 이 링크를 선택하십시오.

이 매개변수 조정은 여러 메시지 구동 Bean을 같은 Application Server에서 전개하고 최대 세션 값의 합계가 기본값인 10을 초과할 때입니다.

데이터 유형 정수
단위 적용되지 않음
기본값 최소: 10, 최대 50
범위 적용되지 않음
권장 메시지 구동 Bean 모두의 최대 세션 값의 합을 최소로 설정하십시오. 최소 이상인 경우는 최대로 설정하십시오.
사용자 정의 등록 정보
메시지 리스너 서비스의 사용자 정의 등록 정보입니다. JMS 서버의 운영 모드는 두 가지인데, ASF(Application Server Facilities)와 비ASF입니다. ASF는 응용프로그램에 동시성과 트랜잭션 지원을 제공하도록 되어 있습니다. 비ASF는 경로 길이를 최적화하려고 그러한 지원을 생략합니다.

다음 경우에는 비ASF 모드를 사용하십시오.

  • 엄격한 메시지 순서가 요구사항입니다
  • 동시 PTP 메시지가 바람직합니다
  • non.asf.receive.timeout 등록 정보가 존재하고 값이 0보다 큽니다
ASF의 처리량이 더 많기 때문에 동시 공개 및 등록 메시지가 바람직할 경우는 비ASF 모드를 사용하지 마십시오.

데이터 유형 정수
단위 밀리초
기본값 ASF 모드(사용자 정의 등록 정보를 작성하지 않음)
범위
권장 값은 메시지 전달에 소요되는 시간을 밀리초 단위로 나타냅니다. 제한시간에 도달하면 과정을 반복해야 하므로 추가 작업이 생깁니다. 따라서 트랜잭션 제한시간보다 낮게 설정해야 하는데, 수 초(10초 이상) 여유를 두고 트랜잭션 제한시간에 가깝게 설정하십시오. 스레드가 CPU 사이클을 얻기 위해 오랫동안 대기 중일 정도로 로드가 과도할 경우에는, 여유로 더 많은 초를 지정하십시오.
