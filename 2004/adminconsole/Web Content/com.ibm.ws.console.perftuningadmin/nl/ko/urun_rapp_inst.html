엔터프라이즈 응용프로그램 설정

이 페이지를 참조하여 엔터프라이즈 응용프로그램을 구성하십시오.

이 관리 콘솔 페이지를 보려면 응용프로그램 > 엔터프라이즈 응용프로그램> application_name을 클릭하십시오.

구성 탭

이름
응용프로그램에 대한 논리 이름을 지정합니다. 응용프로그램 이름은 셀 내에서 고유해야 하며 오브젝트 이름에서 허용되지 않는 문자를 포함할 수 없습니다.
데이터 유형 문자열
시작 가중치
서버가 시작할 때 응용프로그램을 시작할 순서를 지정합니다. 가장 낮은 시작 비중이 있는 응용프로그램이 먼저 시작합니다.
데이터 유형 정수
기본값 1
범위 0 - 100
응용프로그램 아카이브
응용프로그램 EAR 파일을 설치할 디렉토리를 지정합니다. 기본값은 APP_INSTALL_ROOT/ cell_name의 값입니다. 여기서 APP_INSTALL_ROOT 변수는 install_root/installedApps;(예: C:\WebSphere\AppServer\installedApps\cell_name)입니다.

절대 경로를 지정하거나 pathmap 변수(예: ${MY_APPS})를 사용할 수 있습니다. , 어떤 설치에나현재 셀 이름을 선언하는 WebSphere Application Server 변수 ${CELL}은 pathmap 변수 내에 있을 수도 있습니다(예: ${MY_APP}/${CELL}.

데이터 유형 문자열
단위 전체 경로 이름
2진의 메타데이터 사용
Application Server가 응용프로그램 전개 문서, deployment.xml 파일(기본값) 또는 엔터프라이즈 응용프로그램의 EAR 파일에 있는 바인딩, 확장자 및 전개 설명자를 사용할지 여부를 지정합니다.
데이터 유형 부울
기본값 true
분배 사용 가능
WebSphere Application Server가 설치 목적지에서 응용프로그램 아카이브를 펼치거나 삭제하는지 여부를 지정합니다. 기본값은 응용프로그램 분배를 사용 가능하게 하는 것입니다. 설치된 응용프로그램에 대한 응용프로그램 아카이브는 지정된 디렉토리로 펼쳐집니다. 2진은 또한 구성 변경사항 설치 제거 및 저장 시 삭제됩니다. 이 옵션을 사용 불가능하게 하는 경우, 응용프로그램이 실행되는 모든 노드의 목적지 디렉토리에 응용프로그램 아카이브가 올바르게 펼쳐졌는지 확인해야 합니다.
데이터 유형 부울
기본값 true
클래스 로더 모드
클래스를 로드하기 위해 클래스 로더가 첫 번째로 상위 클래스 로더 또는 응용프로그램 클래스 로더에서 검색하는지 여부를 지정합니다. JDK 클래스 로더 및 WebSphere Application Server 클래스 로더에 대한 표준은 PARENT_FIRST입니다. PARENT_LAST를 지정하면 응용프로그램이 상위 클래스 로더에 들어 있는 클래스를 대체할 수 있지만 이 조치는 대체 클래스와 비 대체 클래스를 혼합하여 사용한 경우 ClassCastException 또는 LinkageErrors를 유발할 수 있습니다.

옵션은 PARENT_FIRST 및 PARENT_LAST입니다.기본값은 클래스를 로드하기 위해 응용프로그램 클래스 로더를 검색하기 전에 상위 클래스 로더에서 검색하는 것입니다.

데이터 유형 문자열
기본값 PARENT_FIRST
WAR 클래스 로더 정책
이 응용프로그램의 모든 WAR 파일을 로드하기 위해 단일 클래스 로더를 사용할지 아니면 각 WAR 파일에 대해 상이한 클래스 로더를 사용할지 여부를 지정합니다.

옵션은 APPLICATION 및 MODULE입니다. 기본값은 별도의 클래스 로더를 사용하여 각 WAR 파일을 로드하는 것입니다.

데이터 유형 문자열
기본값 MODULE
자원에 대한 MBean 작성
다양한 자원(Servlet 또는 JSP 파일)에 대한 MBean 파일을 응용프로그램에 작성 여부를 저정합니다.
데이터 유형 부울
기본값 true
재로드 사용 가능
응용프로그램 파일이 갱신될 때 클래스 재로드 사용 가능 여부를 지정합니다.
데이터 유형 부울
기본값 true
재로드 간격
갱신된 파일에 대한 응용프로그램 파일 시스템 스캔 시간(초)을 지정합니다. 기본값은 EAR 파일의 IBM 확장(META-INF/ibm-application-ext.xmi) 파일 내 재로드 간격 속성 값입니다. 이 설정은 클래스 재로드를 사용할 수 있는 경우에만 유효합니다.

여기서 지정한 재로드 간격은 EAR 파일의 각 모듈에 대한 IBM 확장에서 지정한 값을 대체합니다. 이 값은 다시 EAR 파일 내 응용프로그램에 대한 IBM 확장에서 지정한 재로드 간격을 대체합니다.
