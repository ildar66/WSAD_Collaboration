모니터링 정책 설정

이 페이지를 참조하여 Node Agent가 프로세스를 모니터하고 다시 시작하는 방법을 제어하는 설정을 보거나 변경하십시오.

이 관리 콘솔 페이지를 보려면 서버 > Application Servers > server_name > 프로세스 정의 > 모니터링 정책을 클릭하십시오.

구성 탭

시작 시도의 최대 수
포기하기 전에 Application Server 시작 시도의 최대 수.
데이터 유형 정수
핑 간격
Node Agent와 같은 상위 프로세스와 Application Server와 같은 생성된 프로세스 간 통신 시도 빈도를 지정하십시오. 실패한 서버를 다시 시작하는 데 필요한 요구조건에 따라 이 값을 조정하십시오. 값을 줄이면 장애를 그만큼 빨리 감지할 수 있지만, 값을 늘리면, Ping 빈도가 줄어들어 시스템 오버헤드도 감소됩니다.
데이터 유형 정수
핑 제한시간
프로세스 관리자가 서버를 만들 때처럼 상위 프로세스가 하위 프로세스를 만들 때, 상위 프로세스는 하위 프로세스를 Ping하여 하위 프로세스가 성공적으로 만들어졌는지 확인합니다. 이 값은 하위 프로세스를 Ping한 다음 하위 프로세스가 실패했다고 가정할 때까지 상위 프로세스가 대기해야 시간(단위: 초)을 지정합니다.
데이터 유형 정수
단위 초
자동 다시 시작
프로세스가 실패시 자동으로 다시 시작하는지 여부를 지정합니다. 기본값으로 프로세스가 자동으로 다시 시작합니다.
데이터 유형 부울
기본값 true
노드 다시 시작 상태
노드가 완전히 종료하고 다시 시작한 후, 프로세스에 대한 희망 상태를 지정합니다. 옵션은 STOPPED, RUNNING, PREVIOUS입니다. 기본값은 STOPPED입니다.
데이터 유형 문자열
기본값 STOPPED
범위 유효한 값은 STOPPED, RUNNING, 또는 PREVIOUS입니다.
