데이터베이스 설정

이 페이지를 참조하여 데이터베이스 세션 지원에 대해 설정을 지정하십시오.

이 관리 콘솔 페이지를 보려면 서버 > Application Server > server_name > 웹 컨테이너 > 세션 관리 > 분산 환경 설정 > 데이터베이스를 클릭하십시오.

구성 탭

데이터소스 JNDI 이름
데이터소스 설명 지정

세션 관리자가 데이터베이스 연결을 확보할 비 XA 사용 데이터소스의 JNDI 이름입니다. 예를 들어, 데이터소스의 이름이 "jdbc/sessions"이면 "jdbc/sessions"을 지정하십시오. 데이터소스는 데이터베이스 연결 풀과 그 풀에 대한 구성을 표시합니다(예: 풀 크기). 환경에서 데이터소스는 이미 자원으로 구성되어 있어야 합니다.

사용자 ID  
데이터베이스 액세스에 대한 사용자 ID를 지정합니다.
암호  
데이터베이스 액세스에 대한 암호를 지정합니다.
암호 확인  
올바르게 기록되었는지 확인하기 위해 암호를 다시 지정합니다.
DB2 행 크기  
DB2 데이터베이스를 사용하는 경우 세션 테이블에 대하여 구성된 테이블 공간 페이지 크기를 지정합니다. 가능한 값은 4, 8, 16 및 32킬로바이트(K)입니다. 기본 행 크기는 4K입니다.

기본 행 크기는 4K입니다. DB2에서 이 크기는 더 큰 값으로 갱신할 수 있습니다. 일부 환경에서는 이렇게 갱신함으로써 데이터베이스 성능을 높일 수 있습니다. 이 값이 4가 아니면, 사용할 테이블 공간 이름을 지정해야 합니다. 4K 페이지의 경우 테이블 공간 이름은 선택적입니다.

테이블 공간 이름  
세션 테이블에 사용될 테이블 공간 이름을 지정합니다.

이 값은 DB2 페이지 크기가 4K가 아닐 때 필요합니다.

다중 행 세션 사용
응용프로그램 데이터의 각 인스턴스가 데이터베이스에서 별도의 행에 위치함을 지정하며, 이를 통해 각 세션에 대해 보다 많은 양의 데이터를 저장할 수 있습니다. 이 조치는 특정 사용법 시나리오에서 보다 우수한 성능을 제공합니다. 다중행 스키마 사용이 불가능하면, 응용프로그램 데이터의 인스턴스를 동일 행에 놓을 수 있습니다.
