EJB 컨테이너 설정

이 페이지에서 특정 EJB 컨테이너를 구성하고 관리하십시오.

이 관리 콘솔 페이지를 보려면 서버 > Application Server > 서버 이름 > EJB 컨테이너를 클릭하십시오.

구성 탭

비활성화 디렉토리
컨테이너가 비활성화된 stateful 세션 Bean의 지속 상태를 저장할 디렉토리를 지정합니다.

활성 Bean 인스턴스의 수가 컨테이너 구성에 지정된 캐시 크기보다 커질 때 Bean이 비활성화됩니다. Stateful Bean이 비활성화되는 경우, 컨테이너는 비활성화 디렉토리에 파일로 Bean 인스턴스를 직렬화하고 Bean 캐시로부터 인스턴스를 제거합니다. 이 이후에 아카이브된 Bean 인스턴스에 대한 요청이 있으면, 컨테이너는 아카이브 디렉토리로부터 이것을 가져와서 직렬화 해제한 후 캐시로 리턴하고 요청을 디스패치 합니다. 이 단계 중 어느 하나라도 실패하게 되는 경우(예를 들어, Bean 인스턴스가 더 이상 비활성화 디렉토리에 있지 않는 경우), 메소드 호출이 실패합니다.

비활성 풀 정리 간격
컨테이너가 메모리 사용을 줄이기 위해 일부 인스턴스를 삭제할 수 있는지를 판별하기 위해 사용 가능한 Bean 인스턴스의 풀을 검사하는 간격을 지정합니다.
데이터 유형 정수
단위 밀리초
범위 0보다 큼
기본 데이터소스 JNDI 이름
응용프로그램 전개 중 데이터소스가 지정되지 않을 경우 사용할 데이터소스의 JNDI 이름을 지정합니다. 이 설정은 EJB 2.x 준수 CMP Bean에 적용할 수 없습니다.

Servlet 및 Enterprise Bean은 이 연결을 설정하기 위해 데이터소스를 사용합니다. 컨테이너를 구성할 때 컨테이너의 기본 데이터소스를 지정할 수 있습니다. 이 데이터소스가 기본 데이터소스가 되는데, 이 기본 데이터소스는 CMP(Container-Managed Persistence)를 사용하는 컨테이너에 설치된 엔티티 Bean이 사용하는 것입니다.

컨테이너의 기본 데이터소스는 보안이 유지됩니다. 이를 지정할 때 데이터소스에 액세스하기 위한 사용자 ID 및 암호를 제공해야 합니다.

컨테이너의 각 CMP 엔티티 Bean이 구성에 지정된 데이터소스를 가지고 있을 경우에는 기본 데이터소스를 지정하는 것은 선택적입니다. 기본 데이터소스가 지정되지 않고 CMP 엔티티 Bean이 해당 Bean을 위한 데이터소스를 지정하지 않은 상태로 컨테이너에 설치된 경우 해당 CMP 엔티티 Bean을 사용할 수 없습니다.
