Einstellungen für verteilte Umgebung

Auf dieser Seite können Sie einen Typ zum Speichern einer Sitzung in einer verteilten Umgebung angeben.

Klicken Sie, um diese Seite der Administrationskonsole anzuzeigen, auf Server > Anwendungsserver > Servername > Webcontainer > Sitzungsverwaltung > Einstellungen der verteilten Umgebung.

Register "Konfiguration"

Verteilte Sitzungen
Der Typ der verteilten Umgebung, die zum Speichern von Sitzungen verwendet werden soll.

Keine Gibt an, dass die Sitzungsverwaltung die Sitzungsdaten beim Herunterfahren des Servers löscht.
Datenbank Die Sitzungsverwaltung speichert Sitzungsdaten in der Datenquelle, die in den Verbindungseinstellungen der Datenquelle angegeben ist. Klicken Sie auf Datenbank, um die Einstellungen dieser Datenquelle zu ändern.
Replikation zwischen Speichern Die Sitzungsverwaltung speichert Sitzungsdaten in einer Datenquelle im Hauptspeicher. Die Sitzungsdaten werden zwecks Fehlerbehebung an andere Sitzungsverwaltungsfunktionen kopiert. Klicken Sie auf Replikation zwischen Speichern, um den zu verwendenden Replikator anzugeben und um diese Speicher-zu-Speicher-Einstellugen zu ändern. (Gilt nur für WebSphere Application Server Network Deployment.)
