Benutzerdefinierte Einstellungen für Optimierungsparameter

Auf dieser Seite können Sie Optimierungsparameter für verteilte Sitzungen anpassen.

Klicken Sie zum Anzeigen dieser Seite der Administrationskonsole auf Server > Anwendungsserver > Servername > Webcontainer > Sitzungsverwaltung > Einstellungen für verteilte Umgebung > Benutzerdefinierte Optimierungsparameter > Benutzerdefinierte Einstellungen.

Register "Konfiguration"

Schreibfrequenz
Legt fest, wann die Sitzung in den persistenten Speicher geschrieben wird.

Ende des Servlet-Service Eine Sitzung führt einen Schreibzugriff auf eine Datenbank oder eine andere Instanz des WebSphere Application Server durch, nachdem das Servlet die Ausführung abschließt.
Manuelle Aktualisierung Die Synchronisierung des IBMSession-Objekts durch das Programm ist erforderlich, um die Sitzungsdaten in die Datenbank oder eine andere Instanz des WebSphere Application Server zu schreiben.
Zeitgestützt Eine Sitzung führt einen Schreibzugriff auf eine Datenbank oder eine andere Instanz des WebSphere Application Server auf der Basis des angegebenen Werts für das Schreibintervall durch. Standardwert: 10 Sekunden.
Inhalt des Schreibvorgangs
Gibt an, ob nur die aktualisierten Attribute oder alle Sitzungsattribute in den externen Speicher geschrieben werden sollen, unabhängig davon, ob sie geändert wurden. Das externe Verzeichnis kann eine Datenbank oder eine andere Anwendungsserverinstanz sein.
Nur aktualisierte Attribute Nur aktualisierte Attribute werden in den persistenten Speicher geschrieben.
Alle Sitzungsattribute Alle Attribute werden in den persistenten Speicher geschrieben.
Plan für die Bereinigung von Sitzungen erstellen
Gibt an, wann die ungültigen Sitzungen aus einer Datenbank oder einer anderen Anwendungsserverinstanz gelöscht werden sollen.
Plan für die Bereinigung von verteilten Sitzungen erstellen Aktiviert den geplanten Bereinigungsprozess für das Löschen der invalidierten HTTP-Sitzungen aus dem externen Verzeichnis. Wenn Sie diese Option aktivieren, wird die Anzahl der Aktualisierungen einer Datenbank oder einer anderen Anwendungsserverinstanz, die erforderlich sind, um die HTTP-Sitzungen aufrechtzuerhalten, verringert. Wenn diese Option nicht aktiviert ist, wird im Abstand von wenigen Minuten ein Prozess ausgeführt, der invalidierte HTTP-Sitzungen entfernt.

Wenn diese Option aktiviert ist, gibt die Einstellung die beiden Stunden am Tag an, zu denen der Prozess die invalidierten Sitzungen aus dem externen Verzeichnis löscht. Geben Sie die Zeiten an, zu denen die Aktivität auf den Anwendungsservern am geringsten ist. Ein externes Verzeichnis kann eine Datenbank oder eine andere Anwendungsserverinstanz sein.

Erste Uhrzeit (0 - 23) Gibt die erste Stunde in Westeuropäischer Zeit an, in der die invalidierten Sitzungen aus dem externen Verzeichnis gelöscht werden. Dieser Wert muss eine positive ganze Zahl zwischen 0 und 23 sein. Dieser Wert ist nur gültig, wenn die Option "Gültigkeitsdauer planen" aktiviert ist.
Zweite Uhrzeit (0 - 23) Gibt die zweite Stunde in Westeuropäischer Zeit an, in der die invalidierten Sitzungen aus dem externen Verzeichnis gelöscht werden. Dieser Wert muss eine positive ganze Zahl zwischen 0 und 23 sein. Dieser Wert ist nur gültig, wenn die Option "Gültigkeitsdauer planen" aktiviert ist.
