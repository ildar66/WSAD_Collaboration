Thread-Pool-Einstellungen

Auf dieser Seite können Sie eine Thread-Gruppe, die von einem Anwendungsserver verwendet wird, konfigurieren. Die Anforderungen werden über einen der HTTP-Transporte an den Server gesendet. Ein Thread-Pool ermöglicht den Serverkomponenten, Threads wiederzuverwenden. Auf diese Weise wird verhindert, dass zur Laufzeit neue Threads erstellt werden müssen. Das Erstellen neuer Threads ist zeit- und ressourcenintensiv.

Klicken Sie zum Aufrufen dieser Seite der Administrationskonsole auf Server > Anwendungsserver verwalten > Servername > ORB-Service > Thread-Pool. (Sie können diese Seite über verschiedene Optionskombinationen aufrufen.)

Register "Konfiguration"

Mindestgröße
Die Mindestanzahl der Threads im Pool.
Datentyp Integer
Standardwert 10
Maximale Größe
Die maximal zulässige Anzahl der Threads im Pool.

Falls der Tivoli Performance Viewer anzeigt, dass die Metrik Percent Maxed konstant im zweistelligen Bereich liegt, sollten Sie die maximale Größe erhöhen. Die Metrik Percent Maxed gibt Aufschluss über die Verwendungsdauer der konfigurierten Threads. Wenn mehrere Clients gleichzeitig eine Verbindung zum serverseitigen ORB herstellen, können Sie den Parameter für eine Unterstützung von bis zu 1000 Clients erhöhen.

Datentyp Integer
Standardwert 50
Empfohlener Wert 50 (25 auf Linux-Systemen)
Zeitlimit für Thread-Inaktivität
Gibt den Zeitraum der Inaktivität an (in Millisekunden). Nach Ablauf dieses Zeitraums wird ein Thread wiederhergestellt. Der Wert 0 steht für keine Wartezeit, ein negativer Wert (kleiner als 0) steht für eine unbegrenzte Wartezeit.
Datentyp Integer
Einheiten Millisekunden
Standardwert 3500
Ist erweiterbar
Bei Auswahl dieser Option kann die Anzahl der Threads den für den Thread-Pool konfigurierten Maximalwert übersteigen.
Datentyp Boolescher Wert
Standardwert Nicht aktiviert (false)
Bereich Die gültigen Werte sind Thread-Zuordnung über maximale Größe des Thread-Pools hinaus zulassen und Nicht aktiviert.
