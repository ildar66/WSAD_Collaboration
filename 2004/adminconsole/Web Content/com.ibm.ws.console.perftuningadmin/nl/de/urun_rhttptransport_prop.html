Einstellungen für HTTP-Transport

Auf dieser Seite können Sie einen HTTP-Transport anzeigen und konfigurieren. Der Name der Seite kann zu einer SSL-Einstellung gehören, wie z. B. DefaultSSLSettings.

Klicken Sie, um diese Seite der Administrationskonsole anzuzeigen, auf Server > Anwendungsserver > Servername > Webcontainer > HTTP-Transporte > Hostname.

Register "Konfiguration"

Host
Gibt die Host-IP-Adresse für die Bindung beim Transport an.

Ist der Anwendungsserver eine lokale Maschine, lautet der Hostname möglicherweise localhost.

Datentyp String
Port
Gibt den Port an, an den die Bindung für den Transport erfolgen soll. Geben Sie eine Port-Nummer zwischen 1 und 65535 an. Die Port-Nummer muss für jeden Anwendungsserver auf einer Maschine eindeutig sein.
Datentyp Integer
Bereich 1 bis 65535
SSL aktiviert
Gibt an, ob die Verbindungen zwischen dem WebSphere-Plug-in und dem Anwendungsserver mit SSL (Secure Sockets Layer) geschützt werden sollen. Der Standardwert sieht vor, dass SSL nicht verwendet wird.
Datentyp Boolescher Wert
Standardwert false
SSL
Gibt den Typ der SSL-Einstellungen für die Verbindungen zwischen dem WebSphere-Plug-in und dem Anwendungsserver an. Zu den Optionen gehören eine oder mehrere SSL-Einstellungen im Sicherheitscenter: z. B. DefaultSSLSettings, ORBSSLSettings oder LDAPSSLSettings.
Datentyp String
Standardwert Eine im Sicherheitscenter definierte SSL-Einstellung.
