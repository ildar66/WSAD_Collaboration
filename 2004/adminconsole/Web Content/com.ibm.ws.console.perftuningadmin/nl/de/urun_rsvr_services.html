Benutzerdefinierte Services

Auf dieser Seite können Sie eine Liste von Services anzeigen, die für den Anwendungsserver verfügbar sind. Sie können auf dieser Seite sehen, ob die Services aktiviert sind. Ein benutzerdefinierter Service bietet Ihnen die Möglichkeit, einen WebSphere-Anwendungsserver anzuschließen und Code zu definieren, der beim Starten oder Herunterfahren des Servers ausgeführt wird.

Klicken Sie, um diese Seite der Administrationskonsole anzuzeigen, auf Server > Anwendungsserver > Servername > Benutzerdefinierte Services.

URL für externe Konfiguration
Gibt den URL für eine benutzerdefinierte Servicekonfigurationsdatei an.

Wenn für die Klasse der benutzerdefinierten Services eine Konfigurationsdatei erforderlich ist, geben Sie für den Wert den vollständigen Pfad zu dieser Konfigurationsdatei ein. Dieser Dateiname wird an die Klasse des benutzerdefinierten Service übergeben.

Klassenname
Gibt den Klassennamen der Serviceimplementierung an. Diese Klasse muss das Interface des benutzerdefinierten Service implementieren.
Anzeigename
Gibt den Namen des Service an.
Start
Gibt an, ob der Server versucht, den Service zu starten und zu initialisieren, wenn der übergeordnete Prozess (Server) gestartet wird. Standardmäßig wird der Service nicht aktiviert, wenn der übergeordnete Prozess gestartet wird.
