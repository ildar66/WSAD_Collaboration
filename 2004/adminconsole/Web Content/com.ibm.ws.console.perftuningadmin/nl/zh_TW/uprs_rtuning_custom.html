調整參數自訂設定

請利用這個頁面來自訂分散式階段作業的調整參數。

如果要檢視這個管理主控台頁面，請按一下伺服器 > 應用程式伺服器 > server_name > Web 儲存區 > 階段作業管理 > 分散式環境設定 > 自訂調整參數 > 自訂設定。

「配置」標籤

寫入頻率
指定階段作業寫入持續儲存庫的時機。

Servlet 服務結束 階段作業在 Servlet 執行完成之後，寫入資料庫或另一個 WebSphere Application Server 實例。
手動更新 必須執行 IBMSession 物件的程式化同步作業，才能將階段作業資料寫入資料庫或另一個 WebSphere Application Server 實例中。
以時間為基礎 以指定的寫入間隔值為基礎，將階段作業資料寫入資料庫或另一個 WebSphere Application Server 實例。 預設值：10 秒
寫入內容
指定只有更新的屬性寫入外部位置，或不論有沒有變更過，所有階段作業屬性都要寫入外部位置。 外部位置可以是資料庫或另一個應用程式伺服器實例。
僅更新的屬性 只將更新的屬性寫入持續儲存庫中。
所有階段作業屬性 所有屬性都寫入持續儲存庫中。
排定階段作業清除時程表
指定從資料庫或另一個應用程式伺服器實例中清除無效的階段作業的時間。
指定分散式階段作業清除時程表 啟用排定的失效程序來從外部位置中清除失效的 HTTP 階段作業。 請啟用這個選項來減少使 HTTP 階段作業保持作用中，所需要的資料庫或另一個應用程式伺服器實例的更新次數。 當沒有啟用這個選項時，失效程序會每隔幾分鐘執行一次，以移除失效的 HTTP 階段作業。

當啟用這個選項時，請指定一天中的兩小時，讓程序清除外部位置中的失效階段作業。 請指定應用程式伺服器中活動最少的時間。 外部位置可以是資料庫或另一個應用程式伺服器實例。

一日的第一次 (0 - 23) 表示從外部位置清除無效階段作業的第一小時（格林威治標準時間，GMT）。 請將這個值指定為 0-23 的正整數。 這個值只在啟用了排程失效時才有效。
一日的第二次 (0 - 23) 表示從外部位置清除無效階段作業的第二小時（格林威治標準時間，GMT）。 請將這個值指定為 0-23 的正整數。 這個值只在啟用了排程失效時才有效。
