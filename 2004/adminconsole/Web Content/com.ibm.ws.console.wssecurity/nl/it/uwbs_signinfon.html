Impostazioni di configurazione delle informazioni sulla firma

Utilizzare questa pagina per configurare nuovi parametri di firma.

Le specifiche elencate in questa pagina relative ai metodi signing, digest e canonicalization vengono individuate nel documento W3C, dal titolo "XMLSignature Syntax and Specification: W3C Recommendation 12 Feb 2002".

Per visualizzare questa pagina della console di gestione, fare clic su Applicazioni Enterprise > nome_applicazione. In Elementi correlati, fare clic su Moduli Web > nome_file_URI > Servizi Web: collegamenti sicurezza server. Nella colonna Collegamento ricevente richiesta, fare clic su Modifica > Informazioni sulla firma > Nuovo.

Fare clic su Nuovo per creare un parametro di firma.

Fare clic su Elimina per eliminare un parametro di firma.

Metodo Signature
Specifica l'URI dell'algoritmo del metodo di firma. Questo metodo contiene il valore reale della firma digitale codificata utilizzando base64.

Sono supportati i seguenti algoritmi:

  • http://www.w3.org/2000/09/xmldsig#rsa-sha1
  • http://www.w3.org/2000/09/xmldsig#dsa-sha1

Metodo Digest
Specifica l'URI dell'algoritmo del metodo di Digest.

Viene supportato l'algoritmo http://www.w3.org/2000/09/xmldsig#sha1.

Metodo Canonicalization
Specifica l'URI dell'algoritmo del metodo di canonicalization.

Sono supportati i seguenti algoritmi:

  • http://www.w3.org/2001/10/xml-exc-c14n#
  • http://www.w3.org/2001/10/xml-exc-c14n#WithComments
  • http://www.w3.org/TR/2001/REC-xml-c14n-20010315
  • http://www.w3.org/TR/2001/REC-xml-c14n-20010315#WithComments

Chiave di firma
Specifica le informazioni sulla chiave utilizzata per la firma. Questi campi vengono ignorati nella configurazione lato ricevente.

Se si specifica un Nome chiave e un Riferimento indicatore di posizione della chiave, selezionare Nessuno per il Percorso certificato.

Percorso certificato
Specifica le impostazioni per la convalida del percorso del certificato. Se si seleziona Considera attendibili tutti, questa convalida viene ignorata e tutti i certificati in entrata vengono considerati attendibili. Questi campi vengono ignorati nella configurazione lato mittente.

Se si seleziona Considera attendibili tutti o Ancoraggio sicuro e un Archivio certificati, selezionare Nessuno per la chiave di firma nel campo precedente.

Ancoraggio sicuro

Le selezioni disponibili per Ancoraggio sicuro vengono specificate facendo clic su Server > Server delle applicazioni > nome_server. In Ulteriori proprietà, fare clic su Servizi Web: collegamenti predefiniti per la sicurezza dei servizi Web (WS-Security, Web Services Security) > Ancoraggi sicuri.

Archivio certificati

Le selezioni disponibili per Archivio certificati vengono specificate selezionando Server > Server delle applicazioni > nome_server. In Elementi correlati, fare clic su Servizi Web: collegamenti predefiniti per la sicurezza dei servizi Web (WS-Security, Web Services Security) > Raccolta archivio certificati.
