配置单元级别的现时标志

此任务提供有关如何使用 WebSphere Application Server 管理控制台配置单元级别的现时标志的说明。您可以在应用程序级别、服务器级别和单元级别配置现时标志。但是，您必须考虑优先顺序。以下列表显示优先顺序：
  1. 应用程序级别
  2. 服务器级别
  3. 单元级别
如果您在应用程序级别和服务器级别上配置现时标志，则应用程序级别指定的值优先于服务器级别指定的值。同样地，应用程序级别指定的值优先于服务器级别和单元级别指定的值。在 WebSphere Application Server Network Deployment 中，现时标志高速缓存超时、现时标志最大寿命和现时标志时钟偏移字段对于有效使用现时标志是必需的。但是，这些字段在服务器级别上是可选的。完成下列步骤以在单元级别上配置现时标志：

此任务的步骤

  1. 除非您更改了端口号，否则通过在您的 Web 浏览器中输入 http://localhost:9090/admin 以连接到管理控制台。
  2. 单击服务器 > 应用程序服务器 > server1。
  3. 在“其它属性”下，单击 Web 服务：Web 服务安全性的缺省绑定。
  4. 为现时标志高速缓存超时字段指定值（以秒计）。
    为现时标志高速缓存超时字段指定的值表明在除去现时标志之前保留高速缓存现时标志多长时间。您必须指定至少 300 秒。然而，如果您未指定值，则缺省是 600 秒。此字段在服务器级别上是可选的，但在单元级别上是必需的。
  5. 为现时标志最大寿命字段指定值（以秒计）。
    为现时标志最大寿命字段指定的值表明现时标志有效多长时间。您必须至少指定 300 秒，但值不能超过前一步骤中为现时标志高速缓存超时字段指定的秒数。如果您未指定值，则缺省是 600 秒。在 WebSphere Application Server Network Deployment 环境、WebSphere Application Server for z/OS 环境中，如果在服务器级别为现时标志高速缓存超时字段指定的值，则该值不能超过在单元级别为现时标志高速缓存超时字段指定的值。此字段在服务器级别上是可选的，但在单元级别上是必需的
  6. 为现时标志时钟偏移字段指定值（以秒计）。
    为现时标志时钟偏移字段指定的值指定当消息接收方检查值的新鲜度时要考虑的时间量（以秒计）。当您设置这个值的时候，考虑以下信息：
    • 如果时钟是不同步的，消息发送方和消息接收方之间的时间差异。
    • 加密和发送消息所需的时间。
    • 网络阻塞所需的时间。
    您必须在此字段中最少指定 0 秒。然而，最大值不能超过在现时标志最大寿命字段中表明的秒数。如果您未指定值，则缺省是 0 秒。此字段在服务器级别上是可选的，但在单元级别上是必需的。
  7. 重新启动服务器。
    如果您更改现时标志高速缓存超时值且没有重新启动服务器，此更改无法被服务器识别。
