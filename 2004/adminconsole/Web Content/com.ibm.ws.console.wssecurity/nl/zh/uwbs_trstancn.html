信任锚配置设置

使用此信息配置信任锚。信任锚指向包含可信根或自签署证书的密钥存储。此信息使您能够指定信任锚的名称，以及访问密钥存储所需的信息。应用程序绑定使用此名称来引用绑定文件中预定义的信任锚定义（或缺省）。

要查看此管理控制台页面，完成以下步骤：

  1. 单击服务器 > 应用程序服务器 > server_name。
  2. 在“其它属性”下，单击 Web 服务：Web 服务安全性的缺省绑定 > 信任锚 > 新建。

信任锚名
指定应用程序绑定用于引用缺省绑定中的预定义信任锚定义的唯一名称。
密钥存储密码
指定访问密钥存储文件所需的密码。
密钥存储路径
指定密钥存储文件的位置。

使用 ${USER_INSTALL_ROOT} 作为此路径扩展到您机器上的 WebSphere Application Server 路径。

密钥存储类型
指定密钥存储文件的类型。

此字段中的值是 JKS 或 JCEKS。

JKS
如果您未使用 Java 密码术扩展（JCE），则指定此选项。
JCEKS
如果您使用 Java 密码术扩展，则指定此选项。尽管 JCEKS 密钥存储格式更安全，但是它减少的性能。

数据类型 String
缺省 JKS
范围 JKS，JCEKS
