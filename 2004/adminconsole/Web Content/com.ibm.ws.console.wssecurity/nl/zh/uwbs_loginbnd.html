登录绑定配置设置

使用此页面配置加密和解密参数。

可插入令牌使用 Java 认证和授权服务（JAAS）CallBackHandler（javax.security.auth.callback.CallBackHandler）接口生成可插入消息的令牌。以下列表描述 CallBack 支持实现：

com.ibm.wsspi.wssecurity.auth.callback.BinaryTokenCallback
此实现用于生成作为 <wsse:BinarySecurityToken/@ValueType> 插入消息的二进制令牌。
javax.security.auth.callback.NameCallback and javax.security.auth.callback.NameCallback
此实现用于生成作为 <wsse:UsernameToken> 插入消息的用户名令牌。
com.ibm.wsspi.wssecurity.auth.callback.XMLTokenSenderCallback
此实现用于生成可扩展标记语言（XML）标记，并作为 <SAML: Assertion> 元素插入消息。
com.ibm.wsspi.wssecurity.auth.callback.PropertyCallback
此实现用于获取绑定文件中指定的属性。

要查看此管理控制台页面，完成以下步骤：

  1. 单击应用程序 > 企业应用程序 > application_name。
  2. 在“相关项”下，单击 Web 模块 > URI_file_name > Web 服务：客户机安全性绑定。
  3. 在“请求发送程序绑定”下，单击编辑 > 登录绑定。

如果加密信息不可用，选择无。

如果加密信息可用，选择专用登录绑定，并在下列字段中指定配置：

认证方法
指定认证方法的唯一名称。
回调处理程序
指定回调处理程序的名称。回调处理程序必须实现 javax.security.auth.callback.CallbackHandler 接口。
Basic Auth 用户标识
指定用于基本认证的用户名。Basic Auth 认证方法提供在绑定文件中定义用户标识和密码的能力。
Basic Auth 密码
指定用于基本认证的密码。
令牌类型 URI
指定标记类型的统一资源标识（URI）。此信息作为 XML 令牌 <SAML: Assertion> 的 <wsse:BinarySecurityToken>/ValueType 插入。
令牌类型本地名
指定令牌类型的本地名。此信息作为 XML 令牌 <SAML: Assertion> 的 <wsse:BinarySecurityToken>/ValueType 插入。
