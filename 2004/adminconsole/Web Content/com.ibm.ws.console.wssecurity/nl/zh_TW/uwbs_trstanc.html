信任錨點集合

請利用這個頁面來檢視含有最高授信使用者憑證的金鑰儲存庫物件清單。 這些物件用在送入 X.509 格式安全記號的憑證路徑驗證上。 信任錨點內的金鑰儲存庫物件含有 CertPath API 用來驗證憑證鏈授信的最高授信使用者憑證。

如果要建立金鑰儲存庫檔，請利用在 install_dir\java\jre\bin\keytool 目錄中的金鑰工具。

如果要檢視這個管理主控台頁面，請按一下伺服器 > 應用程式伺服器 > server_name。 在「其他內容」之下，按一下 Web 服務：Web 服務安全的預設連結 > 信任錨點。

按一下新建來建立新的信任錨點。

按一下刪除來刪除信任錨點。

如果您按一下更新執行時期，就會利用先前儲存的 ws-security.xml 檔所包含的預設連結資訊來更新 Web 服務安全執行時期。 如果您在這個畫面中進行變更，您必須完成下列步驟：

  1. 按一下管理主控台頂端的儲存來儲存您的變更。 當您按一下儲存時，您會返回管理主控台起始畫面。
  2. 返回「信任錨點」集合畫面，按一下更新執行時期。

    附註: 當您按一下更新執行時期時，也會在 Web 服務安全執行時期中更新其他 Web 服務的配置變更。

信任錨點名稱
指定用來識別信任錨點的唯一名稱。
金鑰儲存庫路徑
指定信任錨點所在的金鑰儲存庫檔的位置。
金鑰儲存庫類型
指定金鑰儲存庫檔案的類型。

這個欄位的值是 JKS 或 JCEKS。
