콜렉션 인증 저장 콜렉션

이 페이지를 참조하여 유효성 검증을 기다리는 신뢰가 없는 중간 인증 파일이 포함된 인증 저장의 목록을 보십시오. 유효성 검증은 인증 철회 목록(CRL)에 인증이 있는지, 인증이 만기되지 않았는지 및 신뢰 있는 서명자가 인증을 발행했는지 여부를 확인하는 검사로 구성될 수 있습니다.

이 관리 콘솔 페이지를 보려면 다음 단계를 완료하십시오.

  1. 서버 > Application Servers server_name를 클릭하십시오.
  2. 관련 항목에서 웹 서비스: 웹 서비스 보안에 대한 기본 바인딩 > 콜렉션 인증 저장을 클릭하십시오.
  3. 새로 작성을 클릭하여 새 콜렉션 인증 저장의 저장 이름 및 프로바이더를 지정하십시오.
  4. 삭제를 클릭하여 콜렉션 인증 저장을 삭제하십시오.

이 패널을 사용하여 다음 단계를 완료하십시오.

  1. 인증 저장 이름과 인증 저장 프로바이더를 지정하십시오.
  2. 관리 콘솔의 맨 위에서 저장을 클릭하여 변경사항을 저장하십시오. 저장을 클릭하면 관리 콘솔 홈 패널로 되돌아 갑니다.
  3. ws_security.xml 파일에 있는 기본 바인딩 정보를 사용하여 웹 서비스 보안 런타임을 갱신하려면, 콜렉션 인증 저장 콜렉션 패널로 되돌아가 런타임 갱신을 클릭하십시오.

참고: 런타임 갱신을 클릭하면, 다른 웹 서비스에 작성한 구성 변경사항도 웹 서비스 보안 런타임에서 갱신됩니다.
