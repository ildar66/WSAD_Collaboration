웹 서비스: 클라이언트 보안 바인딩 콜렉션

이 페이지를 참조하여 웹 서비스 보안을 위한 클라이언트측 바인딩 구성의 목록을 보십시오. 이들 바인딩은 웹 서비스가 다른 웹 서비스에 대한 클라이언트일 때 사용됩니다.

이 관리 콘솔 페이지를 보려면, 다음 단계를 완료하십시오.

  1. 엔터프라이즈 응용프로그램 > application_name을 클릭하십시오.
  2. 관련 항목 아래에서 웹 모듈 > URI_file_name> 웹 서비스: 클라이언트 보안 바인딩을 클릭하십시오.

포트
서버에서 메시지를 수신 및 송신하는 데 사용되는 포트를 지정합니다.
웹 서비스
웹 서비스의 이름을 지정합니다.
요청 송신자 바인딩
요청 수신자에 요청 메시지를 전송하는 데 사용되는 바인딩 구성을 지정합니다.

편집을 클릭하여 요청 전송자의 서명 정보, 암호화 정보 및 로그인 바인딩을 구성하고 키 저장에 있는 키 위치 지정자의 목록을 보십시오.

클라이언트에 지정된 요청 전송자의 바인딩 정보는 서버에 지정된 요청 수신자의 바인딩 정보와 일치해야 합니다.

응답 수신자 바인딩
응답 송신자로부터 응답 메시지를 수신하는 데 사용되는 바인딩 구성을 지정합니다.

편집을 클릭하여 서명 및 암호 정보를 구성하고, 신뢰 앵커 목록, 콜렉션 인증 저장에 있는 중간 인증 및 응답 수신자의 키 위치 지정자 오브젝트를 보십시오.

클라이언트에 지정된 응답 수신자의 바인딩 정보는 서버에 지정된 응답 전송자의 바인딩 정보와 일치해야 합니다.

HTTP 기본 인증
HTTP 전송 레벨 기본 인증과 함께 이 포트에 사용할 사용자 ID 및 암호를 지정합니다. 메시지 레벨 보안과 별도로 전송 레벨 인증 보안이 사용 가능하도록 설정할 수 있습니다.

전송 레벨 인증용 기본 인증 ID와 암호를 구성하려면 편집을 클릭하십시오.

HTTP SSL 구성
이 포트에 전송 레벨 SSL 보안이 사용 가능하도록 한 후 구성하십시오. 메시지 레벨 보안과는 별도로 전송 레벨 SSL 보안을 사용할 수 있습니다.

이 포트의 HTTP SSL 구성에 대한 설정을 지정하려면 편집을 클릭하십시오.
