웹 서비스 서버 전개 설명자 보기

이 페이지를 참조하여 서버 전개 설명자 설정을 보십시오.

이 관리 콘솔 페이지를 보려면, 다음 단계를 완료하십시오.

  1. 응용프로그램 > 엔터프라이즈 응용프로그램 > application_name을 클릭하십시오.
  2. 관련 항목 아래에서 웹 모듈 > URI_file_name > 웹 서비스 서버 전개 설명자 보기를 클릭하십시오.

WebSphere Application Server에는 응용프로그램 레벨 및 서버 레벨의 두 가지 바인딩 레벨이 있습니다. WebSphere Application Server Network Deployment에는 응용프로그램 레벨, 서버 레벨 및 셀 레벨의 세 가지 바인딩 레벨이 있습니다. 다음 구현 설명의 정보는 응용프로그램 레벨 바인딩을 구성하는 방법을 표시합니다. 기본값인 서버 레벨 바인딩을 구성하려면, 다음 단계를 완료하십시오.

  1. 서버 > Application Servers > server_name을 클릭하십시오.
  2. 관련 항목 아래에서 웹 서비스: 웹 서비스 보안을 위한 기본 바인딩을 클릭하십시오.
셀 레벨 바인딩을 구성하려면, 보안 > 웹 서비스를 클릭하십시오.

요청 디지털 서명 검증

무결정 제한조건(필수 서명)이 정의되어 있는 경우, 바인딩 파일에 서명 정보를 구성했는지 검증하십시오.

서명 매개변수를 구성하려면, 다음 단계를 완료하십시오.

  1. 응용프로그램 > 엔터프라이즈 응용프로그램 > application_name을 클릭하십시오.
  2. 관련 항목 아래에서 웹 모듈 > URI_file_name > 웹 서비스: 서버 보안 바인딩을 클릭하십시오.
  3. 요청 수신자 바인딩 열에서 편집 > 서명 정보를 클릭하십시오.

신뢰 앵커를 구성하려면, 다음을 완료하십시오.

  1. 서버 > Application Servers > server_name을 클릭하십시오.
  2. 추가 등록 정보 아래에서 웹 서비스: 웹 서비스 보안을 위한 기본 바인딩 > 신뢰 앵커를 클릭하십시오.

콜렉션 인증 저장을 구성하려면, 다음 단계를 완료하십시오.

  1. 서버 > Application Servers > server_name을 클릭하십시오.
  2. 관련 항목 아래에서 웹 서비스: 웹 서비스 보안을 위한 기본 바인딩 > 콜렉션 인증 저장을 클릭하십시오.

키 위치 지정자를 구성하려면, 다음 단계를 완료하십시오.

  1. 서버 > Application Servers > server_name을 클릭하십시오.
  2. 추가 등록 정보 아래에서 웹 서비스: 웹 서비스 보안을 위한 기본 바인딩 > 키 위치 지정자를 클릭하십시오.

요청 암호 해독

신뢰성 제한조건(암호화)이 지정되는 경우, 암호화 정보가 정의되어 있는지 검증하십시오.

암호화 정보 매개변수를 구성하려면, 다음 단계를 완료하십시오.

  1. 엔터프라이즈 응용프로그램 > application_name을 클릭하십시오.
  2. 관련 항목 아래에서 웹 모듈을 클릭하십시오.
  3. 추가 등록 정보 아래에서 웹 서비스: 서버 보안 바인딩을 클릭하십시오. 요청 수신자 바인딩에서 편집 > 암호화 정보를 클릭하십시오.

키 위치 지정자를 구성하려면, 다음 단계를 완료하십시오.

  1. 서버 > Application Servers > server_name을 클릭하십시오.
  2. 추가 등록 정보 아래에서 웹 서비스: 웹 서비스 보안을 위한 기본 바인딩 > 키 위치 지정자를 클릭하십시오.

BasicAuth 인증

BasicAuth가 필수 보안 토큰으로 구성되어 있는 경우, 바인딩 파일에 CallbackHandler를 지정하여 기본 인증 데이터를 수집하십시오. 다음은 콜백 지원 구현의 목록입니다.

com.ibm.wsspi.wssecurity.auth.callback.GuiPromptCallbackHandler
구현은 인터페이스에서 BasicAuth 정보(사용자 이름 및 암호)를 입력하라는 프롬프트를 표시합니다.
com.ibm.wsspi.wssecurity.auth.callback.NonPromptCallbackHandler
이 구현은 바인딩 파일에서 BasicAuth 정보를 읽습니다.
com.ibm.wsspi.wssecurity.auth.callback.StdPromptCallbackHandler
이 구현은 표준 입력(stdin) 프롬프트를 사용하여 사용자 이름 및 암호를 입력하라는 프롬프트를 표시합니다.

로그인 맵핑 정보를 구성하려면, 다음 단계를 완료하십시오.

  1. 서버 > Application Servers > server_name을 클릭하십시오.
  2. 추가 등록 정보 아래에서 웹 서비스: 웹 서비스 보안을 위한 기본 바인딩 > 로그인 맵핑을 클릭하십시오.

TrustMode가 BasicAuth인 ID 확정 인증

com.ibm.wsspi.wssecurity.auth.callback.NonPromptCallbackHandler 구현으로 바인딩 파일에서 로그인 바인딩을 구성하십시오. 다운스트림 서버의 TrustedIDEvaluator가 신뢰하는 BasicAuth 사용자 ID 및 암호를 지정하십시오.

로그인 맵핑 정보를 구성하려면, 다음 단계를 완료하십시오.

  1. 서버 > Application Servers > server_name을 클릭하십시오.
  2. 추가 등록 정보 아래에서 웹 서비스: 웹 서비스 보안을 위한 기본 바인딩 > 로그인 맵핑을 클릭하십시오.

TrustMode가 Signature인 ID 확정 인증

키 위치 지정자를 가리키는 단일 키로 바인딩 파일에서 서명 정보를 구성하십시오. 키 위치 지정자는 다운스트림 서버에서 신뢰하는 X.509 인증을 포함합니다.

로그인 맵핑 정보를 구성하려면, 다음 단계를 완료하십시오.

  1. 서버 > Application Servers > server_name을 클릭하십시오.
  2. 추가 등록 정보 아래에서 웹 서비스: 웹 서비스 보안을 위한 기본 바인딩 > 로그인 맵핑을 클릭하십시오.

WSLogin은 JAAS(Java Authentication and Authorization Service)에서 사용되는 로그인 구성의 이름입니다. JAAS를 구성하려면 보안 > JAAS 구성 > 응용프로그램 로그인을 클릭하십시오.

바인딩의 <TrustedIDEvaluatorRef> 태그 값은 <TrustedIDEvaluator> 이름의 값과 일치해야 합니다.

신뢰 ID 평가자를 구성하려면, 다음 단계를 완료하십시오.

  1. 서버 > Application Servers > server_name을 클릭하십시오.
  2. 추가 서비스 아래에서 웹 서비스: 웹 서비스 보안을 위한 기본 바인딩 > 신뢰 ID 평가자를 클릭하십시오.
