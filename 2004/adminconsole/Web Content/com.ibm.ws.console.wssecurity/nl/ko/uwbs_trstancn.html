신뢰 앵커 구성 설정

이 정보는 신뢰 앵커를 구성하는 데 사용합니다. 신뢰 앵커는 신뢰 있는 루트 또는 자체 서명 인증이 포함된 키 저장을 가리킵니다. 이 정보를 사용하여 신뢰 앵커의 이름과 키 저장에 액세스하는 데 필요한 정보를 지정할 수 있습니다. 응용프로그램 바인딩에서는 바인딩 파일(또는 기본값)에서 사전 정의된 신뢰 앵커 정의를 참조하는 데 이 이름을 사용합니다.

이 관리 콘솔 페이지를 보려면, 다음 단계를 완료하십시오.

  1. 서버 > Application Servers > server_name을 클릭하십시오.
  2. 추가 등록 정보 아래에서 웹 서비스: 웹 서비스 보안을 위한 기본 바인딩 > 신뢰 앵커 > 새로 작성을 클릭하십시오.

신뢰 앵커 이름
기본 바인딩에서 사전 정의된 신뢰 앵커 정의를 참조하기 위해 응용프로그램 바인딩에서 사용하는 고유 이름을 지정합니다.
키 저장 암호
키 저장 파일에 액세스하는 데 필요한 암호를 지정합니다.
키 저장 경로
키 저장 파일의 위치를 지정합니다.

${USER_INSTALL_ROOT}를 이 경로로 사용하면 시스템의 WebSphere Application Server 경로로 확장됩니다.

키 저장 유형
키 저장 파일의 유형을 지정합니다.

이 필드의 값은 JKS 또는 JCEKS입니다.

JKS
JCE(Java Cryptography Extensions)를 사용하고 있지 않은 경우에 이 옵션을 지정하십시오.
JCEKS
Java Cryptography Extensions를 사용하고 있는 경우 이 옵션을 지정하십시오. JCEKS 키 저장 형식이 보다 안전하지만 이는 성능을 저하시킵니다.

데이터 유형 문자열
기본값 JKS
범위 JKS, JCEKS
