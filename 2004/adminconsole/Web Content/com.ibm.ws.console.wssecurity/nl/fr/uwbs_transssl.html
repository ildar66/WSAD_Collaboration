Collection de configurations SSL HTTP

Cette page permet de configurer la sécurité SSL (Secure Sockets Layer) au niveau du transport. Cette configuration est utile lorsqu'un service Web est client d'un autre.

Vous pouvez utiliser la sécurité au niveau du transport pour activer SSL HTTP (HTTPS). Ce type de sécurité peut être activée ou désactivée indépendamment de celle au niveau des messages. Sachant qu'elle suppose une sécurité minimale, utilisez celle au niveau des message lorsque la protection d'une application de service Web est essentielle.

Pour afficher cette page de la console d'administration, suivez la procédure ci-après.

  1. Cliquez sur Applications > Applications d'entreprise > nom_application.
  2. Dans la section Articles liés, cliquez sur Module Web > nom_fichier_URI > Services Web : Liaisons de sécurité du client.
  3. Dans la section Authentification SSL HTTP, cliquez sur Editer.

Vous pouvez également définir l'authentification de base en suivant la procédure ci-après :

  1. Cliquez sur Applications > Applications d'entreprise > nom_application.
  2. Dans la section Articles liés, cliquez sur Module Web > nom_fichier_URI > Services Web : Liaisons de sécurité du client.
  3. Dans Configuration HTTP de base, cliquez sur Editer.

HTTP SSL activé
Sélectionnez cette option pour activer les communications sécurisées par socket pour le transport HTTP concernant ce port. Après activation, WebSphere Application Server utilise le paramètre de configuration SSL HTTP.
Configuration SSL HTTP
Indique l'alias de la configuration SSL à utiliser avec le transport HTTP pour ce port.

Cette option est utilisée si vous sélectionnez HTTP SSL activé. Les alias SSL sont définis dans le répertoire de configuration Secure Sockets Layer, modifiable en cliquant sur Sécurité > SSL.
