HTTP-SSL-Konfiguration

Verwenden Sie diese Seite, um die SSL-Sicherheit (Secure Sockets Layer) auf Transportebene zu konfigurieren. Sie können diese Konfiguration verwenden, wenn ein Webservice ein Client eines anderen Webservice ist.

Zum Aktivieren von HTTP-SSL (oder HTTPS) können Sie die Sicherheit auf Transportebene verwenden. Die Sicherheit auf Transportebene kann unabhängig von der Sicherheit auf Nachrichtenebene aktiviert und inaktiviert werden. Weil die Sicherheit auf Transportebene den niedrigsten Sicherheitsgrad bietet, sollten Sie die Sicherheit auf Nachrichtenebene verwenden, falls eine Webservice-Anwendung unbedingt geschützt werden muss.

Gehen Sie zum Anzeigen dieser Seite der Administrationskonsole wie folgt vor:

  1. Klicken Sie auf Anwendungen > Enterprise-Anwendungen > Anwendungsname.
  2. Klicken Sie unter "Zugehörige Elemente" auf Webmodule > Name_der_URI-Datei > Web Services: Sicherheitsbindungen des Clients.
  3. Klicken Sie unter "HTTP-SSL-Konfiguration" auf Bearbeiten.

Sie können die HTTP-SSL-Basisauthentifizierung auch wie folgt festlegen:

  1. Klicken Sie auf Anwendungen > Enterprise-Anwendungen > Anwendungsname.
  2. Klicken Sie unter "Zugehörige Elemente" auf Webmodule > Name_der_URI-Datei > Web Services: Sicherheitsbindungen des Clients.
  3. Klicken Sie unter HTTP-Basiskonfiguration auf Bearbeiten.

HTTP-SSL aktiviert
Wählen Sie diese Option aus, um für den HTTP-Transport an diesem Port die Übertragung über sichere Sockets zu aktivieren. Wenn diese Option aktiviert ist, verwendet WebSphere Application Server die HTTP-SSL-Konfiguration.
HTTP-SSL-Konfiguration
Der für den HTTP-Transport an diesem Port zu verwendende Aliasname der SSL-Konfiguration.

Diese Option wird verwendet, wenn Sie HTTP-SSL aktiviert auswählen. Die SSL-Aliasnamen werden im SSL-Konfigurationsrepertoire definiert, das Sie unter Sicherheit > SSL konfigurieren können.
