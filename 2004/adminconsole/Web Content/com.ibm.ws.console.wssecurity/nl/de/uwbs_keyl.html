Key Locator

Verwenden Sie diese Seite, um eine Liste der verfügbaren Key Locator anzuzeigen. Key Locator geben die erforderlichen Schlüssel für die digitale Signatur und die Verschlüsselung an. Ein Key Locator muss das Interface com.ibm.wsspi.wssecurity.config.KeyLocator implementieren. Die zwei Standardimplementierungen sind com.ibm.wsspi.wssecurity.config.WSldKeyStoreMapKeyLocator und com.ibm.wsspi.wssecurity.config.KeyStoreKeyLocator.

Gehen Sie zum Anzeigen dieser Seite der Administrationskonsole wie folgt vor:

  1. Klicken Sie auf Server > Anwendungsserver > Servername.
  2. Klicken Sie unter "Weitere Merkmale" auf Web Services: Standardbindungen für die Sicherheit von Web Services > Key Locator.
  3. Klicken Sie auf Neu, um einen Key Locator zu erstellen. Klicken Sie auf Löschen, um einen Key Locator zu löschen.

Führen Sie in dieser Anzeige die folgenden Schritte aus:

  1. Geben Sie in der Anzeige den Namen und den Klassennamen des Key Locator an.
  2. Speichern Sie Ihre Änderungen. Klicken Sie dazu oben in der Administrationskonsole auf Speichern. Nachdem Sie auf Speichern geklickt haben, erscheint die Hauptanzeige der Administrationskonsole.
  3. Kehren Sie nach dem Speichern Ihrer Änderungen in die Anzeige Key Locator zurück, um die Laufzeit der Web-Services-Sicherheit mit den Standardbindungen zu aktualisieren, die in der Datei ws_security.xml definiert sind.
  4. Klicken Sie zum Aktualisieren der Laufzeit der Web-Services-Sicherheit auf Laufzeit aktualisieren. Wenn Sie auf Laufzeit aktualisieren klicken, werden die an den anderen Webservices vorgenommenen Konfigurationsänderungen auch in der Sicherheitslaufzeit von Web Services aktualisiert.
  5. Nachdem Sie die Key Locator definiert haben, klicken Sie auf den Namen des Key Locator, um weitere Merkmale und Schlüssel unter Weitere Merkmale anzugeben.

Name des Key Locator
Der eindeutige Name des Key Locator.
Klassenname des Key Locator
Der Klassenname des Key Locator in der Keystore-Datei.
