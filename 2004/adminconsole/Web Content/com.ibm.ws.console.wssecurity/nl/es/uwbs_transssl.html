Colección de configuración HTTP SSL

Utilice esta página para configurar la seguridad SSL (Secure Sockets Layer) a nivel de trasporte. Puede utilizar esta configuración cuando un servicio Web es un cliente de otro servicio Web.

Puede utilizar la seguridad a nivel de transporte para habilitar HTTP SSL (o HTTPS). La seguridad a nivel de transporte puede habilitarse o inhabilitarse independientemente de la seguridad a nivel de mensajes. Como la seguridad a nivel de transporte proporciona una seguridad mínima, utilice la seguridad a nivel de mensajes cuando la seguridad sea esencial para una aplicación de servicios Web.

Para ver esta página de la consola administrativa, realice los pasos siguientes:

  1. Pulse Aplicaciones > Aplicaciones de empresa > nombre_aplicación.
  2. En Elementos relacionados, pulse Módulo Web > nombre_archivo_URI> Servicios Web: Enlaces de seguridad del cliente.
  3. En Configuración HTTP SSL, pulse Editar.

También puede especificar una autenticación básica HTTP SSL realizando los pasos siguientes:

  1. Pulse Aplicaciones > Aplicaciones de empresa > nombre_aplicación.
  2. En Elementos relacionados, pulse Módulo Web > nombre_archivo_URI> Servicios Web: Enlaces de seguridad del cliente.
  3. En Configuración básica de HTTP, pulse Editar.

HTTP SSL habilitado
Seleccione esta opción para habilitar las comunicaciones de socket seguro para el transporte HTTP de este puerto. Cuando esta opción esta habilitada, WebSphere Application Server utiliza el valor Configuración HTTP SSL.
Configuración HTTP SSL
Especifica qué alias de la configuración SSL debe utilizarse con el transporte HTTP para este puerto.

Esta opción se utiliza si se selecciona HTTP SSL habilitado. Los alias SSL se definen en el repertorio de configuraciones SSL, que se pueden configurar pulsando Seguridad > SSL.
