WSDL ファイル設定のパブリッシュ

このページを使用して、Web サービス記述言語 (WSDL) ファイルをパブリッシュします。

このページを表示するには、「アプリケーション」>「エンタープライズ・アプリケーション」>「application_instance」>「Publish WSDL」をクリックします。

「OK」をクリックすると、アプリケーション内のすべての Web サービス対応モジュールの zip ファイルが作成されます。パブリッシュする zip ファイルの名前は、application_name_WSDLFiles.zip です。パブリッシュする zip ファイル内のディレクトリー構造は、application_name/module_name/[META-INF|WEB-INF]/wsdl/wsdl_file_name です。

パブリッシュする WSDL ファイル内の、サービス soap:address スタンザの location 属性には、Web サービスがアクセスする際の URL が含まれています。各モジュール内の Web サービスに使用される URL の一部を指定できます。モジュール内の Web サービスには、HTTP トランスポートまたは JMS トランスポート、あるいはその両方を介してアクセスできます。どちらのタイプのトランスポートにも URL 情報を指定できます。

「構成」タブ

Web サービスの URL 接頭部の指定
URL で使用されるプロトコル (http または https のいずれか)、host_name、および port_number を指定します。

URL の接頭部の形式は、プロトコル://host_name:port_number (例えば、http://myHost:9045) です。パブリッシュする WSDL ファイルに使用される実際の URL には、そのモジュールのコンテキスト・ルートおよび Web サービスの URL パターンの前に接頭部が付加されます (例えば、http://myHost:9045/services/myService)。

HTTP URL 接頭部の選択
デフォルトの URL 接頭部リストに関連付けるドロップダウン・リストを指定します。このリストは、モジュールの仮想ホストの一連のポートと、 モジュールのアプリケーション・サーバーの一連のポートとの交点です。Web サービス・アプリケーション・サーバーに直接アクセスする場合は、このリストの項目を使用してください。

HTTP 接頭部を設定するには、「HTTP URL prefix」または「Custom HTTP URL prefix」を選択して、値を入力し、接頭部を使用するモジュールのチェック・ボックスを選択して 「適用」をクリックします。「適用」をクリックすると、選択した「Select HTTP URL prefix」または 「Custom HTTP URL prefix」フィールドの入力は、チェック・ボックス (左端の列にあります) が選択されているすべてのモジュールの「HTTP URL prefix」フィールドにコピーされます。HTTP 接頭部は「JMS URL prefix」列のフィールドには適用されません。

カスタム HTTP URL 接頭部
モジュール内の Web サービスが中間ノード (例えば Web サービス・ゲートウェイや IHS サーバー) を経由してアクセスされる場合は、 中間サービスのプロトコル、ホスト、および port_number を指定します。

HTTP 接頭部を設定するには、「HTTP URL prefix」または「Custom HTTP URL prefix」を選択して、値を入力し、接頭部を使用するモジュールのチェック・ボックスを選択して 「適用」をクリックします。「適用」をクリックすると、選択した「Select HTTP URL prefix」または 「Custom HTTP URL prefix」フィールドの入力は、チェック・ボックス (左端の列にあります) が選択されているすべてのモジュールの「HTTP URL prefix」フィールドにコピーされます。HTTP 接頭部は「JMS URL prefix」列のフィールドには適用されません。

JMS URL 接頭部
モジュールごとに使用する JMS URL 接頭部ストリングを指定します。

指定した URL 接頭部は、宛先および connectionFactory プロパティーを含んでいる必要があります。 他のプロパティー値のペアを含んでいる場合がありますが、targetService プロパティーは含まないようにしてください。 このプロパティーは、パブリッシュされた WSDL ファイルの作成時にシステムが追加します。JMS URL の接頭部の形式は、jms:/[queue&topic]?destination=target_queue_or_topic_jndi_name&connectionFactory=factory_jndi_name (例えば、jms:/queue?destination=jms/Q1&connectionFactory=jms/QCF) になります。 パブリッシュする WSDL ファイルに使用される実際の URL には、Web サービスの targetService の前に接頭部が付加されます (例えば、jms:/queue?destination=jms/Q1&connectionFactory=jms/QCF&targetService=StockQuote)。
