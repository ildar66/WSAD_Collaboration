HTTP SSL 構成コレクション

このページを使用して、トランスポート・レベルの Secure Sockets Layer (SSL) のセキュリティーを構成します。Web サービスが別の Web サービスのクライアントになっている場合に、この構成を使用できます。

トランスポート・レベルのセキュリティーを使用して、HTTP SSL (または HTTPS) を使用可能にできます。トランスポート・レベルのセキュリティーは、 メッセージ・レベルのセキュリティーから個別に使用可能とすることも使用不可とすることもできます。トランスポート・レベルの提供するセキュリティーは 最低限のセキュリティーであるため、セキュリティーが Web サービス・アプリケーションに対して必要不可欠である場合は、 メッセージ・レベルのセキュリティーを使用します。

この管理コンソール・ページを表示するには、以下のステップを実行します。

  1. 「アプリケーション」>「エンタープライズ・アプリケーション」>「application_name 」をクリックします。
  2. 「Related Items」の下の「Web モジュール」>「URI_file_name」>「Web Services: Client Security Bindings」をクリックします。
  3. 「HTTP SSL Configuration」の下の「編集」をクリックします。

以下のステップを完了して、HTTP SSL 基本認証を指定することもできます。

  1. 「アプリケーション」>「エンタープライズ・アプリケーション」>「application_name 」をクリックします。
  2. 「Related Items」の下の「Web モジュール」>「URI_file_name」>「Web Services: Client Security Bindings」をクリックします。
  3. 「HTTP Basic Configuration」の下の「編集」をクリックします。

HTTP SSL Enabled
このオプションを選択して、このポートに対する HTTP トランスポート用のセキュア・ソケット通信を 使用可能にします。使用可能にした場合、WebSphere Application Server は HTTP SSL 構成設定を使用します。
HTTP SSL Configuration
このポートに対する HTTP トランスポートに使用する SSL 構成のエイリアスを指定します。

このオプションは、「HTTP SSL Enabled」を選択した場合に使用されます。SSL エイリアスは、 「Secure Sockets Layer configuration repertoire」で定義されます。 これは「Security」>「SSL」をクリックすると構成できます。
