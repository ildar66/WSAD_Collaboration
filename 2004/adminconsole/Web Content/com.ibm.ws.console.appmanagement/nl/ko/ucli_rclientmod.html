응용프로그램 클라이언트 어셈블리 설정

이 페이지를 참조하여 J2EE 응용프로그램 클라이언트에 대한 어셈블리 등록 정보를 지정하십시오.

파일 이름(필수, 문자열)
EAR(Enterprise 아카이브) 파일의 최상위 레벨과 관련하여 응용프로그램 클라이언트 모듈의 파일 이름을 지정합니다.

독립형 모듈인 경우, 파일 이름은 아카이브의 전체 경로 이름입니다.

대체 DD
모듈의 Java 아카이브(JAR) 파일의 원래 전개 설명자 파일 대신 사용할 수 있는 대체 전개 설명자 파일의 파일 이름을 지정합니다.

이 파일은 특정 응용프로그램에 대한 전개 설명자의 차기 어셈블리 버전입니다. (원래 전개 설명자 파일을 편집하여 종속성 및 보안 정보를 분석할 수 있습니다. 대체 전개 설명자 사용을 명령하여 원래의 전개 설명자 파일을 손상되지 않은 채로 유지할 수 있습니다.) 대체 DD 값은 모듈의 루트 디렉토리와 관련된 전개 설명자 파일의 전체 경로 이름이어야 합니다. 규칙에 따라 파일은 ALT-INF 디렉토리에 있습니다. 이 등록 정보가 지정되지 않은 경우 모듈의 JAR 파일에서 직접 전개 설명자 파일을 읽습니다.

클래스 경로
응용프로그램 클라이언트 JAR 파일에 포함되지 않는 종속 코드를 포함하는 전체 클래스 경로를 지정합니다.

EAR 파일 루트와 관련된 값을 지정하고 공백을 이용해 값을 구분합니다. 하드 드라이브의 디렉토리나 참조 파일을 무시하는 절대값입니다. JAR 파일에는 없지만 EAR 파일의 루트에 있는 클래스를 지정하려면, 마침표와 정방향 슬래시(./)를 사용하십시오. 다음은 myapp.ear 파일에 client.jar로 네이밍된 응용프로그램 클라이언트 JAR 파일이 포함되어 있는 예 디렉토리 구조입니다. 추가 클래그는 class1.jar와 class2.zip 파일에 들어 있습니다. xyz.class로 네이밍된 클래스는 JAR 파일이 아닌 EAR 파일 myapp.ear/client.jar myapp.ear/class1.jar myapp.ear/class2.zip myapp.ear/xyz.class 루트에 패키지되어 있습니다. 클래스 경로 등록 정보의 값으로 class1.jar class2.zip ./를 지정하십시오. (.class 파일의 디렉토리 이름만 지정하십시오.)

표시 이름(필수, 문자열)
GUI에 표시되도록 축약형 이름을 지정합니다.

작은 아이콘
작은 이미지(16x16 픽셀)를 포함하는 JPEG 또는 GIF 파일을 지정합니다.

이 이미지는 GUI에서 응용프로그램 클라이언트를 나타내는 아이콘으로 사용됩니다.

큰 아이콘
큰 이미지(32x32 픽셀)를 포함하는 JPEG 또는 GIF 파일을 지정합니다.

이 이미지는 GUI에서 응용프로그램 클라이언트를 나타내는 아이콘으로 사용됩니다.

설명
응용프로그램 클라이언트를 설명하는 텍스트가 있습니다.

기본 클래스(필수, 문자열)
이 응용프로그램에 대한 기본 클래스의 전체 경로 이름을 지정합니다.
