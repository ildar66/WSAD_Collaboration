Einstellungen für Zielzuordnungen

Auf dieser Seite können Sie eine implementierte Anwendung oder ein implementiertes Modul (Deployment) einem Server oder einem Cluster zuordnen.

Klicken Sie, um diese Seite der Administrationskonsole anzuzeigen, auf Anwendungen > Enterprise-Anwendungen > Anwendungsname > Zielzuordnungen > Zielname.

Register "Konfiguration"

Ziel
Gibt den Namen des Zielservers oder des Ziel-Cluster an, dem die Anwendung oder das Modul zugeordnet wird. Sie geben das Ziel auf der Seite "Module Anwendungsservern zuordnen" an, auf die über die Einstellungen für eine Anwendung zugegriffen wird.
Datentyp String
Aktiviert
Gibt an, ob die Anwendungsmodule, die auf dem Zielserver installiert sind, beim Start des Servers gestartet (oder aktiviert) werden. Legt den Anfangsstatus der Anwendungsmodule fest. Der Wert "true" zeigt an, dass die entsprechenden Module beim Start des Servers aktiviert werden und daher verfügbar sind. Der Wert "false" zeigt an, dass die entsprechenden Module beim Start des Servers nicht aktiviert werden und daher nicht verfügbar sind.
Datentyp Boolescher Wert
Standardwert true
