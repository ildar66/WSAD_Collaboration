Einstellungen für EJB-Module

Auf dieser Seite können Sie ein bestimmtes implementiertes (Deployment) EJB-Modul konfigurieren und verwalten.

Anmerkung: Ein einzelnes EJB-Modul kann nicht gestartet oder zwecks Änderung gestoppt werden. Sie müssen die gesamte Anwendung starten oder stoppen, zu der das Modul gehört.

Klicken Sie zum Anzeigen dieser Seite der Administrationskonsole auf Anwendungen > Enterprise-Anwendungen > Anwendungsname > EJB-Module > Modulname.

Register "Konfiguration"

URI
Der URI gibt die Position des Modularchivinhalts in einem Dateisystem an, wenn er im Verhältnis zum Anwendungs-URL aufgelöst wird. Der URI muss mit dem URI eines ModuleRef-URI im Deployment-Deskriptor der implementierten Anwendung (EAR) übereinstimmen.
Alternativer DD
Gibt einen Deployment-Deskriptor an, der anstelle des im Modul installierten Deployment-Deskriptors zur Laufzeit verwendet werden soll.
Anfangswertigkeit
Gibt die Reihenfolge an, in der Module beim Start des Servers gestartet werden. Das Modul, das beim Start die niedrigste Wertigkeit hat, wird zuerst gestartet.
Datentyp Integer
Standardwert 5000
Bereich Größer als 0
