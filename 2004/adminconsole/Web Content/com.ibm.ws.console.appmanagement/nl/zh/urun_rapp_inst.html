企业应用程序设置

使用本页面配置企业应用程序。

要查看此管理控制台页面，单击应用程序 > 企业应用程序> application_name。

“配置”选项卡

名称
指定应用程序的逻辑名。应用程序名称在单元内必须是唯一的，并且不能包含对象名中不允许的字符。
数据类型 String
启动权值
指定服务器启动时启动应用程序的顺序。具有最低启动权值的应用程序第一个启动。
数据类型 整型
缺省 1
范围 0 到 100
应用程序二进制文件
指定将安装应用程序 EAR 文件的目录。缺省值是 APP_INSTALL_ROOT/cell_name 的值，其中 APP_INSTALL_ROOT 变量是 install_root/installedApps；例如，C:\WebSphere\AppServer\installedApps\cell_name。

您可指定绝对路径，或使用 pathmap 变量，如 ${MY_APPS}。您可在任何安装中使用 pathmap 变量。表示当前单元名的 WebSphere Application Server 变量 ${CELL} 也在 pathmap 变量中；例如 ${MY_APP}/${CELL}。

数据类型 String
单位 完整路径名
使用二进制的元数据
指定应用程序服务器是否使用位于应用程序部署文档、deployment.xml 文件（缺省）或那些位于企业应用程序资源文件（EAR）文件中的绑定、扩展和部署描述符。
数据类型 Boolean
缺省 true
启用分布
指定 WebSphere Application Server 在安装目标中是展开还是删除应用程序二进制文件。缺省为启用应用程序分布。已安装应用程序的应用程序二进制文件展开到指定的目录。当您卸载并保存更改到配置时，二进制文件也删除。如果您禁用此选项，那么必须确保在应用程序运行的所有节点的目标目录中正确地展开了应用程序二进制文件。
数据类型 Boolean
缺省 true
类装入程序方式
指定类装入程序是否首先在父类装入程序或应用程序类装入程序中搜索来装入类。JDK 类装入程序和 WebSphere Application Server 类装入程序的标准为 PARENT_FIRST。通过指定 PARENT_LAST，应用程序可以覆盖包含在父类装入程序中的类，但如果您混合使用覆盖或非覆盖类，此操作可以潜在结果为 ClassCastException 或 LinkageErrors。

选项是 PARENT_FIRST 和 PARENT_LAST。缺省是在应用程序类装入程序装入一个类中搜索之前，在父类装入程序中搜索。

数据类型 String
缺省 PARENT_FIRST
WAR 类装入程序策略
指定是否使用单个类装入程序装入此应用程序的所有 WAR 文件，还是为每个 WAR 文件使用不同的类装入程序。

选项是 APPLICATION 和 MODULE。缺省是分别使用类装入程序装入每个 WAR 文件。

数据类型 String
缺省 MODULE
为资源创建 MBean
指定是否在应用程序内创建各种资源（如 servlet 或 JSP 文件）的 MBean 文件。
数据类型 Boolean
缺省 true
启用重新装入
指定在更新应用程序文件时是否启用类重新装入。
数据类型 Boolean
缺省 true
重新装入间隔
指定用于扫描应用程序文件系统的已更新文件的秒数。缺省为 EAR 文件的 IBM 扩展（META-INF/ibm-application-ext.xmi）文件中重新装入时间间隔属性的值。仅当启用了类重新装入时此设置才生效。

此处指定的重新装入时间间隔覆盖 EAR 文件中每个 Web 模块的 IBM 扩展中指定的值（其反过来覆盖 EAR 文件中应用程序的 IBM 扩展中指定的重新装入时间间隔）。

数据类型 整型
单位 秒
