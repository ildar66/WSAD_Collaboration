Impostazioni di assemblaggio del client applicativo

Utilizzare questa pagina per specificare le proprietà di assemblaggio per i client applicativi J2EE.

Nome file (Obbligatorio, Stringa)
Specifica il nome file del modulo client applicativo, relativo al livello superiore del file EAR (Enterprise Archive).

Se questo è un modulo autonomo, il nome file è il nome del percorso completo dell'archivio.

Descrittore di distribuzione alternativo
Specifica il nome file di un file del descrittore di distribuzione alternativo da utilizzare al posto di quello originale nel file JAR (Java Archive) del modulo.

Questo file è la versione post assemblaggio del file del descrittore di distribuzione. (E' possibile modificare il file del descrittore di distribuzione originale per risolvere le informazioni sulle dipendenze e sulla sicurezza. L'indirizzamento dell'utilizzo del descrittore di distribuzione consente di mantenere intatto il file originale del descrittore di distribuzione originale). Il valore del descrittore di distribuzione (DD) alternativo deve essere il percorso completo del file del descrittore di distribuzione relativo alla directory root del modulo. Per convenzione, il file si trova nella directory ALT-INF. Se questa proprietà non viene specificata, il file del descrittore di distribuzione viene letto direttamente dal file JAR del modulo.

Classpath
Specifica il percorso classe contenente il codice dipendente non contenuto nel file JAR del client applicativo.

Specificare i valori relativi alla root del file EAR e separare i valori con degli spazi. Vengono ignorati i valori assoluti che si riferiscono a file o directory sul disco fisso. Per specificare le classi non presenti nei file JAR ma presenti nella root del file EAR, utilizzare un punto e la barra (./). Considerare la seguente struttura di directory di esempio in cui il file myapp.ear contiene un file JAR del client applicativo di nome client.jar. Le classi aggiuntive risiedono nei file class1.jar e class2.zip. Una classe di nome xyz.class non è inclusa in un file JAR bensì è contenuta nella root del file EAR myapp.ear/client.jar myapp.ear/class1.jar myapp.ear/class2.zip myapp.ear/xyz.class. Specificare class1.jar class2.zip ./ quale valore della proprietà Percorso classe. (Denominare solo la directory dei file .class.)

Nome della visualizzazione (Obbligatorio, Stringa)
Specifica un nome breve da visualizzare nelle interfacce utente grafiche (GUI).

Icona piccola
Specifica un file JPEG o GIF contenente un'immagine piccola (16x16 pixel).

L'immagine viene utilizzata come un'icona per rappresentare il client applicativo in una GUI.

Icona grande
Specifica un file JPEG o GIF contenente un'immagine grande (32x32 pixel).

L'immagine viene utilizzata come un'icona per rappresentare il client applicativo in una GUI

Descrizione
Contiene il testo che descrive il client applicativo.

Classe principale (Obbligatorio, Stringa)
Specifica il nome del percorso completo della classe principale per questo client applicativo.
