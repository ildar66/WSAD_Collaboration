Valores de ensamblaje del cliente de aplicaciones

Utilice esta página para especificar las propiedades de ensamblaje de los clientes de aplicaciones J2EE.

Nombre de archivo (Necesario, Serie)
Especifica el nombre de archivo del módulo del cliente de aplicaciones, relativo al nivel superior del archivo EAR (Enterprise Archive).

Si es un módulo autónomo, el nombre de archivo es el nombre completo de la vía de acceso del archivador.

DD alternativo
Especifica el nombre de archivo de un archivo descriptor de despliegue alternativo que se debe utilizar en lugar del archivo descriptor de despliegue original en el archivo JAR (Java Archive) del módulo.

Este archivo es la versión posterior al ensamblaje del archivo descriptor de despliegue. (Puede editar el archivo descriptor de despliegue original para que resuelva las dependencias y la información de seguridad. Si dirige el uso del descriptor de despliegue alternativo puede conservar el archivo descriptor de despliegue original intacto). El valor de la propiedad DD alternativo debe ser el nombre completo de la vía de acceso del archivo descriptor de despliegue relativo al directorio raíz del módulo. Por convenio, el archivo se encuentra en el directorio ALT-INF. Si no se especifica esta propiedad, el archivo descriptor de despliegue se lee directamente desde el archivo JAR del módulo.

Classpath
Especifica la classpath completa que contiene el código dependiente no incluido en el archivo JAR del cliente de aplicaciones.

Especifique los valores relativos al directorio raíz del archivo EAR y separe los valores con espacios. Se ignorarán los valores absolutos que hagan referencia a archivos o directorios de la unidad de disco duro. Para especificar clases que no están en archivos JAR, pero sí en el directorio raíz del archivo EAR, utilice un punto y una barra inclinada (./). Observe la estructura de directorio del ejemplo siguiente en la que el archivo myapp.ear contiene el archivo JAR del cliente de aplicaciones llamado client.jar. Las clases adicionales residen en los archivos class1.jar y class2.zip. Una clase denominada xyz.class no se empaqueta en un archivo JAR sino que se encuentra en el directorio raíz del archivo EAR myapp.ear/client.jar myapp.ear/class1.jar myapp.ear/class2.zip myapp.ear/xyz.class. Especifique class1.jar class2.zip ./ como valor de la propiedad Classpath. (Dé sólo el nombre del directorio de los archivos .class)

Nombre de visualización (Necesario, Serie)
Especifica un nombre abreviado que está previsto para ser visualizado por las GUI.

Icono pequeño
Especifica un archivo JPEG o GIF que contiene una imagen pequeña (16x16 píxeles).

La imagen se usa como icono para representar el cliente de aplicaciones en una GUI.

Icono grande
Especifica un archivo JPEG o GIF que contiene una imagen grande (32x32 píxeles).

La imagen se usa como icono para representar el cliente de aplicaciones en una GUI.

Descripción
Contiene el texto que describe el cliente de aplicaciones.

Clase principal (Necesario, Serie)
Especifica el nombre completo de la vía de acceso de la clase principal de este cliente de aplicaciones.
