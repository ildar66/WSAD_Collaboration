Valores de aplicación de empresa

Utilice esta página para configurar una aplicación de empresa.

Para ver esta página de la consola administrativa, pulse Aplicaciones > Aplicaciones de empresa> nombre_aplicación.

Pestaña Configuración

Nombre
Especifica un nombre lógico para la aplicación. Los nombres de aplicación deben ser exclusivos en una célula y no deben contener caracteres no permitidos en los nombres de objeto.
Tipo de datos String
Peso inicial
Especifica el orden en que se inician las aplicaciones cuando se inicia el servidor. La aplicación con el peso inicial menor se inicia primero.
Tipo de datos Integer
Valor por omisión 1
Rango 0 a 100
Binarios de aplicación
Especifica el directorio donde se va a instalar el archivo EAR de la aplicación. El valor por omisión es el valor de APP_INSTALL_ROOT/nombre_célula, donde la variable APP_INSTALL_ROOT es raíz_instalación/installedApps; por ejemplo, C:\WebSphere\AppServer\installedApps\nombre_célula.

Puede especificar una vía de acceso absoluta o utilizar una variable Pathmap, como por ejemplo ${MY_APPS}. Puede utilizar una variable Pathmap en cualquier instalación,. Una variable ${CELL} de WebSphere Application Server que denote el nombre de célula actual también puede estar en la variable Pathmap, como por ejemplo ${MY_APP}/${CELL}.

Tipo de datos String
Unidades Nombre completo de la vía de acceso
Usar metadatos de los binarios
Especifica si el servidor de aplicaciones va a utilizar el enlace, las extensiones y los descriptores de despliegue situados en el documento de despliegue de la aplicación, el archivo (por omisión) deployment.xml, o los archivos situados en el archivo EAR (Enterprise Application Resource).
Tipo de datos Boolean
Valor por omisión true
Habilitar la distribución
Especifica si WebSphere Application Server expande o suprime los binarios de aplicación del destino de instalación. El valor por omisión es habilitar la distribución de aplicaciones. Los binarios de aplicación de las aplicaciones instaladas se expanden en el directorio especificado. Los binarios también se suprimen cuando se desinstala y se guardan los cambios de la configuración. Si inhabilita esta opción, debe asegurarse de que los binarios de aplicación se expandan correctamente en los directorios de destino de todos los nodos donde se ejecute la aplicación.
Tipo de datos Boolean
Valor por omisión true
Modalidad de cargador de clases
Especifica si el cargador de clases debe buscar en el cargador de clases padre o en el cargador de clases de la aplicación antes de cargar una clase. El estándar para los cargadores de clases JDK y los cargadores de clases de WebSphere Application Server es PADRE_ÚLTIMO. Al especificar PADRE_ÚLTIMO, la aplicación puede alterar temporalmente las clases incluidas en el cargador de clases padre, pero esta acción potencialmente puede dar lugar a ClassCastException o LinkageErrors si ha mezclado clases alteradas temporalmente con no alteradas temporalmente.

Las opciones son PADRE_PRIMERO y PADRE_ÚLTIMO. El valor por omisión es buscar en el cargador de clases padre antes de buscar en el cargador de clases de la aplicación para cargar la aplicación.

Tipo de datos String
Valor por omisión PADRE_PRIMERO
Política del cargador de clases WAR
Especifica si se debe utilizar un sólo cargador de clases para cargar todos los archivos WAR de esta aplicación o un cargador de clases diferente para cada archivo WAR.

Las opciones son APLICACIÓN y MÓDULO. El valor por omisión es utilizar un cargador de clases diferente para cargar cada archivo WAR.

Tipo de datos String
Valor por omisión MÓDULO
Crear MBeans para los recursos
Especifica si se deben crear archivos MBean para varios recursos (por ejemplo, servlets o archivos JSP) de una aplicación.
Tipo de datos Boolean
Valor por omisión true
Recarga habilitada
Especifica si de debe habilitar la recarga de clases cuando se actualicen los archivos de aplicación.
Tipo de datos Boolean
Valor por omisión true
Intervalo de recarga
Especifica el número de segundos para explorar el sistema de archivos de la aplicación en el caso de los archivos actualizados. El valor por omisión es el del atributo de intervalo de recarga del archivo (META-INF/ibm-application-ext.xmi) de extensión de IBM del archivo EAR. Este valor entra en vigor sólo si la recarga de clases está habilitada.

El intervalo de recarga especificado aquí altera temporalmente el valor especificado en las extensiones de IBM de cada Módulo Web del archivo EAR (que, a su vez, altera temporalmente el intervalo de recarga especificado en las extensiones de IBM de la aplicación del archivo EAR).

Tipo de datos Integer
Unidades Segundos
