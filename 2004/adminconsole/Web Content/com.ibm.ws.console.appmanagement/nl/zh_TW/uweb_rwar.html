「Web 模組部署」設定

請利用這個頁面來配置 Web 模組部署的實例。

如果要檢視這個管理主控台頁面，請按一下應用程式 > 企業應用程式 > application_instance > Web 模組 > Web Module_instance。

URI
指定一個 URI，以在相對於應用程式 URL 而解析時，指定模組保存檔內容在檔案系統中的位置。 如果模組套裝在部署應用程式或企業保存檔 (EAR) 中，這個 URI 就必須符合應用程式部署描述子中的 ModuleRef URI。
替代 DD
指定用來取代模組 JAR 檔中的原始部署描述子檔之替代部署描述子檔名稱。

這個檔案是部署描述子檔的後組合版本。 您可以編輯原始部署描述子檔來解析相依性和安全資訊。 指定使用替代的部署描述子會將原始部署描述子檔保持不變。

替代 DD 內容的值必須是相對於模組根目錄的部署描述子檔的完整路徑名稱。 依照慣例，檔案會在 ALT-INF 目錄中。 如果沒有指定這個內容，就會讀取模組 JAR 檔中的部署描述子檔。

啟動加權
指定模組的啟動次序。 加權較低的模組比較高的模組先啟動。
類別載入器模式
指定類別載入器應該先在母別載入器中搜尋，或先在應用程式類別載入器中搜尋，來載入類別。 JDK 類別載入器和 WebSphere 類別載入器的標準都是 PARENT_FIRST。 您的應用程式可以指定 PARENT_LAST 來置換母類別載入器中所包含的類別，但如果您混合使用已置換的類別及非置換的類別，這個動作可能會造成 ClassCastException 或 LinkageErrors。

選項是 PARENT_FIRST 和 PARENT_LAST。預設值是在應用程式類別載入器中搜尋前，先在母類別載入器中搜尋來載入類別。

資料類型 字串
預設值 PARENT_FIRST
