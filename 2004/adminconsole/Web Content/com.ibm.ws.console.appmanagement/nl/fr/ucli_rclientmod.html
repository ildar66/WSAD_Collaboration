Paramètres d'assemblage de l'application client

Cette page permet de spécifier les propriétés d'assemblage des clients d'application J2EE.

Nom de fichier (requis, chaîne)
Spécifie le nom de fichier du module client d'application par rapport au niveau supérieur du fichier EAR.

S'il s'agit d'un module autonome, le nom de fichier est le chemin d'accès complet de l'archive.

Alternative DD
Indique le nom de fichier d'un autre descripteur de déploiement pouvant être utilisé dans le fichier JAR du module à la place du descripteur d'origine.

Ce fichier est la version postérieure à l'assemblage du fichier de descripteur de déploiement. Il est possible d'éditer le descripteur d'origine pour résoudre les dépendances et les informations de sécurité. L'utilisation d'un autre descripteur vous permet de conserver intact le descripteur d'origine. La valeur de la propriété Alternative DD doit être indiquée sous la forme du chemin complet d'accès au fichier de descripteur relativement au répertoire racine du module. Par convention, ce fichier se trouve dans le répertoire ALT-INF. Si cette propriété n'est pas spécifiée, le fichier descripteur de déploiement est lu directement dans le fichier JAR du module.

Chemin d'accès aux classes
Spécifie le chemin d'accès complet des classes contenant le code dépendant qui ne figure pas dans le fichier JAR du client d'application.

Indiquez des valeurs relatives au répertoire racine du fichier EAR, séparées par des espaces. Les valeurs absolues faisant référence à des fichiers ou des répertoires du disque dur sont ignorées. Utilisez un point et une barre oblique (./) pour indiquer des classes absentes des fichiers JAR mais présentes à la racine du fichier EAR. Prenons l'exemple d'arborescence ci-dessous, dans lequel le fichier monapp.ear contient un fichier JAR de client d'application nommé client.jar. Les autres classes se trouvent dans class1.jar et class2.zip. La classe nommée xyz.class est absente du fichier JAR, mais se trouve bien à la racine du fichier EAR : myapp.ear/client.jar myapp.ear/class1.jar myapp.ear/class2.zip myapp.ear/xyz.class. Spécifiez class1.jar class2.zip ./ comme valeur de la propriété Chemin d'accès aux classes. (Ne désignez que le répertoire des fichiers .class).

Nom affiché (requis, chaîne)
Indique un nom abrégé destiné à être affiché par les interfaces graphiques.

Petite icône
Indique un fichier JPEG ou GIF contenant une image de petite taille (16x16 pixels).

L'image permet de représenter le client d'application sous forme d'icône dans une GUI.

Grande icône
Indique le fichier JPEG ou GIF contenant une image de grande taille (32x32 pixels).

L'image permet de représenter le client d'application sous forme d'icône dans une GUI.

Description
Contient un texte décrivant le client d'application.

Classe main (requis, chaîne)
Spécifie le nom de chemin complet de la classe main de ce client d'application.
