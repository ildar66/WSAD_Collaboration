Paramètres de l'application d'entreprise

Cette page permet de configurer une application d'entreprise.

Pour afficher cette page de la console d'administration, cliquez sur Applications > Applications d'entreprise > nom_application.

Onglet Configuration

Nom
Indique le nom logique de l'application. Les noms d'application doivent être uniques dans une cellule et ne contenir aucun caractère non autorisé dans les noms d'objet.
Type de données Chaîne
Pondération de démarrage
Indique l'ordre de lancement des applications lors du démarrage du serveur. L'application dotée de la valeur de lancement la plus faible est lancée la première.
Type de données Entier
Valeur par défaut 1
Plage de valeurs 0 à 100
Fichiers binaires d'application
Indique le répertoire dans lequel le fichier EAR de l'application doit être installé. La valeur par défaut est APP_INSTALL_ROOT/nom_cellule, où la variable APP_INSTALL_ROOT correspond à racine_install/installedApps ; par exemple, C:\WebSphere\AppServer\installedApps\nom_cellule.

Vous pouvez indiquer un chemin d'accès absolu ou utiliser une variable de mappe des chemins d'accès telle que ${MY_APPS}. Si une variable de mappe des chemins d'accès peut être utilisée dans tout type d'installation. Une variable WebSphere Application Server ${CELL} qui indique le nom de cellule en cours peut également être comprise dans la variable de mappe des chemins d'accès. Exemple : ${MY_APP}/${CELL}.

Type de données Chaîne
Unité Nom de chemin d'accès complet
Utiliser les métadonnées des fichiers binaires
Indique si le serveur d'applications utilise les liaisons, les extensions et les descripteurs de déploiement situés au même endroit que le document de déploiement de l'application, le fichier deployment.xml (par défaut), ou ceux qui se trouvent dans le fichier EAR (enterprise application resource) de l'application.
Type de données Booléen
Valeur par défaut true
Activer la distribution
Indique si WebSphere Application Server développe ou supprime les fichiers binaires de l'application dans la destination de l'installation. Par défaut, la distribution de l'application est activée. Les fichiers binaires des applications installées sont développés dans le répertoire indiqué. Les fichiers binaires sont également supprimés lors de la désinstallation et les modifications sont sauvegardées dans la configuration. Si vous désactivez cette option, vous devez vous assurer que les fichiers binaires d'application sont développés de manière appropriée dans les répertoires de destination de tous les noeuds sur lesquels l'application est exécutée.
Type de données Booléen
Valeur par défaut true
Mode du chargeur de classe
Indique si le chargeur de classe doit d'abord effectuer la recherche dans le chargeur de classe parent ou dans le chargeur de classe de l'application afin de charger une classe. PARENT_FIRST est la norme pour les chargeurs de classe JDK et les chargeurs de classe WebSphere Application Server. Si vous indiquez PARENT_LAST, votre application peut substituer des classes contenues dans le chargeur de classe parent, mais cette action peut éventuellement aboutir à l'exception ClassCastException ou à des erreurs LinkageErrors si vous avez utilisé conjointement des classes substituées et des classes non substituées.

Les options possibles sont PARENT_FIRST et PARENT_LAST. Par défaut, la recherche s'effectue d'abord dans le chargeur de classe parent, puis dans le chargeur de classe d'application pour charger une classe.

Type de données Chaîne
Valeur par défaut PARENT_FIRST
Règle de chargeur de classe WAR
Indique si un seule chargeur de classe doit être utilisé pour charger tous les fichiers WAR de l'application ou si un chargeur de classe différent doit être utilisé pour chaque fichier WAR.

Les options possibles sont APPLICATION et MODULE. Par défaut, chaque fichier WAR est chargé par un chargeur de classe distinct.

Type de données Chaîne
Valeur par défaut MODULE
Créer des MBeans pour les ressources
Indique s'il convient de créer des fichiers MBean pour les différentes ressources (telles que les servlets ou les fichiers JSP) dans une application.
Type de données Booléen
Valeur par défaut true
Rechargement activé
Indique si le rechargement des classes est activé lors de la mise à jour des fichiers d'application.
Type de données Booléen
Valeur par défaut true
Intervalle de rechargement
Indique le nombre de secondes nécessaire au scannage du système de fichiers de l'application dans le but de rechercher les fichiers mis à jour. La valeur par défaut correspond à celle de l'attribut de l'intervalle de rechargement dans le fichier d'extension IBM (META-INF/ibm-application-ext.xmi) du fichier EAR. Ce paramètre prend effet uniquement si le rechargement des classes est activé.

L'intervalle de rechargement indiqué remplace la valeur spécifiée dans les extensions IBM de chaque module Web du fichier EAR (qui remplace lui-même l'intervalle de rechargement indiqué dans les extensions IBM du fichier EAR).

Type de données Entier
Unité Seconde
