Paramètres de module Web

Utilisez cette page pour configurer les paramètres de vos modules Web.

Pour accéder à cette page, recherchez le chemin d'accès suivant dans la console d'administration :

Applications > Application d'entreprise >
instance_application > Module Web
URI
Spécifie un URI qui, une fois résolu par rapport à l'URL de l'application, indique l'emplacement du contenu de l'archive du module dans un système de fichiers. Si ce module constitue l'un des composants intégrés dans l'archive d'une application déployée (fichier EAR), l'URI indiqué ici doit correspondre à une référence de module (ModuleRef) spécifiée dans le descripteur de déploiement de cette application.
Nom
Spécifie le nom unique par lequel le module sera désigné dans les outils à interface graphique.
Descripteur de déploiement secondaire
Spécifie le nom de fichier d'un autre descripteur de déploiement à utiliser à la place du descripteur original se trouvant dans le fichier JAR du module.

Ce fichier est la version post-assemblage du fichier descripteur de déploiement. Vous pouvez éditer le fichier descripteur de déploiement original pour résoudre les interdépendance et les informations de sécurité. L'utilisation d'un autre descripteur vous permet de conserver intact le descripteur d'origine.

La valeur de la propriété Autre DD doit être le nom complet du fichier descripteur de déploiement, spécifié par rapport au répertoire racine du module. Par convention, ce fichier se trouve dans le répertoire ALT-INF. Si cette propriété n'est pas spécifiée, le fichier descripteur de déploiement est lu dans le fichier JAR du module.

Pondération de démarrage
Détermine l'ordre de démarrage des différents modules de l'application. Les modules sont démarrés par ordre croissant de poids.
Préférer les classes WEB-INF
Indique les classes à charger dans WEB-INF avant les autres. Il est conseillé d'implémenter le chargeur de classes de l'application de telle sorte que les classes et les ressources intégrées dans le fichier WAR du module soient chargées avant celles qui résident dans les fichiers JAR des bibliothèques utilisées à l'échelle du conteneur.
Etat initial
Spécifie l'état par défaut de l'application au démarrage du serveur.
