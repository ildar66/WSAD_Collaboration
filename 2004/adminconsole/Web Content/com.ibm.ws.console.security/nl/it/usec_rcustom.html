Impostazioni registro utenti personalizzato

Utilizzare questa pagina per configurare il registro utenti personalizzato.

Per visualizzare questa pagina della console di gestione, fare clic su Sicurezza > Registri utenti > Personalizza.

Dopo aver impostato le proprietà su questo pannello, fare clic su Applica. Utilizzare il pannello Proprietà per ulteriori proprietà richieste dal registro personalizzato. Quando viene abilitata la sicurezza ed una qualsiasi di queste proprietà viene modificata, andare al pannello Sicurezza globale e fare clic su Applica per convalidare le modifiche.

Scheda Configurazione

ID utente server
Specifica l'ID utente su cui il server è in esecuzione, per scopi di sicurezza.

Questo ID server rappresenta un utente valido nel registro personalizzato.

Tipo dati: Stringa
Password utente del server
Specifica la password corrispondente all'ID server di sicurezza.
Tipo dati: Stringa
Nome classe del registro personalizzato
Specifica un nome classe, separato da punti, che implementa l'interfaccia com.ibm.websphere.security.UserRegistry.

Inserire il nome della classe registro personalizzato nel percorso classe. Un'ubicazione suggerita è la directory %root_installazione%/lib/ext. Sebbene il registro personalizzato implementi l'interfaccia com.ibm.websphere.security.UserRegistry, per compatibilità con le versioni precedenti, un registro può implementare, alternativamente, l'interfaccia com.ibm.websphere.security.CustomRegistry.

Tipo dati: Stringa
Predefinito: com.ibm.websphere.security.FileRegistrySample
Ignora maiuscolo/minuscolo
Specifica che viene eseguito un controllo autorizzazione che ignora la distinzione tra maiuscolo e minuscolo.
Predefinito: Abilitato
Intervallo: Abilitato o disabilitato
Utilizzare il collegamento Proprietà personalizzate per aggiungere ulteriori proprietà necessarie per inizializzare il registro personalizzato. La seguente proprietà è predefinita dal prodotto; quindi impostarla solo se necessario:
  • WAS_UseDisplayName--Se impostata su true, i metodi getCallerPrincipal(), getUserPrincipal(), getRemoteUser() restituiscono il nome visualizzato. Per impostazione predefinita, viene restituito il securityName dell'utente. Questo viene introdotto per supportare la compatibilità delle versioni precedenti con il registro personalizzato della Versione 4.0.
