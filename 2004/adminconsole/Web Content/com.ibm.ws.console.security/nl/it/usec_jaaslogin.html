Impostazioni del modulo login JAAS

Utilizzare questa pagina per definire il modulo login di una configurazione login JAAS (Java Authentication Authorization Service).

Per visualizzare questa pagina di gestione, fare clic su Sicurezza > Configurazione JAAS > Login applicazione > nome_alias > Moduli login JAAS.

Scheda Configurazione

Nome classe del modulo
Specifica il nome classe del modulo login.

I moduli login predefiniti, definiti dal prodotto WebSphere, utilizzano una classe LoginModule proxy, com.ibm.ws.security.common.auth.module.WSLoginModuleProxy. Questa classe proxy carica il modulo login WebSphere con il programma di caricamento classi di contesto thread e delega tutte le operazioni all'implementazione del modulo login reale. Quest'ultima viene specificata come opzione di delega nella configurazione delle opzioni. La classe proxy è necessaria perché i programmi di caricamento classi delle applicazioni Developer Kit non possono visualizzare i programmi di caricamento classi del prodotto WebSphere.

Tipo dati: Stringa
Strategia autenticazione
Specifica il funzionamento dell'autenticazione mentre quest'ultima avanza nello stack dei moduli login.

Un provider di autenticazione JAAS fornisce la strategia di autenticazione. In JAAS, una strategia di autenticazione viene implementata tramite l'interfaccia LoginModule.

Tipo dati: Stringa
Predefinito: Necessario
Intervallo: Obbligatorio, Richiesto, Sufficiente e Facoltativo

Specificare ulteriori opzioni in Ulteriori proprietà opzioni. Queste coppie di nomi e valori vengono inviate ai moduli login durante l'inizializzazione. Questo è uno dei meccanismi utilizzati per inviare informazioni ai moduli di login.
