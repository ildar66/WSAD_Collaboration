Impostazioni del registro utenti del sistema operativo locale

Utilizzare questa pagina per configurare le impostazioni del registro utenti del sistema operativo locale.

Per visualizzare questa pagina della console di gestione, fare clic su Sicurezza > Registri utenti > OS locale.

Scheda Configurazione

ID utente server
Specifica un ID utente valido nel registro OS locale.

Questo è l'ID del server di sicurezza che viene utilizzato solo per la sicurezza WebSphere Application Server e non è associato al processo di sistema sul quale è in esecuzione il server. Il server chiama il registro OS locale per autenticare e ottenere le informazioni sui privilegi relative agli utenti che richiamano le API native in quel registro particolare. L'accesso alle API native generalmente è limitato agli utenti con privilegi speciali (ad esempio, root nei sistemi UNIX Agisci come parte del sistema operativo nei sistemi Windows). Per utilizzare la sicurezza nel server delle applicazioni, l'ID processo (non l'ID server di sicurezza), su cui è in esecuzione WebSphere Application Server, deve disporre di privilegi sufficienti per chiamare le API di sistema. Il privilegio speciale indica che il processo, che esegue WebSphere Application Server, deve essere parte del gruppo di Amministratori e deve disporre del privilegio Agisci come parte del sistema operativo sui sistemi Windows, oltre a essere root, o a disporre dell'autorizzazione root sui sistemi UNIX.

Se si usa un registro del sistema Windows, questo ID non può essere uguale al nome della macchina Windows. I sistemi Windows considerano il nome della macchina bob come avente un account simile al bob dell'utente.

Tipo dati: Stringa
Unità: Caratteri alfanumerici
Password utente server
Specifica una password utente che corrisponde a un ID utente valido nel registro OS locale.
