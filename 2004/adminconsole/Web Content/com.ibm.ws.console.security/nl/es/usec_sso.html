Valores de inicio de sesión único

Utilice esta página para establecer los valores de configuración del inicio de sesión único (SSO).

Para ver esta página de la consola administrativa, pulse Seguridad > Mecanismos de autenticación > LTPA > Inicio de sesión único (SSO).

Pestaña Configuración

Necesita SSL
Especifica que el inicio de sesión único sólo se habilita cuando las peticiones se realizan a través de conexiones SSL HTTPS.
Tipo de datos: Boolean
Valor por omisión: Inhabilitar
Rango: Habilitar o Inhabilitar
Nombre de dominio
Especifica el nombre de dominio (por ejemplo, .ibm.com) para todos los sistemas principales de inicio de sesión único.

Si no se especifica ningún valor, el navegador Web toma como valor por omisión el nombre de sistema principal en el que está ejecutándose la aplicación Web. Esto limitará la cookie HTTP (generado para fines SSO) solamente al sistema principal que lo ha originado. Limitar la cookie HTTP puede no ser aconsejable si hay más de un sistema principal que participe en el dominio SSO. Dejar vacío el atributo de nombre de dominio sólo es aconsejable si se están ejecutando varios sistemas principales virtuales con diferentes nombres de dominio en el mismo sistema principal físico. Dejar vacío este campo permite que su navegador Web tome como valor por omisión el nombre de dominio de cada sistema principal virtual diferente. Si no se especifica explícitamente un nombre de dominio en este campo, entonces se utilizará este valor para todos los sistemas principales virtuales y, de este modo, quedarán limitados a un sólo dominio, lo que en algunas situaciones no resulta recomendable.

Si un dominio se especifica explícitamente, todos los URL utilizados para acceder a los recursos Web protegidos contienen el nombre de sistema principal de DNS (Sistema de nombre de dominio) del servidor. Por ejemplo, una vez que se haya configurado la seguridad global para LTPA y se haya especificado un nombre de dominio SSO explícitamente, se puede acceder a la consola administrativa mediante el siguiente URL: http://susistppal.austin.ibm.com:9090/admin, donde susistppal.austin.ibm.com se sustituye por el nombre de sistema principal de DNS del servidor.

Tipo de datos: String
Habilitado
Especifica que la función de inicio de sesión único está habilitada.

Las aplicaciones Web que utilizan las páginas de inicio de sesión de estilo FormLogin de J2EE (como por ejemplo, la consola administrativa de WebSphere Application Server) requieren que el inicio de sesión único (SSO) esté habilitado. Sólo debe inhabilitar SSO para ciertas configuraciones avanzadas donde las cookies de tipo SSO LTPA no son necesarias.

Tipo de datos: Boolean
Valor por omisión: Habilitado
Rango: Habilitado o Inhabilitado
