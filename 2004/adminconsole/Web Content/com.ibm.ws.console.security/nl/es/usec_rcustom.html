Valores del registro de usuario personalizado

Utilice esta página para configurar el registro de usuario personalizado.

Para ver esta página de la consola administrativa, pulse Seguridad > Registros de usuario > Personalizado.

Una vez que las propiedades estén establecidas en este panel, pulse Aplicar. Utilice el panel Propiedades para acceder a las propiedades adicionales que necesite el registro de usuario. Cuando se habilite la seguridad y se modifique una de estas propiedades, vaya al panel Seguridad Global y pulse Aplicar para validar los cambios.

Pestaña Configuración

ID de usuario de servidor
Especifica el ID de usuario bajo el que se ejecuta el servidor a efectos de seguridad.

Este ID de servidor representa un usuario válido del registro personalizado.

Tipo de datos: String
Contraseña de usuario de servidor
Especifica la contraseña correspondiente al ID de servidor de seguridad.
Tipo de datos: String
Nombre de clase de registro personalizado
Especifica un nombre de clase separada por puntos que implementa la interfaz com.ibm.websphere.security.UserRegistry.

Coloque el nombre de clase del registro personalizado en al classpath. Una ubicación sugerida es el directorio %raíz_instalación%/lib/ext. Aunque el registro personalizado implementa la interfaz com.ibm.websphere.security.UserRegistry, por motivos de compatibilidad con versiones anteriores, un registro de usuario puede implementar de forma alternativa la interfaz com.ibm.websphere.security.CustomRegistry.

Tipo de datos: String
Valor por omisión: com.ibm.websphere.security.FileRegistrySample
Ignorar mayúsculas/minúsculas
Especifica que se está realizando una comprobación de autorización que ignora las mayúsculas y minúsculas.
Valor por omisión: Habilitado
Rango: Habilitado o Inhabilitado
Utilice el enlace Propiedades personalizadas para añadir todas las propiedades necesarias para inicializar el registro personalizado. La siguiente propiedad está predefinida por el producto; y sólo se debe establecer cuando sea necesario:
  • WAS_UseDisplayName--Cuando se establece en true, los métodos getCallerPrincipal(), getUserPrincipal(), getRemoteUser() devolverán el nombre de visualización. Por omisión, se devuelve el securityName del usuario. Esta propiedad está presente para dar soporte a la compatibilidad con versiones anteriores del registro personalizado, como la versión 4.0.
