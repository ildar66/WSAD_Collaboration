Valores de módulo de inicio de sesión JAAS (Java Authentication Authorization Service)

Utilice esta página para definir el módulo de inicio de sesión para una configuración de inicio de sesión JAAS (Java Authentication Authorization Service).

Para ver esta página administrativa, pulse Seguridad > Configuración JAAS > Inicios de sesión de la aplicación > nombre_alias > Módulos de inicio de sesión de JAAS.

Pestaña Configuración

Nombre de clase de módulo
Especifica el nombre de clase del módulo de inicio de sesión.

Los módulos de inicio de sesión por omisión definidos por el producto WebSphere utilizan una clase LoginModule del proxy, com.ibm.ws.security.common.auth.module.WSLoginModuleProxy. Esta clase del proxy carga el módulo de inicio de sesión de WebSphere con el cargador de clases del contexto de hebra y delega todas las operaciones en la implementación del módulo de inicio de sesión real. La implementación del módulo de inicio de sesión real se especifica como la opción de delegación cuando se configuran las opciones. La clase del proxy es necesaria ya que los cargadores de clases de la aplicación Developer Kit no pueden visualizar los cargadores de clases del producto WebSphere.

Tipo de datos: String
Estrategia de autenticación
Especifica el comportamiento de autenticación a medida que prosigue la autenticación por la pila de módulos de inicio de sesión.

Un proveedor de autenticación JAAS proporciona la estrategia de autenticación. En JAAS, una estrategia de autenticación se implementa a través de la interfaz LoginModule.

Tipo de datos: String
Valor por omisión: Necesaria
Rango: Obligatoria, Necesaria, Suficiente y Opcional

Especifique las opciones adicionales en Propiedades adicionales de opciones. Estos pares de nombre y valor se pasan a los módulos de inicio de sesión durante la inicialización. Este es uno de los mecanismos que se utiliza para pasar información a los módulos de inicio de sesión.
