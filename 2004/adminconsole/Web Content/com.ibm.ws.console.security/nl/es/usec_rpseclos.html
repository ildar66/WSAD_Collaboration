Valores del registro de usuario del sistema operativo local

Utilice esta página para configurar los valores del registro de usuario del sistema operativo local.

Para ver esta página de la consola administrativa, pulse Seguridad > Registros de usuario > Sistema operativo local.

Pestaña Configuración

ID de usuario de servidor
Especifica un ID de usuario válido en el registro de sistema operativo local.

Éste ID es el ID de servidor de seguridad que sólo se utiliza en la seguridad de WebSphere Application Server y no está asociado con el proceso del sistema que ejecuta el servidor. El servidor llama al registro de sistema operativo local para autenticar y obtener información de privilegios sobre los usuarios llamando a las API de ese determinado registro. El acceso a las API originales está restringido normalmente a los usuarios que tienen privilegios especiales (por ejemplo, root en sistemas UNIX y actuar como parte del sistema operativo en sistemas Windows). Para utilizar la seguridad en el servidor de aplicaciones, el ID del proceso (no el ID del servidor de seguridad) en el que se ejecuta WebSphere Application Server debe tener suficientes privilegios para llamar a las API del sistema. El privilegio especial significa que es necesario que el proceso que está ejecutando WebSphere Application Server forme parte del grupos de Administradores y tenga el privilegio actuar como parte del sistema operativo en sistemas Windows y sea root o tenga autoridad root en sistemas UNIX.

Al utilizar un registro del sistema Windows, este ID no puede coincidir con el nombre de la máquina Windows. Los sistemas Windows tratan el nombre de la máquina bob como si tuviera una cuenta similar a la del usuario bob.

Tipo de datos: String
Unidades: Caracteres alfanuméricos
Contraseña de usuario de servidor
