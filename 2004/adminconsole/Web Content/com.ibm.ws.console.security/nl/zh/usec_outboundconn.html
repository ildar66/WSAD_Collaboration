公共安全互操作性传输出站设置

使用此页面指定当为出站请求与下行服务器进行通信时此服务器使用的传输和安全套接字层（SSL）设置。

要查看此管理控制台页面，单击安全性 > 认证协议 > CSI 出站传输。

“配置”选项卡

传输
指定客户机进程是否使用已连接传输的一台服务器连接到服务器。

您可选择使用 SSL、TCP/IP 或两者作为服务器支持的出站传输。如果指定 TCP/IP，服务器仅支持 TCP/IP 并且无法启动与下行服务器的 SSL 连接。如果您指定 SSL 支持的，则此服务器可以启动 TCP/IP 或 SSL 连接。如果您指定 SSL 必需的，则此服务器必须使用 SSL 启动与下行服务器的连接。当您确实指定 SSL 时，确定您要将哪个 SSL 配置设置集用于出站配置。此决策确定用于到下行服务器的出站连接的密钥文件和信任文件。例如，考虑下列情况：

TCP/IP
此服务器将仅打开与下行服务器的 TCP/IP 连接。
SSL 支持的
此服务器将打开与任何支持它们的下行服务器的 SSL 连接，以及与任何不支持它们的下行服务器的 TCP/IP 连接。
SSL 必需的
此服务器总是打开与下行服务器的 SSL 连接。

缺省： SSL 支持的
范围： TCP/IP、SSL 必需的、SSL 支持的
SSL 设置
指定出站连接的预定义的 SSL 设置列表。这些设置在“SSL 配置指令表”面板中配置。
数据类型： String
缺省： DefaultSSLSettings
范围： 在“SSL 配置指令表”面板中配置的任何 SSL 设置
