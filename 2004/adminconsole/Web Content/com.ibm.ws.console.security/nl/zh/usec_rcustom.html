定制用户注册表设置

使用此页面配置定制用户注册表。

要查看此管理控制台页面，单击安全性 > 用户注册表 > 定制。

一旦在此面板中设置了属性，单击应用。事宜定制注册表所需其它属性的属性面板。当启用了安全性并且更改任何这些属性时，请转至全局安全性面板并单击应用来确认更改。

“配置”选项卡

服务器用户标识
指定为安全起见用于运行服务器的用户标识。

此服务器标识表示定制注册表中有效用户。

数据类型： String
服务器用户密码
指定与安全性服务器标识相应的密码。
数据类型： String
定制注册表类名
指定一个实现 com.ibm.websphere.security.UserRegistry 接口的以点分隔的类名。

将定制注册表类名放入类路径。建议的位置是 %install_root%/lib/ext 目录。尽管定制注册表实现了com.ibm.websphere.security.UserRegistry 接口，对于反向兼容性，用户注册表可以交替地实现 com.ibm.websphere.security.CustomRegistry 接口。

数据类型： String
缺省： com.ibm.websphere.security.FileRegistrySample
忽略大小写
指定执行不区分大小写的授权检查。
缺省： 启用的
范围： 启用的或禁用的
使用定制属性链接添加初始化定制注册表必需的任何其它属性。以下属性由产品预定义；仅当必需时才设置此属性：
  • WAS_UseDisplayName － 当设置为 true 时，方法 getCallerPrincipal()、getUserPrincipal()、getRemoteUser() 返回显示名称。缺省情况下，返回用户的 securityName。引进它的主要目的是为了支持与 V4.0 定制注册表的反向兼容性。
