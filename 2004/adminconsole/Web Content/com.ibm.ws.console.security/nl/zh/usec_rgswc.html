服务器安全性设置

使用此页面配置服务器安全性并且设全局安全性设置。如果您需要回复到全局安全性缺省值，那么取消选择管理控制台中适当的复选框。

要查看此管理控制台页面，单击服务器 > 应用程序服务器 > server_name > 服务器安全性。注意，这些设置仅适用于 Network Deployment。

当全局安全性启用时，您可以禁用独立的应用程序服务器上的安全性。然而您无法在禁用全局安全性时启用独立的应用程序服务器上的安全性。为用户请求而禁用应用程序服务器安全性时，管理和命名安全性对于该应用程序服务器仍是启用的，这样管理和命名基础结构可以保持安全。要避免问题，验证命名安全性对于命名功能具有 Everyone 访问权，可以在您的用户 代码内使用该命名功能。您不必配置管理安全性，因为用户代码不直接访问管理功能。用户代码通过支持的编制脚本工具访问管理功能。

“配置”选项卡

服务器级别安全性
指定服务器是否覆盖安全性的单元缺省值。

要回复调用服务器级别安全性的缺省，单击使用单元安全性。单击应用，然后选择保 存来确认服务器级别的更改。

缺省 False
CSI
指定服务器是否覆盖 CSI 协议的单元缺省值。

要回复到 CSI 协议的单元缺省，单击使用单元 CSI。单击应用，然后选择保 存来确认服务器级别的更改。

缺省 False
SAS
指定服务器是否覆盖 协议的单元缺省。

要回复到 SAS 协议的单元缺省，单击使用单元 SAS。单击应用，然后选择保 存来确认服务器级别的更改。

缺省 False
