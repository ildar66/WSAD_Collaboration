单次注册设置

使用此页面配置单次注册（SSO）的配置值。

要查看此管理控制台页面，单击安全性 > 认证机制 > LTPA > 单次注册（SSO）。

“配置”选项卡

需要 SSL
指定仅当请求是在 HTTPS 安全套接字层（SSL）连接上完成时才启用单次注册功能。
数据类型： Boolean
缺省： 禁用
范围： 启用或禁用
域名
指定所有单次注册主机的域名（例如，.ibm.com）。

如果未指定值，用户的 Web 浏览器把运行 Web 应用程序的主机名作为缺省值。这限制了 HTTP cookie（为 SSO 目的而生成的）仅针对产生它的主机。如果有多个主机参与 SSO 域，则对 HTTP cookie 的限制可能是不需要的。仅当在相同物理主机上正在运行拥有不同域名的多个虚拟主机时，才期待保留域名属性为空。保留此字段为空允许您的 Web 浏览器将域名缺省设置为每个不同的虚拟主机。如果此字段中已明确指定域名，则该值将用于所有的虚拟主机，从而把它们限制到单个域，在某些情况下可能不需要如此。

如果明确地指定域名，则所有用于访问受保护的 Web 资源的 URL 都包含服务器域名系统（DNS）主机名。例如，一旦为 LTPA 配置全局安全性，并指定显式 SSO 域名后，使用以下 URL 访问管理控制台：http://yourhost.austin.ibm.com:9090/admin，其中 yourhost.austin.ibm.com 用您的服务器 DNS 主机名替换。

数据类型： String
启用的
指定启用单次注册功能。

使用 J2EE 表单登录样式登录页面的 Web 应用程序（如 WebSphere Application Server 管理控制台）需要启用单次注册（SSO）。仅在不需要 LTPA SSO 类型 cookie 的地方禁用某些高级配置的 SSO。

数据类型： Boolean
缺省： 启用的
范围： 启用的或禁用的
