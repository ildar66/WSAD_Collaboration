本地操作系统用户注册表设置

使用此页面配置本地操作系统用户注册表设置。

要查看此管理控制台页面，单击安全性 > 用户注册表 > 局部操作系统。

“配置”选项卡

服务器用户标识
在本地 OS 注册表中指定有效的用户标识。

互标识是仅用于 WebSphere Application Server 安全性的安全性服务器标识，而与运行服务器的系统进程无关。服务器调用本地 OS 注册表认证和获取有关用户的特权信息（通过调用特定注册表中的本机 API）。访问本机 API 通常受到具有特殊特权的用户的限制（例如，UNIX 系统中的 root用户和 Windows 系统中担当操作系统的一部分）。要在应用程序服务器中使用安全性，运行 WebSphere Application Server 的进程标识（非安全性服务器标识）需要足够的特权来调用系统 API。特殊特权意味着运行 WebSphere Application Server 的进程需要作为管理员组的一部分，对于 Windows 系统具有担当操作系统的一部分特权，而在 UNIX 系统中作为 root 用户或具有根权限。

当使用 Windows 系统注册表时，此标识不能与 Windows 机器的名称匹配。Windows 系统将机器名 bob 视为具有类似于用户 bob 的帐户。

数据类型： String
单位： 字母数字字符
服务器用户密码
指定与本地 OS 注册表中的有效用户标识相应的有效用户密码。
数据类型 String
