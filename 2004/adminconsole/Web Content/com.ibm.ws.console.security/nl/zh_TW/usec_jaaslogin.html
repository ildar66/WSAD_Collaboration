Java 鑑別授權服務登入模組設定

您可以利用這個頁面，來定義 Java 鑑別授權服務 (JAAS) 登入配置的登入模組。

如果要檢視這個管理頁面，請按一下安全 > JAAS 配置 > 應用程式登入配置 > alias_name > JAAS 登入模組。

「配置」標籤

模組類別名稱
指定給定登入模組的類別名稱。

Websphere 產品所定義的預設登入模組使用 Proxy LoginModule 類別 com.ibm.ws.security.common.auth.module.WSLoginModuleProxy。 這個 Proxy 類別會利用執行緒環境定義類別載入器來載入 WebSphere 登入模組，並且會將所有作業委派給真正的登入模組實作。 在選項配置中，真正的登入模組實作是指定為委派選項。 您還得具備 Proxy 類別，因為 Developer Kit 應用程式類別載入器看不見 Websphere 產品類別載入器。

資料類型： 字串
鑑別策略
將鑑別行為指定為在登入模組堆疊中向下鑑別。

JAAS 鑑別提供者會提供鑑別策略。 在 JAAS，鑑別策略是透過 LoginModule 介面來實作。

資料類型： 字串
預設值： 必要的
範圍： 必要、必備、充足和選用

請在選項其他內容中指定其他選項。在起始設定期間，會將這些名稱和值配對傳遞給登入模組。 這是將資訊傳遞給登入模組的機制之一。
