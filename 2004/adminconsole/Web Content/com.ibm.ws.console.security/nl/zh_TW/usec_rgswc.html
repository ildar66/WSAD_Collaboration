伺服器安全設定

請利用這個頁面來配置伺服器安全，置換廣域安全設定。如果您必須回復為預設的廣域安全，請取消選取管理主控台中適當的勾選框。

如果要檢視這個管理主控台頁面，請按一下伺服器 > 應用程式伺服器 > server_name > 伺服器安全。 請注意，這些設定只適用於 Network Deployment。

您可以停用個別應用程式伺服器的安全，但同時啟用廣域安全。 不過，您不能啟用個別應用程式伺服器的安全，但同時停用廣域安全。 當停用使用者要求的應用程式伺服器安全時，仍會啟用這個應用程式伺服器的管理和命名安全，因而管理和命名基礎架構仍然安全。 如果要避免問題，請確認命名安全有您的使用者程式碼所用的命名功能的每個人存取權。 您不需要配置管理安全，因為使用者程式碼不會直接存取管理功能。 使用者程式碼是利用支援的 Scripting 工具來存取管理功能。

「配置」標籤

伺服器層次安全
指定伺服器要不要置換安全的 Cell 預設值。

如果要回復伺服器層次安全的 Cell 預設值，請按一下使用 Cell 安全。 請按一下套用，再選取儲存，來驗證伺服器層次的變更。

預設值 False
CSI
指定伺服器要不要置換 CSI 通訊協定的 Cell 預設值。

如果要回復 CSI 通訊協定的 Cell 預設值，請按一下使用 Cell CSI。 請按一下套用，再選取儲存，來驗證伺服器層次的變更。

預設值 False
SAS
指定伺服器要不要置換 通訊協定的 Cell 預設值。

如果要回復 SAS 通訊協定的 Cell 預設值，請按一下使用 Cell SAS。 請按一下套用，再選取儲存，來驗證伺服器層次的變更。

預設值 False
