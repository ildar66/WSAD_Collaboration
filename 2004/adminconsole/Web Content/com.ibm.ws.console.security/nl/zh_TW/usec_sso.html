單一登入設定

請利用這個頁面來設定單一登入 (SSO) 的配置值。

如果要檢視這個管理主控台頁面，請按一下安全 > 鑑別機制 > LTPA > 單一登入 (SSO)。

「配置」標籤

需要 SSL
指定只有在利用 HTTPS Secure Socket Layer (SSL) 連線發出要求時，才啟用單一登入功能。
資料類型： Boolean
預設值： 停用
範圍： 啟用或停用
領域名稱
指定所有單一登入主機的領域名稱（如 .ibm.com）。

如果沒有指定任何值，使用者的 Web 瀏覽器會以 Web 應用程式執行其中的主機名稱為預設值。 這會將 HTTP Cookie（為了 SSO 而產生）限制在產生它的主機。 如果有多個主機參與 SSO 領域，限制 HTTP Cookie 有可能是不恰當的。 只有在相同實體主機中執行多個有不同領域名稱的虛擬主機時，將領域名稱屬性保留空白才是恰當的。 將這個欄位保留空白可讓您的 Web 瀏覽器以每個不同的虛擬主機為預設領域名稱。 如果在這個欄位中明確指定領域名稱，所有虛擬主機都會使用這個值，也因而會將它們限制在單一領域中，在某些情況下，這是不恰當的。

如果明確指定領域名稱的話，用來存取受保護的 Web 資源的所有 URL 都包含伺服器領域名稱系統 (DNS) 主機名稱。 比方說，將廣域安全配置給 LTPA 且已指定了明確的 SSO 領域名稱之後，就可以利用下列 URL 來存取管理主控台：http://yourhost.austin.ibm.com:9090/admin，其中 yourhost.austin.ibm.com 要改成伙的伺服器 DNS 主機名稱。

資料類型： 字串
啟用
指定已啟用單一登入功能。

使用 J2EE FormLogin 樣式登入頁面的 Web 應用程式（如 WebSphere Application Server 管理主控台）需要啟用單一登入 (SSO)。 請只停用不需要 LTPA SSO 類型 Cookie 的特定進階配置的 SSO。

資料類型： Boolean
預設值： 啟用
範圍： 啟用或停用
