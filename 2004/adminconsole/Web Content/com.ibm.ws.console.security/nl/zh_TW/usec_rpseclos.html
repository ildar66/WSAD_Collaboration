本端作業系統使用者登錄設定

請利用這個頁面來配置本端作業系統使用者登錄設定。

如果要檢視這個管理主控台頁面，請按一下安全 > 使用者登錄 > 本端 OS。

「配置」標籤

伺服器使用者 ID
指定本端 OS 登錄中的有效使用者 ID。

這個 ID 是安全伺服器 ID，只用在 WebSphere Application Server 安全上，與執行伺服器的系統程序無關。 伺服器會在本端 OS 登錄中呼叫原生 API，以呼叫這個特定登錄來鑑別和取得使用者的相關專用權資訊。 通常只有具備特殊專用權的使用者（如 UNIX 系統中的 root，以及 Windows 系統中的作為部分作業系統），才能存取原生 API。 如果要在應用程式伺服器中使用安全，用來執行 WebSphere Application Server 的程序 ID（不是安全伺服器 ID）必須有足以呼叫系統 API 的專用權。 特殊專用權表示執行 WebSphere Application Server 的程序必須是管理者群組的一部分，且在 Windows 系統中必須有作為部分作業系統專用權，在 UNIX 系統中必須是 root 或具有最高權限。

在使用 Windows 系統登錄時，這個 ID 不能與 Windows 機器同名。 Windows 系統會將機器名稱 bob 視為擁有類似使用者 bob 的帳戶。

資料類型： 字串
單位： 英數字元
伺服器使用者密碼
指定對應於本端 OS 登錄中之有效使用者 ID 的有效使用者密碼。
資料類型 字串
