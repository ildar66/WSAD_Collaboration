Common Secure Interoperability 傳輸離埠設定

請利用這個頁面來指定在與離埠要求的下游伺服器通訊時，這個伺服器要使用哪些傳輸和 Secure Socket Layer (SSL) 設定。

如果要檢視這個管理主控台頁面，請按一下安全 > 鑑別通訊協定 > CSI 離埠傳輸。

「配置」標籤

傳輸
指定用戶端程序要不要利用伺服器所連接的傳輸，來連接伺服器。

您可以選擇利用 SSL、TCP/IP 或兩者來作為伺服器支援的離埠傳輸。 如果您指定 TCPIP，伺服器只會支援 TCP/IP，無法起始 SSL 連線來連接下游伺服器。 如果您指定支援 SSL，這個伺服器可以起始 TCP/IP 或 SSL 連線。 如果您指定需要 SSL，這個伺服器必須利用 SSL 來起始通往下游伺服器的連線。 當您指定 SSL 時，必須決定離埠配置要使用哪一組 SSL 配置設定。 這個決定會確定和下游伺服器之離埠連線所要使用的金鑰檔和信任檔。 比方說，我們設想下列情況：

TCP/IP
這個伺服器只會開啟 TCP/IP 連線來連接下游伺服器。
支援 SSL
這個伺服器會開啟 SSL 連線來連接任何支援 SSL 連線的下游伺服器，開啟 TCP/IP 連線來連接任何不支援 SSL 連線的下游伺服器。
需要 SSL
這個伺服器只會開啟 SSL 連線來連接下游伺服器。

預設值： 支援 SSL
範圍： TCP/IP、需要 SSL、支援 SSL
SSL 設定
指定離埠連線預定 SSL 設定的清單。 這些設定是在「SSL 配置儲存庫」畫面中配置的。
資料類型： 字串
預設值： DefaultSSLSettings
範圍： 「SSL 配置儲存庫」畫面中所配置的任何 SSL 設定
