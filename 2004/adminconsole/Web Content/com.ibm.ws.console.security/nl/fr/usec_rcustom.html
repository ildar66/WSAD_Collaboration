Paramètres personnalisés du registre d'utilisateurs

Cette page permet de configurer le registre d'utilisateurs personnalisé.

Pour afficher cette page de la console d'administration, cliquez sur Sécurité > Registres d'utilisateurs > Personnalisé.

Une fois les propriétés définies dans ce panneau, cliquez sur Valider. Utilisez le panneau Propriétés pour les propriétés supplémentaires requises par le registre personnalisé. Lorsque la sécurité est activée et que certaines propriétés sont modifiées, accédez au panneau Sécurité globale et cliquez sur Valider pour valider les modifications.

Onglet Configuration

ID utilisateur administrateur
Indique l'ID utilisateur sous lequel s'exécuter le serveur pour les besoins de sécurité.

Cet ID serveur représente un utilisateur valide dans le registre personnalisé.

Type de données : Chaîne
Mot de passe administrateur
Indique le mot de passe correspondant à l'ID de serveur de sécurité.
Type de données : Chaîne
Nom de classe du registre personnalisé
Indique un nom de classe séparé par des points et implémentant l'interface com.ibm.websphere.security.UserRegistry.

Placez le nom de la classe du registre personnalisé dans le chemin d'accès aux classes. Il est conseillé d'utiliser le répertoire %install_root%/lib/ext. Même si le registre personnalisé implémente l'interface com.ibm.websphere.security.UserRegistry, un registre d'utilisateurs peut également, pour une compatibilité amont, implémenter l'interface com.ibm.websphere.security.CustomRegistry.

Type de données : Chaîne
Valeur par défaut : com.ibm.websphere.security.FileRegistrySample
Ignorer maj/min
Indique qu'une vérification sans prise en compte des majuscules et des minuscules est effectuée.
Valeur par défaut : Activée
Portée : Activée ou Désactivée
Utilisez le lien Propriétés personnalisées pour ajouter les propriétés supplémentaires requises pour l'initialisation du registre personnalisé. La propriété suivante est pré-définie par le produit. Définissez cette propriété uniquement lorsque cette opération est requise :
  • WAS_UseDisplayName : si cette propriété a pour valeur true, les méthodes getCallerPrincipal(), getUserPrincipal(), getRemoteUser() renvoient le nom affiché. Par défaut le nom de sécurité de l'utilisateur est renvoyé. Cela permet avant tout de prendre en charge le registre personnalisé de la version 4.0 (compatibilité arrière).
