Paramètres du module de connexion JAAS (Java Authentication and Authorization Service)

Cette page permet de définir le module de connexion pour une configuration de connexion JAAS.

Pour afficher cette page de la console d'administration, cliquez sur Sécurité > > Configuration JAAS > Connexions de l'application > nom_alias > Modules de connexion JAAS.

Onglet Configuration

Nom de classe du module
Indique le nom de classe d'un module de connexion spécifique.

Les modules de connexion par défaut définis par le produit WebSphere utilisent une classe LoginModule de proxy, com.ibm.ws.security.common.auth.module.WSLoginModuleProxy. Cette classe de proxy charge le module de connexion WebSphere avec le chargeur de classe de contexte d'unité d'exécution et délègue toutes les opérations à l'implémentation du module de connexion real. L'implémentation du module de connexion real est indiquée en tant qu'option déléguée dans la configuration des options. La classe de proxy est nécessaire car les chargeurs de classes de l'application Developer Kit n'ont pas connaissance des chargeurs de classes du produit WebSphere.

Type de données : Chaîne
Stratégie d'authentification
Indique le comportement de l'authentification à mesure que l'authentification s'effectue sur la pile des modules de connexion.

Un fournisseur d'authentification JAAS fournit la stratégie d'authentification. Dans JAAS, les stratégies d'authentification sont implémentées via l'interface du module de connexion.

Type de données : Chaîne
Valeur par défaut : Obligatoire
Portée : Obligatoire, Requise, Suffisante et Facultative

Spécifiez des options supplémentaires dans Propriétés supplémentaires des options. Ces paires nom/valeur sont transmises aux modules de connexion lors de l'initialisation. Il s'agit de l'un des mécanismes utilisés pour transmettre les informations aux modules de connexion.
