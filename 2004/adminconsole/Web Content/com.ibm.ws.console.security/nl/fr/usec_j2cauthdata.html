Paramètres d'entrée de données d'authentification pour le connecteur Java 2

Cette page constitue l'emplacement principal à partir duquel les administrateurs peuvent définir les données d'authentification comprenant les identités et les mots de passe des utilisateurs. Ces données peuvent référencer des entrées de données d'authentification par adaptateurs de ressources, sources de données et autres configurations nécessitant des données d'authentification, à l'aide d'un alias.

Vous pouvez afficher cette page directement à partir de la page de configuration JAAS ou à partir d'autres pages pour les ressources qui utilisent les entrées de données d'authentification J2C. Par exemple, pour afficher cette page d'administration, vous pouvez cliquer sur Sécurité > Configuration JAAS > Entrées des données d'authentification J2C ou sur Ressources > Fournisseur JMS WebSphere > Fabriques de connexions de file d'attente WebSphere > fabrique_connexions > Entrées des données d'authentification J2C.

Important: La suppression des entrées de données d'authentification est une procédure délicate. Si les données supprimées sont utilisées par d'autres configurations, le processus d'initialisation des ressources échoue.

Définissez une nouvelle entrée de données d'authentification en cliquant sur Nouveau.

Alias
Indique le nom de l'entrée de donnée d'authentification.
Type de données : Chaîne
Unités : Chaîne
Valeur par défaut : Aucune
ID utilisateur
Indique l'identité de l'utilisateur.
Type de données : Chaîne
Description
Spécifie une description facultative de l'entrée de donnée d'authentification. Par exemple, cette entrée de données d'authentification permet de se connecter à DB2.
Type de données : Chaîne
