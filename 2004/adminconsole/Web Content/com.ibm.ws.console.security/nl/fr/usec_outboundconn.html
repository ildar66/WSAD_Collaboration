Paramètres des communications sortantes pour le transport CSI

Cette page permet d'indiquer les transports et les paramètres SSL (Secure Sockets Layer) que ce serveur utilise lors de la communication avec des serveurs en aval pour les demandes sortantes.

Pour afficher cette page de la console d'administration, cliquez sur Sécurité > Protocole d'authentification > Transport des communications sortantes CSI.

Onglet Configuration

Transport
Indique si les processus client se connectent au serveur à l'aide de l'un des transports connectés au serveur.

Vous pouvez choisir d'utiliser SSL, TCP/IP ou Les deux en tant que protocole de transport sortant pris en charge par un serveur. Si vous indiquez TCP/IP, le serveur prend en charge uniquement TCP/IP et ne peut pas initialiser les connexions SSL avec les serveurs en aval. Si vous indiquez que SSL est pris en charge, ce serveur peut initialiser des connexions TCP/IP ou SSL. Si vous indiquez que SSL est requis, ce serveur doit utiliser SSL pour initialiser les connexions vers les serveurs en aval. Si vous spécifiez SSL, choisissez l'ensemble de paramètres de configuration SSL à utiliser pour la configuration des communications sortantes. Ce choix détermine le fichier de clés et le fichier des tiers dignes de confiance à utiliser pour les connexions sortantes sur les serveurs en aval. Prenez en compte, par exemple, les éléments suivants :

TCP/IP
Ce serveur ouvre uniquement les connexions TCP/IP avec les serveurs situés en aval.
SSL pris en charge
Ce serveur ouvre les connexions SSL avec les serveurs situés en aval qui prennent en charge ces connexions et les connexions TCP/IP connections avec les serveurs situés en aval ne prenant pas en charge ce type de connexion.
SSL requis
Ce serveur ouvre toujours des connexions SSL avec des serveurs situés en aval.

Valeur par défaut : SSL pris en charge
Portée : TCP/IP, requis par SSL, pris en charge par SSL
Paramètres SSL
Indique une liste de paramètres SSL prédéfinis pour les connexions sortantes. Ces paramètres sont définis dans la sous-fenêtre Répertoires de configuration SSL.
Type de données : Chaîne
Valeur par défaut : Paramètres SSL par défaut
Portée : Tous les paramètres SSL configurés dans le panneau des répertoires de configuration SSL
