Paramètres du registre d'utilisateurs du système d'exploitation local

Cette page permet de configurer les paramètres du registre d'utilisateurs du système d'exploitation local.

Pour afficher cette page de la console d'administration, cliquez sur Sécurité > Registres d'utilisateurs > Système d'exploitation local.

Onglet Configuration

ID utilisateur administrateur
Indique un ID utilisateur valide dans le registre du système d'exploitation local.

Cet ID correspond à l'ID de serveur de sécurité, qui est utilisé uniquement pour la sécurité WebSphere Application Server et qui n'est pas associé au processus système qui exécute le serveur. Le serveur appelle le registre du système d'exploitation local pour authentifier et obtenir des informations sur les droits des utilisateurs en appelant les API natives dans ce registre. L'accès API est généralement limité aux utilisateurs disposant de droits spéciaux (par exemple, root dans les systèmes UNIX et Agir comme partie du système d'exploitation sous Windows). Pour utiliser la sécurité dans le serveur d'applications, l'ID de processus (et non l'ID du serveur de sécurité) sur lequel WebSphere Application Server s'exécute doit disposer des droits suffisants pour appeler les API système. Les droits spéciaux signifient que le processus exécutant WebSphere Application Server doit appartenir au groupe Administrateurs et disposer du droit Agir comme partie du système d'exploitation sous Windows et être root ou disposer du droit root sur des systèmes UNIX.

Si vous utilisez un registre système Windows, cet ID ne peut pas correspondre au nom de la machine Windows. En effet, pour les systèmes Windows, le nom de machine bob possède un compte similaire à l'utilisateur bob.
