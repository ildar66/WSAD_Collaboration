LTPA(Lightweight Third Party Authentication) 설정

이 페이지를 참조하여 LPTA(Lightweight Third Party Authentication) 설정을 구성하십시오.

이 관리 콘솔 페이지를 보려면 보안 > 인증 메커니즘 > LTPA를 클릭하십시오.

처음으로 보안을 구성 중인 경우 암호만 필요합니다. 암호가 입력되면 적용을 클릭하십시오. SSO(>Single Sign On)를 클릭하고 도메인 이름을 입력하십시오. SSO가 사용 가능한지 확인하십시오. 적용을 클릭하십시오. 보안 설정을 완료하려면 해당 레지스트리가 설정되어 있는지 확인하고 글로벌 보안 패널에서 적용을 클릭하십시오. 보안이 사용 가능하고 이들 등록 정보 중 하나가 변경되는 경우 글로벌 보안 패널로 이동하여 적용을 클릭하여 변경사항의 유효성을 검증하십시오.

구성 탭

키 생성
서버가 새 LTPA 키를 생성할지 지정합니다.

LTPA를 인증 메커니즘으로 사용하여 처음으로 보안이 켜질 때 LTPA 키가 패널에 입력된 암호와 함께 자동으로 생성됩니다. 이전에 설정한 암호를 사용하여 새 키 세트를 생성해야 하는 경우 키 생성을 클릭하십시오. 새 암호를 사용하는 경우 이 단추를 클릭하지 마십시오. 새 암호를 입력하고 확인 또는 적용을 클릭하면 새 키 세트가 생성됩니다. 새 키 세트 생성 시, 먼저 저장해야 사용할 수 있습니다.

키 가져오기
서버가 새 LTPA 키를 가져올지 지정합니다.

복수 WebSphere 도메인(셀) 사이에서 WebSphere 제품의 SSO(Single Sign On)를 지원하려면 도메인 사이에 LTPA 키와 암호를 공유하십시오. 키 가져오기 단추를 사용하여 다른 도메인에서 LTPA 키를 가져올 수 있습니다. LTPA 키는 한 셀에서 파일로 이전에 내보낸 상태여야 합니다. 새 LTPA 키 세트를 가져오려면, LPTA 키가 위치하는 파일 이름 및 해당 암호를 입력하십시오. 키 가져오기만을 클릭하고 확인 또는 적용은 클릭하지 마십시오.

키 내보내기
서버가 LTPA 키를 내보낼지 지정합니다.

복수 WebSphere 도메인(셀) 사이에서 WebSphere 제품에서 SSO(Single Sign On)를 지원하려면 도메인 사이에 LTPA 키와 암호를 공유하십시오. 키 내보내기 단추를 사용하여 LTPA 키를 다른 도메인으로 내보낼 수 있습니다.

LTPA 키를 내보내려면, 보안을 사용 가능하게 하여 시스템이 실행되고 LPTA를 사용하고 있는지 확인하십시오. 키 파일 이름 필드에 파일 이름을 입력한 후 키 내보내기를 클릭하십시오. 암호화된 키가 지정된 파일에 저장됩니다.

암호
LTPA 키를 암호화 및 암호 해독할 암호를 지정합니다. 이러한 키를 다른 WebSphere Application Server 도메인 구성으로 가져올 때(있는 경우) 및 Domino 서버의 SSO를 구성할 때 이 암호를 사용하십시오.

키를 생성되거나 가져온 후 키는 LTPA 토큰을 암호화 및 암호 해독하는 데 사용할 수 있습니다. 암호를 변경할 때마다 확인 또는 적용 클릭 시 새 LPTA 키 세트가 자동 생성됩니다. 이 새 키 세트는 저장해야 사용할 수 있습니다.

데이터 유형 문자열
암호 확인
LTPA 키를 암호화하고 암호 해독하는 데 사용되는 확인 암호를 지정합니다.

이러한 키를 다른 WebSphere Application Server 도메인 구성으로 가져올 때(있는 경우) 및 Domino 서버의 SSO를 구성할 때 이 암호를 사용하십시오.

데이터 유형 문자열
제한시간
LTPA 토큰이 만기될 기간(분)을 지정합니다. 이 기간은 글로벌 보안 패널에서 구성된 캐시 제한시간보다 길어야 합니다.
데이터 유형 정수
단위 분
기본값 120
키 파일 이름
키를 가져오거나 내보낼 때 사용되는 파일의 이름을 지정합니다.

완전한 파일 이름을 입력한 후 키 가져오기 또는 키 내보내기를 클릭하십시오.
