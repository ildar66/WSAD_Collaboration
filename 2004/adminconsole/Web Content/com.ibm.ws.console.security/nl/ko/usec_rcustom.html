사용자 정의 사용자 레지스트리 설정

이 페이지를 참조하여 사용자 정의 사용자 레지스트리를 구성하십시오.

이 관리 콘솔 페이지를 보려면 보안 > 사용자 레지스트리 > 사용자 정의를 클릭하십시오.

이 패널에서 등록 정보가 설정된 후 적용을 클릭하십시오. 사용자 정의 레지스트리에서 필요한 추가 등록 정보를 위해 등록 정보 패널을 사용하십시오. 보안이 사용 가능하고 이 등록 정보 중 하나가 변경되는 경우 글로벌 보안 패널로 이동하여 적용을 클릭하여 변경사항의 유효성을 검증하십시오.

구성 탭

서버 사용자 ID
서버가 실행될 때 보안 목적으로 사용되는 사용자 ID를 지정합니다.

이 서버 ID는 사용자 정의 레지스트리의 유효한 사용자를 표시합니다.

데이터 유형: 문자열
서버 사용자 암호
보안 서버 ID에 해당하는 암호를 지정합니다.
데이터 유형: 문자열
사용자 정의 레지스트리 클래스 이름
com.ibm.websphere.security.UserRegistry 인터페이스를 구현하는 점 분리 클래스 이름을 지정합니다.

사용자 정의 레지스트리 클래스 이름을 클래스 경로에 넣으십시오. 제안되는 위치는 %install_root%/lib/ext 디렉토리입니다. 사용자 정의 레지스트리가 뒤로 호환성을 위해 com.ibm.websphere.security.UserRegistry 인터페이스를 구현하는 경우에도 사용자 레지스트리가 대신 com.ibm.websphere.security.CustomRegistry 인터페이스를 구현할 수 있습니다.

데이터 유형: 문자열
기본값: com.ibm.websphere.security.FileRegistrySample
대소문자 구분 안함
권한 확인 시 대소문자 구분 여부를 지정합니다.
기본값: 사용 가능
범위: 사용 가능 또는 사용 불가능
사용자 정의 등록 정보 링크를 사용하여 사용자 정의 레지스트리 초기화에 필요한 추가 등록 정보를 추가하십시오. 다음 등록 정보는 제품에 의해 사전 정의되며 필요할 때에만 이 등록 정보를 설정합니다.
  • WAS_UseDisplayName--true로 설정하는 경우, getCallerPrincipal(), getUserPrincipal(), getRemoteUser() 메소드가 표시 이름을 리턴합니다. 기본적으로 사용자의 securityName이 리턴됩니다이는 주로 버전 4.0 사용자 정의 레지스트리와의 백워드 호환성을 지원하기 위해 사용됩니다.
