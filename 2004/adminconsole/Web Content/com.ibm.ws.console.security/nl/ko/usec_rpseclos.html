로컬 운영 체제 사용자 레지스트리 설정

이 페이지를 참조하여 로컬 운영 체제 사용자 레지스트리 설정을 구성하십시오.

이 관리 콘솔 페이지를 보려면 보안 > 사용자 레지스트리 > 로컬 OS를 클릭하십시오.

구성 탭

서버 사용자 ID
로컬 OS 레지스트리에 올바른 사용자 ID를 지정합니다.

이 ID는 WebSphere Application Server 보안에만 사용되고 서버를 실행하는 시스템 프로세스와 연관되지 않는 보안 서버 ID입니다. 서버는 해당 특정 레지스트리에 있는 기본 API를 호출하여 사용자에 관한 특정 정보를 인증하고 확보하도록 로컬 OS 레지스트리를 호출합니다. 기본 API에 대한 액세스는 일반적으로 특별 권한(예: UNIX의 경우 루트, Windows의 경우 운영 체제의 일부로 활동)이 있는 사용자로 제한됩니다. Application Server에서 보안을 사용하려면 WebSphere Application Server가 실행하는 프로세스 ID(보안 서버 ID가 아님)에 시스템 API를 호출할 수 있는 충분한 특권이 있어야 합니다. 특별 권한이란 WebSphere Application Server를 실행하는 프로세스가 관리자 그룹의 일부여야 하며, Windows의 경우 운영 체제의 일부로 활동 권한이 있고 루트여야 하고, UNIX 시스템의 경우 루트 권한이 있어야 함을 의미합니다.

Windows 시스템 레지스트리를 사용하는 경우, 이 ID는 Windows 시스템의 이름과 일치할 수 없습니다. 시스템 이름 bob이 사용자 bob과 유사한 계정을 가지는 것으로 취급하기 때문입니다.

단위: 영숫자
서버 사용자 암호
로컬 OS 레지스트리의 올바른 사용자 ID에 해당하는 올바른 사용자 암호를 지정합니다.
