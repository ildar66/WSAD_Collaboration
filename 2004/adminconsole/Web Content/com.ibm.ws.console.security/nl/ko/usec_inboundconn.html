CSI(Common Secure Interoperability) 전송 인바운드 설정

이 페이지를 참조하여 열 리스너 포트 및 사용할 SSL(Secure Socket Layer) 설정값을 지정하십시오. 이 스펙은 클라이언트 또는 업스트림 서버가 수신 요청을 위해 이 서버와 통신하는 데 사용할 전송을 판별합니다.

이 관리 콘솔 페이지를 보려면 보안 > 인증 프로토콜 > CSI 인바운드 전송을 클릭하십시오.

구성 탭

전송
클라이언트가 연결된 전송 중 하나를 사용하여 서버에 대한 연결을 처리할지 여부를 지정합니다.

서버가 지원하는 인바운드 전송으로 SSL(Secure Socket Layer), TCP/IP 또는 둘다를 사용하도록 선택할 수 있습니다. TCP/IP를 지정하면 서버가 TCP/IP만 지원하며 SSL 연결을 승인할 수 없습니다. SSL 지원을 지정하면, 이 서버가 TCP/IP 또는 SSL 연결을 지원할 수 있습니다. SSL-필수를 지정하면 이 서버와 통신 중인 서버가 SSL을 사용해야 합니다.

SSL-지원 또는 SSL-필수를 지정하는 경우 인바운드 구성에 대해 사용하려는 SSL 구성 설정 세트를 결정하십시오. 이 결정이 이 서버에 대한 인바운드 연결에 사용되는 키 파일과 신뢰 파일을 판별합니다.

기본적으로, CSIv2(Common Secure Interoperability Version 2) 및 SAS(Security Authentication Service)용 SSL 포트는 동적으로 생성됩니다. Application Server에서 SSL 포트 수정이 필요한 경우, 서버 > Application Servers > server_name > 엔드포인트를 클릭하십시오. 수정할 다음 포트를 구성하십시오. 영(0) 포트 번호는 동적 지정이 런타임 시 작성됩니다. 

CSIV2_SSL_MUTUALAUTH_LISTENER_ADDRESS 
CSIV2_SSL_SERVERAUTH_LISTENER_ADDRESS 
SAS_SSL_SERVERAUTH_LISTENER_ADDRESS

  • TCP/IP: TCP/IP 리스너 포트만 열리고 모든 요청 인바운드는 SSL 보호되지 않습니다.
  • SSL-지원: TCP/IP 및 SSL 리스너 포트 모두가 열리고 대부분의 요청이 SSL을 사용하여 인바운드됩니다.
  • SSL-필수: SSL 리스너 포트만 열리고 모든 요청이 SSL 연결을 통해 들어옵니다. SSL-필수를 선택한 경우, 또한 활성 인증 프로토콜로서 CSI를 선택해야 합니다. CSI 및 SAS를 선택한 경우, SAS는 일부 특별 요청에 대해 TCP/IP 소켓이 열려 있도록 요청합니다.

기본값: SSL 지원
범위: TCP/IP, SSL 필수, SSL-지원
SSL 설정
인바운드 연결을 위해 선택할 사전 정의된 SSL 설정값 목록을 지정합니다. 이 설정은 SSL 레퍼토리 패널에서 구성됩니다.

데이터 유형: 문자열
기본값: DefaultSSLSettings
범위: SSL 구성 레퍼토리에 구성되는 모든 SSL 설정
