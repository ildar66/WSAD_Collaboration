서버 보안 설정

서버 보안을 구성하고 글로벌 보안 설정을 대체하려면 이 페이지를 참조하십시오. 글로벌 보안 기본값을 되돌려야 할 경우, 관리 콘솔에서 해당 선택란을 선택 취소하십시오.

이 관리 콘솔 페이지를 보려면 서버 > Application servers > server_name > 서버 보안을 클릭하십시오. 이 설정은 Netowrk Deployment에만 적용된다는 것을 알아두십시오.

글로벌 보안이 사용 가능하면 개별 Application Server에서 보안을 사용 불가능하게 할 수 있습니다. 그러나, 글로벌 보안이 사용 불가능한 경우에는 개별 Application Server에서 보안을 사용 가능하게 할 수 없습니다. Application Server가 사용자 요청에 대해 사용 불가능한 동안 관리 및 네이밍 보안이 해당 Application Server에서 사용 가능하므로 관리 및 네이밍 기본구조가 보안 상태로 유지됩니다. 문제점을 방지하려면, 네이밍 보안이 사용자 코드에서 사용되는 네이밍 기능에 대해 Everyone 액세스 권한이 있는지 검증하십시오. 사용자 코드는 직접 관리 기능에 액세스하지 않기 때문에 관리 보안을 구성하지 않아도 됩니다. 사용자 코드는 지원되는 스크립트 도구를 통해 관리 기능에 액세스합니다.
