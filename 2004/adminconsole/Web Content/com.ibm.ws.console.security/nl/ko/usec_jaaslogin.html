Java 인증 권한 서비스 로그인 모듈 설정

이 페이지를 참조하여 JAAS(Java Authentication Authorization Service) 로그인 구성의 로그인 모듈을 정의하십시오.

이 관리 페이지를 보려면 보안 > JAAS 구성 > 응용프로그램 로그인 > alias_name > JAAS 로그인 모듈을 클릭하십시오.

구성 탭

모듈 클래스 이름
해당 로그인 모듈의 클래스 이름을 지정합니다.

WebSphere 제품에서 정의하는 기본 로그인 모듈은 프록시 LoginModule 클래스, com.ibm.ws.security.common.auth.module.WSLoginModuleProxy를 사용합니다. 이 프록시 클래스는 스레드 컨텍스트 클래스 로더를 사용하여 WebSphere 로그인 모듈를 로드하며 모든 조작을 실제 로그인 모듈 구현에 위임합니다. 실제 로그인 모듈 구현은 옵션 구성에서 전달 옵션으로 지정됩니다. 개발자 킷 응용프로그램 클래스 로더는 WebSphere 제품 클래스 로더를 볼 수 없으므로 프록시 클래스가 필요합니다.

데이터 유형: 문자열
인증 계획
인증이 로그인 모듈의 스택 아래로 진행하면서 인증 작동을 지정합니다.

JAAS 인증 프로바이더는 인증 전략을 제공합니다. JAAS에서 인증 계획은 LoginModule 인터페이스를 통해 구현됩니다.

데이터 유형: 문자열
기본값: 필요
범위: 필수, 전제 조건, 충분 조건 및 선택적

옵션 추가 등록 정보에서 추가 옵션을 지정하십시오.이름 및 값 쌍이 초기화 동안 로그인 모듈에 전달됩니다. 이것은 로그인 모듈에 정보를 전달하는 데 사용되는 메커니즘 중 하나입니다.
