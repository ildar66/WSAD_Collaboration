共通セキュア・インターオペラビリティー・トランスポート・アウトバウンド設定

このページを使用して、このサーバーがアウトバウンド要求でダウンストリーム・サーバーと通信するときに使用する トランスポートと Secure Sockets Layer (SSL) の設定を指定します。

この管理コンソール・ページを表示するには、「セキュリティー」>「認証プロトコル」>「CSI Outbound Transport」をクリックします。

「構成」タブ

トランスポート
クライアント・プロセスが、サーバーの接続済みトランスポートの 1 つを使用して サーバーに接続するかどうかを指定します。

サーバーがサポートするアウトバウンド・トランスポートとして、「SSL」、「TCP/IP」、 または「Both」のいずれかを選択し、使用できます。 「TCP/IP」を指定すると、 サーバーは TCP/IP のみをサポートし、ダウンストリーム・サーバーとの SSL 接続は開始できません。 「SSL サポート」を指定すると、このサーバーは TCP/IP 接続または SSL 接続を開始できます。 「SSL 必須」を指定すると、 このサーバーは SSL を使用して、ダウンストリーム・サーバーへの接続を開始する必要があります。 「SSL」を指定する場合、SSL 構成設定のどのセットをアウトバウンド構成に使用するかを決定する必要があります。 この決定により、 ダウンストリーム・サーバーへのアウトバウンド接続で使用する鍵ファイルとトラスト・ファイルが決まります。 例えば、次の例について考えてみます。

TCP/IP
このサーバーはダウンストリーム・サーバーとの接続について、TCP/IP 接続だけをオープンします。
SSL サポート
このサーバーは、SSL をサポートしているダウンストリーム・サーバーについて は SSL 接続を、SSL をサポートしていないダウンストリーム・サーバーについては TCP/IP 接続をオープンします。
SSL 必須
このサーバーはダウンストリーム・サーバーとの接続について、常に SSL 接続をオープンします。

デフォルト: SSL サポート
範囲: TCP/IP、SSL 必須、SSL サポート
SSL 設定
アウトバウンド接続用に事前定義された SSL 設定のリストを指定します。 これらの設定は、「SSL 構成レパートリー」パネルで構成します。
データ型: ストリング
デフォルト: DefaultSSLSettings
範囲: 「SSL 構成レパートリー」パネルで構成される任意の SSL 設定
