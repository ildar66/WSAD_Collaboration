Java Authentication Authorization Service のログイン・モジュール設定

このページを使用して、Java Authentication Authorization Service (JAAS) ログイン構成のログイン・モジュールを定義します。

この管理ページを表示するには、「セキュリティー」>「JAAS 構成」>「Application Logins」>「alias_name」>「JAAS ログイン・モジュール」をクリックします。

「構成」タブ

モジュール・クラス名
指定されたログイン・モジュールのクラス名を指定します。

WebSphere 製品によって定義されたデフォルトのログイン・モジュールは、プロキシー LoginModule クラス の com.ibm.ws.security.common.auth.module.WSLoginModuleProxy を使用します。 このプロキシー・クラスは、スレッド・コンテキスト・クラス・ローダーとともに WebSphere ログイン・モジュールをロードし、 実際の ログイン・モジュール・インプリメンテーションにすべてのオペレーションを委任します。 実際のログイン・モジュール・インプリメンテーションは、オプション構成における代行オプションとして指定されます。 Developer Kit のアプリケーション・クラス・ローダーには、WebSphere 製品のクラス・ローダーの可視性がないため、 プロキシー・クラスが必要です。

データ型: ストリング
認証ストラテジー
認証がログイン・モジュールのスタックを下に進むときの認証の動作を指定します。

JAAS 認証プロバイダーは、認証ストラテジーを提供します。JAAS では、認証ストラテジーは LoginModule インターフェースを介してインプリメントされます。

データ型: ストリング
デフォルト: 必要
範囲: 必須、必要、十分、およびオプション

「Options Additional Properties」で追加のオプションを指定します。 これらの名前と値のペアは、初期化中にログイン・モジュールに渡されます。 これは、ログイン・モジュールに情報を渡すために使用するメカニズムの 1 つです。
