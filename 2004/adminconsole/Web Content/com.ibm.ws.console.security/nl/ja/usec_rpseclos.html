ローカル・オペレーティング・システムのユーザー・レジストリー設定

このページを使用して、ローカル・オペレーティング・システムのユーザー・レジストリー設定を構成します。

この管理コンソール・ページを表示するには、「セキュリティー」>「ユーザー・レジストリー」>「ローカル OS」をクリックします。

「構成」タブ

サーバー・ユーザー ID
ローカル OS レジストリー内の有効なユーザー ID を指定します。

この ID はセキュリティー・サーバー ID であって、WebSphere Application Server のセキュリティー のためにのみ使用され、サーバーを実行するシステム・プロセスとは関連していません。 このサーバーは、認証のためにローカル OS レジストリーを呼び出し、 その特定のレジストリー内のネイティブな API を呼び出してユーザーに関する特権情報を入手します。 ネイティブ API へのアクセスは、特別な特権 (例えば、UNIX の場合の「root」や Windows システムの場合の「 オペレーティング・システムの一部として機能する (Act as part of operating system)」) を持つユーザーのみが行うことができます。アプリケーション・ サーバーでセキュリティーを使用するには、(セキュリティー・サーバー ID ではなく) WebSphere Application Server が実行される プロセス ID に、システム API を呼び出すための十分な特権が必要です。この特別な特権は、WebSphere Application Server を実行 するプロセスが管理者グループに属している必要があり、Windows システムの場合には「 オペレーティング・システムの一部として機能する (Act as part of operating system)」特権を、UNIX では root であるかあるいは root 権限を持っていることを意味します。

Windows システム・レジストリーを使用する際に、この ID を Windows マシンの名前と同じにすることはできません。 これは、Windows システムでは、マシン名 bob が、ユーザー bob と同様のアカウントを持つものとして扱われる ためです。

データ型: ストリング
単位: 英数字
サーバー・ユーザー・パスワード
ローカル OS レジストリー内の有効なユーザー ID に対応する有効なユーザー・パスワードを指定します。
データ型 ストリング
