Einstellungen der Benutzer-Registry des lokalen Betriebssystems

Auf dieser Seite können Sie die Einstellungen der Benutzer-Registry des lokalen Betriebssystems konfigurieren.

Klicken Sie zum Anzeigen dieser Seite der Administrationskonsole auf Sicherheit > Benutzer-Registrys > LocalOS.

Register "Konfiguration"

Benutzer-ID des Servers
Gibt eine gültige ID in der Registry des lokalen Betriebssystems an.

Diese ID ist die ID des Sicherheitsservers, die nur für die Sicherheitseinrichtung des WebSphere Application Server verwendet und nicht dem Systemprozess zugeordnet wird, der den Server ausführt. Der Server ruft die Registry des lokalen Betriebssystems auf, um Berechtigungsinformationen zu Benutzern zu authentifizieren und abzurufen, indem er die nativen APIs in dieser bestimmten Registry aufruft. Der Zugriff auf native APIs ist normalerweise auf Benutzer beschränkt, die spezielle Berechtigungen haben (z. B. root auf UNIX-Systemen und Einsetzen als Teil des Betriebssystems auf Windows-Systemen). Die Prozess-ID (nicht die ID des Sicherheitsservers), unter der der WebSphere Application Server ausgeführt wird, muss, um die Sicherheitseinrichtung auf dem Anwendungsserver verwenden zu können, ausreichende Berechtigungen zum Aufrufen der System-APIs besitzen. Die Sonderberechtigung bedeutet, dass der Prozess, der WebSphere Application Server ausführt, auf Windows-Systemen zur Gruppe Administratoren gehört und die Berechtigung Einsetzen als Teil des Betriebssystems hat und auf UNIX-Systemen root ist oder Root-Berechtigung hat.

Wenn Sie die System-Registry von Windows verwenden, darf diese ID nicht mit dem Namen der Windows-Maschine übereinstimmen. Windows-Systeme behandeln den Einheitennamen bob wie einen Account des Benutzers bob.

Datentyp String
Einheiten Alphanumerische Zeichen
