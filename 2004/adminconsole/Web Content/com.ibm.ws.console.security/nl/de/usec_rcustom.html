Einstellungen für eigene Benutzer-Registry

Auf dieser Seite können Sie eine eigene Benutzer-Registry konfigurieren.

Klicken Sie zum Anzeigen dieser Seite der Administrationskonsole auf Sicherheit > Benutzer-Registrys > Benutzerdefiniert.

Wenn Sie die Merkmale in dieser Anzeige definiert haben, klicken Sie auf Anwenden. In der Anzeige Merkmale können Sie weitere Merkmale angeben, die die benutzerdefinierte Registry benötigt. Wenn bei aktivierter Sicherheitseinrichtung eines dieser Merkmale geändert wird, müssen Sie in die Anzeige Globale Sicherheit wechseln und auf Anwenden klicken, um die Änderungen zu überprüfen.

Register "Konfiguration"

Benutzer-ID des Servers
Gibt die Benutzer-ID an, unter der der Server aus Sicherheitsgründen ausgeführt wird.

Die Server-ID stellt einen gültigen Benutzer in der benutzerdefinierten Registry dar.

Datentyp String
Benutzerkennwort des Servers
Gibt das Kennwort an, das der ID des Sicherheitsservers entspricht.
Datentyp String
Klassenname der benutzerdefinierten Registry
Ein Klassenname, der Punkte als Trennzeichen verwendet und das Interface com.ibm.websphere.security.UserRegistry implementiert.

Geben Sie im Klassenpfad den Klassennamen der benutzerdefinierten Registry ein. Ein Vorschlag für das Verzeichnis ist %install_root%/lib/ext. Obwohl die angepasste Registry das Interface com.ibm.websphere.security.UserRegistry implementiert, um die Abwärtskompatibilität zu gewährleisten, kann eine Benutzer-Registry auch das Interface com.ibm.websphere.security.CustomRegistry implementieren.

Datentyp String
Standardwert com.ibm.websphere.security.FileRegistrySample
Groß-/Kleinschreibung ignorieren
Gibt an, dass eine Berechtigungsprüfung ohne Berücksichtigung der Groß-/Kleinschreibung durchgeführt wird.
Standardwert Aktiviert
Bereich Aktiviert oder inaktiviert
Verwenden Sie den Link Benutzerdefinierte Merkmale, wenn Sie weitere Merkmale hinzufügen möchten, die zum Initialisieren der angepassten Registry erforderlich sind. Das folgende Merkmal ist vom Produkt bereits vordefiniert und sollte nur bei Bedarf gesetzt werden:
  • ^WAS_UseDisplayName -- Wenn dieses Merkmal auf true eingestellt ist, geben die Methoden getCallerPrincipal(), getUserPrincipal(), getRemoteUser() den Anzeigenamen zurück. Standardmäßig wird der securityName des Benutzers zurückgegeben. Dieses Verfahren wurde in erster Linie für die Unterstützung der Abwärtskompatibilität mit der angepassten Registry der Version 4.0 eingeführt.
